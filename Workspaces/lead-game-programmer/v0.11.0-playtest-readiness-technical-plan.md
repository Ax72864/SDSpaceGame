## 3. D1 / D2 / D7：面板状态机边界（核心）

### 3.1 主面板优先级常量

新增（落地在 `Game/game.js` 内一段连续常量区，靠近 `META_CATEGORY_LABELS` 等 UI 常量定义处）：

```javascript
// 数字越小优先级越高；同时只能有一个主面板可见
const MAIN_PANEL_PRIORITY = {
  runSettlement: 1,    // 结算（失败/通关）最高优先级：玩家必须看到结果
  galaxyMap: 2,        // 跃迁选择：玩家主动触发，且涉及不可逆推进
  objectiveChoice: 3,  // 任务完成提示：信息层级，非阻塞操作
  meta: 4,             // 局外天赋购买与协议选择：可随时打开关闭
  researchTree: 5      // 局内研发：v0.5 的局内能力提升，已稳定
};
```

`objectiveChoicePanel` 是"信息层级"而非"阻塞层级"：玩家可继续局内操作；它的显示由 `state.run.awaitingObjectiveChoice / objectiveChoiceDismissed` 驱动，**不**进入"互斥关闭"集合，仅在更高优先级面板打开时 CSS 隐藏（保留状态机不变）。

### 3.2 互斥矩阵

行 = 即将打开的面板，列 = 当前已打开的面板，单元格 = 应执行的动作：

| 即将打开 \ 当前已开 | runSettlement | galaxyMap | objectiveChoice | meta | researchTree |
| --- | --- | --- | --- | --- | --- |
| runSettlement | 已是自身，no-op | 关闭 galaxyMap | 隐藏 objectiveChoice（不改 state） | 关闭 meta | 关闭 researchTree |
| galaxyMap | 拒绝打开（保留结算） | no-op | 隐藏 objectiveChoice | 关闭 meta | 关闭 researchTree |
| objectiveChoice | 不显示（settlement 优先） | 不显示（galaxyMap 优先） | no-op | 不显示（meta 优先） | 不显示（researchTree 优先） |
| meta | 拒绝打开（保留结算） | 拒绝打开（保留跃迁选择） | 关闭 meta 时恢复 objectiveChoice | no-op | 关闭 researchTree |
| researchTree | 拒绝打开 | 拒绝打开 | 同 meta | 关闭 meta | no-op |

**关键不变量**：

1. `runSettlementPanel` 打开期间，**禁止**打开 `galaxyMapPanel` / `metaPanel` / `researchTree`。`closeSettlementForExplore` / `nextLevel` 关闭结算后才允许其他面板打开。
2. `galaxyMapPanel` 打开期间，**禁止**打开 `metaPanel` / `researchTree`。
3. `objectiveChoicePanel` 是"低优先级信息层"，被更高层面板打开时只做 CSS 隐藏（`panel.classList.add("hidden")`），不修改 `state.run.awaitingObjectiveChoice` 与 `objectiveChoiceDismissed`；高层面板关闭后由 `updateObjectiveChoiceUi()` 自然恢复显示。
4. **拒绝打开**时调用 `showToast` 提示"请先关闭当前结算/跃迁选择面板"，**不**强制关闭高优先级面板。

### 3.3 helper 接口

T2 落地时新增 3 个 helper（建议放在 `bindInput` 之前的工具函数区），调用方对状态机的访问全部通过它们：

```javascript
function getActiveMainPanel() {
  if (runSettlementPanelEl && !runSettlementPanelEl.classList.contains("hidden")) return "runSettlement";
  if (isGalaxyMapPanelOpen()) return "galaxyMap";
  if (isMetaPanelOpen()) return "meta";
  if (state.hud.researchTreeOpen) return "researchTree";
  return null;
}

function requestOpenMainPanel(panelId, options = {}) {
  const active = getActiveMainPanel();
  if (active === panelId) return { ok: true, alreadyOpen: true };

  const priority = MAIN_PANEL_PRIORITY;
  if (active && priority[active] < priority[panelId]) {
    if (!options.silent) showToast("请先关闭当前界面再进行此操作。");
    return { ok: false, reason: "blocked_by_higher_priority", blockedBy: active };
  }

  if (active) closeMainPanel(active, { reason: "preempted_by_higher_priority" });
  return { ok: true, alreadyOpen: false };
}

function closeMainPanel(panelId, options = {}) {
  switch (panelId) {
    case "runSettlement": runSettlementPanelEl?.classList.add("hidden"); break;
    case "galaxyMap":     hideGalaxyMapPanel(); break;
    case "meta":          toggleMetaPanel(false); break;
    case "researchTree":  toggleResearchTree(false); break;
    case "objectiveChoice": /* 仅 CSS 隐藏，由 updateObjectiveChoiceUi 恢复 */
      document.getElementById("objectiveChoicePanel")?.classList.add("hidden");
      break;
  }
  if (!options.skipObjectiveRefresh) updateObjectiveChoiceUi();
}
```

**调用边界硬规则**：

1. `toggleMetaPanel(true)`、`openGalaxyMapPanel(choices)`、`endRunSettlement` / `gameOver`、`toggleResearchTree(true)` 等入口必须先调 `requestOpenMainPanel(panelId)`，根据返回结果决定是否继续。
2. **不允许**实现方为绕过互斥而直接 `classList.remove("hidden")` 操作面板根元素；必须走入口函数。
3. `requestOpenMainPanel` 返回 `ok: false` 时，调用方**不要**继续执行打开逻辑（包括 `renderMetaPanel()`、`renderGalaxyMapCards()` 等渲染调用）。
4. 现有 `toggleMetaPanel(open)` 中"open 时关闭 researchTree 与 galaxyMap"的硬编码（`Game/game.js:3807-3810`）改为统一走 `requestOpenMainPanel("meta")` + `closeMainPanel(active)`，避免双重路径。
5. **关键防回归**：`endRunSettlement` / `gameOver` 已经强制 `state.paused = true`，互斥校验时这两个调用是"结算被动出现"而非"玩家主动打开"，可对它们传 `options.force = true` 跳过 `requestOpenMainPanel` 的 block，但**仍**必须主动 `closeMainPanel("galaxyMap")` / `closeMainPanel("meta")` / `closeMainPanel("researchTree")`。

### 3.4 Escape 关闭优先级（D2）

新规则覆盖 `Game/game.js:6657-6724` 中的 Escape 分支（不动 capture 阶段的 galaxyMap 优先消费）：

```text
Escape →
  if (galaxyMapPanel 已被 capture 阶段消费) return;   // 兼容现有
  active = getActiveMainPanel();
  switch (active) {
    case "runSettlement":
      // 结算不允许 Escape 关闭：避免误触发"开始新局"或"留在终结星系"
      // 仅 showToast("请通过按钮关闭结算面板。")，不 preventDefault
      return;
    case "meta":
      toggleMetaPanel(false); event.preventDefault(); return;
    case "researchTree":
      toggleResearchTree(false); event.preventDefault(); return;
    default:
      // objectiveChoicePanel：当前 awaitingObjectiveChoice 时按 stayBtn 等价行为切换为 dismissed
      // 但不主动跳转、不触发跃迁；继续保持现有"清 selectedBuild"路径
      if (state.run.awaitingObjectiveChoice) {
        state.run.objectiveChoiceDismissed = true;
        state.run.awaitingObjectiveChoice = false;
        updateObjectiveChoiceUi();
        event.preventDefault();
        return;
      }
      // 现有：清 selectedBuild / selectedCell / clearBuildError / updateBuildButtons / updateHud
  }
```

**Escape 必须遵守**的硬规则：

1. **永远不触发**：购买（`purchaseMetaTalent`）、确认协议（`selectStartProtocol`）、确认跃迁（`confirmGalaxyJump`）、开始新局（`startNewRun`）、留在终结星系（`closeSettlementForExplore`）等不可逆动作。
2. **结算面板期间 Escape 是 no-op**（仅 toast 提示）：制作人方向 §2 明确"Escape 应按当前最上层面板优先级关闭"，但结算结果必须被玩家主动确认，避免误关键路径。
3. `objectiveChoicePanel` 的 Escape 等价于"暂时停留"语义（`stayBtn`）：把 `awaitingObjectiveChoice` 切到 `dismissed`，保留任务完成态，玩家可再次点击 jumpBtn 重新弹出 galaxyMap。**不**触发 `confirmGalaxyJump`。
4. Escape 处理函数内部**不读写** `state.meta` 字段；仅读写 `state.run.awaitingObjectiveChoice / objectiveChoiceDismissed` 和 UI class。

### 4.3 派生函数

```javascript
function countSimultaneouslyOpenMainPanels() {
  let count = 0;
  if (!runSettlementPanelEl?.classList.contains("hidden")) count++;
  if (isGalaxyMapPanelOpen()) count++;
  if (isMetaPanelOpen()) count++;
  if (state.hud?.researchTreeOpen) count++;
  return count;
}

function summarizeMetaForSnapshot(meta) {
  if (!isMetaObject(meta)) return { schemaVersion: null, points: 0, talents: {}, unlockedProtocolsCount: 0, selectedProtocol: null };
  return {
    schemaVersion: meta.schemaVersion,
    points: meta.points,
    talents: { ...meta.talents },                          // 浅拷贝，rank 数字快照
    unlockedProtocolsCount: (meta.unlockedProtocols || []).length,
    selectedProtocol: meta.selectedProtocol
  };
}
```

### 4.4 错误环形缓冲（D9）

```javascript
const PLAYTEST_ERROR_CAPACITY = 50;
const playtestErrorBuffer = [];
let playtestErrorBound = false;

function bindPlaytestErrorListeners() {
  if (playtestErrorBound) return;
  playtestErrorBound = true;
  window.addEventListener("error", (event) => {
    pushPlaytestError({
      type: "error",
      message: event.message,
      source: event.filename,
      line: event.lineno,
      col: event.colno,
      timestamp: Date.now()
    });
  });
  window.addEventListener("unhandledrejection", (event) => {
    pushPlaytestError({
      type: "unhandledrejection",
      message: String(event.reason?.message || event.reason || "unknown"),
      timestamp: Date.now()
    });
  });
}

function pushPlaytestError(entry) {
  playtestErrorBuffer.push(entry);
  if (playtestErrorBuffer.length > PLAYTEST_ERROR_CAPACITY) {
    playtestErrorBuffer.splice(0, playtestErrorBuffer.length - PLAYTEST_ERROR_CAPACITY);
  }
}

function getRecentRuntimeErrors(limit = 10) {
  const safeLimit = Math.max(1, Math.min(PLAYTEST_ERROR_CAPACITY, limit | 0));
  return playtestErrorBuffer.slice(-safeLimit);
}

function getRuntimeErrorCount() { return playtestErrorBuffer.length; }
```

边界约束：

1. `bindPlaytestErrorListeners` 在 `initGame` 流程结束时调用一次（与 `bindInput` 类似），**不**在 snapshot 时绑定（避免反复绑定 listener）。
2. **不**包装 `console.error`：避免影响开发期 stack trace；不静默错误，让错误仍正常抛出到浏览器 console。
3. 环形缓冲容量固定 50，避免长会话内存膨胀；超过容量丢弃最早条目。
4. `clearErrors()` 仅清空 buffer，不影响浏览器原生 error 状态。
