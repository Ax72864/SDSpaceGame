## 1. 背景

### 1.1 版本目标摘要

制作人方向把 v0.9.0 定为「局间星系图系统」——在 v0.7.0 战斗操控 + v0.8.0 局内事件之上，引入**平行 GALAXY_TYPES 注册表 + 跃迁路径选择 UI**，玩家每次完成主任务点击「跃迁下一星系」时弹出 2-3 个候选星系卡片，每个星系有独立的任务 / 事件 / 敌人 / 资源偏好和视觉签名；7 关一局结构不变，玩家走出的路径成为「独属于本局的轨迹」。

整个版本是「权重覆盖层 + 跃迁 UI 重做」而非「现有系统重构」——OBJECTIVE_TYPES / `createObjective` / `getObjectiveWeightTable` / v0.7.0 锁敌战斗 / v0.6.0 hostile-station / v0.8.0 ENCOUNTER_TYPES 全部完整保留；玩家选择 `emptyVoid` galaxyType 后游戏体验**精确等价 v0.8.0**（所有乘子均为 1.0，paletteKey 与默认一致）。

### 1.3 关键设计风险（高优先级）

1. **跃迁 UI 流程改造可能破坏 `nextLevel` 无参调用契约**：现有 `nextLevel()` 完全无参数，被 4 处调用：玩家 jumpToNextGalaxy / `__gameTest.resetRun` 间接（其实 resetRun 用 createObjective+seedEncountersForLevel，不调 nextLevel）/ 内部潜在的关卡推进。如果 v0.9.0 把 nextLevel 改成 `nextLevel(galaxyType)` 必传，会破坏所有内部调用。**必须**：保留 `nextLevel()` 无参签名（向后兼容），增加可选参数 `nextLevel(galaxyType = "emptyVoid")`，传入或缺省都正确写入 `state.run.currentGalaxyType`。同时**强烈推荐** handler hook 模式：在 `window.gameActions.jumpToNextGalaxy()` 内部分支（弹 galaxyMapPanel 或直接 nextLevel），不让 nextLevel 自己分支。
2. **`__gameTest.getObjectiveType(seed, level)` 必须保持 galaxyType-free 等价**：现有此钩子是纯函数，按 OBJECTIVE_LEVEL_WEIGHTS[lvl] 抽，不读 state.run.currentGalaxyType。若 v0.9.0 把 galaxyType 乘子直接塞到 OBJECTIVE_LEVEL_WEIGHTS 表里（**禁止**），会让此钩子在不同 galaxyType 下输出不同 type 类型，破坏全部 v0.4.0/v0.7.0/v0.8.0 已知 escort seed 穷举。**必须**：乘子作用在 `getObjectiveWeightTable(level)` 内部（state.run.currentGalaxyType 决定的运行时乘子），`__gameTest.getObjectiveType(seed, level, galaxyType?)` 新增可选第 3 参（默认 `emptyVoid` 乘子 1.0），等价 v0.8.0。
3. **`generateGalaxy` 的 paletteKey 决定渲染色调，与 currentGalaxyType 协同时序**：现有 `nextLevel()` 流水线是「先 applyGalaxy → 后 createObjective + seedEncountersForLevel」。v0.9.0 必须在 applyGalaxy **之前**把 `state.run.currentGalaxyType` 设置好（否则 generateGalaxy 拿不到 galaxyType 决定的 paletteKey 覆盖），但又必须在 nextLevel 入参的最早期赋值。**必须**：`nextLevel(galaxyType = "emptyVoid")` 函数第一行（`ensureRunRuntimeState` 之后、`state.run.level += 1` 之后、`generateGalaxy` 之前）写 `state.run.currentGalaxyType = galaxyType`；`generateGalaxy` 内部接受可选 `galaxyTypeOverride` 参数（或读 `state.run.currentGalaxyType`）决定 paletteKey。
4. **galaxyMapPanel 与 objectiveChoicePanel 的状态竞争**：现有任务完成后弹 `#objectiveChoicePanel`（含「跃迁下一星系」+「暂时停留」按钮）；v0.9.0 新流程是「玩家点击 jumpBtn → 弹 #galaxyMapPanel」。**风险**：两个 panel 同时显示，或玩家在 galaxyMapPanel 上点击「暂时停留」逻辑被绕过。**必须**：jumpToNextGalaxy handler 内部，先按 `state.run.currentGalaxyType === undefined`（首关 level 0 兜底）或 `levelIndex(state.run.level + 1) >= ENDGAME_LEVEL`（终末跳）直接 `nextLevel(warFront)`；其他情况弹 #galaxyMapPanel，**关闭 #objectiveChoicePanel**，玩家在 galaxyMapPanel 上点击候选卡片 → `confirmGalaxyJump(galaxyType)` → `nextLevel(galaxyType)`；Escape 关闭 galaxyMapPanel → 重新打开 #objectiveChoicePanel（awaitingObjectiveChoice 保持 true）。
5. **权重乘子叠加可能让某些 OBJECTIVE / ENCOUNTER type 完全消失**：现有 OBJECTIVE_LEVEL_WEIGHTS / ENCOUNTER_LEVEL_WEIGHTS 是绝对权重；如果 galaxyType 乘子 = 0（如 emptyVoid 把 ambush 乘 0），会让此 type 完全不能抽到。**必须**：数值方案约束乘子 ≥ 0.3（与 §"数值边界建议"一致），ADR 在 `applyGalaxyWeightMod(baseWeights, galaxyMod)` 内做 `Math.max(0, base * mod)` 保护，但**不要**额外底数兜底（保留乘子语义）。
6. **`hostileStationSpawnedThisGalaxy` 与 warFront ×2 assault 乘子冲突**：v0.6.0 设计：`hostileStationSpawnedThisGalaxy` 在跨关后 reset，本关内 assault 任务只能触发一次；v0.9.0 warFront galaxyType 让 hostile-station 概率 ×2。**风险**：v0.6.0 设计是按 OBJECTIVE 抽到 assault → 触发；如果 v0.9.0 把 hostile-station 概率乘子直接塞到 `getObjectiveWeightTable` 的 assault 项，会让 warFront galaxyType 的 assault 任务出现频率翻倍（与方向 §"hostile-station 触发乘子" 一致），符合预期。但**注意**：spawnHostileStation 还有其他触发路径（如 assault 失败后再触发），数值方案需明确此乘子仅影响 assault 任务**抽到的概率**，不影响 assault 任务**进行中** spawn 的 hostile-station 数量。
7. **路径预览 HUD 持久显示，遮挡视野风险**：右上角 7 个圆点 + 当前节点亮 + 已走节点淡；CSS 必须保证圆点直径 ≤ 14px、整体宽度 ≤ 220px、透明度 ≥ 0.6（保证可读性同时不抢戏）；不允许超出 HUD 资源条边界。
8. **Escape 取消 galaxyMapPanel 的事件冲突**：现有 Escape 已被 v0.4.0 / v0.5.0 用于关闭研发树 / 选择面板等；v0.9.0 必须**优先级最高**——galaxyMapPanel 打开时 Escape 立即关闭它（不冒泡到其他 Escape handler）。`event.stopPropagation() + event.preventDefault()` 显式处理。
9. **跨关 buffer 时间一致性**：现有 `state.spawnTimer = getLevelTransitionBuffer(state.run.level)` 在 nextLevel 末尾设置；v0.9.0 emptyVoid 不缩短 buffer（避免触发 spawnTimer / 跨关清理时序问题，确保 OBJECTIVE 跳跃后玩家有缓冲）。仅在 HUD 提示文字上做 galaxyType 显示差异。
10. **存档兼容 + ensureRunRuntimeState**：state.run.galaxyMap / currentGalaxyType / galaxyChoicesShown 全部走 runtime 路径，不进 SAVE_KEY；旧存档启动时 ensureRunRuntimeState 必须默认 `state.run.galaxyMap = { nodes: [], currentNodeId: null, pendingChoices: [] }`、`state.run.currentGalaxyType = "emptyVoid"`、`state.run.galaxyChoicesShown = false`。

  - **enemySpawnMod / resourceMod / hostileStationMod 默认 1.0**：未列出时（如 mysteryZone 无 hostileStationMod）按 1.0 处理。
- **回滚**：若节点 B 实测 10 字段冗余或某字段不必要（如 cssClass 与 paletteKey 重叠），可在 v0.9.1 重构；v0.9.0 范围保持完整字段集。

### 2.3 权重乘子作用时机（决定 3）

- **决定**：叠加式 `finalWeight = baseWeight × galaxyMod[type]`，乘子 ≥ 0，galaxyMod 默认 1.0 时**精确等价 v0.8.0**。
- **应用位置**：
  1. **OBJECTIVE 抽池**：在 `getObjectiveWeightTable(level)` 返回前应用乘子；`createObjective` 调用此函数后直接 pickWeighted。
  2. **ENCOUNTER 抽池**：在 `seedEncountersForLevel(level)` 内的 `baseWeights = ENCOUNTER_LEVEL_WEIGHTS[lvl]` 之后应用乘子。
  3. **敌人 spawn rate**：在 `scheduleNextSpawnTimer` 或 `spawnWave` 内（具体位置见 §2.7）应用 `enemySpawnMod` 调整间隔。
  4. **资源乘子**：在 4 个资源给予点（采矿 / enemy 击杀 / objective reward / encounter reward）inline 乘 `resourceMod`。
  5. **hostile-station 触发乘子**：仅作用于 `getObjectiveWeightTable` 内 `assault` 项，乘 `hostileStationMod`（warFront ×2，其他 1.0）。
- **`applyGalaxyWeightMod(baseWeights, galaxyMod)` 通用 helper**：
  ```js
  // 通用乘子叠加 helper：base × mod，base 缺失时按 1.0 兜底，乘子可缺失（默认 1.0），结果 ≥ 0
  function applyGalaxyWeightMod(baseWeights, galaxyMod) {
    if (!galaxyMod || typeof galaxyMod !== "object") return { ...baseWeights };
    const result = {};
    for (const [type, baseWeight] of Object.entries(baseWeights)) {
      const mod = galaxyMod[type];
      if (mod == null) {
        result[type] = baseWeight;
      } else {
        result[type] = Math.max(0, baseWeight * mod);
      }
    }
    return result;
  }
  ```
- **`getObjectiveWeightTable(level)` 改造**：
  ```js
  function getObjectiveWeightTable(level) {
    const baseWeights = {
      ...(OBJECTIVE_LEVEL_WEIGHTS[level] || OBJECTIVE_LEVEL_WEIGHTS[OBJECTIVE_LEVEL_WEIGHTS.length - 1])
    };
    if (state.run.hostileStationSpawnedThisGalaxy) {
      delete baseWeights.assault;
    }
    // v0.9.0：叠加 galaxyType 权重乘子；emptyVoid 全 1.0 等价 v0.8.0
    const galaxyType = state.run.currentGalaxyType || "emptyVoid";
    const galaxyDef = GALAXY_TYPES[galaxyType];
    if (!galaxyDef) return baseWeights;
    const modded = applyGalaxyWeightMod(baseWeights, galaxyDef.objectiveWeightMod);
    // hostile-station 触发乘子：warFront ×2 让 assault 任务出现频率翻倍
    if (modded.assault != null && Number.isFinite(galaxyDef.hostileStationMod)) {
      modded.assault = Math.max(0, modded.assault * galaxyDef.hostileStationMod);
    }
    return modded;
  }
  ```
- **`seedEncountersForLevel(level)` 改造**：
  ```js
  function seedEncountersForLevel(level) {

    if (availableTypes.length === 0) return;

    const rng = getEncounterRng(lvl);
    const targetCount = counts.min + Math.floor(rng() * (counts.max - counts.min + 1));

    // 后续逻辑与 v0.8.0 完全相同（cooldown 检查、makeEncounter、push 等）
    // ...
  }
  ```
- **理由**：
  - **叠加式不破坏 OBJECTIVE_LEVEL_WEIGHTS / ENCOUNTER_LEVEL_WEIGHTS 数值表**：现有 weight 表是 level 难度曲线基线，叠加 galaxyType 乘子让"按 level 决定整局节奏 + 按玩家选择微调当前关"两层叠加。
  - **`__gameTest.getObjectiveType(seed, level)` 等价**：钩子内部不读 state.run.currentGalaxyType，直接走 OBJECTIVE_LEVEL_WEIGHTS（决定 8 一致），所以同 seed 同 level 输出与 v0.8.0 完全相同。
  - **乘子可缺失**：未列出的 type 默认 1.0；emptyVoid 显式列出所有 type 全 1.0 是为了让"等价"语义在代码层可见。
  - **乘子 ≥ 0 保护**：避免负权重让 pickWeighted 异常；但数值方案约束乘子 ≥ 0.3 实际不会触发该保护，仅作为代码层安全网。
- **回滚**：若节点 B 实测某种 galaxyType（如 warFront）让 assault 任务出现频率过高（影响 v0.6.0 hostile-station 体验），数值方案微调 hostileStationMod ≤ 1.5；不需要架构改动。

### 2.4 跃迁流程改造点（决定 4）

- **决定**：在 `window.gameActions.jumpToNextGalaxy()` handler 内部 hook（**不改 `nextLevel()` 的 API 兼容性**），保留 `nextLevel(galaxyType = "emptyVoid")` 可选参数签名供 confirmGalaxyJump 调用。

  ```
    ├─ state.run.galaxyMap.pendingChoices 保留（玩家可再次点 jumpBtn 重新生成）

    ├─ 重新打开 #objectiveChoicePanel
    └─ updateHud()
  ```
- **`nextLevel(galaxyType = "emptyVoid")` 改造**：

### 2.6 取消跃迁的状态恢复（决定 6）

- **决定**：Escape 取消后 `state.run.awaitingObjectiveChoice` **保持 true**（不修改），玩家可再次点击 jumpBtn 重新弹出 galaxyMapPanel；同时 `state.run.galaxyChoicesShown = false`（重新生成候选）；`state.run.galaxyMap.pendingChoices` 保留（玩家心理预期"刚才看到的候选还在"）。
- **完整 cancelGalaxyJump 实现**：

  ```js
    state.run.galaxyChoicesShown = false;

    // awaitingObjectiveChoice 保持 true（不修改）
    // objectiveChoiceDismissed 不修改（玩家自己点暂时停留才会触发 dismissed）
    updateObjectiveChoiceUi();
    updateHud();
  }
  ```
- **Escape 全局监听**：
  ```js
  document.addEventListener("keydown", (event) => {
    if (event.key !== "Escape") return;
    const panel = document.getElementById("galaxyMapPanel");
    if (!panel || panel.classList.contains("hidden")) return;
    event.stopPropagation();
    event.preventDefault();
  ```

- **`window.gameActions.jumpToNextGalaxy()` 重新弹出**：
  ```js

  // galaxyChoicesShown=false → 重新走完整流程 → 生成候选（可能同 seed 同 level 仍是同一组）→ 弹 #galaxyMapPanel
  ```
- **理由**：
  - **保持 awaitingObjectiveChoice=true**：玩家任务已完成，跃迁 pending 状态合理；与 v0.4.0 "暂时停留" 后 awaitingObjectiveChoice=false + dismissed=true 的语义区分开（暂时停留是放弃跃迁继续在本关玩；Escape 取消是延迟决策）。
  - **重置 galaxyChoicesShown=false**：让 jumpBtn 能再次触发流程；同时保留 pendingChoices 让数据连续性更好。
  - **pendingChoices 保留 + 再次点 jumpBtn 时同 seed 重新生成**：玩家不会"突然看到不同的候选列表"，体验稳定。

### 2.9 跨关 buffer 时间是否调整（决定 9）

- **决定**：**保持现有 buffer 时间不变**（`state.spawnTimer = getLevelTransitionBuffer(state.run.level)`），不为 emptyVoid 缩短 buffer。
- **理由**：
  - **避免触发 spawnTimer / 跨关清理时序问题**：现有 buffer 是给玩家"切换关卡时整理舰船 / 重新部署 turret / 缓冲一下"的体验窗口；缩短 buffer 会让 emptyVoid 关卡"敌人立即来"，可能让"低风险"变成"措手不及"。
  - **方向 §G7 emptyVoid 提到"跨关 buffer ×0.7（更快跃迁）"**：但仔细分析，方向作者所说的"跨关 buffer"在玩家心理上是"切关动画 / 等待时间"，与 `state.spawnTimer`（敌人首次 spawn 等待时间）不完全等同。spawnTimer 影响**关内**节奏，与"跃迁速度"无关。
  - **HUD 层做 galaxyType 提示足够**：在 nextLevel 后 `showToast` 中显示当前 galaxyType 名称（如"跃迁完成：已进入空旷星域。事件密度减半，敌人减少。"），玩家心理预期"低风险关卡"不需要靠时间缩短实现。
- **若数值方案后续要求调整**：可在 `getLevelTransitionBuffer(level)` 内追加 `* galaxyDef.spawnDelayMod`（默认 1.0），但目前不做。
- **回滚**：若节点 C 实测玩家觉得 emptyVoid 关卡 buffer 偏长，数值方案微调；不需要架构改动。

## 4. 关键算法伪代码

### 4.1 `generateGalaxyChoices(targetLevel, rng)` 候选生成

```js
function generateGalaxyChoices(targetLevel, rng) {
  const lvl = levelIndex(targetLevel);
  // 边界：level 0 起点 / level 6 终末固定
  if (lvl <= 0) return [{ galaxyType: "emptyVoid", reason: "start" }];
  if (lvl >= ENDGAME_LEVEL) return [{ galaxyType: "warFront", reason: "endgame" }];

  const candidateCount = GALAXY_CANDIDATE_COUNT[lvl] || 2;
  const levelBias = GALAXY_LEVEL_BIAS[lvl - 1] || GALAXY_CHOICE_BASE_WEIGHTS;

  // 收集所有可用 galaxyType（排除 emptyVoid / warFront；它们是固定起终点）
  const allTypes = Object.keys(GALAXY_TYPES).filter((t) => t !== "emptyVoid" && t !== "warFront");

  // 排除当前 galaxyType（避免连续两关同类型）
  let availableTypes = allTypes.slice();
  if (state.run.currentGalaxyType && state.run.currentGalaxyType !== "emptyVoid") {
    availableTypes = availableTypes.filter((t) => t !== state.run.currentGalaxyType);
  }

  // 按 risk × levelBias 计算每个 type 的最终权重
  const weights = {};
  for (const t of availableTypes) {
    const risk = GALAXY_TYPES[t].riskLevel || "mid";
    const baseW = GALAXY_CHOICE_BASE_WEIGHTS[risk] || 0.3;
    const biasW = levelBias[risk] || 1.0;
    weights[t] = baseW * biasW;
  }

  // 抽 candidateCount 个候选（不放回；避免同关重复同 type）
  const choices = [];
  for (let i = 0; i < candidateCount && Object.keys(weights).length > 0; i++) {
    const picked = pickWeightedGalaxyType(rng, weights);
    if (!picked) break;
    choices.push({ galaxyType: picked, reason: "random" });
    delete weights[picked];
  }
  return choices;
}

function pickWeightedGalaxyType(rng, weightTable) {
  const entries = Object.entries(weightTable).filter(([, w]) => Number.isFinite(w) && w > 0);
  if (!entries.length) return null;
  const total = entries.reduce((sum, [, w]) => sum + w, 0);
  let roll = rng() * total;
  for (const [type, w] of entries) {
    roll -= w;
    if (roll <= 0) return type;
  }
  return entries[entries.length - 1][0];
}
```

```js
function applyGalaxyWeightMod(baseWeights, galaxyMod) {
  if (!galaxyMod || typeof galaxyMod !== "object") return { ...baseWeights };
  const result = {};
  for (const [type, baseWeight] of Object.entries(baseWeights)) {
    const mod = galaxyMod[type];
    result[type] = mod == null ? baseWeight : Math.max(0, baseWeight * mod);
```
