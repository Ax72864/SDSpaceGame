## 1. 体验目标

v0.11.0 的 UI 目标是让空存档玩家在第一局走完最小闭环：

1. 知道先建立采矿能力。
2. 知道为什么要靠近带彩色外环的资源点。
3. 知道当前星系任务如何通过局内行动完成。
4. 完成任务后知道可以跃迁，也知道可以暂时停留。
5. 失败或通关后知道本局结果、获得了什么、下一步在哪里。
6. 第一次打开 Meta 时知道这是局外成长，用于让下一局更强或更稳定。
7. 选择开局协议时知道协议只影响下一局，不会改写当前局。

UI 优先级：

| 优先级 | 信息 | 承载位置 | 原因 |
| --- | --- | --- | --- |
| P0 | 当前目标与下一步 | `guideBox` | 首局行动入口，必须单一清楚 |
| P0 | 资源外环与采矿站关系 | `resourceGuide` + 建造 hint | 玩家理解采集方式的关键 |
| P0 | 任务完成后的跃迁选择 | `objectiveChoicePanel` / `galaxyMapPanel` | 闭合局内推进 |
| P0 | 结算结果、局外天赋点、下一步入口 | `runSettlementPanel` | 闭合局内到局外 |
| P0 | Meta 点数、可购买、协议只影响下一局 | `metaPanel` / `metaPanelToggle` | 建立跨局成长意图 |
| P1 | Esc / 关闭说明 | 主面板标题区 / footer | 降低卡住感 |
| P1 | 小屏滚动和按钮可触达 | CSS / DOM 状态 | 保证真实浏览器路径可走 |
| P2 | 装饰动效、路线颜色、扫光 | 现有样式增强 | 只服务识别，不抢可读性 |

## 2. 首局引导文案结构

### 2.1 总体结构

`guideBox` 保持双行结构，但每个阶段只出现一个主目标和一个下一步，不同时抛出多个同级任务。

```text
当前目标：建立采矿能力
下一步：选择“采矿站”，放在矿石或金属资源点旁；驾驶工站进入彩色外环即可采集。
```

建议将首局提示拆成以下节点，按玩家进度替换，不需要长教程弹窗：

| 节点 | 触发状态 | `currentGoal` 建议 | `nextStep` 建议 |
| --- | --- | --- | --- |
| 建造采矿站 | 空存档 / 首局早期 / 未建立稳定采集 | `先建立采矿能力` | `选择“采矿站”，放在矿石或金属资源点旁。采矿站会帮助你持续获得建造资源。` |
| 靠近资源外环 | 玩家尚未进入资源点外环或资源采集不足 | `靠近带彩色外环的资源点` | `橙色外环产矿石，银色外环产金属；驾驶工站进入外环后，采矿站和工站采集更有效。` |
| 完成任务 | 任务条件未完成 | `完成当前星系任务` | `根据任务提示补齐采集、建造或生存目标；完成后会出现跃迁选择。` |
| 任务完成 | `objectiveChoicePanel` 可见 | `星系任务已完成` | `可以跃迁下一星系，也可以暂时停留整理防御和资源。` |
| 跃迁 | `galaxyMapPanel` 可见 | `选择下一跳目的地` | `比较候选星系的奖励和风险；按 Esc 或“暂时停留”可返回当前星系。` |
| 结算 | `runSettlementPanel` 可见 | `查看本局结算` | `确认本局结果、获得的局外天赋点，以及下一步是新局、Meta 购买或协议选择。` |
| 购买 Meta | 结算后 / 首次打开 Meta | `使用局外天赋点强化下一局` | `购买可用天赋会让下一局更强或更稳定；局内科研点不会变成局外天赋点。` |
| 选择协议 | Meta 协议页 / 新局前 | `选择下一局开局协议` | `协议只影响下一局开局。未解锁协议会显示需要的局外天赋。` |

### 2.2 `resourceGuide` 文案结构

`resourceGuide` 应服务“靠近资源外环”这件事，不要变成百科：

- 第一行：当前最近或最相关资源点，例如 `附近资源：矿石外环，适合补采矿站成本。`
- 第二行：行动建议，例如 `把采矿站放在外环旁，并驾驶工站进入外环。`
- 第三行可选：颜色说明，例如 `橙=矿石，银=金属，绿=气体，紫=等离子。`

当玩家已经稳定采集后，`resourceGuide` 可以降级为简短状态，不再持续抢主提示：

```text
资源采集已建立：继续完成星系任务，或补充防御后准备跃迁。
```

### 2.3 结算与 Meta 首次解释

结算面板应在结果之后、主按钮之前保留 `局外成长` 区块：

```text
局外成长
本局获得局外天赋点：+3
当前可用局外天赋点：5
下一步：打开局外天赋，购买能让下一局更稳定的节点。
```

失败和通关只调整语气，不改变结构：

- 失败：`这局仍积累了局外天赋点，可以强化下一局再挑战。`
- 通关：`通关奖励已转化为局外天赋点，可以规划下一条路线。`

Meta 面板首次说明使用短句：

```text
局外天赋点来自结算，可购买永久天赋；局内科研点只在本局用于研发，两者互不转换。
```

## 3. 多面板互斥视觉规则

### 3.1 主流程面板定义

本版本纳入互斥管理的主流程面板：

- `runSettlementPanel`
- `galaxyMapPanel`
- `metaPanel`
- `objectiveChoicePanel`

推荐视觉优先级：

| 优先级 | 面板 | 原则 |
| --- | --- | --- |
| 1 | `runSettlementPanel` | 结算最高优先，打开期间不被 Meta 或星图覆盖关键结果 |
| 2 | `galaxyMapPanel` | 星图是跃迁确认层，打开时关闭 Meta，隐藏右侧任务选择提示 |
| 3 | `metaPanel` | Meta 是局外成长抽屉，打开时关闭星图；若结算仍在，应作为结算入口的后续层，不盖住结算结果 |
| 4 | `objectiveChoicePanel` | 任务完成后的轻量选择，可被星图替代或隐藏 |

### 3.2 打开和关闭规则

同一时间最多展示一个主流程大面板。`objectiveChoicePanel` 是右侧内嵌选择层，若 `galaxyMapPanel` 或 `runSettlementPanel` 打开，应隐藏或降级为状态文案。

打开行为建议：

- 打开 `runSettlementPanel`：关闭 `galaxyMapPanel`，关闭 `metaPanel`，隐藏 `objectiveChoicePanel`。
- 打开 `galaxyMapPanel`：关闭 `metaPanel`，隐藏 `objectiveChoicePanel`，不关闭结算；若结算已打开则阻止星图打开。
- 打开 `metaPanel`：关闭 `galaxyMapPanel`；若 `runSettlementPanel` 正在显示，Meta 应从结算按钮进入，并提供清楚返回路径。
- 打开 `objectiveChoicePanel`：仅在没有结算、星图、Meta 大面板时展示。

关闭行为建议：

- `runSettlementPanel`：关闭按钮应是明确动作，例如 `开始新局`、`留在终结星系`，不建议只靠 Esc 关闭结算。
- `galaxyMapPanel`：关闭按钮文案 `暂时停留（Esc）`，关闭后回到当前星系，`objectiveChoicePanel` 显示 `任务已完成，可随时跃迁`。
- `metaPanel`：关闭按钮文案 `关闭（Esc）`，关闭后焦点回到 `metaPanelToggle` 或结算内的 `查看局外天赋` 按钮。
- `objectiveChoicePanel`：Esc 等同 `暂时停留`，不确认跃迁。

### 3.3 Esc 文案和焦点反馈

每个主面板都应在标题区或 footer 明示 Esc 行为：

| 面板 | Esc 文案 | 关闭后 |
| --- | --- | --- |
| `galaxyMapPanel` | `Esc：暂时停留，不消耗跃迁选择` | 回到右侧任务完成提示 |
| `metaPanel` | `Esc：关闭局外天赋面板` | 回到打开前入口 |
| `objectiveChoicePanel` | `Esc：暂时停留` | 保留任务完成态 |
| `runSettlementPanel` | `Esc 不会开始新局；请使用按钮选择下一步` | 不触发不可逆动作 |

焦点和状态建议：

- 打开主面板时给 `body` 或 `#hud` 加当前状态 class，例如 `.ui-main-panel-open` 和 `.ui-panel-galaxy-open`。
- 主面板根节点加 `.is-open`、`.is-top-panel`，关闭中可加 `.is-closing`。
- 被互斥隐藏的面板不要只靠 opacity，应加 `hidden` 或 `aria-hidden="true"`，避免键盘焦点进入。
- 关闭后焦点回到触发按钮，避免 Tab 跳到不可见按钮。

## 4. 小屏 / 右侧面板策略

### 4.1 断点与布局

沿用当前 `<760px` 断点，但建议补充更明确的主面板策略：

| 视口 | 策略 |
| --- | --- |
| `>= 1024px` | 右侧状态面板常驻；Meta 抽屉右侧滑入；星图居中 overlay；结算在右侧或居中均可，但必须可读 |
| `761px - 1023px` | 右侧状态面板宽度收窄并限制高度；Meta 抽屉可占更大宽度；星图卡片减少列数 |
| `<= 760px` | 右侧状态面板变成半高滚动卡；Meta 抽屉全屏；星图和结算使用可滚动 modal；底部建造区保留可触达 |
| `<= 420px` | 主按钮区域 sticky；候选卡单列；说明文案折叠或降级；禁止出现横向溢出 |

### 4.2 Meta 抽屉

桌面：

- 保持当前右侧抽屉思路，宽度不超过 `min(760px, calc(100vw - 320px))`。
- 顶部保留标题、可用局外天赋点、关闭按钮。
- 内容区滚动，footer 不挤占主要购买卡片。

小屏：

- `.meta-panel-drawer` 全屏，`top: 0; width: 100%; bottom: 0;`。
- header 固定，body 滚动，购买按钮和协议选择按钮高度不低于 `40px`。
- tabs / category chips 可横向滚动，但卡片内容不能横向滚动。
- 关闭按钮固定在右上角，文案或 `aria-label` 明确 `关闭局外天赋面板`。

信息优先级：

1. 当前可用局外天赋点。
2. 可购买项 / 推荐项。
3. 当前协议和下一局影响。
4. 未解锁原因。
5. 版本说明、重置说明、长解释。

### 4.3 结算面板

结算面板应避免被右侧状态面板高度限制截断：

- 桌面可继续作为右侧 panel 内区块，但内容超过高度时必须滚动。
- 小屏建议升级为 modal / drawer 样式，`max-height: calc(100vh - 24px)`，内部滚动。
- 主操作区建议 sticky bottom，至少保留 `开始新局`、`留在当前星系 / 返回`、`查看局外天赋` 中当前可用的主动作。
- 结算的局外成长区块必须在按钮前可见，若空间不足则折叠详细推荐，不折叠点数和下一步。

### 4.4 星图候选卡

桌面：

- 候选卡继续使用多列 wrap。
- 每张卡优先显示名称、风险、奖励 / 代价摘要、确认动作。
- 过长的描述和 foresight 列表进入卡内滚动或折叠。

小屏：

- 候选卡单列或横向 snap 二选一；更推荐单列，降低误选风险。
- `.galaxy-map-content` 使用 `width: calc(100vw - 16px)`、`max-height: calc(100vh - 16px)`、`overflow-y: auto`。
- `.galaxy-map-card` 不使用固定 `min-height: 320px`，改为内容自适应，保留 `min-height` 只到可读下限。
- footer 固定或 sticky，`暂时停留（Esc）` 可触达。

小屏卡片保留信息：

1. 当前候选名称。
2. 风险等级。
3. 奖励 / 惩罚摘要。
4. 确认按钮或整卡点击说明。

可降级信息：

- 长描述。
- 详细 foresight 列表。
- 装饰图标和额外分隔线。

### 4.5 右侧状态面板与遮挡规避

当前 `status-panel` 在小屏下位于右上半屏并可滚动，应继续保留，但需要控制遮挡：

- 小屏首局 `guideBox` 和 `resourceGuide` 保持可见，`moreStatusPanel` 默认收起。
- 当 `galaxyMapPanel`、`metaPanel`、`runSettlementPanel` 打开时，右侧状态面板应隐藏、变暗或被 `aria-hidden`，避免视觉抢焦点。
- `build-panel` 底部建造按钮不能被右侧状态面板覆盖；若视口过低，优先压缩说明文字而不是压缩按钮高度。
- 顶部资源 HUD 可换行，但资源数字不能被主面板标题遮挡。

## 5. 文案原则

### 5.1 术语边界

必须严格区分：

| 系统 | 推荐文案 | 避免文案 |
| --- | --- | --- |
| 局外 Meta | `局外天赋`、`局外天赋点`、`永久天赋`、`下一局更强` | `局外科研点`、`研发天赋`、`本局科研点可购买 Meta` |
| 局内研发 | `局内科研点`、`本局研发`、`本局内生效` | `局外天赋点`、`永久科研`、`跨局研发点` |
| 星系图 | `跃迁`、`下一跳`、`星系候选`、`暂时停留` | `协议选择`、`天赋路线` |
| 协议 | `开局协议`、`下一局生效`、`当前局不变更` | `立即生效`、`当前局协议重置` |

点数解释统一：

- `局外天赋点`：结算获得，跨局保存，用于购买 Meta。
- `局内科研点`：本局获得，本局研发使用，不作为 Meta 货币。

### 5.2 普通玩家与验收辅助隔离

不要把验收辅助暴露给普通玩家：

- 不在默认 UI 显示 `snapshot`、`__gameTest`、DOM 检查、localStorage、console error 等字样。
- debug / playtest 信息必须隐藏在测试钩子、query 参数、控制台方法或 debug 开关下。
- 普通文案只写玩家能理解的行动和结果，例如 `查看局外天赋`，不写 `读取 meta snapshot`。
- 验收失败提示若需要 UI 呈现，应只在 debug 模式显示；默认玩家路径用自然错误文案，例如 `暂时无法读取局外天赋，请刷新后重试`。

### 5.3 语气

- 首局：短句、行动导向，不讲系统史。
- 失败：承认失败但强调仍有成长。
- 通关：强调奖励已进入局外成长。
- 未解锁：说明差什么，不只说不可用。
- Esc：说明会发生什么，尤其强调不会触发不可逆动作。

## 6. DOM / CSS 建议

### 6.1 状态 class

建议集中维护当前主流程面板状态：

| class | 挂载位置 | 用途 |
| --- | --- | --- |
| `.ui-main-panel-open` | `body` 或 `#hud` | 任一主流程面板打开，压低非焦点 HUD |
| `.ui-panel-settlement-open` | `body` 或 `#hud` | 结算优先，隐藏星图 / Meta 冲突入口 |
| `.ui-panel-galaxy-open` | `body` 或 `#hud` | 星图打开，隐藏任务选择内嵌块 |
| `.ui-panel-meta-open` | `body` 或 `#hud` | Meta 打开，冻结右侧状态面板焦点 |
| `.ui-panel-objective-open` | `body` 或 `#hud` | 任务选择可见，提示 Esc 暂时停留 |
| `.is-top-panel` | 当前面板根节点 | 当前视觉焦点 |
| `.is-muted-behind-panel` | 背后 HUD / panel | 被主面板压低但不销毁 |
| `.is-scrollable` | modal content / drawer body | 明确可滚动区域 |
| `.has-sticky-actions` | 面板根节点 | 小屏固定按钮区 |

### 6.2 DOM 属性与焦点

推荐补齐：

- `metaPanel`、`galaxyMapPanel`、结算 modal 化后的 `runSettlementPanel` 使用 `role="dialog"`、`aria-modal="true"`、`aria-labelledby`。
- `guideBox` 中当前目标或下一步可使用 `aria-live="polite"`，但避免每帧刷新造成读屏噪音。
- `resourceGuide` 和 `statusAlerts` 继续使用 `aria-live="polite"`，仅在文本变化时更新。
- 主面板关闭按钮使用可读 label，例如 `aria-label="关闭局外天赋面板"`。
- 打开主面板后焦点进入标题或第一个可操作按钮；关闭后回到触发按钮。
- 隐藏互斥面板时同步 `hidden` class 和 `aria-hidden="true"`。

### 6.3 小屏 media query 建议

建议在现有 `@media (max-width: 760px)` 基础上补充：

```css
@media (max-width: 760px) {
  .ui-main-panel-open .status-panel {
    visibility: hidden;
  }

  .galaxy-map-content,
  .run-settlement-panel.is-modal {
    width: calc(100vw - 16px);
    max-height: calc(100vh - 16px);
    overflow-y: auto;
  }

  .galaxy-map-cards {
    flex-direction: column;
    align-items: stretch;
  }

  .galaxy-map-card {
    width: 100%;
    min-height: auto;
    max-height: none;
  }

  .panel-actions-sticky {
    position: sticky;
    bottom: 0;
    background: rgba(8, 18, 36, 0.96);
  }
}
```

注意：以上是实现建议，不要求本 UI 方案直接改 `Game/style.css`。

### 6.4 视觉层级与颜色

- 继续使用现有深蓝透明玻璃风格，避免新增大色系。
- 可购买 / 可行动使用青蓝高亮；局外天赋点使用暖金；失败 / 危险使用红橙。
- 状态不能只靠颜色传达，必须有文字标签，例如 `可购买`、`点数不足`、`已满级`、`风险高`。
- 遮罩透明度在桌面约 `0.55 - 0.7`；小屏可略高，保证文字压住背景战斗特效。
- 动效控制在 `120ms - 220ms`，不做长时间闪烁。

## 8. UI 验收 / 回归点

以下回归点供 PM、实现成员和审核成员逐条检查，不少于 25 条：

1. 空存档首次进入游戏后，`guideBox` 的主目标优先说明建立采矿能力。
2. 首局 `nextStep` 明确提示选择采矿站并靠近矿石 / 金属资源外环。
3. `resourceGuide` 至少说明一个资源外环颜色与资源类型关系。
4. 玩家进入资源外环或建立稳定采集后，资源提示会降级，不持续压过任务目标。
5. 星系任务未完成时，任务文案能指向局内行动，不只显示抽象条件。
6. 星系任务完成后，`objectiveChoicePanel` 明确提供跃迁和暂时停留两个方向。
7. `objectiveChoicePanel` 中 Esc 文案等同暂时停留，不暗示确认跃迁。
8. 打开 `galaxyMapPanel` 后，`objectiveChoicePanel` 不再同时抢占视觉焦点。
9. `galaxyMapPanel` 标题或副文案说明 Esc / 暂时停留不会消耗跃迁选择。
10. 星图候选卡显示名称、风险、奖励 / 代价摘要和确认动作。
11. 小屏下星图候选卡可滚动，确认动作和暂时停留按钮可触达。
12. 打开 `metaPanel` 时，`galaxyMapPanel` 不应继续显示在其后形成双 overlay。
13. 打开 `galaxyMapPanel` 时，`metaPanel` 不应继续遮挡候选卡或取消按钮。
14. 打开 `runSettlementPanel` 时，Meta 和星图不会盖住结算结果。
15. 结算面板显示本局结果、获得的局外天赋点和当前可用局外天赋点。
16. 失败结算说明失败后仍可通过局外天赋点强化下一局。
17. 通关结算说明通关奖励已进入局外成长或下一步规划。
18. 结算后的主动作不会因 Esc、重复点击或关闭按钮误触发新局。
19. 结算面板提供进入 Meta 或开始新局的明确下一步，具体入口以实现范围为准。
20. `metaPanelToggle` 文案使用 `局外天赋`，不误写为局内研发。
21. Meta 面板顶部说明 `局外天赋点` 与 `局内科研点` 互相独立。
22. Meta 购买按钮的不足原因写为 `局外天赋点不足`，不写 `科研点不足`。
23. 协议页说明开局协议只影响下一局，当前局不变更。
24. 未解锁协议显示需要的局外天赋或条件，不只显示 `未解锁`。
25. 普通玩家默认 UI 不出现 `__gameTest`、`snapshot`、localStorage、DOM 检查等验收辅助字样。
26. debug / playtest 辅助即使存在，也必须通过测试通道读取，不占用默认 HUD。
27. 每个主流程面板都有可见关闭按钮或明确的下一步按钮。
28. 主流程面板打开后，Tab 焦点不会进入被隐藏的互斥面板。
29. 关闭 `metaPanel` 后焦点回到 `metaPanelToggle` 或结算中的 Meta 入口。
30. 关闭 `galaxyMapPanel` 后，任务完成态仍保留，玩家可再次选择跃迁。
31. 连续打开 Meta、星图、结算、任务选择，不出现两个以上主流程大面板同时遮挡主视野。
32. 连续按 Esc 不触发购买、确认跃迁、选择协议、新局开始等不可逆动作。
33. 小屏 375x667 下，首局主提示、资源提示和当前任务至少可通过滚动读到。
34. 小屏 375x667 下，Meta 抽屉正文可滚动，购买按钮可触达。
35. 小屏 375x667 下，结算内容可滚动，主操作按钮可见或可通过明确滚动触达。
36. 小屏 375x667 下，星图候选卡不被右侧状态面板长期遮挡。
37. 小屏下右侧 `moreStatusPanel` 默认不抢占首局引导和任务区域。
38. 顶部资源 HUD 在桌面和小屏下仍能读到关键资源数字。
39. 底部建造按钮在小屏下不被右侧状态面板或主面板按钮区遮挡。
40. 可购买、未解锁、已满级、风险高等状态均有文字标签，不只依赖颜色。
41. 面板遮罩不会阻挡当前顶层面板按钮点击。
42. `aria-live` 区域只在文案变化时更新，不因每帧刷新产生读屏噪音。
43. 主面板隐藏时同步 `hidden` / `aria-hidden`，避免键盘进入不可见内容。
44. 桌面宽屏布局不因小屏样式回归导致 Meta 抽屉过窄或星图卡片错位。
45. 控制台无因空 DOM、焦点返回或隐藏面板状态造成的报错。
46. 旧存档进入后，缺少新增 UI 状态字段不导致面板打不开。
47. 局外天赋点、局内科研点和资源 HUD 数字在文案中不混名。
48. v0.9 星系图路径、v0.10 Meta 面板和 v0.5 局内研发的文案边界不回潮。
49. UI 验收辅助不改变 `SAVE_KEY`、不写入 localStorage、不修改普通玩家流程。
50. 有头浏览器人工体验能完成 `采矿目标 -> 跃迁 / 结算 -> 购买 Meta 或选择协议 -> 新局` 的最小路径，并记录是否看懂每一步。
