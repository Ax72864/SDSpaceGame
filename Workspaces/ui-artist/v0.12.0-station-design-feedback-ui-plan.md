## 1. 体验目标

v0.12.0 的 UI 目标是让玩家从“能放建筑”升级为“能理解自己的空间站设计是否合理”。玩家打开建造面板、准备放置设施、选中设施或查看右侧状态时，应能回答：

1. 这个设施属于采矿、供电、推进、武器、防御、维修还是支撑。
2. 当前造不造得起，缺什么资源，建造后会不会造成电力压力。
3. 采矿设施是否靠近资源外环，武器 / 护盾 / 维修设施的作用边界是否可读。
4. 当前空间站最明显的 1-2 个短板是什么。
5. 什么时候需要立即处理，什么时候只是普通低效。

信息优先级：

| 优先级 | 信息 | 承载位置 | 原因 |
| --- | --- | --- | --- |
| P0 | 不可负担原因、断电、未连接核心 | 建造卡 / selected panel / guide | 直接阻断玩家行动 |
| P0 | 设施角色、成本、电力、用途、已建数量 | 建造卡 | 帮助比较下一步建造选择 |
| P0 | 电力、采矿、推进、火力、防御、维修短板 | 右侧健康摘要 | 让玩家不用逐个点设施也能读懂设计状态 |
| P1 | 采矿距离、资源外环、武器射线、覆盖范围 | 放置预览 / selected panel | 帮助玩家理解位置和边界 |
| P1 | 推荐 / 警告短句 | 建造卡 / 健康摘要 | 给出方向，但不替玩家决策 |
| P2 | 颜色、图标、轻动效、装饰边框 | CSS / DOM class | 增强辨识度，不能压过文字 |

## 2. 建造面板设施卡

### 2.1 卡片结构

建造入口从按钮列表升级为“可比较的设计选择”。每张设施卡建议使用以下固定信息顺序：

```text
[角色标签] 设施名称                         已建 x2
一句话用途：持续采集附近资源外环内的矿物。
成本：矿石 40 / 金属 20        电力：-2
状态：可建造 / 缺金属 8 / 建造后电力紧张
推荐：当前无采矿能力，优先补 1 座。
```

字段说明：

| 字段 | 必要性 | 显示规则 |
| --- | --- | --- |
| 角色标签 | 必须 | 单个主标签优先，必要时追加副标签，如 `采矿`、`供电`、`推进`、`武器`、`防御`、`维修`、`支撑` |
| 设施名称 | 必须 | 保持现有名称，不为了方案新增建筑名 |
| 已建数量 | 必须 | `已建 x0`、`已建 x2`；小屏可压缩为 `x2` |
| 核心用途 | 必须 | 一句话解释“为什么造它”，不解释完整规则 |
| 成本 | 必须 | 显示实际展示成本；有 Meta / 协议影响时展示变化后的结果 |
| 电力 | 必须 | 产出用 `+`，消耗用 `-`，无影响显示 `0` 或隐藏但测试字段保留 |
| 可负担状态 | 必须 | `可建造`、`资源不足`、`位置受限`、`缺电风险` 等短状态 |
| 推荐 / 警告文案 | 必须 | 最多 1 条主句，避免一张卡同时出现多个红色警告 |

### 2.2 角色标签与视觉语义

| 角色 | 颜色建议 | 图标语义 | 卡片短用途示例 |
| --- | --- | --- | --- |
| 采矿 | 青绿 / 矿物绿 | 镐、矿石、资源环 | `建立持续资源收入，靠近资源外环更有效。` |
| 供电 | 黄色 / 电弧蓝 | 闪电、电池 | `提高电力余量，降低关键设施断电风险。` |
| 推进 | 紫蓝 / 引擎蓝 | 喷口、箭头 | `提升移动与跃迁准备能力。` |
| 武器 | 橙红 | 准星、炮塔 | `提供火力，但需要电力与可读攻击边界。` |
| 防御 | 护盾蓝 | 盾牌、装甲板 | `提升基础防护，降低敌袭前风险。` |
| 维修 | 暖绿 | 扳手、十字 | `帮助受损设施恢复，关注覆盖范围。` |
| 支撑 | 中性灰蓝 | 节点、连接 | `补足连接、结构或辅助能力。` |

标签形态：

- 角色标签使用低饱和实底 + 高对比文字，不只依赖颜色。
- 卡片左上角只放主标签；副标签可以在用途行后以小 chip 展示。
- 推荐卡使用轻描边或小角标，不使用大面积闪烁。
- 警告卡只在关键风险时使用橙 / 红边框；普通不可负担使用灰化 + 缺口文案。

### 2.3 成本、电力与可负担状态

成本显示应优先回答“现在为什么能 / 不能造”：

| 状态 | 主文案 | 视觉 | 说明 |
| --- | --- | --- | --- |
| 可建造 | `可建造` | 正常亮度，按钮可用 | 成本和位置条件满足 |
| 资源不足 | `缺金属 8` | 成本缺口高亮，按钮禁用 | 多资源不足时显示最大缺口或最关键缺口 |
| 电力紧张 | `建造后电力 -1` | 橙色状态，不一定禁用 | 只提示风险，不改变建造规则 |
| 当前断电 | `当前电力不足，建后可能不可用` | 橙 / 红状态 | 若该设施依赖电力，应明确影响 |
| 位置待选 | `选择放置位置` | 中性提示 | 建造卡不替代放置诊断 |
| 已达软建议 | `已有 x3，先检查电力` | 中性 / 蓝灰 | 只是建议，不禁用 |

成本变化显示：

- 若 Meta / 协议 / 折扣影响成本，卡片显示实际成本，并可在成本旁加 `已受加成` 小标。
- 不展示未实现的“原价对比”动画，除非实现能保证与实际扣费一致。
- 可负担状态以实际展示成本为准，不能基于原始成本提示缺口。

### 2.4 推荐 / 警告文案

推荐文案服务“下一步优先级”，警告文案服务“当前关键风险”。每张卡最多显示一条。

推荐触发示例：

| 条件 | 文案示例 |
| --- | --- |
| 没有采矿能力 | `推荐：当前无采矿能力，先建立资源收入。` |
| 电力余量偏低 | `推荐：先补供电，避免关键设施断电。` |
| 无推进能力 | `推荐：跃迁前补推进能力。` |
| 有武器但无防御 | `推荐：已有火力，补一层基础防护更稳。` |
| 有受损设施且无维修 | `推荐：维修能力不足，受损后恢复慢。` |

警告触发示例：

| 条件 | 文案示例 |
| --- | --- |
| 资源不足 | `警告：缺金属 8，当前不能建造。` |
| 建造后核心能力断电 | `警告：建造后电力余量为负，武器可能断电。` |
| 敌袭前无武器 / 防御 | `警告：敌袭前缺少基础火力或防御。` |
| 采矿站全不可达 | `警告：现有采矿站未覆盖资源外环。` |

避免文案：

- 避免 `自动优化`、`一键修复`、`系统会帮你调整`。
- 避免绝对承诺，如 `这样一定安全`、`一定能打到敌人`。
- 避免把普通低效写成红色危机，如 `已有 1 个采矿站但效率一般` 不应刷红。

## 3. 设施选择 / 放置反馈

### 3.1 放置预览层级

准备放置设施时，画面上最多同时出现三层信息：

1. 主合法性：能否放置、是否连接核心、资源是否足够。
2. 角色相关预览：采矿距离 / 电力压力 / 武器射线 / 护盾或维修覆盖。
3. 轻量建议：更近资源外环、更稳电力、更清晰边界。

推荐结构：

```text
放置预览：采矿站
状态：可放置
距离：最近矿石外环 2 格，覆盖良好
电力：建造后余量 +1
建议：靠近外环边缘可更快建立采集链。
```

不可放置时：

```text
放置预览：采矿站
状态：不可放置
原因：未连接空间站核心
下一步：先放在核心相邻结构上，或补支撑设施。
```

### 3.2 采矿距离与资源外环

采矿反馈应让玩家理解“资源外环”而不是背公式：

- 资源外环用稳定描边或半透明环强调，颜色与资源类型一致。
- 准备建造采矿设施时，最近相关资源外环高亮；其他资源外环降级显示。
- 预览文案优先使用距离等级，不强依赖精确格数：`覆盖良好`、`距离偏远`、`未覆盖资源外环`。
- 若技术侧能提供格数，可显示 `最近矿石外环 2 格`；小屏可压缩为 `矿石外环：近`。

采矿状态文案：

| 状态 | 文案 | 视觉 |
| --- | --- | --- |
| 覆盖良好 | `覆盖资源外环，适合采集。` | 绿色 / 青色短线 |
| 距离偏远 | `距离资源外环偏远，采矿效率可能不足。` | 橙色虚线 |
| 未覆盖 | `未覆盖资源外环，当前采不到资源。` | 红色断线 / 警告 chip |
| 无相关资源 | `附近未发现可采资源外环。` | 灰色 |

### 3.3 电力压力

电力提示应区分“当前断电”和“建造后可能紧张”：

| 场景 | 承载位置 | 文案示例 |
| --- | --- | --- |
| 当前电力充足 | 建造卡 / 预览 | `电力：建后余量 +3` |
| 建造后接近 0 | 建造卡 / selected panel | `电力紧张：建后余量 0` |
| 建造后为负 | guide / status + 预览 | `建造后电力为负，关键设施可能断电。` |
| 已经断电 | status + selected panel | `当前电力不足，采矿 / 武器 / 防御可能受影响。` |

视觉建议：

- 电力数字使用 `+` / `-` 和颜色双编码。
- 电力压力不应把所有耗电卡刷红；只在核心能力受影响时升级为 guide / status。
- 卡片内电力信息固定位置，方便横向比较。

### 3.4 武器射线 / 火力

选中武器、导弹或准备放置攻击设施时，反馈重点是“是否可用”和“作用边界”：

- 用细线 / 扇形 / 边界框表示射线或作用范围，不使用高频闪烁。
- 可攻击状态用 `可用`、`缺电`、`边界受限`、`暂无目标` 四类短状态。
- 若没有精确射线可视化，至少在 selected panel 里说明 `武器方向 / 范围由现有战斗判定决定`，并提供可读状态。

文案示例：

| 状态 | 文案 |
| --- | --- |
| 可用 | `火力可用：当前有供电，进入射线范围的敌人可被攻击。` |
| 缺电 | `火力断电：补供电后才会参与战斗。` |
| 边界受限 | `射线受限：当前方向覆盖不足，敌人可能绕开火力。` |
| 暂无目标 | `暂无目标：武器就绪，等待敌人进入范围。` |

### 3.5 护盾 / 装甲 / 维修覆盖

防御与维修提示要避免让玩家误以为“有一个设施就完全安全”：

| 能力 | 可读提示 | 视觉 |
| --- | --- | --- |
| 护盾 | `基础护盾存在 / 护盾缺电 / 护盾覆盖不足` | 蓝色半透明圈或护盾 chip |
| 装甲 | `装甲结构存在 / 防御薄弱` | 深蓝 / 灰蓝边框 |
| 维修 | `维修覆盖内 / 未被维修覆盖 / 维修缺电` | 绿色虚线圈或扳手 chip |

selected panel 建议显示：

```text
防御：基础护盾存在，但当前电力紧张。
维修：该设施不在维修覆盖内。
建议：补供电或靠近维修设施，避免战后长期停摆。
```

## 4. 空间站设计健康摘要

### 4.1 摘要定位

健康摘要放在右侧状态或更多状态中，目标是快速暴露关键 1-2 个短板。它不是任务列表，不替玩家自动优化，也不展开完整百科。

推荐标题：`空间站设计`

桌面结构：

```text
空间站设计
[电力 正常] [采矿 风险] [推进 缺失]
[火力 可用] [防御 薄弱] [维修 缺失]
重点：采矿站未覆盖资源外环；跃迁前建议补推进。
```

小屏结构：

```text
设计：采矿风险 / 推进缺失
展开后显示 6 项短摘要
```

### 4.2 六类短项

健康摘要固定覆盖 6 类短项，每项只显示状态词 + 一句可选短解释。

| 类别 | 主要问题 | 状态词 | 文案示例 |
| --- | --- | --- | --- |
| 电力 | 余量、断电影响 | `充足` / `紧张` / `断电` | `电力断电：武器和采矿可能受影响。` |
| 采矿 | 是否有采矿、是否覆盖资源 | `缺失` / `偏远` / `有效` | `采矿风险：采矿站未覆盖资源外环。` |
| 推进 | 是否具备移动 / 跃迁准备能力 | `缺失` / `不足` / `就绪` | `推进缺失：跃迁前建议补推进设施。` |
| 火力 | 是否有武器、是否缺电、边界是否受限 | `缺失` / `断电` / `可用` | `火力断电：补供电后才能参与战斗。` |
| 防御 | 护盾 / 装甲基础能力 | `薄弱` / `基础` / `稳固` | `防御薄弱：敌袭前缺少基础防护。` |
| 维修 | 是否有维修、覆盖是否可读 | `缺失` / `未覆盖` / `覆盖` | `维修缺失：受损设施恢复能力不足。` |

### 4.3 视觉层级

健康摘要使用三层视觉，而不是把所有问题同等刷红：

| 层级 | 使用条件 | 视觉 | 示例 |
| --- | --- | --- | --- |
| 严重 | 核心能力不可用且影响当前行动 | 红 / 橙边框，进入 guide / status | `武器断电`、`采矿站全不可达` |
| 注意 | 即将影响战斗、跃迁或资源循环 | 橙色 chip，进入摘要重点 | `电力余量 0`、`防御薄弱` |
| 信息 | 普通状态或轻微低效 | 蓝灰 / 绿色 chip | `推进存在`、`维修覆盖内` |

摘要排序：

1. 当前阻断行动的问题，如资源不足、未连接核心、断电。
2. 当前目标相关的问题，如采矿链、敌袭、跃迁。
3. 其他普通短板。

摘要最多突出两条重点。第三条以后留在展开详情或 selected panel。

## 5. 低噪声提示优先级

### 5.1 guide / status 显示

`guideBox` / 右侧 status 只显示会影响当前关键行动的短提醒：

| 条件 | guide / status 文案示例 |
| --- | --- |
| 资源不足且玩家正在尝试建造 | `资源不足：缺金属 8，暂时不能建造采矿站。` |
| 放置位置未连接核心 | `无法放置：该位置未连接空间站核心。` |
| 全部采矿站未覆盖资源 | `采矿风险：现有采矿站未覆盖资源外环。` |
| 大量核心能力断电 | `电力风险：采矿 / 武器 / 防御中至少一项断电。` |
| 敌袭前无武器或防御 | `敌袭风险：当前缺少基础火力或防御。` |
| 跃迁前推进 / 防御明显缺口 | `跃迁前检查：推进或防御存在明显短板。` |

guide / status 不显示：

- 普通效率提示，如 `采矿距离不是最优`。
- 已经在 selected panel 中清楚展示的局部细节。
- 玩家没有触发相关行动时的长解释。
- 重复的卡片级缺口列表。

### 5.2 selected panel 显示

selected panel 承担局部诊断和边界解释：

- 选中设施：显示当前设施的角色、状态、供电、连接、覆盖或作用边界。
- 准备放置：显示当前位置合法性、资源外环距离、电力压力和角色相关预览。
- 选中资源 / 空格：显示附近资源外环、推荐设施类型或不可放置原因。

selected panel 可以显示更细的信息，但每组仍建议不超过 3 行：

```text
采矿诊断
最近资源外环：矿石，距离 2 格
状态：覆盖良好
建议：保持供电，继续完成采矿链。
```

### 5.3 不提示或降级显示

以下内容默认不主动提示：

- 轻微非最优布局，但不影响采矿、战斗、跃迁或维修。
- 玩家已经明确解决的问题，如补电后仍重复提示旧断电。
- 无法给出可信原因的推测性问题。
- 只服务开发调试的字段，如内部 id、坐标、helper 名称。
- 与当前版本无关的系统解释，如完整 Meta 规则、完整战斗公式。

## 6. 小屏 / 右侧面板兼容

### 6.1 布局原则

v0.12.0 增加的信息量较大，小屏必须优先保证操作可触达和问题可读：

| 视口 | 策略 |
| --- | --- |
| `>= 1024px` | 建造面板可多列；右侧健康摘要常驻；selected panel 展示完整诊断 |
| `761px - 1023px` | 建造卡压缩为 2 列或窄列；右侧摘要显示 6 个 chip + 重点 1 条 |
| `<= 760px` | 建造卡单列或横向滚动；右侧摘要默认折叠；selected panel 置于可滚动面板 |
| `<= 420px` | 只露出主状态和按钮；长用途折叠；成本 / 电力 / 状态必须保留 |

### 6.2 建造卡小屏降级

小屏保留：

1. 角色标签。
2. 设施名称。
3. 成本。
4. 电力。
5. 可负担状态。
6. 已建数量。

小屏可折叠：

- 长用途说明。
- 推荐理由的第二句。
- 成本变化来源说明。
- 装饰图标和副标签。

建议卡片压缩示例：

```text
[采矿] 采矿站        x0
矿石 40 / 金属 20    电力 -2
缺金属 8
```

### 6.3 右侧摘要与 selected panel

- 健康摘要在小屏默认显示 `设计：采矿风险 / 电力紧张`，点击或展开后显示 6 项。
- selected panel 与更多状态面板不能同时抢占底部建造区；打开 selected panel 时，健康摘要可以折叠为一行。
- 任何 overlay 打开时，建造按钮、关闭按钮和主要确认按钮都要保持可触达。
- 避免使用仅靠 hover 才能读到的信息；触屏必须能点击展开。

## 7. DOM / CSS 落地建议

本节只提供实现参考，不要求本方案直接修改 `Game/`。

建议状态 class：

| 用途 | class 示例 |
| --- | --- |
| 建造卡角色 | `.facility-card.role-mining`、`.facility-card.role-power` |
| 可负担状态 | `.is-affordable`、`.is-unaffordable`、`.has-power-risk` |
| 推荐 / 警告 | `.is-recommended`、`.has-critical-warning` |
| 放置预览 | `.placement-preview.is-valid`、`.placement-preview.is-invalid` |
| 健康摘要 | `.design-health-item.level-ok`、`.level-warning`、`.level-critical` |
| 小屏折叠 | `.is-compact`、`.is-expanded` |

建议可读字段：

- 建造卡 DOM 或只读输出应包含 `role`、`cost`、`powerDelta`、`affordable`、`reason`、`builtCount`、`recommendation`。
- 健康摘要 DOM 或只读输出应包含 `power`、`mining`、`propulsion`、`firepower`、`defense`、`repair`。
- selected diagnostics 应包含 `connection`、`resourceReachability`、`powerImpact`、`rangeOrCoverage`、`primaryReason`。

可访问性：

- 颜色状态必须配文字，如 `缺金属 8`，不能只变红。
- 禁用按钮保留原因说明，并可被 headless DOM 读取。
- 关键提示更新时可以使用轻量 `aria-live="polite"`，避免连续刷屏。

## 8. UI 回归点

### A. 建造面板设施卡

- [ ] A01. 空存档打开建造面板，采矿、供电、推进、武器、防御、维修、支撑角色能被区分。
- [ ] A02. 每张设施卡显示角色标签、设施名称、成本、电力和已建数量。
- [ ] A03. 采矿设施卡的一句话用途能说明与资源外环相关。
- [ ] A04. 供电设施卡显示电力产出或对电力余量的正向作用。
- [ ] A05. 武器设施卡显示火力角色，不暗示一定能打到所有敌人。
- [ ] A06. 防御设施卡能区分护盾 / 装甲或基础防护语义。
- [ ] A07. 维修设施卡显示维修用途，并提示关注覆盖范围。
- [ ] A08. 资源不足时，卡片在点击前显示缺口或不可负担原因。
- [ ] A09. 多资源不足时，卡片不刷多条红字，至少显示一个主缺口。
- [ ] A10. 建造折扣、Meta 或协议影响成本时，卡片显示变化后的实际成本。
- [ ] A11. 已建数量随建造后刷新，且小屏仍可见。
- [ ] A12. 推荐 / 警告文案每张卡最多显示一条主句，不挤占核心成本和状态。

### B. 放置与选择反馈

- [ ] B01. 准备建造采矿站时，最近资源外环被高亮或在 selected panel 中说明。
- [ ] B02. 采矿站放在资源外环附近时，反馈显示覆盖良好或等价文案。
- [ ] B03. 采矿站远离资源外环时，反馈显示距离偏远或未覆盖。
- [ ] B04. 附近无可采资源外环时，反馈明确说明附近未发现资源外环。
- [ ] B05. 准备放置在未连接核心的位置时，提示原因是未连接核心。
- [ ] B06. 资源不足导致不能建造时，不只表现为按钮无响应。
- [ ] B07. 建造后电力余量为负时，预览或卡片显示电力压力。
- [ ] B08. 当前断电设施被选中时，selected panel 显示缺电影响。
- [ ] B09. 选中武器设施时，能看到可用、缺电、边界受限或暂无目标状态之一。
- [ ] B10. 武器射线 / 范围视觉不遮挡核心按钮和资源 HUD。
- [ ] B11. 选中护盾或防御设施时，玩家能识别当前是否具备基础防护。
- [ ] B12. 选中维修设施或受损设施时，能区分维修覆盖内 / 未覆盖 / 缺电。

### C. 健康摘要与低噪声提示

- [ ] C01. 右侧健康摘要包含电力、采矿、推进、火力、防御、维修 6 类短项。
- [ ] C02. 健康摘要默认突出最关键的 1-2 个短板，而不是显示长列表。
- [ ] C03. 电力充足、紧张、断电三类状态在视觉和文字上可区分。
- [ ] C04. 采矿缺失、偏远、有效三类状态在摘要中可读。
- [ ] C05. 推进缺失或不足时，摘要提示跃迁前检查。
- [ ] C06. 火力缺失、断电或可用状态不互相混淆。
- [ ] C07. 防御薄弱时，摘要不承诺“安全”，只提示基础防护不足。
- [ ] C08. 维修缺失或未覆盖时，摘要和 selected panel 不互相矛盾。
- [ ] C09. 全部采矿站采不到资源时，guide / status 出现关键风险提示。
- [ ] C10. 普通轻微低效不进入 guide / status 持续刷屏。
- [ ] C11. 敌袭前无武器或防御时，提示能落到具体短板。
- [ ] C12. 跃迁前存在明显短板时，提示使用检查语气，不替玩家自动决策。

### D. 小屏与右侧面板

- [ ] D01. `<= 760px` 视口下建造卡核心字段仍可见：角色、成本、电力、状态、已建数量。
- [ ] D02. `<= 420px` 视口下建造面板不出现横向溢出。
- [ ] D03. 小屏健康摘要可折叠为一行，并能展开查看 6 项。
- [ ] D04. selected panel 打开时不遮挡底部主要建造按钮。
- [ ] D05. 右侧状态、更多状态、selected panel 同时存在时，视觉焦点明确。
- [ ] D06. 触屏场景不依赖 hover 才能查看推荐 / 警告原因。
- [ ] D07. 关键提示在小屏不超过两行，长说明进入展开区。
- [ ] D08. 武器射线、资源外环、维修覆盖等视觉层在小屏不遮挡关闭按钮或主要操作。

### E. 一致性与只读验证

- [ ] E01. 建造卡字段可被 headless DOM 或只读 build palette 读取。
- [ ] E02. 健康摘要 6 类短项可被 design health 读取。
