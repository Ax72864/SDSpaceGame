## 1. 体验目标

v0.13.0 的 UI 目标是让玩家在战斗前、战斗中、战斗后都能读懂“威胁来自哪里、当前防御是否有效、武器为什么打或不打、维修是否跟上、下一步该修正什么设计”。玩家不需要理解底层算法，但应该能回答：

1. 当前是否有战斗压力，最近威胁是什么。
2. 敌人数量、类型、方向、距离和危险等级是否需要立刻处理。
3. 选中炮塔 / 导弹井时，能否看到目标、射程、LOS、缺电、装填和齐射状态。
4. 选中护盾 / 维修站时，能否看到防御存在、缺电、护盾恢复、维修目标和无目标原因。
5. 战斗结束后，能否从 2-4 条短句知道主要伤害来源、损坏最重区域、防御 / 维修短板和下一步建议。

信息优先级：

| 优先级 | 信息 | 承载位置 | 说明 |
| --- | --- | --- | --- |
| P0 | 敌袭临近、最近威胁、关键防御 / 武器断电、严重受损 | `statusAlerts` / 顶部 HUD 短 chip | 直接影响当下生存和操作 |
| P0 | 选中设施的目标、状态、缺电、LOS / 射程、装填 / 齐射、防御 / 维修目标 | `selectedCellDiagnostics` | 解释单个设施为什么有效或无效 |
| P1 | 敌人数量、类型、方向、距离、危险等级 | `statusAlerts` 首条或 `moreStatusPanel` 摘要 | 帮助玩家判断威胁来源 |
| P1 | 战斗后 2-4 条复盘短句 | `statusAlerts` 战斗刚结束段 / `moreStatusPanel` 复盘卡 | 帮助玩家修正设计，不刷屏 |
| P2 | 普通效率、单一火力、维修覆盖一般、导弹未满齐射 | `designHealthSummary` / 细节面板 | 不用红字打断 |

## 2. HUD / 右侧状态威胁摘要

### 2.1 承载位置

推荐采用“一个主摘要 + 必要时一个细节卡”的最小结构：

1. 顶部 `weaponsRow` 保持武器即时概览，后续可追加极短战斗态，例如 `威胁 3 · 最近 海盗 420m ↗`，只在有敌时出现。
2. 右侧 `statusAlerts` 放战斗摘要的首要短句，顺序高于普通建造 / 效率问题，但不压过任务失败、终局结算等硬状态。
3. `moreStatusPanel` 中新增或复用“战斗状态”小卡，展开后显示完整字段；桌面默认可见，移动端收进折叠区。
4. 不建议新增居中大弹窗；中央告警只保留给极少数事件，如敌袭首次进入危险距离、核心严重受损或战斗结束复盘入口。

### 2.2 字段结构

安全态字段：

```text
战斗：安全
说明：暂无接近威胁
```

安全态只显示一行或完全静默。若右侧已经有大量信息，安全态可以只在 `moreStatusPanel` 里显示，不占用 `statusAlerts`。

有敌态字段：

```text
威胁：3 个敌对目标 · 危险 中
最近：海盗 420m ↗ 右上接近
构成：海盗 x2 / 小行星 x1
建议：右上外缘火力不足，检查炮塔 LOS
```

字段优先级：

| 顺序 | 字段 | 必要性 | 显示规则 |
| --- | --- | --- | --- |
| 1 | 危险等级 | 必须 | `安全`、`注意`、`危险`、`危急`；只代表 UI 展示分级，不改变战斗规则 |
| 2 | 最近威胁 | 必须 | 类型 + 距离 + 方向；多敌时优先最近或最危险 |
| 3 | 敌人数量 | 必须 | `1 个敌对目标`、`3 个敌对目标`；避免展开完整列表 |
| 4 | 类型构成 | 必须 | 小行星、海盗、空间站、守护者、敌站；最多 3 类，超出用 `等` |
| 5 | 方向 / 距离 | 必须 | 方向用屏幕方位或箭头 + 文案；距离用统一单位 `m` 或现有游戏距离单位 |
| 6 | 下一步建议 | 条件显示 | 只有缺防御、武器 / 护盾断电、严重受损、敌袭升级时显示 |

### 2.3 危险等级与视觉

| 等级 | 触发语义 | HUD 表现 | 文案示例 |
| --- | --- | --- | --- |
| 安全 | 无敌人或敌人未形成压力 | 静默或蓝灰一行 | `战斗：安全，暂无接近威胁` |
| 注意 | 有敌但距离较远 / 尚未造成损伤 | 蓝灰 / 黄边，非闪烁 | `威胁接近：海盗 680m ↗` |
| 危险 | 敌人进入交火距离、无防御、关键武器断电 | 橙色 | `危险：3 个敌人接近，最近敌站 420m →` |
| 危急 | 核心附近严重受损、无火力且承压、护盾 / 维修关键断电 | 红色，短时强调 | `危急：核心附近受损，武器全部断电` |

视觉原则：

- 危险色只用于敌袭、缺防御、武器 / 护盾断电、严重受损或战斗刚结束。
- 普通效率问题、单一火力、导弹未满齐射、维修空闲不使用红字。
- 同一威胁持续存在时保持摘要更新，不重复刷同一句 alert。
- 多条并发时 `statusAlerts` 最多显示 2 条战斗相关主句，其余放入 `moreStatusPanel`。

## 3. 选中设施反馈结构

选中面板应继续复用 `selectedCellPanel`、`selectedCellInfo`、`selectedCellDiagnostics`。已有结构已经支持连接、电力、火力、护盾、维修等诊断行，v0.13.0 只需要补齐战斗语义字段，不改变面板层级。

通用顺序：

```text
已选：设施名 启用 / 优先级 50（运作中）
战斗目标：目标类型 / 无目标原因
状态：工作中 / 缺电 / 装填中 / LOS 阻挡 / 射程不足
关键数值：距离、射程、装填、护盾值、维修速率
建议：一句行动建议
```

### 3.1 炮塔

炮塔选中反馈：

```text
火力：正在瞄准 海盗 · 目标 360m / 射程 480m
射线：可开火 / LOS 被自机遮挡 / 目标超出射程
状态：工作中 / 缺电停火 / 手动关闭 / 无可攻击目标
建议：外缘留空可减少遮挡
```

字段要求：

- `目标`：显示目标类型；没有目标时显示原因，不只写 `暂无目标`。
- `状态`：优先显示阻断原因，顺序为缺电、手动关闭、脱落、LOS 阻挡、射程不足、无目标、装填 / 冷却。
- `LOS / 射程`：LOS 阻挡用橙色；超距用黄色；缺电用红色。
- `装填`：可沿用现有 reload bar，炮塔就绪时不额外刷红。

### 3.2 导弹井与齐射

导弹井选中反馈：

```text
导弹：目标 海盗 · 560m / 射程 720m
装填：就绪 2/3 · 下一发 1.2s
齐射：当前档 3 · 可发射 2 · F=齐射
状态：资源不足 / 缺电 / 锁敌失效 / 无目标
```

字段要求：

- `装填`：显示就绪数和下一发时间；单个导弹井选中时显示自身装填。
- `齐射`：显示当前齐射档、可发射数量和按钮价值；可发射为 0 时不鼓励玩家按。
- `锁敌`：若右键锁敌目标死亡、超距或不可攻击，显示 `锁敌失效：目标已消失 / 超出射程 / 无可攻击目标`。
- `发射后`：状态回到装填 / 冷却，不继续显示上一轮已发射目标。

### 3.3 护盾

护盾选中反馈：

```text
护盾：存在 · 160/220
覆盖：范围 130 · 朝向主要来袭方向
状态：拦截中 / 缺电失效 / 护盾破裂恢复中
建议：承压方向缺护盾时，补供电或调整外缘防护
```

字段要求：

- `防御存在`：有护盾且工作时明确显示 `护盾存在`，避免玩家只看到数值。
- `缺电`：缺电失效为 P0 红色；不暗示护盾会改变武器攻击逻辑。
- `破盾`：破裂恢复中用橙色，不写成永久失效。
- `无护盾`：整体摘要可提示短板，但不在选中其他设施时强插红字。

### 3.4 维修站

维修站选中反馈：

```text
维修：无人机前往 左翼炮塔 / 正在修复 核心附近护甲
目标：最近受损模块 · 280m
状态：工作中 / 空闲无可修 / 缺电停摆 / 有损伤但无覆盖
能力：设施 18/s · 骨架 12/s
```

字段要求：

- `维修目标`：优先显示正在前往或正在维修的对象；无目标时解释原因。
- `无目标原因`：区分无可修对象、缺电、脱落 / 手动关闭、无覆盖或现有规则无法维修。
- `严重受损`：若核心附近或外缘武器区严重受损，维修状态可给一句建议，但不替玩家自动修复。
- `无人机状态`：`前往`、`维修中`、`返回 / 空闲` 三类足够，不展开完整路径。

## 4. 战斗后复盘摘要

### 4.1 展示位置

战斗结束后 6-10 秒内，在 `statusAlerts` 显示 2-4 条短句；若玩家打开 `moreStatusPanel`，可看到同样内容的“本次战斗复盘”卡。不要使用长弹窗阻断移动和建造。

推荐结构：

```text
本次战斗：主要伤害来自海盗火力。
损坏最重：右上外缘武器区。
短板：护盾缺电，维修未覆盖受损区。
建议：先补供电，再把维修站靠近核心链路。
```

### 4.2 短句规则

- 最多 3-4 条；无足够数据时宁可 2 条，不编造。
- 每条 18-28 个汉字为宜，小屏可折行一次。
- 第一条讲结果来源，第二条讲损坏区域，第三条讲短板，第四条给建议。
- 复盘是建议，不写 `自动修复`、`系统已优化`、`一定安全`。
- 新一轮战斗开始后，旧复盘降级或隐藏，避免污染当前判断。

### 4.3 复盘字段

| 字段 | 显示规则 | 示例 |
| --- | --- | --- |
| 主要伤害来源 | 取本次运行时最高贡献来源；并列时合并为两类 | `主要伤害来自小行星碰撞和海盗火力。` |
| 损坏最重区域 | 按核心附近 / 方位 / 设施群归纳，不列全部格子 | `损坏最重：左翼外层采矿区。` |
| 防御 / 维修短板 | 只挑最影响行动的 1-2 个 | `短板：炮塔被遮挡，维修跟不上。` |
| 下一步建议 | 面向设计修正，不改变规则 | `建议：外缘留出射线，补 1 座维修站。` |

## 5. 低噪声原则

### 5.1 允许突出显示的情况

以下情况可进入 `statusAlerts` 或使用橙 / 红强调：

- 敌袭接近、敌人进入危险距离或数量明显增加。
- 空间站无武器 / 无防御且敌袭正在发生。
- 武器、导弹、护盾、维修设施因缺电停摆。
- 炮塔 LOS 阻挡导致当前目标打不到。
- 核心附近、武器区、护盾区、维修区严重受损。
- 战斗刚结束，需要短复盘提醒玩家修正设计。

### 5.2 不应突出显示的情况

以下情况应放到 `designHealthSummary`、`moreStatusPanel` 或选中细节，不用红字打断：

- 安全态下火力单一、导弹未满齐射、维修空闲。
- 普通资源效率、采矿距离一般、建造推荐。
- 护盾存在但没有承压、维修站无损伤可修。
- 同一敌人持续接近但危险等级没有变化。
- 玩家正在建造模式中出现的非战斗建议，除非会导致关键战斗能力断电。

### 5.3 并发信息裁剪

右侧同时出现多类问题时，优先级为：

1. 战斗生命线：核心 / 关键结构严重受损、所有武器断电、护盾 / 维修断电。
2. 当前威胁：最近敌人、数量、方向、距离、危险等级。
3. 当前设施有效性：选中设施目标、LOS、射程、装填、维修目标。
4. 战斗后复盘：战斗刚结束时保留短时间。
5. 普通设计健康：火力单一、防御偏弱、资源效率。

## 6. 小屏 / 移动端可读性

现有移动端 `status-panel` 宽度约 `min(240px, calc(100vw - 16px))`，且 `more-status-body` 有滚动区域。v0.13.0 文案必须按窄栏设计。

小屏规则：

- 主摘要最多两行：第一行 `威胁 3 · 海盗 420m ↗`，第二行 `危险：护盾缺电`。
- 类型构成压缩为 `海盗x2 / 小行星x1`，不写长列表。
- 方向使用短词：`上`、`右上`、`右`、`右下`、`下`、`左下`、`左`、`左上`，可配箭头但不能只靠箭头。
- 距离取整数，避免 `420.34m`。
- 战斗复盘每条允许折行一次；超过两行应缩短文案。
- `statusAlerts` 最多显示 2 条战斗主提示；其余折叠到 `moreStatusPanel`。
- 打开建造 / Meta 主面板时，右侧状态已有透明处理，战斗 P0 提示仍可保留短句，但不要覆盖建造按钮。
- 触控环境不依赖 hover；锁敌 / 齐射提示必须有文字，不只显示图标。
- 小屏不新增固定底部大条，避免遮挡空间站和敌人接近方向。

## 7. DOM / CSS 落地建议

### 7.1 优先复用

优先复用现有容器：

- `#weaponsRow`：保留导弹井 / 炮塔 / 齐射档 / LOS 计数，可在有敌时追加极短 `威胁` chip。
- `#statusAlerts.status-alerts`：承载低噪声战斗主提示和战斗后复盘短句。
- `#moreStatusPanel.more-status`：承载完整战斗状态卡，桌面展开，移动端折叠。
- `#designHealthSummary.design-health-summary`：保留设计健康，不把普通效率问题升级为战斗红字。
- `#selectedCellDiagnostics.selected-cell-diagnostics`：承载选中设施的战斗有效性细节。

### 7.2 可选最小新增容器

若实现需要清晰分离战斗卡，可只在 `moreStatusPanel` 内新增一个最小容器：

```html
<div id="combatStatusSummary" class="combat-status-summary" aria-live="polite"></div>
```

新增容器规则：

- 放在 `stationData` 与 `designHealthSummary` 之间，作为“当前战斗态”。
- 空内容时隐藏。
- 不新增独立弹窗系统。
- 不改变现有 DOM id 名称，避免 v0.12 回归脚本失效。

### 7.3 状态 class 建议

可复用或补充以下 class 语义：

- `alert-good`：安全态、战斗解除、复盘中正向结果。
- `alert-warn`：注意、射程不足、装填中、维修空闲但有轻微损伤。
- `alert-danger`：危急、缺电停摆、严重受损、无防御承压。
- `diag-ok`：设施正在工作。
- `diag-warning`：LOS 阻挡、射程不足、装填中、护盾恢复中。
- `diag-critical`：缺电、脱落、手动关闭造成关键能力停摆。

若新增样式，应保持 12px 左右字号、1.45 行高、低饱和背景、细边框；不要引入大面积闪烁。危急提示可短时强调，但持续状态应稳定显示。

## 8. 文案示例

威胁摘要：

- `战斗：安全，暂无接近威胁。`
- `威胁接近：海盗 680m 右上，2 个敌对目标。`
- `危险：最近敌站 420m 正面接近，护盾缺电。`
- `危急：核心附近受损，武器全部断电。`

选中炮塔：

- `火力：目标海盗 360m / 射程 480m，LOS 可用。`
- `火力：LOS 被自机遮挡，外缘留空可改善射线。`
- `火力：缺电停火，补供电或提高优先级。`

选中导弹井：

- `导弹：就绪，齐射档 3，可发射 2。`
- `导弹：装填中，下一发 1.2s。`
- `锁敌失效：目标已消失，等待新目标。`

选中护盾 / 维修：

- `护盾：存在，160/220，范围 130。`
- `护盾：缺电失效，补供电后恢复拦截。`
- `维修：无人机前往右翼炮塔。`
- `维修：有损伤但无覆盖，调整维修站位置。`

- `建议：先补供电，再外移炮塔射线。`

## 9. UI 回归点

### A. HUD 威胁摘要

- [ ] A01. 无敌人安全态下，`statusAlerts` 不显示夸张战斗警报。
- [ ] A02. 安全态摘要最多一行，或仅在 `moreStatusPanel` 中低噪声显示。
- [ ] A03. 单个敌人接近时，摘要显示敌人数量。
- [ ] A04. 单个敌人接近时，摘要显示最近威胁类型。
- [ ] A05. 单个敌人接近时，摘要显示方向。
- [ ] A06. 单个敌人接近时，摘要显示整数距离。
- [ ] A07. 多个敌人接近时，摘要优先显示最近或最危险目标。
- [ ] A08. 类型构成能区分小行星、海盗、空间站、守护者和敌站。
- [ ] A09. 敌人死亡或远离后，摘要不残留旧目标。
- [ ] A10. 危险等级变化时，颜色能从安全 / 注意 / 危险 / 危急正确切换。
- [ ] A11. 同一威胁持续存在时，不重复刷同一句提示。
- [ ] A12. 威胁解除后，右侧提示能降级或隐藏。

### B. 低噪声与视觉层级

- [ ] B01. 敌袭、缺防御、关键断电、严重受损和战斗刚结束才使用红 / 橙强调。
- [ ] B02. 普通效率问题不以红字插入 `statusAlerts`。
- [ ] B03. 多条战斗提示并发时，`statusAlerts` 最多展示 2 条主句。
- [ ] B04. 其余细节能在 `moreStatusPanel` 或选中诊断中查看。
- [ ] B05. 建造模式提示与战斗提示同时存在时，战斗 P0 信息不被完全挤掉。
- [ ] B06. 任务完成 / 失败等硬状态不被普通战斗建议覆盖。
- [ ] B07. 危急提示没有持续闪烁造成阅读疲劳。
- [ ] B08. 战斗提示不遮挡顶部资源、武器行和右侧任务选择按钮。

### C. 选中炮塔反馈

- [ ] C01. 选中炮塔时显示当前目标或无目标原因。
- [ ] C02. 炮塔缺电时显示缺电停火，并使用 critical 语义。
- [ ] C03. 炮塔手动关闭时不误写为缺电。
- [ ] C04. 炮塔 LOS 阻挡时显示被自机遮挡。
- [ ] C05. 炮塔目标超出射程时显示射程不足。
- [ ] C06. 炮塔有目标且 LOS 可用时不显示无效原因。
- [ ] C07. 炮塔 reload bar 与战斗诊断不互相矛盾。
- [ ] C08. 读取炮塔诊断不触发真实攻击或改目标。

### D. 选中导弹 / 齐射反馈

- [ ] D01. 选中导弹井时显示自身装填状态。
- [ ] D02. 导弹就绪时显示齐射档位和可发射数量。
- [ ] D03. 导弹全部装填中时，齐射提示不鼓励玩家立即发射。
- [ ] D04. 发射后状态能回到装填或冷却。
- [ ] D05. 锁敌目标死亡时显示锁敌失效原因。
- [ ] D06. 锁敌目标超距时显示超出射程。
- [ ] D07. 无可攻击目标时不显示上一轮目标。
- [ ] D08. 导弹反馈不改变导弹伤害、冷却、命中、资源消耗或结算。

### E. 护盾反馈

- [ ] E01. 选中工作中的护盾时显示护盾存在。
- [ ] E02. 护盾显示当前值 / 最大值和范围。
- [ ] E03. 护盾缺电时显示失效原因。
- [ ] E04. 护盾破裂恢复中使用 warning，不误写为永久失效。
- [ ] E05. 无护盾时，整体摘要可指出短板但不刷屏。
- [ ] E06. 护盾文案不暗示改变武器攻击逻辑。
- [ ] E07. 护盾状态读取不改变护盾阻挡判定。

### F. 维修反馈

- [ ] F01. 选中维修站时显示维修能力数值。
- [ ] F02. 维修无人机前往目标时显示目标设施或区域。
- [ ] F03. 维修无人机正在维修时状态可读。
- [ ] F04. 无损伤可修时显示空闲无目标。
- [ ] F05. 维修缺电时显示停摆原因。
- [ ] F06. 有损伤但无目标时能区分无覆盖、缺电、无可修对象或现有规则原因。
- [ ] F07. 维修提示不承诺自动修复所有损伤。
- [ ] F08. 维修状态读取不改变维修目标选择或维修进度。

### G. 战斗后复盘

- [ ] G01. 战斗结束后显示 2-4 条短句。
- [ ] G02. 复盘包含主要伤害来源或明确说明数据不足。
- [ ] G03. 复盘包含损坏最重区域或关键受损设施群。
- [ ] G04. 复盘包含防御 / 维修短板之一。
- [ ] G05. 复盘包含下一步设计建议。
- [ ] G06. 单条复盘在小屏最多折行一次。
- [ ] G07. 复盘不会在新一轮战斗中长期残留。
- [ ] G08. 复盘文案不写自动修复、自动优化或绝对安全。

### H. 小屏 / 移动端

- [ ] H01. 240px 右侧面板宽度下，威胁主摘要不横向溢出。
- [ ] H02. 方向和距离在小屏仍可读，不只依赖颜色或图标。
- [ ] H03. 类型构成在小屏可压缩显示，不展开长列表。
- [ ] H04. `moreStatusPanel` 折叠时，P0 威胁仍能在主提示看到。
- [ ] H05. `moreStatusPanel` 展开时，战斗状态卡可滚动查看。
- [ ] H06. 打开主建造 / Meta 面板时，战斗提示不挡住主要按钮。
- [ ] H07. 触控环境没有 hover 依赖，齐射 / 锁敌说明有文字。
- [ ] H08. 小屏不新增遮挡空间站主体的固定大条。

### I. DOM / 回归兼容

- [ ] I01. `statusAlerts` 现有空内容隐藏行为保持有效。
- [ ] I02. `moreStatusPanel` 桌面默认展开、移动端折叠策略不被破坏。
- [ ] I03. `selectedCellDiagnostics` 的 `diag-ok` / `diag-warning` / `diag-critical` 语义一致。
- [ ] I04. `designHealthSummary` 仍显示 v0.12 设计健康摘要。
- [ ] I05. `weaponsRow` 仍显示导弹井、炮塔、齐射档和 LOS 计数。
- [ ] I06. 若新增 `combatStatusSummary`，空内容时必须隐藏。
- [ ] I07. 新增 class 不改变旧 DOM id 名称。
- [ ] I08. Headless 读取既有 selected diagnostics 不因新增战斗字段报错。

### J. 文案与可访问性

- [ ] J01. 所有关键提示只靠文字也能理解，不只靠颜色。
- [ ] J02. 危险等级文案不暗示数值规则变化。
- [ ] J03. 距离单位一致，整数化展示。
- [ ] J04. 方向词和箭头一致，不出现相反含义。
- [ ] J05. 文案不使用未实现的敌人、武器或系统名。
- [ ] J06. `aria-live="polite"` 区域不因高频刷新造成读屏刷屏。
- [ ] J07. 战斗复盘没有超过 4 条。
- [ ] J08. 普通效率建议不使用 `danger` 语义。
