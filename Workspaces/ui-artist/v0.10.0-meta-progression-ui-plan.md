## 0. 文档定位与边界

本方案只定义 v0.10.0 Meta 成长的 UI / 信息架构：右侧入口、树 / 卡片面板、开局协议选择、结算反馈、文案规则、CSS / DOM 建议、动效与回归点。

严格范围：

- 只新增 / 修改 `Workspaces/ui-artist/v0.10.0-meta-progression-ui-plan.md`。
- 不修改 `Game/`、ADR、PM 看板、数值方案、`42.md`、`Releases/`、`Docs/` 或其他成员工作区。
- 不 stage / commit 当前 dirty 的 `42.md`。

本方案不决定 schema、helper、数值或协议实际生效逻辑；这些以 ADR 与数值方案为准。UI 只消费 `META_TALENT_TREE`、`START_PROTOCOLS`、`getMetaPurchaseState`、`getMetaSettlementHints`、`selectStartProtocol` 等 helper 的结果。

## 1. 体验目标

v0.10.0 的 UI 目标不是把右侧面板塞满，而是让玩家在三个关键时刻知道自己该做什么：

1. 局中打开 Meta 面板时：知道当前有多少 **局外天赋点**，每条路线在追什么，哪些节点现在能买。
2. 开新局前 / 局中查看协议时：知道当前 **开局协议** 是什么，下一局会受到什么影响，为什么某些协议还没解锁。
3. 结算时：知道本局获得了多少局外天赋点，现在能买什么，还差什么，下一局建议尝试哪种协议或路线。

UI 优先级：

| 优先级 | 信息 | 原因 |
| --- | --- | --- |
| P0 | 可用局外天赋点、节点购买状态、未解锁原因 | 直接决定玩家能不能行动 |
| P0 | 当前开局协议、默认 `balanced`、下一局影响 | 建立跨局意图 |
| P0 | 结算获得点数、推荐节点、下一局建议 | 闭合失败 / 通关后的去处 |
| P1 | 分类筛选、前置连线、效果预览 | 帮助规划路线 |
| P1 | 旧成长整理提示、暂不支持重置 | 降低迁移和点错焦虑 |
| P2 | 路线图标、轻量动效、进度装饰 | 增强辨识度，但不能压过可读性 |

## 2. 总体入口与面板层级

### 2.1 入口改造

当前右侧“局外天赋”按钮区升级为一个单一入口：

- 入口文案：`局外天赋`
- 副文案：`可用 {points} 点`
- 小状态：
  - 有可购买节点：`有可购买`
  - 无可购买但有推荐差额：`还差 {n} 点`
  - 全部当前可见节点满级：`当前路线已满`

入口行为：

- 点击打开 `Meta 面板`，不再在右侧直接展示旧的 5 个购买按钮。
- 旧按钮必须移除或转接到新面板，不能出现“旧按钮 + 新节点卡”两套购买入口。
- 面板默认停留在 `天赋` 标签；如果结算刚结束并有推荐节点，则默认高亮推荐节点。

### 2.2 面板布局

建议使用右侧抽屉式大面板，桌面端覆盖右侧状态栏的扩展层，但不遮挡核心 HUD 的资源 / 任务 / 星系路径摘要：

```text
┌─────────────────────────────────────────────┐
│ 局外成长 Meta                     ESC 关闭 │
│ 可用局外天赋点: 12      当前协议: 标准开局 │
├──────────┬──────────┬──────────┬──────────┤
│ 天赋树   │ 开局协议 │ 结算建议 │ 说明      │
├─────────────────────────────────────────────┤
│ [路线筛选 chips]  全部 资源 防御 武器 探索 │
│                                             │
│ 资源型 tier 1        ── 前置线 ── tier 2   │
│ ┌节点卡┐ ┌节点卡┐       ┌节点卡┐          │
│                                             │
│ 防御型 / 武器型 / 探索型 ...               │
├─────────────────────────────────────────────┤
│ 当前版本暂不支持局外天赋重置...             │
└─────────────────────────────────────────────┘
```

桌面端推荐尺寸：

- 宽度：`min(760px, calc(100vw - 320px))`
- 高度：`calc(100vh - 72px)`
- 位置：右侧贴边，顶部避开主资源 HUD。
- 背景：深色半透明底 + 面板实体底，避免和星空 / 战斗特效混在一起。

小屏 / 移动端：

- 面板切换为全屏底层 overlay，顶部固定标题和关闭按钮。
- 路线筛选 chips 横向滚动。
- 节点卡改为单列，前置连线改为“前置：xxx rank n”文本，不强画复杂线。
- 底部购买按钮 sticky，保证玩家不需要来回滚动找操作。

## 3. 天赋树 / 节点卡信息架构

### 3.1 点数摘要

面板顶部固定摘要条：

| 字段 | 文案示例 | 说明 |
| --- | --- | --- |
| 可用点数 | `局外天赋点：12` | 必须使用“局外天赋点”，禁止简称“研发点” |
| 已投入 | `已投入：8 / 50` | 来自 ranks 与成本表；没有总成本 helper 时可先隐藏 |
| 推荐 | `推荐：采集校准、框架强化` | 最多 2 个，避免顶栏过载 |
| 当前协议 | `下一局协议：标准开局` | 点击切到协议页 |

点数变化反馈：

- 购买成功后数字从旧值滑动 / 闪烁到新值，颜色使用局外成长金色。
- 扣点失败不改变数字，只在按钮旁显示失败原因。
- 结算后首次打开面板，点数摘要出现 `+{gained}` 小浮标，2 秒后淡出。

### 3.2 分类筛选

路线分类与数值方案一致：

| 分类 | id | 颜色建议 | 图标语义 |
| --- | --- | --- | --- |
| 全部 | `all` | 中性灰蓝 | 网格 |
| 资源型 | `resource` | 矿物绿 / 青绿 | 矿石 / 箱子 |
| 防御型 | `defense` | 护盾蓝 | 护盾 / 核心 |
| 武器型 | `weapon` | 炮火橙 | 炮塔 / 准星 |
| 探索型 | `exploration` | 星图紫 | 星图 / 研究 |

筛选行为：

- 默认 `全部`。
- 玩家点击某分类后，只显示该分类节点，但保留跨路线前置提示。例如 `buildDiscount` 需要 `metalRefining` 时，防御筛选下仍显示“还需资源型：金属精炼 1级”。
- 有可购买节点的分类 chip 显示小点。
- 已满的分类 chip 显示 `已满` 小标签。

### 3.3 节点卡内容

每个节点卡建议信息密度如下：

| 区域 | 内容 | 示例 |
| --- | --- | --- |
| 顶部 | 图标、节点名、路线、tier | `采集校准 · 资源型 T1` |
| 等级 | 当前等级 / 最大等级 | `等级 1 / 3` |
| 效果 | 当前效果 + 下一级预览 | `当前：采集 +10%` / `下级：采集 +20%` |
| 成本 | 下一级花费 | `升级消耗：1 局外天赋点` |
| 前置 | 未满足时显示缺口 | `需要：采集校准 2级` |
| 状态 | available / locked / maxed | 颜色、按钮、说明一致 |
| 操作 | 购买按钮 | `购买` / `点数不足` / `已满级` |

效果预览原则：

- 数值型：显示当前总效果和购买后效果，避免只显示“+10% / 级”。
- 解锁型：显示 `未解锁` / `已解锁`，购买后预览写清解锁对象。
- 协议解锁型：在节点卡上显示 `解锁协议：采矿起步`，购买成功后协议页对应卡片亮起。
- 暂不支持复杂公式展开；折扣类直接显示玩家能懂的结果，例如 `建造金属消耗约 -10%`。

### 3.4 前置连线与轻量层级

v0.10.0 不需要复杂技能树画布，建议用“分层卡片 + 轻量连线”：

- 桌面端按 `tier` 分列，category 分段。
- 同一路线内相邻前置用短线连接。
- 跨路线前置不画长斜线，改用节点卡内 `跨路线前置` 标签，避免视觉混乱。
- 小屏端不画线，只显示前置文本。

线条状态：

| 状态 | 视觉 |
| --- | --- |
| 前置已满足 | 实线，使用目标路线颜色 |
| 前置未满足 | 虚线，灰色 |
| 推荐路径 | 外发光弱高亮，持续不超过 1.5 秒 |

## 4. 节点交互状态

节点卡和按钮统一使用以下状态：

| 状态 | class | 视觉 | 操作 |
| --- | --- | --- | --- |
| 可购买 | `available` | 高亮边框，按钮实色 | 点击购买 |
| 点数不足 | `locked not-enough-points` | 灰化但内容可读 | 禁用，显示还差点数 |
| 前置不足 | `locked prereq-unmet` | 灰化 + 前置标签 | 禁用，列出缺口 |
| 已满级 | `maxed` | 金色 / 完成角标 | 禁用，显示已满级 |
| 推荐 | `recommended` | 轻量脉冲描边 | 可叠加其他状态 |
| 新解锁 | `newly-unlocked` | 2 秒扫光 | 购买后一次性展示 |

禁用原因文案：

| reason | 玩家文案 |
| --- | --- |
| `not_enough_points` | `局外天赋点不足，还差 {n} 点` |
| `prereq_unmet` | `前置未满足：需要 {nodeName} {rank}级` |
| `max_rank_reached` | `已满级` |
| `unknown_node` | `节点数据异常，暂不可购买` |
| `save_failed` | `存档写入失败，未扣除点数` |

购买行为按 ADR：点击即生效，不做预览态。由于 v0.10.0 暂不支持重置，UI 需要在购买按钮附近放置常驻说明：

`当前版本暂不支持局外天赋重置；后续平衡期可能提供一次性重置入口。`

为降低误触，建议：

- 鼠标 / 桌面端：购买按钮点击后立即购买，但按钮文案包含成本，例如 `购买：1点`。
- 触屏端：第一次点击节点卡只展开详情；点击 sticky 购买按钮才购买。
- 双击不会重复扣点，实际防护由 helper 负责；UI 在购买请求期间给按钮加 `is-busy` 直到刷新完成。

## 5. 开局协议 UI

### 5.1 协议页结构

协议页放在 Meta 面板第二个标签，顶部显示当前选择：

```text
当前开局协议
┌─────────────────────────────────────────────┐
│ 标准开局 balanced                           │
│ 下一局影响：无额外补给，按基础规则开局。     │
│ 状态：已选择                                │
└─────────────────────────────────────────────┘

可选协议
[标准开局] [采矿起步] [防御起步] [武器起步] [星图探索]
```

默认协议：

- `balanced` 必须永远显示，空存档可用。
- 文案：`标准开局：无额外补给，按基础规则开局。`
- 选中后显示：`下一局将使用标准开局。`

### 5.2 协议卡内容

每张协议卡显示：

| 区域 | 内容 |
| --- | --- |
| 名称 | `采矿起步` / `防御起步` / `武器起步` / `星图探索` |
| 状态 | 已选择 / 可选择 / 未解锁 |
| 解锁来源 | `需要：启程储备 1级` |
| 预计开局影响 | `金属 +25，矿石 +10，采矿设施 -15%` |
| 适合意图 | `快速滚资源` / `稳住核心` / `更早部署炮塔` |

未解锁原因：

- `采矿起步`：`需要局外天赋「启程储备」1级`
- `防御起步`：`需要局外天赋「核心加固」1级`
- `武器起步`：`需要局外天赋「武器校准」2级`
- `星图探索`：`需要局外天赋「星图洞察」1级`

选择反馈：

- 选择已解锁协议后 toast：`已保存为下一局开局协议：采矿起步。当前局不变更。`
- 局中切换协议只影响下一局，必须明确 `当前局不变更`。
- 结算面板中的协议建议点击后打开协议页，不自动选择。

### 5.3 协议状态 class

| 状态 | class | 说明 |
| --- | --- | --- |
| 当前选择 | `selected` | 选中描边 + 勾标 |
| 已解锁未选 | `available` | 可点击 |
| 未解锁 | `locked` | 禁用但展示影响与解锁条件 |
| 推荐 | `recommended` | 结算建议来源 |

## 6. 结算反馈 UI

### 6.1 结算面板新增区块

结算面板增加一个 `局外成长` 区块，位置应在本局结果摘要之后、重新开始按钮之前：

```text
局外成长
本局获得局外天赋点：+7
当前可用局外天赋点：12

现在可购买
· 采集校准 2级（资源型，消耗 1点）
· 框架强化 1级（防御型，消耗 1点）

下一局建议
· 想更快滚资源：购买「启程储备」后选择「采矿起步」
· 想更稳通关：优先补「核心加固」
```

失败与通关使用同结构，不同语气：

- 失败：`这局仍获得了局外天赋点，可以强化下一局开局。`
- 通关：`通关奖励已转化为局外天赋点，可以规划下一条路线。`

### 6.2 推荐逻辑呈现

结算推荐不自动购买，只提供入口：

- `查看局外天赋`：打开 Meta 面板并定位推荐节点。
- `查看开局协议`：打开协议页。
- 推荐节点最多 3 个，优先级：
  1. 当前可购买且未满级的 tier 1 / tier 2 节点。
  2. 能解锁协议的节点。
  3. 与玩家当前已投入最高路线同类的下一节点。
- 如果没有可购买节点，显示最近目标：`还差 {n} 点可购买「{nodeName}」`。
- 不推荐已满级节点，不推荐前置长期不可达节点。

### 6.3 下一局协议建议

结算区块可显示一条协议建议：

| 情况 | 文案 |
| --- | --- |
| 已解锁采矿协议且资源路线投入最高 | `下一局建议尝试「采矿起步」，更快完成早期采集设施。` |
| 已解锁防御协议且本局核心受损严重 | `下一局建议尝试「防御起步」，给核心更多容错。` |
| 已解锁武器协议且本局敌人压力高 | `下一局建议尝试「武器起步」，更早部署炮塔。` |
| 仅 balanced 可用 | `当前默认「标准开局」可用；购买协议节点后会出现更多开局路线。` |

UI 只展示建议，不自动切换协议，不自动购买节点。

## 7. 与现有 HUD / 右侧面板共存

### 7.1 避免遮挡

必须保留可见或可快速返回的信息：

- 资源 HUD：金属、矿石、气体、等离子、局内科研点。
- 当前任务 / objective。
- 星系路径 / galaxyMap 状态摘要。
- 战斗警报 / 核心受损提示。

共存策略：

- Meta 面板打开时，右侧旧状态面板可被覆盖，但主 HUD 顶栏不被遮挡。
- 资源 / 任务 / 星系路径在面板顶部或左侧仍可见；如果布局限制，可提供一行压缩摘要：`资源正常 · 任务进行中 · 星系路径已选择`。
- 战斗中的高危 toast / 核心受损警报层级高于 Meta 面板，但不抢走购买按钮点击。
- 星系图大面板与 Meta 面板互斥；打开一个时另一个收起，避免双 overlay。

### 7.2 小屏可读性

小屏断点建议：

- `>= 1120px`：右侧抽屉 + tier 分列。
- `720px - 1119px`：居中大弹层 + 两列节点卡。
- `< 720px`：全屏弹层 + 单列节点卡 + sticky 操作区。

移动端最小可读性：

- 正文字号不低于 `13px`。
- 关键数字不低于 `16px`。
- 按钮触控高度不低于 `40px`。
- 节点卡间距不低于 `8px`。
- 路线颜色不能单独承载状态，必须配合文字 / 图标 / 标签。

## 8. 文案原则

### 8.1 术语边界

必须严格区分：

| 系统 | 推荐文案 | 禁用 / 避免文案 |
| --- | --- | --- |
| v0.10 Meta | `局外天赋`、`局外天赋点`、`开局协议`、`下一局` | `局外研发`、`研发天赋`、`研究点` |
| v0.5 Research | `研发`、`局内科研点`、`本局研发` | `永久研发`、`局外研究` |
| v0.9 Galaxy | `星系路径`、`星系候选`、`跃迁` | `天赋路线`、`Meta 路径` |

点数统一：

- `局外天赋点`：跨局持久点数，来自结算，可购买 Meta 节点。
- `局内科研点`：本局研发资源，结算后不作为局外天赋保存。

### 8.2 语气

- 失败结算：强调“仍有成长”，不嘲讽玩家。
- 未解锁：说明“差什么”，不只说“不可用”。
- 满级：肯定完成状态，提示可转向其他路线。
- 旧成长整理：如果出现迁移提示，文案为 `v0.10.0 已整理旧局外成长进度；旧点数保留，旧升级折算为当前天赋等级。`

## 10. 动效与键盘行为

### 10.1 轻量动效

保留轻量、可关闭、短时动效：

- 面板打开：`160ms` 右侧滑入 + 透明度。
- 节点可购买：描边轻微呼吸，每次不超过 `1.5s`，不要无限强闪。
- 购买成功：点数数字轻跳，节点卡扫光一次，协议解锁时对应协议卡亮起。
- 购买失败：按钮轻微 shake 一次，同时显示原因文本。
- 结算推荐：首次展示高亮推荐节点，随后静止。

避免：

- 长时间闪烁。
- 大面积粒子遮挡战斗。
- 购买按钮在可点击前后位置跳动。

### 10.2 键盘 / Escape 行为

- `Escape`：如果 Meta 面板打开，关闭面板；如果有嵌套 tooltip，先关 tooltip。
- `Tab`：按标题、标签页、筛选、节点卡、购买按钮、协议卡顺序循环。
- `Enter / Space`：在聚焦按钮或协议卡时触发选择 / 购买。
- `ArrowLeft / ArrowRight`：在标签页之间切换。
- 关闭面板后焦点回到 `metaPanelToggle`。
- 局中打开面板不暂停游戏时，键盘快捷键不能吞掉核心移动 / 操作；如游戏已有键盘操作冲突，Meta 面板打开期间只接管 UI 焦点内按键。

## 11. 验收标准与 UI 回归点

以下回归点不少于 20 条，供 PM / 实现阶段逐条检查：

1. 右侧只保留一个 `局外天赋` 入口，不再显示旧的 5 个直接购买按钮。
2. `metaPanelToggle` 显示当前可用 `局外天赋点`，点数与存档 / helper 一致。
3. 打开 Meta 面板后，顶部摘要显示可用局外天赋点、当前开局协议和推荐入口。
4. 天赋页默认展示资源、防御、武器、探索四类节点，分类名称与数值方案一致。
5. 分类筛选 `全部 / 资源 / 防御 / 武器 / 探索` 可切换，切换后节点列表和可购买提示同步刷新。
6. 每张节点卡显示节点名、路线、tier、当前等级 / 最大等级、当前效果、下一级效果、成本和前置。
7. `available` 节点按钮可点击，购买后点数、等级、效果预览和按钮状态立即刷新。
8. 点数不足节点显示 `局外天赋点不足，还差 {n} 点`，按钮不可点击。
9. 前置不足节点显示缺少的节点名与 rank，不只显示“未解锁”。
10. 满级节点显示 `已满级`，重复点击不会扣点。
11. 解锁协议的节点卡显示 `解锁协议：{协议名}`，购买成功后协议页对应卡从 locked 变 available。
12. 面板底部显示 `当前版本暂不支持局外天赋重置...`，且没有玩家可见的重置按钮。
13. 协议页永远显示 `balanced / 标准开局`，空存档也可选。
14. 未解锁协议卡显示解锁来源，例如 `需要局外天赋「启程储备」1级`。
15. 选择已解锁协议后，协议卡进入 `selected` 状态，并提示 `下一局生效，当前局不变更`。
16. 协议卡显示预计开局影响，例如金属、矿石、耐久、设施折扣或等离子变化。
17. 结算面板显示本局获得的局外天赋点和当前可用局外天赋点。
18. 失败结算和通关结算都显示局外成长区块，只调整语气，不缺失结构。
19. 结算推荐最多 3 个节点，不推荐已满级节点。
20. 点数不足时，结算显示 `还差 {n} 点可购买「{nodeName}」`。
21. 结算建议协议时只提供入口，不自动购买节点、不自动切换协议。
22. 文案中 `局外天赋点` 与 `局内科研点` 区分清楚，不出现 `局外研发`、`研发天赋` 等混淆词。
23. Meta 面板打开时不遮挡主资源 HUD 的关键数字；小屏下至少能通过摘要看到资源 / 任务状态。
24. Meta 面板与 v0.9 星系图大面板互斥，不出现两个大 overlay 同时覆盖屏幕。
25. 小屏 `<720px` 下节点卡单列显示，购买按钮可触达，正文不小于 13px。
26. 路线颜色不作为唯一状态信息；色弱玩家仍可通过文字 / 图标 / 标签理解状态。
27. `Escape` 能关闭 Meta 面板，关闭后焦点回到 `metaPanelToggle`。
28. `Tab` 顺序可用，购买按钮和协议卡能通过键盘触达。
29. 购买成功动效不超过 2 秒，不遮挡战斗警报。
30. 购买失败时点数显示不变化，并给出失败原因。
31. 旧存档迁移后首次打开面板，旧成长整理提示不遮挡购买按钮。
32. `save_failed` 或 helper 返回异常时，UI 不崩溃，显示 `存档写入失败，未扣除点数`。
33. `scoutStart / 星图探索` 如首版实现，文案明确只提供星系信息提示，不改变星系路径。
34. Meta 面板中所有协议影响都写为“下一局”，不暗示局中立即生效。
35. 控制台无因 Meta UI 渲染造成的 `undefined`、`NaN` 或空节点异常。
