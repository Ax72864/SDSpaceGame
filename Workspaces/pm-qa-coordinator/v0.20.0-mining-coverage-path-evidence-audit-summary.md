## 技术边界审核

任务 #5 边界通过。实际改动只新增只读 `getMiningCoverageDiagnostics` 能力与最小 `resourceGuide` 文案，不改采矿范围/公式、资源布局、移动能力、资源生成公式、建造合法性、数值、存档、Meta 或战斗。

- `computeMiningCoverageDiagnostics(options)` 返回 `readOnly=true`、`miningStatus`、`reachability`、`miners`、`nearestResource`、`targetResource`、`candidates`、`bestCandidate`、`suggestedExpansionPath`、`failure` 与 `rules`。
- `__gameTest.playtest.getMiningCoverageDiagnostics(options)` 通过 `safeDeepCloneForTest` 返回诊断快照，不写入 `state` 或 storage。
- `resourceGuide` 仅在矿站存在但未采集、且最近资源超出范围时追加“当前矿站不会产出；需沿框架扩建到彩色外环内，或驾驶工站靠近资源后重建采矿站。”的解释文案。
- helper 的建议路径只模拟邻接扩建与覆盖关系，不承诺成本、电力、战斗或真实输入一定可达；该限制已在诊断说明与复测结论中记录。

## 通过项

- 方向、看板和证据方案完成：制作人方向明确本阶段只做内部采矿覆盖路径与收益归因证据；PM 看板完成 9 项任务拆分；证据方案定义了 candidate/path/failure、distance/range/gap、等待窗口、输入分类和收益归因字段。
- 只读 helper 能返回 candidate/path/failure：`getMiningCoverageDiagnostics` 可给出入圈候选、`suggestedExpansionPath` 和阻塞 `failure`，足以支持团队判断“采不到 / 往哪扩建 / 是否需要重新选址”。
- `runtime errors = 0`：可见 Chrome/CDP 取证和原路径复测均未记录新的 runtime errors、console runtime errors 或 unhandled rejections。
- `storage signature` 无异常：临时 profile 的 `localStorage/sessionStorage` 为空，未清空或覆盖用户真实存档，未出现存档结构异常。
- v0.19 repair 链不回潮：本轮未修改维修、成本、采矿规则、资源布局、建造合法性、存档或 Meta；v0.19 已关闭的“repair 已建成但下一座资源不足被误读为刚才失败”问题未被重新打开。

## 未通过 / 未覆盖

- 没有取得 `harvestingCount > 0`，采矿收益未闭环；等待窗口内资源增长仍不能归因于采矿站。
- helper 给出的入圈路径在真实铺设中被金属成本阻断；`suggestedExpansionPath` 不等同真实可支付路径。
- 原 `mining=-1,1` 输入稳定性仍有问题：HUD 覆盖、CDP 点击递送和相机/站体移动会导致建造失败或证据不稳定。
- 真实人工鼠标与真实触控未覆盖；可见浏览器/CDP 与只读 hook 只能作为浏览器级或辅助证据。
- 自然交火、战后复盘和 Meta 回流继续阻塞；当前不应进入发布准备、tag 或 `Releases/` 归档。
