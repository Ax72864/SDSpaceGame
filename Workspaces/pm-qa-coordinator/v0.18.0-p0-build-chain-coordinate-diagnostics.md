# v0.18.0-p0-build-chain-retest #4 坐标取证诊断修复说明

## 触发原因

#3 可见 Chrome/CDP 复测未通过，但失败证据更像“声明目标 `gridKey` 与实际命中 `gridKey` 不一致”，而非建造规则缺陷。制作人批准进入 #4 的最小“取证断言/坐标校准”修复，不改建造合法性。

## 修改范围

- `Game/game.js`
  - `Renderer.worldToScreen(world)`：`screenToWorld` 的逆变换，返回 `clientX/clientY` 与 `deviceX/deviceY`。
  - 只读 playtest helper（挂载于 `window.__gameTest.playtest`）：
    - `getGridScreenPoint(gridKeyOrXY)`
    - `getGridInteractionDiagnostics(gridKeyOrXY, options?)`
    - `getPointerGridDiagnostics(screenPoint?)`
    - `compareGridClickAlignment(targetGridKeyOrXY, screenPoint?)`
- 未修改 `buildAt`、资源成本、设施数值、RNG、存档、战斗/Meta 规则。
- 未修改 UI 文案或 CSS。

## Helper 用法

### 1. 目标格 -> 屏幕坐标（CDP 点击校准）

```javascript
window.__gameTest.playtest.getGridScreenPoint("0,-1");
// => { gridKey, gridX, gridY, world, clientX, clientY, deviceX, deviceY, visible, viewport, camera }
```

`gridKeyOrXY` 可为 `"x,y"` 字符串或 `{ x, y }`。CDP mouse click 应使用返回的 `clientX/clientY`。

### 2. 目标格完整交互诊断

```javascript
window.__gameTest.playtest.getGridInteractionDiagnostics("0,-1");
// 含 cell.facility、selectedBuild、placement diagnostics、visible、camera、lastBuildError
```

可选 `options.facility` 覆盖当前选中建造类型，仅影响 placement 诊断读取，不会改 state。

### 3. 屏幕坐标 -> 实际命中格

```javascript
window.__gameTest.playtest.getPointerGridDiagnostics({ x: 640, y: 360 });
// 不传参则读取当前 mouseWorld
```

返回 `gridKey`、`handleCanvasClickWouldResolve`（与 `handleCanvasClick` 同路径的 grid 解析，但不触发建造）。

### 4. 声明目标 vs 实际命中对齐断言

```javascript
window.__gameTest.playtest.compareGridClickAlignment("0,-1", { x: 640, y: 360 });
// => { targetGridKey, pointerGridKey, aligned, screenDelta, target, pointer }
```

PM/CDP 脚本应在每次点击前后调用：先 `getGridScreenPoint` 取点击坐标，点击后再 `compareGridClickAlignment` 或 `getPointerGridDiagnostics` 验证 `aligned === true`。

## 坐标链说明

- 输入：`handleCanvasClick({ x: clientX, y: clientY })` 与 pointer 事件一致。
- 变换：`renderer.screenToWorld` 使用 `client * dpr` 与 canvas 设备像素宽高；逆变换 `worldToScreen` 返回 client/device 双坐标。
- 网格：`worldToGrid(world)` -> `buildAt(grid.x, grid.y, selectedBuild)`。
- Helper 只读克隆返回值，不写入 state、storage 或触发 `buildAt`。

## #4 准入与验收

- 准入：#3 阻塞原因为 CDP/坐标取证不稳定；本修复仅补校准与断言能力，不宣称规则修复。
- 验收：`node --check Game/game.js` 通过；helper 存在且只读；修复后 #5 必须回到原 CDP/可见浏览器路径复测。
- 未覆盖：真实鼠标/触控人工体验仍待 #5 或人工补证。

## 残留风险

- CDP 仍可能因焦点、拖拽误判、DPR 变化或 canvas 布局变化导致偏移；应用 `screenDelta.distance` 阈值判断。
- `getGridScreenPoint` 返回格心坐标，边缘格可能有 1 格以内误差；若 `aligned` 失败应优先校准点击点而非改规则。
- Helper 不能替代人类体验通过，只能证明“点击坐标是否命中声明格”。
