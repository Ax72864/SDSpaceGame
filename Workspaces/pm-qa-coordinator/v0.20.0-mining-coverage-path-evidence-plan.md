# v0.20.0 采矿覆盖路径证据方案

## 基本信息

- 记录人：`/pm-qa-coordinator`
- 时间：2026-06-10
- 范围：只定义证据字段、只读分析口径、浏览器取证步骤；不修改 `Game/`、规则、数值、资源布局、存档或发布资产。
- 入口：`Game/index.html?playtest=1`
- 证据目标：证明矿站是否能在当前规则下覆盖资源外环，并把等待窗口内资源变化归因到采矿。

## 必填证据字段

### 环境与输入

- `entryUrl`：实际打开的 URL。
- `browser`：浏览器名称、端口、是否可见。
- `profile`：临时 profile 路径或说明；必须标明未触碰用户真实存档。
- `viewport`：`width`、`height`、`devicePixelRatio`。
- `inputType`：真实鼠标、真实触控、可见 Chrome/CDP、DOM 按钮、只读 hook；若使用 CDP/hook，必须说明不能等同纯人工体验。
- `storageSignature`：`localStorage.length`、`sessionStorage.length`、key 列表、是否临时 profile。
- `runtimeErrors`：runtime error、console error、unhandled rejection 数量和摘要。

### 当前规则只读字段

- `station.pos`、`station.angle`。
- `facilityGridKey`：目标矿站格，例如 `-1,1` 或候选格。
- `facilityState`：`facility`、`enabled`、`active`、`detached`、`powerStarved`。
- `resourceBody`：资源名、类型、资源种类、中心坐标、半径、剩余量。
- `rangeFormula`：当前规则按 `body.r + MINING_RANGE_OFFSET`，本轮只读记录，不修改公式。
- `distance`：矿站世界坐标到资源天体中心距离。
- `range`：资源外环可采范围。
- `gap`：`ceil(distance - range)`，小于等于 0 视为入圈。
- `reachability.overall`：`minerCount`、`activeCount`、`harvestingCount`、`outOfRangeCount`、`allOutOfRange`、`reasonKey`。
- `target`：`null` 或目标资源 `bodyName/resource/distance/range`。
- `powerMargin`：`used`、`available`、`margin`、是否断电。

### 候选路径字段

- `candidateGridKey`：候选矿站格。
- `candidatePath`：需要铺设的 frame 格列表。
- `frameCount`、`costMetal`、`costOre`。
- `placementBeforeFrame`：候选格直接建造是否合法，若不合法记录 `reasonKey`。
- `reachableByExpansion`：是否可通过 frame 链接入。
- `reachableByMovement`：是否需要推进器/航行进入外环。
- `reachableByReselect`：是否应改选其他候选格。
- `blockingReason`：不可达、资源不足、断电、输入映射、等待不足、UI 提示不足等。

### 浏览器等待窗口字段

- `resourcesBeforeBuild`、`resourcesAfterBuild`、`resourcesBeforeWait`、`resourcesAfterWait`。
- `bodyAmountBeforeWait`、`bodyAmountAfterWait`。
- `waitStartedAt`、`waitEndedAt`、`waitDurationSec`。
- `harvestingCountBeforeWait`、`harvestingCountAfterWait`。
- `targetBeforeWait`、`targetAfterWait`。
- `toast`、`statusAlerts`、`resourceGuideText`。
- `attribution`：资源增长是否来自采矿；若 `harvestingCount=0` 或目标为 `null`，不能归因采矿。

## 成功/失败判定

- 成功：矿站 `facility=mining`、`active=true`、`harvestingCount>0`、`target` 指向明确资源；等待窗口内目标资源 `amount` 下降，玩家资源对应类型上升，且可排除或标注核心自然产出。
- 当前规则未覆盖：矿站存在且 active，但 `target=null`、`harvestingCount=0`、`allOutOfRange=true`，记录最近资源 `distance/range/gap`。
- 输入证据不足：只读候选入圈，但 CDP/点击映射未稳定落到目标格，不能判定玩家路径通过。
- 断电证据不足：矿站建成但 `active=false` 或 `powerStarved=true`，必须补电或另记为电力阻塞，不得包装为采矿路径失败。
- UI 提示不足：规则状态明确，但 toast/statusAlerts/resourceGuide 未说明当前位置采不到、还差距离或下一步选址，转 #5 评估最小反馈修复。

## 本轮执行边界

- 允许：可见 Chrome/CDP 输入、DOM 建造按钮点击、只读 playtest hook 记录、临时 profile。
- 禁止：直接改 `state`、调用 `buildAt` 伪造建造、改采矿范围/公式/布局/移动能力/建造规则、清空用户真实存档。
- CDP/hook 只作为浏览器级辅助证据；若成功，也需在结论里区分“浏览器级通过”和“未覆盖真实人工鼠标/触控”。
