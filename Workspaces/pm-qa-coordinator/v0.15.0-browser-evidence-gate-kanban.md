# v0.15.0-browser-evidence-gate PM 看板

## 本轮目标

版本名：`v0.15.0-browser-evidence-gate`。

本轮目标不是新增玩法，也不是正式发布，而是把 v0.14.0 已建立的 release readiness checks、隐藏人工 checklist 和小屏/HUD 密度打磨真正跑进真实浏览器，形成可复查的证据包。PM 需要用真实浏览器运行态 JSON、P0 人工路径、三视口/小屏、触控假设、帧耗时和复制摘要证据支撑制作人判断：继续补证、进入小修 polish，或进入发布准备交接。

## 背景

- 制作人方向文档：`Workspaces/game-producer-lead-designer/v0.15.0-browser-evidence-gate-direction.md`。
- v0.14.0 已建立取证入口、只读 release readiness checks、隐藏人工 checklist 和小屏/HUD 密度打磨，但 Chrome headless `--dump-dom` 未形成有效 browser 证据。
- 当前缺口是真实浏览器运行态 JSON、真实 P0 人工路径、真实触控或触控假设边界、三视口遮挡/滚动/可点性证据、真实帧耗时粗采样和复制摘要路径。
- 没有真实浏览器运行态 JSON 时，结论只能是“继续补证”；没有真实 P0 人工路径记录时，也不能进入发布准备交接。
- 证据必须区分真实浏览器、headless、Node/VM、人工体验和未覆盖路径，不能用自动检查替代手感、节奏和可读性判断。

## 制作人方向摘要

- v0.15.0 的交付物是“证据与判断”，不是“新内容”。
- 真实浏览器取证优先使用现有 Edge/Chrome/DevTools/CDP/远程调试或人工 DevTools，不安装依赖、不改系统设置。
- 必须尝试在真实浏览器调用 `runReleaseCandidateChecks({ sampleFrames: 30 })` 并归档结构化摘要。
- P0 人工路径覆盖空存档首局、建造、采矿、资源不足、断电、敌人接近、武器/护盾/维修、战斗后复盘、星系跳转和 Meta 回到新局。
- 三视口建议覆盖 `760x640`、`420x740`、`390x844` 或等价 viewport；真实触控若未覆盖，必须标为残留风险。
- 只允许修复证据路径中发现的 S0/S1/S2 明确缺陷；手感 polish、文案微调、视觉密度优化和非阻塞小屏舒适度优化由制作人另行判断。

## 风险定级

- 风险等级：中低风险。
- 中低风险来源：本版本默认聚焦真实浏览器取证、PM 证据包、人工记录、三视口记录和必要低风险缺陷修复，不主动改变核心玩法、数值、存档和发布产物。
- 中风险来源：真实浏览器取证可能受本机浏览器、DevTools 操作、远程调试端口、视口设置和复制摘要路径影响；人工记录如果不结构化，容易被误读为“证据充分”。
- 高风险升级触发：若触及核心规则、数值、战斗判定、敌人 AI、RNG、资源公式、维修/护盾/脱落判定、存档结构、`SAVE_KEY`、meta schema、依赖安装、系统配置、发布产物、tag 或 `Releases/` 归档，必须暂停并升级为高风险流程。
- 用户确认触发：若涉及安装依赖、浏览器驱动、系统配置、全局环境、权限、服务、删除文件、覆盖文件或其他本机环境危险操作，必须先向用户说明风险并等待确认。
