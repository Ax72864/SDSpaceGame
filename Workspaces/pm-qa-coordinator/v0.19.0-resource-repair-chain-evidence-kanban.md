# v0.19.0-resource-repair-chain-evidence PM 看板

## 本轮目标

版本名：`v0.19.0-resource-repair-chain-evidence`。

本轮目标是在不新增玩法、不改数值、不发布的前提下，补齐 v0.18.0 遗留的资源、维修与采矿证据链：确认 `repair` 建成条件或失败原因，证明 `mining` 覆盖资源外环后能产生可归因收益，并判断资源不足提示是否足够可理解。PM 负责把证据方案、执行任务、最小修复准入、复测、专项审核和制作人最终决定组织成可执行闭环。

## 背景

- 制作人方向文档：`Workspaces/game-producer-lead-designer/v0.19.0-resource-repair-chain-evidence-direction.md`。
- v0.18.0 已闭环 `power` / `mining` / `turret` 的 `frame` 替换路径，并建立坐标取证能力。
- v0.18.0 仍未闭环：`repair` 因资源不足未建成；`mining x1` 未覆盖资源外环，采矿收益未形成；真实人工鼠标与真实触控未覆盖。
- 自然交火、战后复盘和 Meta 回流仍阻塞于资源 / 维修 / 采矿链条，不能提前解锁。
- 本阶段不是发布阶段，只做内部证据闭环、PM 记录、只读诊断和必要最小提示/反馈修复判断。

## 风险定级

- 风险等级：中低风险。
- 中低风险前提：仅做资源 / 维修 / 采矿证据、PM 记录、只读诊断、原路径复测，以及证据证明需要时的必要最小提示/反馈修复。
- 高风险升级触发：若触及资源成本或公式、维修规则、采矿范围或公式、建造合法性、存档、`SAVE_KEY`、meta schema、依赖安装、发布流程、tag 或 `Releases/` 归档，必须暂停并升级为高风险流程或等待用户确认。
- 用户确认触发：若涉及安装依赖、浏览器驱动、测试框架、修改系统/浏览器全局环境、权限、服务、删除文件、覆盖文件或其他本机环境危险操作，必须先说明风险并等待用户确认。
