# v0.17.0-p0-touch-combat-meta-gate PM 看板

## 本轮目标

版本名：`v0.17.0-p0-touch-combat-meta-gate`。

本轮目标不是新增玩法、不是调平衡、不是发布，而是补齐 SSDSpaceGame 发布闸门最后几类真实体验证据：完整真人 P0、真实触控或诚实替代说明、自然交火复盘、Meta 点击回流和小屏叠层 polish。PM 需要把证据路径、任务边界、验收口径和风险升级条件提前写清，支撑制作人最终决定：继续补证、小修 polish、进入发布准备交接，或暂不通过闸门。

## 背景

- 制作人方向文档：`Workspaces/game-producer-lead-designer/v0.17.0-p0-touch-combat-meta-gate-direction.md`。
- v0.15.0 已取得真实浏览器 JSON，`ok=true`、runtime errors 0、storage signature 稳定、frameTiming rAF 正常，说明浏览器运行态基础证据已经成立。
- v0.15.0 发布闸门仍未通过：完整真人 P0 未闭环，真实触控未覆盖，武器/护盾/维修实际反馈、战斗后复盘、Meta 点击回流仍缺少完整证据，小屏叠层仍有 polish 风险。
- v0.17.0 必须把缺口收束为真实体验证据和必要低风险 UI polish，不能扩张为新玩法、新系统、新数值或发布流程。
- 证据必须区分真实玩家、真实浏览器、真实触控、DevTools/鼠标替代、hook 辅助和未覆盖；不能用自动检查或 hook 结果替代玩家是否看懂、点到、走通。

## 制作人方向摘要

- v0.17.0 的核心目标是回答：当前纵向切片是否具备进入发布准备的最后一组真实体验证据。
- 真人 P0 主路径尽量不用 test hook；若必须使用辅助，必须标注辅助、原因和它不能证明的体验部分。
- 自然交火与复盘至少要覆盖武器开火、护盾/受击或防御缺口、维修状态、战后 `combatReview`。
- Meta 点击回流必须真实点击结算、Meta、开始新局或等价 UI 回流，并记录 storage 影响。
- 触控/小屏优先真实触控；若无真机，必须诚实记录不可用原因和 DevTools/鼠标替代边界。
- polish 必须克制，只处理 S0/S1/S2 明确阻塞，或制作人判定高价值低风险的小屏、按钮 hit、复盘入口、触控滚动问题。

## 风险定级

- 风险等级：中风险。
- 中风险来源：本轮默认聚焦真实体验补证、PM 证据包和必要低风险 UI polish；真实 P0、交火、复盘、Meta 回流和触控路径可能暴露隐藏阻塞；证据类型容易被误读。
