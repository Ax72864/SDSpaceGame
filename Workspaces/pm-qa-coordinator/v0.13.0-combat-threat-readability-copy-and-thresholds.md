## 0. 版本目标与本表定位

v0.13.0 只做战斗反馈的展示分级和文案。本表的全部阈值和文案 **只用于决定如何呈现现有战斗状态**，不修改任何战斗判定、不改伤害/HP/射程/AI/敌人生成/RNG/护盾/维修/脱落判定，不改 `SAVE_KEY` 或 meta schema。

阈值的作用是把主程 helper 输出的 `severity / reasonKey` 映射到玩家看得见的等级和短句。任何数字调整都是显示层调整：阈值变大变小只影响"何时显示哪条文案"，不影响敌人是否真的造成伤害、武器是否真的命中、护盾是否真的挡住。

本表与主程方案 §3.7 共享 reasonKey 白名单：任何新增 reasonKey 必须先在主程方案 §3.7 列出，再回到本表追加文案。

## 1. 威胁等级阈值（threat severity）

威胁等级 4 档：`safe` / `watch` / `danger` / `critical`，对应主程方案 §9 的 `ok / warning / critical` 严重度，再加一档 `watch` 处理"敌人存在但尚未压迫核心区"的中间态。helper 内 enum 仍是三级；本表 `watch` 是 UI 表现层细分（同 reasonKey 在不同上下文展示密度不同）。

### 1.1 整体威胁等级判定（用于 HUD 顶栏 / statusAlerts 顶级标识）

按"最近敌人距离 + 敌人数量 + 强敌存在 + 核心受损 + 无防御"组合判定。从下到上扫描，命中即停。

| 优先级 | 威胁等级 | 触发组合（任一满足） | 显示密度 | 建议文案前缀 |
| --- | --- | --- | --- | --- |
| P1 最高 | `critical` | 核心 HP 比 < 0.35 且 `enemyCount >= 1`；或 hostile-station 在己方武器射程内；或 guardian 在终末 (`state.run.endgame === true`) 且距离 ≤ 400；或最近敌人距离 ≤ 200 且本站无任何 `active` 武器；或脱落风险 `detachmentRisk.severity === "critical"` 且 `enemyCount >= 1` | 必显（HUD + statusAlerts） | `紧急：` |
| P2 | `critical` | 最近敌人距离 ≤ 300（任意敌种） | 必显（HUD + statusAlerts） | `紧急：` |
| P3 | `danger` | 最近敌人距离 ∈ (300, 500] 且 `kind ∈ {pirate, hostile-station, guardian}`；或 `enemyCount >= 3`；或 hostile-station 存在（不论距离） | HUD 显示 + statusAlerts 高优 | `警戒：` |
| P4 | `watch` | 最近敌人距离 ∈ (500, 800]；或仅 asteroid 且距离 ≤ 400；或 `enemyCount >= 1` 且 `isEnemyPressureLikely() === true` 但未命中以上 | HUD 单段短串；statusAlerts 不刷新 | `留意：` |
| P5 兜底 | `safe` | `isEnemyPressureLikely() === false`（无活体敌人、无 combat encounter、终末守护者不存在或已死） | HUD 不追加威胁短串；statusAlerts 不触发威胁通道 | （不显示） |

**阈值含义**：

- 距离 200 / 300 / 500 / 800 是显示分界，**不影响**敌人何时进入实际武器射程或何时真正撞击空间站。
- 数量阈值 `enemyCount >= 3` 是触发"多敌"显示密度的门槛，**不**改变敌人 spawn 节奏。
- 强敌定义：`hostile-station`（敌方空间站 BOSS）、`guardian`（守护者，仅终末星系）、`pirate`（海盗，主动追击）。
- `asteroid` 与 `station`（普通敌站，悬停 AI）单独存在时默认 `watch` 级，组合命中其他条件可升级。

### 1.2 单敌威胁等级（用于 `threats[]` 数组元素的 severity）

每个 `threats[i].severity` 按本表判定，独立于整体威胁等级：

| 敌种 | safe | watch | danger | critical |
| --- | --- | --- | --- | --- |
| asteroid | 距离 > 400 | 距离 ∈ (200, 400] | 距离 ∈ (100, 200] | 距离 ≤ 100（高碰撞风险） |
| pirate | （不存在 safe，活体即至少 watch） | 距离 > 500 | 距离 ∈ (250, 500] | 距离 ≤ 250 |
| station（普通敌站） | （同上） | 距离 > 600 | 距离 ∈ (350, 600] | 距离 ≤ 350 或在己方武器射程内 |
| guardian | （非终末视为 watch） | 终末外 | 终末内且距离 > 400 | 终末内且距离 ≤ 400 |
| hostile-station（BOSS） | （不存在 safe） | 距离 > 700 | 距离 ∈ (400, 700] | 距离 ≤ 400 或在己方武器射程内 |

**说明**：`inWeaponRange === true` 是 critical 触发的一种快捷条件，含义为"打得到敌人的同时也意味着敌人多半能打到我"。

### 1.3 阈值调优依据

| 数字 | 影响什么显示 | 提示噪声风险 | 调整建议 |
| --- | --- | --- | --- |
| 200（critical 最近敌人距离） | 核心区警告何时刷出 | 过小：玩家来不及反应；过大：所有战斗都 critical | 与玩家平均反应窗口对齐，实测后微调 |
| 300（critical 距离） | HUD 是否进入 critical 显示密度 | 过小：critical 罕见；过大：critical 长期常驻 | 默认按 v0.7-v0.12 战斗节奏推荐 |
| 500 / 800（watch 距离边界） | 玩家是否能提前看见敌人接近 | 过大：敌人接近时 HUD 才反应；过小：远敌长期占用 HUD | 800 与典型 spawn 距离对齐 |
| `enemyCount >= 3`（多敌触发） | 是否进入"多目标"复盘统计 | 过小：单次散兵显示"多敌"；过大：真的多敌还说单敌 | 与典型 pirate wave 规模匹配 |
| `core HP 比 < 0.35`（核心受损 critical） | 核心受损是否升级威胁 | 过敏：每次小擦伤都 critical；过钝：核心快爆才报 | 实测中调整 |

所有数字均为暂定值，需在 T1 实测后由 PM 协调实战回归微调；本表只决定显示分级，不进入存档。

## 2. 敌种文案（kindLabel + 短句）

只覆盖现有敌种，不承诺新机制。短句模板用于 `threat.nearest` 显示和复盘 `primarySources` 归并。

| kind | kindLabel | HUD 短句模板 | 复盘 source label | 触发提示 reasonKey |
| --- | --- | --- | --- | --- |
| `asteroid` | 小行星 | `小行星 {direction} {distance}m`（watch+ 才显示） | `小行星撞击` | `asteroid_collision_risk` |
| `pirate` | 海盗 | `海盗 {direction} {distance}m` | `海盗火力` | `pirate_closing` |
| `station` | 敌方建筑 | `敌方建筑 {direction} {distance}m` | `敌方建筑火力` | `hostile_station_pressure`（沿用） |
| `guardian` | 守护者 | `守护者 {direction} {distance}m` | `守护者火力` | `guardian_pressure` |
| `hostile-station` | 敌方空间站 | `敌方空间站 {direction} {distance}m` | `敌方空间站远程` | `boss_in_weapon_range` / `hostile_station_pressure` |

**说明**：

- `kindLabel` 是稳定中文短词，禁止泛化为"未知敌人"；helper enum 缺失对应敌种时 fallback 到 `敌方目标`（兜底，不宣传）。
- `direction` 8 向中文 enum 见 §3。
- `distance` 单位与现有 HUD 一致，显示 `{Math.floor(distance)}m`，不引入新单位。
- 复盘 source label 用于战斗后摘要（§5），避免和 HUD 短句重复同一词。
- BOSS 类两个 reasonKey：进入武器射程时优先 `boss_in_weapon_range`，仅接近但未进射程时 `hostile_station_pressure`。

## 4. 武器 / 导弹 / 护盾 / 维修状态文案

每条 reasonKey 给出 4 个字段：触发条件、HUD 短串、`statusAlerts` 文案、选中面板文案。短串用于 HUD 顶栏 `weaponsRow` 末尾追加；`statusAlerts` 受 v0.12 频率控制器约束（同 reasonKey 8 秒内不重复）；选中面板用于 `buildSelectedCellDiagnosticsHtml` 内对应 facility。

### 4.1 炮塔（turret）

| reasonKey | 触发条件 | HUD 短串 | statusAlerts（warn 级） | 选中面板补充 |
| --- | --- | --- | --- | --- |
| `weapon_ready` | 有目标、在射程内、不缺电、LOS 通透 | （不显示，正常态） | （不触发） | `当前目标 {distance}m，正在攻击` |
| `weapon_no_target` | `selectTargetReadOnly` 返回 null 且 `isEnemyPressureLikely() === true` | （仅当全炮塔无目标时显示）`炮塔暂无可攻击目标` | `炮塔暂无可攻击目标` | `当前无可攻击目标` |
| `weapon_no_power` | `enabled && !active && powerUse > 0` | `炮塔缺电` | `炮塔缺电，无法开火` | `缺电停火，建议补电` |
| `weapon_los_blocked` | 该炮塔最近 `LOS_BLOCK_WARN_DURATION` 内被遮挡 | （不单独显示，进入 §4.1 总览） | `炮塔射线被结构遮挡` | `射线被空间站结构遮挡，不是武器损坏` |
| `all_turrets_los_blocked` | 全部 active turret 均 `losBlockedRecent === true` | `炮塔全部被遮挡` | `所有炮塔射线被遮挡` | （在每个 turret 选中面板沿用 `weapon_los_blocked`） |

**禁止表达**：`已平衡调整射程`、`命中已修正`、`真实难度降低`、`自动避开遮挡` 等暗示判定改动的话术。

### 4.2 导弹（missile）与齐射（salvo）

| reasonKey | 触发条件 | HUD 短串 | statusAlerts | 选中面板补充 |
| --- | --- | --- | --- | --- |
| `missile_reloading` | 全部 missile cell `reload > 0`，至少有一个在装填 | `导弹装填 {Math.ceil(nextReloadSec)}s` | （仅当 `isEnemyPressureLikely()` 且无 ready 时）`导弹全部装填中` | `下一发 {Math.ceil(cell.reload)}s 后就绪` |
| `missile_no_target` | 有 ready missile 但 `selectTargetReadOnly` 返回 null | `导弹无可攻击目标` | `导弹暂无可攻击目标` | `就绪但当前无目标` |
| `missile_no_resource` | 有 ready missile 但 `state.resources.gas/metal` 不足以发射齐射 | `导弹资源不足` | `导弹齐射缺资源` | `资源不足，无法齐射` |
| `priority_target_active` | `state.input.priorityTarget` 有效，距离在射程内 | `已锁定 {kindLabel}` | （不触发） | `已锁定，齐射将集火该目标` |
| `priority_target_stale` | `getPriorityTargetCandidate` 返回 `stale === true` 但 enemy 仍存在 | `锁敌即将过期` | （不触发，仅 HUD watch） | `锁敌即将过期，按 R 取消或重新锁定` |
| `priority_target_out_of_range` | 锁敌目标距离 > 该 missile cell range | `锁敌目标超出射程` | `锁敌目标超出射程` | `目标已脱离射程` |

**齐射档位文案**（`state.missile.salvoSize`）：

| salvoSize | 显示文案 |
| --- | --- |
| `1` | `单发` |
| `2` | `齐射 2` |
| `3` | `齐射 3` |
| `999` | `齐射全部` |

**齐射可发射状态**（`canFireNow`）：

- `canFireNow === 0 && ready missile > 0`：HUD 显示 `齐射 0/{salvoSize}（无目标或缺资源）`，文案来源由 reasonKey 决定（`missile_no_target` / `missile_no_resource` 优先级 > 一般装填）。
- `canFireNow > 0`：HUD 显示 `齐射 {canFireNow}/{salvoSize}`，不附加 reasonKey。

**禁止表达**：`必中`、`完全锁定`、`自动追踪保证命中` 等承诺命中类话术；锁敌只是优先级提示，不改变 missile 实际命中判定。

### 4.3 护盾（shield）

| reasonKey | 触发条件 | HUD 短串 | statusAlerts | 选中面板补充 |
| --- | --- | --- | --- | --- |
| `shield_active` | 至少 1 个 shield cell `active === true` 且 `shield > 0` | （不显示，正常态） | （不触发） | `护盾可用，可减轻部分远程伤害` |
| `shield_no_power` | shield cell `enabled && !active && powerUse > 0` | `护盾缺电` | `护盾缺电，防御失效` | `缺电断盾，建议补电或暂时撤离` |
| `shield_broken` | shield cell `active && shield <= 0`（充能中） | `护盾充能中` | （仅当 `isEnemyPressureLikely()` 持续）`护盾充能中` | `护盾值耗尽，正在恢复` |
| `shield_missing` | `total === 0` 且 `isEnemyPressureLikely() === true` | （由整体威胁文案承载） | `当前没有护盾覆盖`（仅当 critical 级） | （不适用） |

**关键文案约束**：

- 护盾文案不得使用 `绝对安全`、`完全拦截`、`100% 防御` 等表述。
- 默认表达：`护盾可用，可减轻部分远程伤害` —— 明确"部分"，不承诺全防。
- `shield_missing` 不得写成 `必须建造护盾`；只能描述短板，不替玩家决定。

### 4.4 维修（repair）

| reasonKey | 触发条件 | HUD 短串 | statusAlerts | 选中面板补充 |
| --- | --- | --- | --- | --- |
| `repair_active` | `activeRepairerCount > 0 && damagedCellCount > 0 && uncoveredDamagedCount === 0` | （不显示，正常态） | （不触发） | `正在派出无人机修复 {coveredDamagedCount} 处` |
| `repair_no_target` | `activeRepairerCount > 0 && damagedCellCount === 0` | （不显示） | （不触发） | `当前无需修复` |
| `repair_no_power` | `poweredOffCount > 0 && damagedCellCount > 0` | `维修缺电` | `维修站缺电，未派出无人机` | `缺电停摆，建议补电` |
| `repair_uncovered` | `activeRepairerCount > 0 && uncoveredDamagedCount > 0` | `维修覆盖不足` | `部分受损模块未在维修范围内` | `本站射程内无可修对象` |
| `repair_missing` | `repairerCount === 0 && damagedCellCount > 0` | `无维修覆盖` | `当前无维修设施覆盖受损区` | （不适用） |

**关键文案约束**：

- 不写 `自动修复`、`完全恢复`、`保证修复完毕`；维修只是基于现有规则的状态描述。
- `repair_missing` 不得写成 `必须建造维修站`；只描述短板。
- 单设施选中面板的"派出/完成数"由主程 §3.5 helper 提供，文案模板：`近 12 秒派出 {n} 次，完成 {m} 次`。

## 5. 战斗后复盘短句模板

复盘最多 3-4 条短项，对应主程 §3.6 `shortItems`。每条短项有 `kind / severity / reasonKey / payload` 四字段，文案按 `reasonKey` 查表填充。

### 5.1 主要伤害来源（`kind === "source"`）

模板：`主要伤害来自{label}（{totalDamage} 点，{hitCount} 次）`。

| reasonKey | label 填充 |
| --- | --- |
| `main_damage_pirate` | 海盗火力 |
| `main_damage_asteroid` | 小行星撞击 |
| `main_damage_hostile_station` | 敌方空间站远程 |
| `main_damage_guardian` | 守护者火力 |
| `main_damage_station` | 敌方建筑火力 |

**显示规则**：

- 仅当 `primarySources[0].totalDamage >= 5`（暂定阈值，避免单发擦伤上榜）才显示。
- 显示密度：始终 1 条，对应 `primarySources[0]`；其余在选中复盘项时展开。

### 5.2 损坏最重区域（`kind === "worstArea"`）

模板：`{roleLabel}受损最重（{damageTaken} 点，{cellCount} 块）`。

| reasonKey | roleLabel |
| --- | --- |
| `worst_area_thrust` | 推进 |
| `worst_area_weapon` | 武器 |
| `worst_area_defense` | 护盾 / 防御 |
| `worst_area_power` | 电力 |
| `worst_area_core` | 核心 |
| `worst_area_mining` | 采矿 |

**显示规则**：

- 仅当 `worstAreas[0].damageTaken >= 10` 显示。
- 核心区受损（`worst_area_core`）优先级最高，无论 damageTaken 多少都显示。

### 5.3 防御短板（`kind === "gap"`）

模板：直接用 reasonKey 对应短句。

| reasonKey | 文案 |
| --- | --- |
| `shield_offline_during_combat` | 战斗中护盾未生效（缺电 / 无护盾 / 充能中） |
| `missile_never_fired` | 本场战斗导弹未发射 |
| `turret_los_blocked_often` | 炮塔多次被结构遮挡 |
| `repair_overwhelmed` | 维修跟不上受损速度 |
| `thrust_heavy_loss` | 推进器受损较重，机动能力下降 |
| `no_significant_damage` | 本场战斗未受明显损伤 |

**显示规则**：

- 同一场战斗最多 1-2 条 gap；按 severity desc 选择。
- `no_significant_damage` 是兜底（短项 < 2 时填入），不与其他 gap 并列。

### 5.4 下一步建议（`kind === "recommendation"`）

模板：直接用 reasonKey 对应短句。

| reasonKey | 文案 |
| --- | --- |
| `rebuild_shield` | 可考虑补充或修复护盾覆盖 |
| `add_power_source` | 可考虑增加发电以恢复关键设施 |
| `consider_missile_salvo` | 可考虑下场战斗使用导弹齐射 |
| `add_thruster_cover` | 可考虑为推进器增加掩护 |
| `add_repair_station` | 可考虑增加维修站覆盖 |
| `reposition_weapons` | 可考虑调整武器位置，减少遮挡 |

**关键文案约束**：

- 全部以 `可考虑...` 开头，**不**写成 `应该 / 必须 / 推荐 / 强制`，避免替玩家做决定。
- 同一场最多 1 条建议；建议优先级遵循 §6 低噪声优先级。
- 不出现 `已自动修复`、`已自动平衡` 等承诺。

### 5.5 整条复盘组装规则

显示顺序固定：`source → worstArea → gap → recommendation`。

- 最少 2 条（PM 看板 J01）：若不足 2 条，按 `no_significant_damage` + 兜底建议补齐。
- 最多 4 条：若超过，按 severity desc 截断 gap 和 recommendation（source / worstArea 各保留 1 条）。
- duration < `COMBAT_WINDOW_MIN_RECORD_SEC`（2 秒）的战斗不显示复盘（主程 §3.6 已约束）。
- 复盘只显示在 `moreStatusPanel` 内 `combatReviewSummary` 区块（主程 §5.4），不进入 toast 或 modal。

## 6. 低噪声优先级表

当多个 reasonKey 同时触发，按本表优先级展示。statusAlerts 在同一刷新周期内最多显示 3 条（高优先级 critical / warn 优先），HUD 顶栏威胁短串最多 1 段，选中面板按 cell facility 独立展示不抢资源。

| 优先级 | 类别 | 触发条件 | 对应 reasonKey 示例 |
| --- | --- | --- | --- |
| L1 最高 | 敌近且无防御 | 最近敌人距离 ≤ 300 且本站 `weapon active === 0 && shield active === 0` | `weapon_no_power` + `shield_missing` 同时升级 critical |
| L2 | 核心受损 | 核心 cell `hpRatio < 0.35`，或 `state.fragments` 包含核心相邻脱落 | `core_low_hp` / `fragment_detaching` |
| L3 | 武器 / 护盾断电 | turret 或 shield `poweredOffCount > 0` 且 `isEnemyPressureLikely()` | `weapon_no_power` / `shield_no_power` |
| L4 | 严重设施受损 | `damagedCells[0].severity === "critical"` 且非核心 | `worst_area_thrust` / `worst_area_weapon` 等（在 statusAlerts 阶段） |
| L5 | 无维修覆盖 | `repair_missing` 或 `repair_uncovered` 且 `damagedCellCount > 0` | `repair_missing` / `repair_uncovered` |
| L6 最低 | 普通效率问题 | LOS 阻挡、装填中、锁敌过期等可解释但非紧急 | `weapon_los_blocked` / `missile_reloading` / `priority_target_stale` |

**应用规则**：

- statusAlerts 按 L1→L6 顺序选取最多 3 条，且每条 reasonKey 受 v0.12 频率控制器 8 秒冷却。
- 同一类别多 reasonKey 命中时合并为该类别一条（避免同类刷屏）。
- HUD 顶栏威胁短串只反映威胁等级（§1.1），不展开 reasonKey 细节。
- 选中面板独立展示，不受全局优先级影响（玩家主动查看时应提供完整信息）。
- 安全态 (`safe`) 下 L1-L6 全部静默，HUD 不刷威胁短串、statusAlerts 不触发威胁通道。

## 7. 禁止词清单

以下表达在 v0.13 任何位置（HUD、statusAlerts、选中面板、复盘短句、debug 输出）都禁止出现：

| 禁止词 / 表达 | 禁止理由 |
| --- | --- |
| `自动修复` / `自动重建` | 维修不自动决定哪里修；复盘只给建议 |
| `绝对安全` / `完全无损` | 任何战斗状态都可能出现意外伤害 |
| `必中` / `100% 命中` / `保证命中` | 武器结算逻辑未改，命中仍然受现有判定影响 |
| `完全拦截` / `100% 拦截` | 护盾不能拦截所有伤害类型 |
| `已平衡调整` / `已数值优化` | v0.13 不修改任何数值 |
| `真实难度降低` / `已削弱敌人` | v0.13 不改敌人 HP / 伤害 / AI / 生成 |
| `必须建造 X` / `推荐立即 X` / `应当 X` | 建议类文案统一用 `可考虑...` |
| `未知敌人` / `未知威胁` | 敌种 fallback 应为 `敌方目标`，不泛化 |
| `战斗已结束，状态已重置` | 复盘不写存档、不影响 meta |
| `保护层已激活` 等暗示新机制 | v0.13 不引入新机制；只解释现有规则 |
| `修复进度 100%` 等绝对化进度 | 维修速率受现有规则影响，进度可能中断 |

新增禁止词需先回到本表追加，再用于实现。
