# v0.18.0-p0-build-chain-retest #2 正确建造链复测方案

## 目标

验证玩家按当前规则能否完成首局最小建造链：

`frame -> power -> frame -> mining -> frame -> turret -> frame -> repair`

本方案只定义正确操作和记录口径，不修改 `Game/`、规则、数值、存档、战斗或 Meta。

## 前置边界

- 使用真实/可见浏览器或已有 CDP 能力，不安装依赖、不改系统设置。
- 使用临时浏览器 profile，禁止清空或覆盖用户真实存档。
- 输入动作必须来自鼠标、键盘、触控或 CDP input；只读 hook 只用于记录 `gridKey`、`facility`、资源、toast/statusAlerts 和 runtime errors。
- 禁止直接调用 `buildAt()`、直接改 `window.__gameState` 或构造设施状态来包装通过。
- 若目标 `gridKey` 与实际命中 `gridKey` 不一致，本次对应步骤不计通过。

## Game Hook 可用性

已静态确认最新 `Game/game.js` 中与本轮相关的取证能力：

- `window.__gameState` 可只读读取 `selectedBuild`、`resources`、`power`、`station.cells`、`lastBuildError`、`buildHint`、`input.mouseWorld`。
- `window.__gameTest.resetRun(seed, level, galaxyType)` 可在临时 profile 中创建确定性新局；本轮只允许用于临时取证起点，不代表真实存档路径。
- `resolvePlacementArgsForTest()` / `getPlacementDiagnostics()` 可读取放置诊断；该诊断不能替代实际建造成败。
- `handleCanvasClick()` / `handleWorldClick()` 最终会调用 `buildAt()`；复测不能直接调用这些闭包函数，只能通过浏览器输入触发。
- `buildAt(x, y, "frame")` 要求目标为空且有连接邻格；非 `frame` 设施要求目标已存在、未脱落且 `facility === "frame"`。

## 正确操作步骤

每一种设施都必须走两段式操作：

1. 选择 `frame`。
2. 点击一个合法空邻格，记录该格为 `frame` 后的实际 `gridKey`。
3. 选择目标设施。
4. 点击第 2 步实际新增的已有空 `frame`。
5. 记录该格是否从 `frame` 替换成目标设施。

设施顺序：

1. `frame -> power`
2. `frame -> mining`
3. `frame -> turret`
4. `frame -> repair`

## 每步记录字段

- 步骤编号与设施类型。
- 输入方式：真实鼠标、真实触控、可见浏览器 CDP mouse/key、或其他辅助方式。
- 点击前声明目标：`target.gridKey`、屏幕坐标、目标格点击前 `facility`。
- 点击后实际结果：目标格 `facility`、全局设施计数、是否新增/替换成功。
- 资源前后：至少记录 `metal`、`ore`、`gas`、`plasma`、`research`。
- 电力前后：`power.available`、`power.used`。
- UI 反馈：toast、statusAlerts、`lastBuildError`、`buildHint`。
- runtime errors：`error`、`unhandledrejection`、`console.error` 数量和文本；为 0 时明确写 0。
- 未覆盖或不计通过原因：坐标/DPR/相机误差、脚本目标与实际命中不一致、非真实触控、直接状态构造等。

## 通过/失败判定

- 通过：四段链路均能证明“先新增 `frame`，再点击同一实际 `gridKey` 的空 `frame` 替换为目标设施”，且 runtime errors 为 0。
- 部分通过：只能证明部分设施被建成，但操作顺序、目标格或实际命中格不完整。
- 失败：目标设施未建成，或出现“设施只能建造在已连接的空框架上”等阻断提示。
- 阻塞：浏览器无法启动、CDP 无法连接、坐标无法可靠映射、或无法证明输入命中实际目标格。

## 失败后处理

- 若失败来自自动化坐标或 CDP 输入误差，不触发 `Game/` 修复。
- 若真实鼠标/触控按正确步骤仍失败，才进入 #4 最小 UI/反馈/取证断言修复判断。
- 任何规则、数值、存档、战斗或 Meta 修改都必须升级，不在本方案范围内。
