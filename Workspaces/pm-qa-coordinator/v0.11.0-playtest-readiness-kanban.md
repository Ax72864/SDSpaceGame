# v0.11.0 Playtest Readiness PM 看板

版本名：`v0.11.0-playtest-readiness`

制作人方向文档：`Workspaces/game-producer-lead-designer/v0.11.0-playtest-readiness-direction.md`

制作人方向 commit：`a126e5d`

## 本轮目标

把 v0.9 星系图与 v0.10 局外成长从“功能可用、工具回归通过”推进到“首局能真实理解、主面板不互相遮挡、小屏可完成关键路径、PM 和制作人有浏览器级验收证据，可判断是否进入发布准备”。

本轮不新增核心系统，不扩张玩法内容，优先解决 v0.10 未覆盖的真实有头浏览器人工体验、小屏布局、信息密度、手感和主观接受问题。

## 背景

- v0.10 最终状态：开发完成 / 工具回归通过 / 未发布。
- 未发布原因：真实有头浏览器人工体验、小屏布局、信息密度、手感和主观接受未覆盖。
- v0.11 的任务不是继续堆系统，而是补齐首局理解、面板互斥、小屏可读性和浏览器验收证据。
- 如果 v0.11 仍不能证明真实玩家路径可走通，制作人结论应为继续打磨，不进入发布。

## 制作人方向摘要

- 首局路径更清楚：空存档玩家能理解采矿目标、资源靠近、任务完成、跃迁、结算和 Meta 购买之间的关系。
- 信息焦点更稳定：`metaPanel`、`galaxyMapPanel`、`runSettlementPanel`、`objectiveChoicePanel` 不应互相叠加抢视线。
- 失败和通关后有下一步：结算面板需要说明本局结果、获得内容、下一局入口和 Meta 购买意义。
- 小屏体验更可靠：窄屏下右侧面板、候选卡、抽屉和关键按钮可读、可滚动、可点击。
- 发布判断更有证据：PM 必须区分自动检查、headless 检查、真实有头浏览器检查、人工体验和未覆盖路径。

## 风险定级

- 风险等级：中风险偏体验。
- 原因 1：主要修改集中在首局引导、面板互斥、小屏布局和验收辅助，不应触碰存档 key、核心 meta schema 或核心 run 规则。
- 原因 2：体验风险集中在信息密度、焦点管理和真实浏览器手感，工具通过不能替代人工体验。
- 原因 3：若实现触碰 `SAVE_KEY`、核心 meta schema、Meta 生效逻辑、星系图推进规则或任务 / 失败 / 通关判定，风险立即升级为高风险，需要主程专项审核和制作人重新确认。
- 版本默认动作：不安装新依赖、不打 tag、不归档 `Releases/`，除非 #10 制作人另行批准。

## QA 回归点清单

### 空存档首局

- [ ] 清空 localStorage 后首次进入游戏，不出现启动错误或空白画面。
- [ ] 首局主提示优先说明“先建立采矿能力”，而不是同时抛出多个同级目标。
- [ ] 靠近资源点的提示能让玩家理解移动、采集和建造之间的关系。
- [ ] 第一个任务目标文案能指向局内行动，玩家不需要外部说明即可尝试完成。
- [ ] 完成关键目标后，跃迁或继续推进入口可见且语义明确。
- [ ] 首次失败结算能说明失败原因、获得内容和下一步入口。
- [ ] 首次通关结算能说明奖励、下一局入口和局外成长关系。
- [ ] 首次打开 Meta 面板时能理解点数、可购买项和“下一局更强”的意义。

### 旧存档与 localStorage

- [ ] v0.10 旧存档进入 v0.11 后不丢失 `points`、talents、协议选择和已解锁状态。
- [ ] v0.9 / v0.10 旧存档缺少新增 UI 状态字段时能安全启动。
- [ ] 损坏 localStorage 不导致游戏启动崩溃，错误提示或 fallback 不阻断普通开局。
- [ ] 多次刷新不重复赠点、不重复扣点、不重复迁移。
- [ ] 本版本不修改 `SAVE_KEY`，浏览器 localStorage 中无新旧 key 混乱。
- [ ] playtest snapshot 读取 localStorage 摘要时不写入或改写存档。

### Meta 面板

- [ ] `metaPanel` 能从入口打开，并能通过关闭按钮返回游戏或结算上下文。
- [ ] 打开 `metaPanel` 时不会同时遮挡 `galaxyMapPanel`、`runSettlementPanel` 或 `objectiveChoicePanel` 的关键操作。
- [ ] 点数不足、前置不足、满级、可购买状态在小屏和桌面下均可读。
- [ ] 购买成功后 UI、点数、rank 和刷新后的存档保持一致。
- [ ] Meta 文案不把局内研发误称为局外永久天赋。

### galaxyMapPanel

- [ ] 完成任务后打开 `galaxyMapPanel`，候选卡可读且确认按钮可点击。
- [ ] `galaxyMapPanel` 打开时 `metaPanel` 不应覆盖候选卡或确认按钮。
- [ ] 取消星系选择后能回到关内继续游戏，任务完成态不丢失。
- [ ] 确认星系选择后只推进一次，不因 Escape 或重复点击产生双推进。
- [ ] 小屏下候选卡至少保留当前选择、奖励 / 风险、确认按钮。

### runSettlementPanel

- [ ] 失败结算打开时不会被星系图或 Meta 抽屉意外盖住关键结果。
- [ ] 通关结算打开时奖励、局外点数和下一步入口可读。
- [ ] 结算后进入 Meta 面板能看到点数变化，并可完成一次符合条件的购买。
- [ ] 结算面板关闭或继续按钮不会误触发协议选择、购买或重复结算。
- [ ] 小屏下结算内容可滚动，主按钮可见或可通过明确滚动触达。

### objectiveChoicePanel

- [ ] 新局或协议选择需要展示 `objectiveChoicePanel` 时，当前可选项和确认动作明确。
- [ ] `objectiveChoicePanel` 打开时不会与 `runSettlementPanel` 形成阻断叠加。
- [ ] 选择目标或协议后进入新局，HUD 和当前目标状态一致。
