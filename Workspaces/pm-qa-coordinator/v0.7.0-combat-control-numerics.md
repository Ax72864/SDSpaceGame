## §0. 制作人方向回放与硬约束

### §0.1 体验目标三条（按重要程度排序）

1. **看得见战斗节奏**：玩家随时清楚 N 个导弹井中哪些 ready / 哪些还在装填倒计时，turret cell 哪些因 LOS 遮挡而停火；让"我有多少火力可调度"成为一眼可见的信息。
2. **能锁定优先目标**：玩家右键点击敌方 BOSS 关键 cell（如 hostile-station 的 core / 武器 turret）或杂兵核心，所有 turret / missile / shield 都自动优先打它；让 v0.6.0 战术决策（先打武器 / 直击 core）从"靠运气"变成"主动控制"。
3. **能控制齐射时机**：玩家可以选择按 F 键"扣扳机"齐射就绪的导弹井，也可以选择"先发 2 枚试探 / 留 4 枚应急"；让导弹资源管理从"一按全梭"变成"节奏选择"。

## §1. `priorityTarget` 锁敌参数

> 设计意图：让玩家"右键 = 锁敌"成为可预期的低成本操作，锁敌后 12 秒内任意 turret / missile / shield 都自动尊重；超时 / 距离过远 / 敌死三种失效条件覆盖所有边界，避免"幽灵锁定"。

### §1.1 参数表

| 字段 | 推荐值 | PM 可微调范围（±20%）| 设计理由 |
|---|---|---|---|
| `validUntil - setAt` 有效期 | **12.0s** | 9.6 - 14.4 s | 长于 hostile-station 单 turret cell 击破时间（~18s），让玩家"锁一次完成战术段落"；短于 BOSS 决战总时长（30-90s），让玩家被迫每段重新锁敌（避免"锁完就走"破坏紧张度）|
| 锁敌距离半径 | **48 px**（= CELL × 1.5，CELL=32）| 38 - 58 px（CELL × 1.2-1.8）| 让玩家"看到敌人即可右键"，不必精确点中；与 pirate r=20 / station r=42 / hostile-station 单 cell ≈ 32 px 的视觉尺寸匹配 |
| 失效距离 | **800 px** | 640 - 960 px | 略大于 turret base range 480 与 hostile-station turret range 550，让"敌人飞出射程"自动脱锁，避免远端幽灵锁定 |
| `selectTarget` 生效延迟 | **0 帧**（同帧）| 不允许微调 | 锁敌即生效，让玩家立刻看到 turret / missile / shield 改变目标 |

### §1.2 失效条件（三类，任一触发即清空）

```js
function isPriorityTargetValid(now) {
  const pt = state.input.priorityTarget;
  if (!pt || !pt.enemy) return false;
  if (now > pt.validUntil) return false;                    // 超时 12s
  if (pt.enemy.hp <= 0) return false;                       // 敌死
  const dx = pt.enemy.x - state.station.x;
  const dy = pt.enemy.y - state.station.y;
  if (dx * dx + dy * dy > 800 * 800) return false;          // 失效距离
  return true;
}
```

> 实现时建议在 `update()` 主循环开头或 `selectTarget` 内调用 `isPriorityTargetValid(state.time)`，失效则 `state.input.priorityTarget = null`。

### §1.3 `selectTarget(origin, range)` 包装公式

```js
function selectTarget(origin, range) {
  const pt = state.input.priorityTarget;
  if (pt && isPriorityTargetValid(state.time)) {
    const dx = pt.enemy.x - origin.x;
    const dy = pt.enemy.y - origin.y;
    if (dx * dx + dy * dy <= range * range) return pt.enemy;  // 在调用方 range 内才返回
  }
  return nearestEnemy(origin, range);                          // 回退原逻辑
}
```

设计要点：

- `range` 检查在调用方 turret / missile / shield 自己的 range 内完成；优先目标不在该 cell 射程内时仍回退 nearestEnemy。
- 不破坏现有 `nearestEnemy` 函数签名，仅"包装"一层。
- `hasLineOfSight` 检查保留在 `updateTurret` 内调用方，不在 `selectTarget` 内做（保证 turret LOS 战术不被锁敌覆盖）。

### §1.4 锁敌行为的写入路径

| 输入 | 行为 | priorityTarget 字段变化 |
|---|---|---|
| 右键点击敌人（48 px 内）| 写入新锁定 | `{ enemy: foundEnemy, setAt: state.time, validUntil: state.time + 12.0 }` |
| 右键点击空白（48 px 内无敌）| 清空 | `null` |
| 锁定超时 12s | 自动清空 | `null` |
| 锁定敌人 hp ≤ 0 | 自动清空 | `null` |
| 锁定敌人距离 > 800 px | 自动清空 | `null` |
| `resetRun()` | 自动清空 | `null` |

> **§1.4 备注**：右键事件 `event.button === 2` + `event.preventDefault()` + `contextmenu` 抑制由主程 ADR §决定；本方案仅给出锁敌的"输入触发 → 字段变化"映射。

## §2. `salvoSize` 配置

> 设计意图：齐射规模 4 档循环（1 / 2 / 3 / all）让玩家可在"逐发试探"和"一波梭光"间灵活切换；初始 1 与跨 run 重置 1 保持"老玩家默认逐发"无突袭感；持久化仅 run 内，不写入 SAVE_KEY，避免新机制污染存档。

### §2.1 参数表

| 字段 | 推荐值 | 是否允许微调 | 设计理由 |
|---|---|---|---|
| 档位数量 | **4 档**（1 / 2 / 3 / all=999）| ❌ 不允许（制作人 / ADR 拍板）| 4 档已覆盖"逐发 / 双发 / 三发 / 全梭"4 种节奏；多档反而提升认知成本 |
| 初始值 | **1** | ❌ 不允许（制作人拍板）| 与 v0.6.0 体验等价（默认逐发齐射），让老玩家无突袭感 |
| 跨 run 重置 | **重置为 1** | ❌ 不允许（制作人拍板）| `resetRun()` 内显式重置；不写入 SAVE_KEY |
| 跨 level 行为 | **保持当前值**（run 内持久）| ❌ 不允许（制作人拍板）| `nextLevel()` 内不重置，让玩家在 run 内的调档习惯被保留 |
| `[` 减小档位 | **循环递减**（1 → all → 3 → 2 → 1）| ❌ 不允许（制作人拍板）| 循环让玩家不必担心"按多了"|
| `]` 增大档位 | **循环递增**（1 → 2 → 3 → all → 1）| ❌ 不允许（制作人拍板）| 同上 |
| HUD 调档高亮 | **0.5s**（调档后 salvoSize 数值短暂高亮）| PM 可在 0.4-0.6s 微调 | 视觉反馈"我刚改了什么"，避免误触 |

### §2.2 档位枚举

```js
const SALVO_SIZE_VALUES = [1, 2, 3, 999];  // 999 = all（保证 min(999, readyCount) = readyCount）

function cycleSalvoSize(direction) {
  const cur = state.missile.salvoSize;
  const idx = SALVO_SIZE_VALUES.indexOf(cur);
  const next = (idx + direction + SALVO_SIZE_VALUES.length) % SALVO_SIZE_VALUES.length;
  state.missile.salvoSize = SALVO_SIZE_VALUES[next];
  state.missile.salvoChangedAt = state.time;  // HUD 高亮起点
}

// `[` → cycleSalvoSize(-1)
// `]` → cycleSalvoSize(+1)
```

### §2.3 `fireMissiles` 内 salvoSize 应用

```js
function fireMissiles() {
  const now = state.time;
  if (now - (state.input.lastSalvoAt || 0) < 0.3) return;   // F 键防抖（§3）
  state.input.lastSalvoAt = now;

  // 找所有 ready 的导弹井（reload === 0 或 reload <= 0）
  const ready = [];
  for (const cell of state.station.cells.values()) {
    if (cell.facility !== "missile") continue;
    if (cell.reload <= 0) ready.push(cell);
  }
  if (ready.length === 0) return;

  // 按 cell key 字典序排序（ADR 拍板：确定性 + 玩家可预期"左边的先发"）
  ready.sort((a, b) => keyOf(a).localeCompare(keyOf(b)));

  const salvoSize = state.missile.salvoSize || 1;
  const count = Math.min(salvoSize, ready.length);
  for (let i = 0; i < count; i++) {
    fireSingleMissile(ready[i]);  // 等价 #missileBtn 单发逻辑
  }
}
```

### §2.4 持久化策略

| 字段 | 写入 SAVE_KEY | 跨 run 行为 | 跨 level 行为 |
|---|---|---|---|
| `state.missile.salvoSize` | ❌ 否 | 重置为 1 | 保持当前值 |
| `state.missile.salvoChangedAt` | ❌ 否 | 重置为 0 | 保持当前值 |
| `state.input.lastSalvoAt` | ❌ 否 | 重置为 0 | 保持当前值 |
| `state.input.priorityTarget` | ❌ 否 | 重置为 null | 重置为 null（敌人跨 level 已死）|

> **§2.4 备注**：所有 v0.7.0 新增字段均不写入 SAVE_KEY；旧存档读取后这些字段自动按默认值初始化（salvoSize=1 / priorityTarget=null / lastSalvoAt=0），保证向后兼容。

## §3. F 键齐射

> 设计意图：F 键是"扣扳机"的低延迟反馈，必须防抖避免误操作；与 `#missileBtn` 行为完全等价（同走 `fireMissiles()`，同受 `salvoSize` 限制），让"键盘 / UI 按钮"成为玩家偏好选择而非功能差异。

### §3.1 参数表

| 字段 | 推荐值 | 是否允许微调 | 设计理由 |
|---|---|---|---|
| 防抖间隔 | **0.3s** | ❌ 不允许（制作人拍板）| 短于人手按键最快连击间隔（~0.15s）但长于无意误触；防止"按 F 按出 5 连射" |
| 触发函数 | `fireMissiles()` | 不允许微调 | 与 `#missileBtn` 同入口 |
| 与 `#missileBtn` 关系 | **等价**（同走 `fireMissiles()` + 同受 `salvoSize` 限制）| 不允许微调 | 保证两种触发路径行为完全一致 |
| 防抖时间戳 | `state.input.lastSalvoAt` | - | 字段名建议；实现可调 |

### §3.2 防抖路径

```js
// fireMissiles 函数入口（已在 §2.3 给出）：
const now = state.time;
if (now - (state.input.lastSalvoAt || 0) < 0.3) return;
state.input.lastSalvoAt = now;
```

防抖在 `fireMissiles` 内部完成，覆盖 F 键 + `#missileBtn` 两条路径，保证两者行为统一。

## §4. HUD reload 进度条

> 设计意图：选中 missile / turret cell 时，让玩家在 selectedCellPanel 内直接看到 reload 进度，颜色按区间提示"装填阶段"；100% ready 时用亮蓝脉冲让"就绪"信号视觉化，匹配方向 §"导弹井 cell 装填完成时显示亮色发光环"语义。

### §4.1 参数表

| 字段 | 推荐值 | PM 可微调范围 | 设计理由 |
|---|---|---|---|
| 精度 | **1%** | 不允许微调 | 视觉粒度足够细但不抖动 |
| 更新间隔 | **10ms** | 不允许微调 | 每帧（60fps = 16.7ms）即可更新；10ms 是数据精度而非帧率 |
| 显示位置 | **selectedCellPanel 内**（已有面板）| 不允许微调 | 复用 v0.4.0 已有 UI 容器，不引入新 DOM |
| 显示触发 | 选中 missile / turret cell 时 | 不允许微调 | 不为所有 cell 都显示，避免 HUD 信息过载 |
| 高度 | **6 px**（建议）| PM 可在 4-8 px 微调 | 与 v0.5.0 research 进度条视觉风格一致 |
| 宽度 | selectedCellPanel 内 100% | - | 自适应面板宽度 |

### §4.2 颜色区间

| reload 进度区间 | 颜色 | hex | 视觉语义 |
|---|---|---|---|
| 0-25% | 暗红 | `#660000` | "刚开始装填，离 ready 还远" |
| 25-75% | 黄 | `#ddaa33` | "正在装填，过半但未就绪" |
| 75-100%（< 100%）| 亮绿 | `#33cc33` | "快好了" |
| 100%（ready）| 亮蓝（脉冲）| `#5588ff` | "已就绪，可发射"|

进度计算（reload 字段语义：从 maxReload 倒数到 0；0 = ready）：

```js
const maxReload = getCellStat(cell, "reload");   // 1.6s
const progress = 1 - (cell.reload / maxReload);  // 0 = 刚发射，1 = ready
const pct = Math.floor(progress * 100);          // 0-100 integer

let color;
if (cell.reload <= 0) {
  // ready：亮蓝脉冲
  const pulse = 0.7 + 0.3 * Math.sin(state.time * Math.PI * 2);  // 周期 1s, alpha 0.4-1.0
  color = `rgba(85, 136, 255, ${pulse})`;
} else if (pct < 25) {
  color = "#660000";
} else if (pct < 75) {
  color = "#ddaa33";
} else {
  color = "#33cc33";
}
```

### §4.3 主 HUD 武器装填指示器（方向 §"HUD 与视觉反馈"）

| 字段 | 推荐值 | 说明 |
|---|---|---|
| 显示位置 | 资源条下方 | 与方向一致；不挤占任务 / 告警空间 |
| 布局 | 单行水平 | 文字一行流：导弹井 X/Y · 齐射 N · 下一发 Y.Y 秒 · 炮塔 X/Y · K 个 LOS 遮挡 |
| 下一发倒计时精度 | **0.1 秒** | 显示如 "1.2 秒"，精确到一位小数 |

| 末尾提示文字 | "F=齐射 · [/]=调档 · 右键锁敌" | 灰色字（建议 `#888888`），不喧宾夺主 |

### §4.4 cell 内 ready 发光环（导弹井渲染分支）

| 字段 | 推荐值 | 说明 |
|---|---|---|
| 触发 | missile cell `reload <= 0` 时 | 装填完成显示亮色发光环 |
| 颜色 | 与 §4.2 ready 同款亮蓝 `#5588ff` | 视觉一致性 |
| 半径 | cell 自身半径（约 14 px）+ 4 px | 略大于 cell 本体 |
| pulse | 周期 1.0s（与 reload 进度条 ready 同步）| 视觉节奏统一 |

## §5. LOS 遮挡视觉警告

> 设计意图：玩家清楚"这个炮塔被自机挡住了不开火"，但只在"刚被挡上"瞬间警告 0.5 秒，避免持续遮挡产生频闪；5 秒节流确保"持续遮挡"不重复触发。

### §5.1 参数表

| 字段 | 推荐值 | PM 可微调范围（±20%）| 设计理由 |
|---|---|---|---|
| 持续时间 | **0.5s** | 0.4 - 0.6 s | 足够玩家看见，但不持久打扰 |
| alpha 衰减曲线 | **线性 1.0 → 0**（0.5s 内）| 不允许微调（曲线类型） | 简单可预期；非线性曲线增加实现复杂度 |
| 颜色 | 橙色 `#ff8800` | 不允许微调 hex | 警告色但比红色温和（红色保留给优先目标锁定）|
| 防抖间隔 | **5.0s**（同一 cell 5s 内不重复触发）| 4.0 - 6.0 s | 避免持续遮挡时频闪 |
| 触发时机 | turret cell 当前帧因 LOS 遮挡未开火，且距上次触发 > 5s | 不允许微调 | 节流逻辑 |

### §5.2 触发条件

```js
function maybeFlashLosBlocked(cell) {
  const now = state.time;
  if ((cell._losBlockedAt || 0) + 5.0 > now) return;  // 5s 节流
  cell._losBlockedAt = now;                            // 记录闪烁起始
}

// updateTurret 内：
if (!hasLineOfSight(turretOrigin, target)) {
  maybeFlashLosBlocked(cell);
  return;  // 不开火
}
```

### §5.3 渲染（drawCell / renderStation 内）

```js
const flashElapsed = state.time - (cell._losBlockedAt || -Infinity);
if (flashElapsed < 0.5) {
  const alpha = 1.0 - (flashElapsed / 0.5);  // 线性 1.0 → 0
  ctx.strokeStyle = `rgba(255, 136, 0, ${alpha})`;
  ctx.lineWidth = 2;
  ctx.strokeRect(cell.renderX - cell.r, cell.renderY - cell.r, cell.r * 2, cell.r * 2);
}
```

### §5.4 与 v0.6.0 hostile-station 互动

- hostile-station turret cell 不显示玩家 turret 的 LOS 警告（v0.7.0 LOS 警告仅玩家 station 的 turret cell）。
- 玩家 turret 朝 hostile-station 优先 cell 时，若被自机其他 cell 遮挡，正常触发 LOS 警告（节流 5s）。

## §6. 优先目标锁定框

> 设计意图：让玩家"看一眼就知道我锁了谁"——红色十字 + 旋转脉冲圆环区别于普通选中 / hostile-station cell 高亮；视觉风格与威胁度匹配（红色 = 我在追打这个）。

### §6.1 参数表

| 字段 | 推荐值 | PM 可微调范围 | 设计理由 |
|---|---|---|---|
| 红色十字线宽 | **2** | PM 可在 1.5-2.5 微调 | 与 cell 边框线宽一致 |
| 红色十字长 | **20** | PM 可在 16-24 微调 | 中等粗细，清晰但不遮挡敌人本体 |
| 旋转圆环半径 | `enemy.r × 1.5`（默认敌人）；hostile-station 用 **80 px**（固定） | 不允许微调（hostile-station 80）| 普通敌人按比例；BOSS 用固定 80 px 避免按 BOSS 聚合 r 算出过大圆环 |
| 旋转角速度 | **1 rad/s** | PM 可在 0.8-1.2 rad/s 微调 | 约 57°/s = 6.3 秒一圈，缓慢可见但不晕 |
| pulse 周期 | **1.0 秒** | PM 可在 0.8-1.2s 微调 | sin 波 alpha 0.5 → 1.0 → 0.5 |
| 圆环颜色 | 红色 `#ff4444` | 不允许微调 hex | 区别于 cellSelect / hostile-station 的其他高亮色 |
| 渲染层级 | 在 enemy 渲染之上 / HUD 之下 | 不允许微调 | 覆盖在敌人之上但不遮挡 HUD |

### §6.2 渲染公式

```js
function renderPriorityTarget(ctx) {
  if (!state.input.priorityTarget) return;
  const pt = state.input.priorityTarget;
  if (!isPriorityTargetValid(state.time)) return;

  const enemy = pt.enemy;
  const isHostileStation = enemy.kind === "hostile-station";
  const radius = isHostileStation ? 80 : enemy.r * 1.5;

  const pulse = 0.75 + 0.25 * Math.sin(state.time * Math.PI * 2 / 1.0);  // 0.5-1.0
  const rot = (state.time * 1.0) % (Math.PI * 2);                         // 1 rad/s

  ctx.save();
  ctx.translate(enemy.x, enemy.y);

  // 红色十字
  ctx.strokeStyle = `rgba(255, 68, 68, ${pulse})`;
  ctx.lineWidth = 2;
  ctx.beginPath();
  ctx.moveTo(-10, 0); ctx.lineTo(10, 0);
  ctx.moveTo(0, -10); ctx.lineTo(0, 10);
  ctx.stroke();

  // 旋转脉冲圆环（虚线 / 4 段弧）
  ctx.rotate(rot);
  ctx.strokeStyle = `rgba(255, 68, 68, ${pulse})`;
  ctx.lineWidth = 2;
  for (let i = 0; i < 4; i++) {
    ctx.beginPath();
    ctx.arc(0, 0, radius, i * Math.PI / 2 + 0.1, i * Math.PI / 2 + Math.PI / 2 - 0.1);
    ctx.stroke();
  }
  ctx.restore();
}
```

### §6.3 与 v0.6.0 hostile-station cell 视觉的协调

- hostile-station 单 cell 选中（玩家左键点 BOSS 某个 cell 查看 hp）走 v0.6.0 cellSelect 高亮路径，颜色与 v0.7.0 优先目标锁定框无冲突（cellSelect 用其他色系）。
- 玩家右键 hostile-station 整体（不是单 cell）写入优先目标；锁定框圆环半径用固定 80 px（覆盖 BOSS 整体）。
- 锁定框可与 cellSelect 同时显示（玩家既"选了 BOSS 看血"也"锁了 BOSS 优先打"）。

### §6.4 锁定框可见性

| 状态 | 锁定框 |
|---|---|
| 锁定有效 | 显示（按 §6.2 渲染）|
| 锁定超时 | 不显示（已清空）|
| 锁定敌死 | 不显示 |
| 锁定距离 > 800 | 不显示 |
| 锁定敌人移出屏幕 | 不显示（自然不在 viewport 内）|

## §7. 体验目标对应的回归校验（共 23 条，超 ≥ 18 下限）

> 设计意图：本节是 PM 节点 A / B / C 全量回归与节点 C 接受测试的可量化校验点；按方向 §"体验目标"三类 + "不回潮"组织。

### §7.1 看得见战斗节奏（6 条）

1. **导弹井装填指示**：HUD 武器行显示 `导弹井 X/Y 就绪 · 下一发 Y.Y 秒`，X = `reload <= 0` 的 missile cell 数，Y = missile cell 总数，下一发倒计时精度 **0.1 秒**（取所有未 ready 中 reload 最小值）。
2. **炮塔工作 + LOS 遮挡**：HUD 武器行显示 `炮塔 X/Y 工作 · K 个 LOS 遮挡`，X = 当前帧成功开火的 turret cell 数，Y = turret cell 总数，K = 当前帧 `_losBlockedAt` 在 0.5s 内的 turret cell 数。
3. **salvoSize 当前值显示**：HUD 武器行显示 `齐射 N`，N = `state.missile.salvoSize`（1 / 2 / 3 / all）；调档后 0.5 秒内数值短暂高亮。
4. **末尾提示文字**：HUD 武器行末尾显示灰色 `F=齐射 · [/]=调档 · 右键锁敌`，颜色 `#888888`，字号比主信息小（建议 11px vs 主 13px）。
5. **missile cell reload 进度条颜色**：选中 missile cell 时 selectedCellPanel 内显示 reload 进度条，颜色按 §4.2 区间正确变化（0-25% `#660000` / 25-75% `#ddaa33` / 75-100% `#33cc33` / 100% 蓝色脉冲 `#5588ff`）。
6. **turret cell reload 进度条**：选中 turret cell 时同样显示 reload 进度条；已就绪（reload <= 0）显示蓝色脉冲。

### §7.2 能锁定优先目标（6 条）

7. **右键锁敌写入**：右键点击敌人（鼠标 worldPos 距敌人 ≤ 48 px = CELL × 1.5），正确写入 `state.input.priorityTarget = { enemy, setAt: state.time, validUntil: state.time + 12.0 }`。
8. **右键空白清空**：右键点击空白处（48 px 内无敌），`state.input.priorityTarget = null`。
9. **锁定框视觉**：优先目标在世界中显示红色十字 `#ff4444`（线宽 2 / 长 20）+ 旋转脉冲圆环（角速度 1 rad/s / pulse 周期 1.0s / 半径 enemy.r × 1.5；hostile-station 用固定 80 px）。
10. **超时自动失效**：锁定后 12 秒不操作自动失效，`state.input.priorityTarget` 变为 `null`，锁定框消失。
11. **距离自动失效**：锁定敌人距离 station > 800 px 自动清空。
12. **敌死自动失效**：锁定敌人 hp ≤ 0 自动清空。
13. **turret / missile / shield 尊重优先目标**：`updateTurret` / `fireMissiles` / `shieldDirection` 内 `nearestEnemy(origin, range)` 改为 `selectTarget(origin, range)`，优先目标在 range 内即返回；不在 range 内回退 nearestEnemy（命中其他 enemy 路径完全等价 v0.6.0，仅 `selectTarget` 调用点切换）。

### §7.3 能控制齐射时机（6 条）

14. **F 键齐射触发**：按 F 键触发 `fireMissiles()`，仅发射 `min(salvoSize, readyCount)` 个最早装填好的导弹井（按 cell key 字典序排序）。
15. **F 键防抖**：连按 F 键间隔 < 0.3 秒，仅触发一次（`state.input.lastSalvoAt` 节流）。
16. **`[` 减小 salvoSize**：按 `[` 键 salvoSize 循环递减（1 → all → 3 → 2 → 1）。
17. **`]` 增大 salvoSize**：按 `]` 键 salvoSize 循环递增（1 → 2 → 3 → all → 1）。
18. **salvoSize 跨 run 重置**：`resetRun()` 后 `state.missile.salvoSize === 1`；跨 level 保持当前值（`nextLevel()` 不重置）。
19. **`#missileBtn` 等价 F 键**：点击 UI `#missileBtn` 与按 F 键行为完全一致（同走 `fireMissiles()`，同受 salvoSize 限制，同受 0.3s 防抖限制）。

### §7.4 不回潮（5 条）

20. **v0.6.0 BOSS 战难度不下降**：玩家完全不用 v0.7.0 新功能（右键不点 / F 不按 / salvoSize 保持 1）时，level 3-5 hostile-station BOSS 击毁时间与 v0.6.0 基线完全等价（数据采集：同 seed 跑 5 局对比）；玩家用新工具后体验只会更好（锁敌 + 节奏控制），难度不上升。
21. **现有按键完全等价**：Space / Enter / WASD / 1-9 / 0 / - / = / 左键 / 鼠标拖动旋转 station 行为完全等价 v0.6.0；v0.7.0 仅新增右键 / F / `[` / `]` 四个按键，不修改任何现有按键。
22. **8 种 OBJECTIVE_TYPES + 5 种 enemy kind 行为等价**：mine / explore / battle / survive / salvage / escort / guardian / assault 8 种任务行为完全等价 v0.6.0；pirate / scout / station / guardian / hostile-station 5 种敌人 AI 行为完全等价。
23. **SAVE_KEY 不变 + 旧存档兼容**：`SAVE_KEY` 字符串不变；旧存档读取后 `state.input.priorityTarget` 默认 `null`，`state.missile.salvoSize` 默认 `1`，`state.input.lastSalvoAt` 默认 `0`，`cell._losBlockedAt` 默认 `undefined`（按需初始化）；不写入 `state.meta`。

> **§7 校验条数：23 条**（≥ 18 下限）。

## §8. 平衡兜底

> 设计意图：本节是数值方案的"安全网"——即使具体数值微调失误，硬保护体验不爆炸；同时确保 v0.6.0 / v0.5.0 / v0.4.0 等已发布版本的平衡完全不动。

### §8.1 priorityTarget 不破坏现有数值平衡

- `selectTarget` 仅切换"找敌函数"，**不改 damage / range / reload / projectileSpeed / projectileRadius** 任何武器数值。
- 锁敌只是"换目标"，不是"加伤害"；玩家锁 BOSS 后输出 DPS 与不锁完全相同。
- `selectTarget` 内的 range 检查在调用方 cell 的 range 内完成；优先目标不在 range 内时回退 `nearestEnemy`，不会突破 turret base range 480 / hostile-station turret range 550。
- `hasLineOfSight` 保留在 `updateTurret` 调用方，**不在 `selectTarget` 内做**——保证 v0.6.0 hostile-station 侧面装甲战术（玩家绕到 BOSS 侧面让 turret 被 armor 挡住）不被锁敌破坏。

### §8.2 salvoSize 默认 1 保持 v0.6.0 等价

- 初始值 1（与 v0.6.0 单发齐射体验等价）。
- 跨 run 重置 1（防止玩家无意中"上局调到 all 这局突然全梭"）。
- 玩家发现新机制后再升级 salvoSize 才体验差异化；老玩家无突袭感。

### §8.3 F 键 0.3s 防抖避免连发

- F 键最快连击约 0.15-0.20s（人手物理极限），0.3s 防抖足够覆盖。
- 防抖在 `fireMissiles` 内部完成（§3.2），覆盖 F 键 + `#missileBtn` 两条路径；玩家按 F 键 + 同时点 UI 按钮也不会触发双发。

### §8.4 LOS 遮挡警告 5s 节流避免频闪

- 同一 cell 5s 内不重复触发橙色环；持续遮挡场景下视觉上只闪烁一次。
- 0.5s 持续时间 + 5s 节流 = 占空比 10%（5s 内最多 0.5s 亮起），不打扰玩家。

### §8.5 HUD 信息密度不爆炸

- 武器行单行水平布局，不挤占任务 / 告警空间（与方向 §"HUD 与视觉反馈"一致）。
- 提示文字（"F=齐射 · [/]=调档 · 右键锁敌"）使用灰色 `#888888` 弱化，避免喧宾夺主。
- 小屏 / 移动设备的响应式布局留待 v0.7.1 优化（v0.7.0 默认桌面端）。

### §8.6 优先目标渲染性能

- 锁定框最多 1 个（同时只能锁 1 个敌人）；CPU / GPU 开销极小。
- 渲染层级在 enemy 之上 / HUD 之下，不影响 hostile-station 大量 cells 渲染顺序。
- pulse + 旋转计算 O(1)，每帧仅 1 次 sin + 1 次 mod 运算。

### §8.7 跨 level / 跨 run 字段重置

| 字段 | nextLevel() | resetRun() | loadSave() 旧存档兼容 |
|---|---|---|---|
| `state.input.priorityTarget` | null | null | null |
| `state.missile.salvoSize` | 保持 | 1 | 1 |
| `state.missile.salvoChangedAt` | 保持 | 0 | 0 |
| `state.input.lastSalvoAt` | 0 | 0 | 0 |
| `cell._losBlockedAt`（各 turret cell）| 重置（cells 重建）| 重置（cells 重建）| undefined（按需懒初始化）|

### §8.8 与 v0.6.0 hostile-station BOSS 战的具体兼容点

| 场景 | v0.7.0 行为 | v0.6.0 行为 | 兼容性 |
|---|---|---|---|
| 玩家不锁敌打 BOSS | turret / missile / shield 走 `nearestEnemy` 路径，行为等价 v0.6.0 | nearestEnemy 直接调用 | ✅ 等价 |
| 玩家锁定 BOSS turret-A | turret / missile / shield 优先打 turret-A，速度更快 | 无此机制 | ✅ 玩家工具升级，难度只降不升 |
| 玩家锁定 BOSS core | 三种武器优先打 core，速胜路线更可靠 | 无此机制，需手动调 station 角度 | ✅ 玩家工具升级 |
| 玩家锁定 minion | 三种武器优先打 minion，BOSS 暂时不被打 | 无此机制 | ✅ 玩家自主选择，平衡不变 |
| BOSS 在 800 px 外 | 锁定自动失效 | 无锁定概念 | ✅ 等价 |
| BOSS 部分 cell 被自机遮挡 | turret 仍走 LOS 检查 + 显示橙色警告 | LOS 检查存在但无警告 | ✅ 视觉增强，行为等价 |

## §10. 自检结论 + 决策点

### §10.2 待制作人 / 主程拍板的开放决策点

以下决策点本方案给出推荐选项，但**最终由制作人 / 主程决定**（实现前需明确）：

1. **`salvoSize` 是否需要持久化到 SAVE_KEY**：
   - 本方案推荐：**不持久化**（与方向 §"关键技术疑问 #3"一致）；仅 run 内，跨 run 重置 1。
   - 节点 C 体验测试若发现玩家"频繁调档很烦"，可在 v0.7.1 引入"上一 run 末次 salvoSize 写入 meta，下 run 起始读取"语义。

2. **`priorityTarget` 是否需要可视化倒计时**：
   - 本方案推荐：**不显示**（方向 §"HUD 与视觉反馈"未要求；§6 锁定框已通过 pulse 提供"还活着"视觉反馈）。
   - 节点 B 测试若玩家反馈"不知道还剩几秒会失效"，可在 v0.7.1 在锁定框下方添加细圆弧进度条（从满到空表示倒计时）。

3. **F 键防抖时间精确值**：
   - 本方案推荐：**0.3s**（与方向 §"数值边界建议"一致）。
   - 节点 B 测试若发现"按 F 偶尔触发不灵敏"，可调到 0.2s；若发现"误连发太多"，可调到 0.4s；但建议保留在 0.2-0.4s 区间内（不放入 §9.1 ±20% 区间是为了强化"防抖严肃性"）。

4. **HUD reload 进度条 100% ready 是否需要脉冲效果**：
   - 本方案推荐：**需要**（亮蓝 `#5588ff` 脉冲，与导弹井 cell 的 ready 发光环统一）。
   - 节点 B 测试若发现"脉冲让玩家眼花"，可降级为"纯亮蓝固定色"（无 pulse）。

5. **优先目标锁定框对 hostile-station 单 cell 的支持**：
   - 本方案推荐：**右键 hostile-station 整体锁定 BOSS 实体**（圆环用固定 80 px 覆盖整体）；不支持"锁定 BOSS 单 cell"语义。
   - 节点 C 测试若发现"无法精确锁定 BOSS 武器 cell"成为战术痛点，可在 v0.7.1 引入"右键 hostile-station 时按 Shift 锁定具体 cell"扩展。

6. **`#missileBtn` UI 按钮是否需要 salvoSize 显示**：
   - 本方案推荐：**显示**（按钮文字改为 `齐射(N)` 或 `🚀 ×N`）。
   - 节点 B 测试若发现"UI 信息过载"，可保持原按钮文字不变，仅在 HUD 武器行显示 salvoSize。
