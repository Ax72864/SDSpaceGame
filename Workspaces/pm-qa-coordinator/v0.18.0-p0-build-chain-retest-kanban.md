# v0.18.0-p0-build-chain-retest PM 看板

## 本轮目标

版本名：`v0.18.0-p0-build-chain-retest`。

本轮目标是关闭 v0.17.0 遗留的 P0 建造链前置缺口：用正确真实操作复测“先扩 `frame`，再点击已有空 `frame` 替换为 `power` / `mining` / `turret` / `repair`”是否成立。PM 负责把复测方案、证据口径、失败时最小修复准入、原路径复测和制作人最终决定组织成可执行闭环。

## 背景

- 制作人方向文档：`Workspaces/game-producer-lead-designer/v0.18.0-p0-build-chain-retest-direction.md`。
- v0.17.0 不进入发布准备，原因是正确真实操作建造链没有复测关闭。
- v0.17.0 triage 判断更像取证操作问题：脚本或 CDP 坐标可能没有点击回已有空 `frame`，而是继续点击空邻格；该证据不支持修改建造合法性规则。
- v0.18.0 要证明的是玩家按当前规则能否完成首局最小建设动作，而不是辅助脚本能否构造成功状态。
- 若正确操作仍失败，本轮只允许围绕最小新手提示、点击反馈或取证断言做修复判断，并必须按原失败路径复测。

## 风险定级

- 风险等级：中低风险。
- 中低风险前提：仅做正确操作复测、PM 证据、失败时最小新手提示/点击反馈/取证断言修复和原路径复测。
- 高风险升级触发：若触及建造合法性规则、资源成本、设施数值、战斗判定、AI、RNG、存档结构、`SAVE_KEY`、meta schema、依赖安装、发布流程、tag 或 `Releases/` 归档，必须暂停并升级为高风险流程或等待用户确认。
- 用户确认触发：若涉及安装依赖、浏览器驱动、测试框架、修改系统/浏览器全局环境、权限、服务、删除文件、覆盖文件或其他本机环境危险操作，必须先说明风险并等待用户确认。
