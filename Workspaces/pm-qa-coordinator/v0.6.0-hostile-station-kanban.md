# v0.6.0 敌方空间站 PM 看板

## 本轮目标

让玩家在每局 level 3-5 中段遭遇 BOSS 级敌方空间站决战：通过 cells 化多目标击杀形成战术攻击决策，击毁后爆出大量 wreck fragment，并与 v0.3.0 拾荒系统、v0.5.0 研发 build 循环衔接，形成"升级 build -> 决战 -> 拾荒奖励 -> 进入下个星系"的中期高潮。

## 背景与制作人方向

制作人方向文档：`Workspaces/game-producer-lead-designer/v0.6.0-hostile-station-direction.md`，commit `0a6ea69`。基线为 HEAD `0a6ea69`（v0.5.0 发布后），当前版本风险级别为项目高风险。

### 方向要点摘要

- 新增 `enemy.kind = "hostile-station"`，在标准 enemy 字段基础上扩展 `cells: Map<key, cell>`，建议复用 v0.3.0 `fragment.cells` 与 `createCell` 工厂。
- hostile-station 不以整体 HP 为主要击杀路径，HP 显示可由 cells 聚合；击杀判定由 ADR 明确，制作人建议 core cell 被破坏即死。
- 玩家弹道命中 hostile-station 时进入 cell 粒度扣血，支持先打武器、动力、防御或核心的战术选择。
- hostile-station 死亡后复用 `makeWreckFragment` 生成 6-12 个 `origin = "enemy-wreck"` wreck fragment，衔接拾荒桥接奖励。
- AI 采用近静态悬停，不做移动型敌方空间站；武器 cell 复用现有 projectile / missile 路径并独立 reload / fire。
- 新增 `assault` 任务类型，由任务主动生成 hostile-station，注册为第 8 种 `OBJECTIVE_TYPES`，并通过 `notifyObjective("hostileStationKilled")` 完成。
- HUD 需要提供红色警报、HP% / 距离提示和击毁后的"主目标已摧毁"反馈。

### 体验目标

1. **可感知 BOSS 决战**：玩家在 level 3-5 遇到 hostile-station 时，能立刻感受到规模、生命池、攻击密度和 cells 逐块脱落反馈不同于普通杂兵。
2. **可战术攻击决策**：玩家可以选择先拆武器 cell、削弱 power / shield / armor，或直击 core 快速终结，让同一 build 在不同战术下出现不同结果。
3. **可衔接奖励循环**：击毁后生成 6-12 个 wreck fragment，玩家拖去停靠合并并获得设施收益，形成中期决战后的拾荒奖励闭环。

### 必做范围

- 工程基础：新增 hostile-station enemy 类型、cells 数据结构、cell 粒度命中与伤害路径、HP 聚合显示和击杀判定。
- AI 与战斗：近静态悬停、多武器槽独立开火、武器 cell 被破坏停火、召唤 pirate minion、死亡震屏与粒子反馈。
- 死亡与奖励：击毁后生成 6-12 个 `origin = "enemy-wreck"` wreck fragment，爆出位置在站体附近并可按 v0.3.0 salvage 路径桥接。
- 任务与生成：新增 `assault` 任务类型，level 3-5 出现，单星系上限 1 次，奖励含 metal、researchReward 和 rewardMultiplier。
- HUD 与反馈：spawn 红色警报、HP% + 距离显示、击毁后 1-2 秒完成提示。
- 文档与验证：主程 ADR、数值方案、专项审核、PM 浏览器级 + 主观回归、制作人最终决定与发布安排。

### 不做范围

- 不做移动型敌方空间站。
- 不做敌方空间站 cells 升级 / 改造系统。
- 不做敌方 NPC 阵营。
- 不做新武器机制。
- 不做 enemy / npc 抽象重构。
- 不修改 guardian。
- 不修改 v0.5.0 研发系统。
- 不破坏 v0.4.0 任务系统。
- 不修改 `SAVE_KEY` / 局外存档结构。
- 不做 v0.5.1 候选项。
- 不做联机 / 异步对战。

### 兼容要求

- v0.1.x / v0.2.0 / v0.2.1 / v0.3.0 / v0.4.0 / v0.5.0 全部回归点必须通过。
- 现有 4 种 enemy kind（asteroid / pirate / station / guardian）行为完全等价；新增 hostile-station 不影响既有 enemy 路径。
- v0.3.0 fragment 系统（player + wreck origin）行为不破坏；新增 `enemy-wreck` origin 与 `player` / `wreck` 区分但共用 cells 数据结构。
- v0.4.0 任务系统 7 种 `OBJECTIVE_TYPES` 保留；新增 `assault` 为第 8 种。
- v0.5.0 研发系统对玩家攻击 hostile-station 自动生效；hostile-station 武器 cell 可走 `getCellStat` fallback，但不应被玩家研发升级。
- `SAVE_KEY` 不变；旧存档兼容，无 assault / hostileStation 字段时默认安全。
- 终局 guardian 在 level 6 仍然出现，与 hostile-station 在 level 3-5 不冲突。

## 风险定级

- 风险等级：项目高风险。
- 定级原因：涉及核心玩法中期 BOSS 决战、新 enemy 实体类型、hostile-station cells 化、多武器 cell 行为、死亡 wreck 爆出、任务系统新增第 8 种 objective，以及玩家弹道 / enemy 死亡 / fragment / wreck / reward 多条伤害与奖励路径改动。
- 主要风险：新 enemy schema 若处理不当会破坏现有 enemy kind；cells 复用若生命周期不清会污染 fragment / wreck / player station；多个伤害路径改动可能造成玩家弹道误伤 fragment、guardian 回潮或 hostile-station 无法正确击杀；`enemy-wreck` origin 可能破坏 v0.3.0 salvage 桥接；`assault` 任务可能影响 v0.4.0 7 种任务选择、跨关清理和奖励倍率；hostile-station weapon cell 若错误进入玩家研发升级，会破坏 v0.5.0 研发边界。
- 高风险流程：制作人方向 -> PM 拆分 -> 主程技术 ADR -> 数值方案 -> 核心实现 -> 专项审核 -> PM 浏览器级 + 主观回归 -> 制作人最终决定与发布安排。
- 审核基底约束：专项审核使用 `expert-agent-planner-gpt55`，必须与执行不同基底。
