# v0.7.0 战斗操控 PM 看板

## 本轮目标

把已在自动追鼠标的 `state.station.angle` 升级为玩家的战斗操作信号，通过武器装填可视化、右键锁敌优先目标和键盘手动齐射，让 v0.6.0 hostile-station BOSS 决战与日常战斗从"按按钮等敌死"升级为"看节奏选目标卡时机扣扳机"。

## 背景与制作人方向

制作人方向文档：`Workspaces/game-producer-lead-designer/v0.7.0-combat-control-direction.md`，上游指定 commit `9cf2b23`。基线为 HEAD `9cf2b23`（v0.6.0 发布后），当前版本风险级别为项目中高风险。

### 方向要点摘要

- 新增 `state.input.priorityTarget`，玩家右键点击敌人后，turret / missile / shield 通过 `selectTarget(origin, range)` 优先选择锁定目标。
- 新增 `state.missile.salvoSize`，玩家可用 F 键手动齐射，并用 `[` / `]` 在 1 / 2 / 3 / all 四档之间循环调整齐射规模。
- 保留 `nearestEnemy` fallback、`hasLineOfSight` 战术遮挡、现有左键点击、拖动旋转、Space / Enter / 1-9 / WASD 等输入行为。
- HUD 增加武器装填信息、selected cell reload 进度条、salvoSize 当前值与 F + `[` / `]` 可发现性提示。
- 世界视觉增加优先目标锁定框、导弹井 ready 发光环和 turret LOS 遮挡橙色瞬时警告。

### 体验目标

1. **看得见战斗节奏**：玩家随时清楚导弹井 ready / reload、炮塔工作数量和 LOS 遮挡数量。
2. **能锁定优先目标**：玩家可右键锁定 hostile-station core / turret 或其他关键敌人，让武器与护盾优先响应战术意图。
3. **能控制齐射时机**：玩家可用 F 键按当前 salvoSize 发射就绪导弹，选择试探、保留或全梭。

### 必做范围

- 工程基础：`state.input.priorityTarget`、`selectTarget(origin, range)`、`state.missile.salvoSize`、右键输入守卫和 `contextmenu` 抑制。
- 武器系统：`updateTurret`、`updateShield` / `shieldDirection`、`fireMissiles` 接入 `selectTarget` 与 salvoSize。
- 输入系统：右键锁敌 / 清空锁敌、F 键齐射、`[` / `]` 调整齐射规模，旧输入完全保留。
- HUD 与视觉反馈：主 HUD 武器行、selectedCellPanel reload 进度条、优先目标锁定框、导弹井 ready 标识、turret LOS 遮挡警告。
- 文档与验证：主程 ADR、数值方案、核心实现、专项审核、PM 浏览器级 + 主观回归、制作人最终决定与发布安排。

### 不做范围

- 不改炮塔朝向。
- 不改 shield 方向性。
- 不引入新武器类型。
- 不引入弹药 / 能量 / 充能等新资源系统。
- 不修改 v0.5.0 研发系统 / v0.6.0 hostile-station 实体。
- 不修改 `SAVE_KEY` / 局外存档结构。
- 不破坏 v0.6.0 hostile-station BOSS 战平衡。
- 不做联机 / 异步对战。
- 不做局外天赋树重构。
- 不做新探索 / 地图系统。

### 兼容要求

- v0.1.x / v0.2.0 / v0.2.1 / v0.3.0 / v0.4.0 / v0.5.0 / v0.6.0 全部回归点必须通过。
- 现有 4 种 enemy kind + hostile-station 行为完全等价。
- 现有 8 种 `OBJECTIVE_TYPES` 不破坏。
- 现有 fragment / wreck / enemy-wreck / salvage 路径不破坏。
- v0.5.0 研发系统 / `getCellStat` / `hullMod` / `weaponMod` / `techLevel` 完全保留。
- v0.6.0 hostile-station 决战体验不下降，shield 优先朝玩家锁定目标但不改变敌我基础数值。
- 玩家可完全不用 v0.7.0 新功能继续游戏，右键不点、F 不按、salvoSize 保持 1 时玩法等价 v0.6.0 默认逐发齐射。
- `SAVE_KEY` 不变；旧存档兼容，无 `priorityTarget` / `salvoSize` 字段时默认安全。

## 风险定级

- 风险等级：项目中高风险。
- 定级原因：涉及核心战斗系统扩展、输入系统扩展、HUD 重构、多个武器路径接入 `selectTarget`。
- 主要风险：右键与左键 / 拖动旋转事件互相污染；`selectTarget` 若失效规则不清可能让 turret / missile / shield 锁死空引用；salvoSize 改造可能改变导弹默认手感和 BOSS 战平衡；HUD 武器行可能遮挡任务 / 资源信息；LOS 遮挡提示若每帧闪烁会造成视觉噪音；旧存档缺字段必须安全默认。
- 中高风险流程：制作人方向 -> PM 拆分 -> 主程技术 ADR -> 数值方案 -> 核心实现 -> 专项审核 -> PM 浏览器级 + 主观回归 -> 制作人最终决定与发布安排。
- 审核基底约束：专项审核使用 `expert-agent-planner-gpt55`，必须与执行不同基底。
