# v0.12.0-station-design-feedback PM 看板

## 本轮目标

版本名：`v0.12.0-station-design-feedback`。

本轮目标是把现有建造、资源、移动、战斗、研发、星系图和 Meta 成长从“功能可用”推进到“玩家能理解自己的空间站设计是否合理”。重点是空间站设计反馈与建造可读性：玩家打开建造面板、选择设施或检查右侧状态时，应能看懂设施角色、资源/电力短板、采矿范围、武器/护盾/维修状态和关键风险提示。

本版本不新增建筑类型，不修改存档，不修改核心玩法规则，不替玩家自动修复空间站。

## 背景

- 制作人方向文档：`Workspaces/game-producer-lead-designer/v0.12.0-station-design-feedback-direction.md`。
- 上游指定 commit：`8dd5734`。

- v0.12.0 可以继续开发，不把 v0.11.0 人工发布验收作为启动阻塞；但 v0.12.0 是否进入发布准备，仍需 #10 制作人最终决定。

## 制作人方向摘要

- 建造入口从按钮列表升级为可比较的设计选择，至少能显示设施角色、成本、电力、可负担状态和已建数量。
- 放置/选择反馈需要解释资源不足、未连接核心、电力压力、采矿范围、武器/导弹边界、护盾能力和维修覆盖。
- 右侧状态或更多状态中增加空间站设计健康摘要，用 4-6 个短项总结电力、采矿、推进、火力、防御和维修短板。
- 低噪声问题提示只在关键风险出现时提醒，不把普通轻微低效刷成持续红字。
- 增加只读测试/验收钩子，输出 build palette、design health、selected cell diagnostics、resource reachability、power margin 等验证信息。
- 所有诊断只解释现有系统，不改变数值规则、存档结构、RNG、任务/事件注册表或战斗判定。

## 风险定级

- 风险等级：中风险。
- 中风险来源：UI 信息架构、诊断准确性、提示优先级、只读测试钩子字段、文案与实际规则一致性。
- 风险控制：默认只做展示、只读诊断、文案和测试输出；任何触及数值公式、资源/电力结算、战斗/护盾/维修判定、存档结构或 RNG 的行为必须升级为高风险并暂停执行。
- 高风险升级触发：修改 `SAVE_KEY`、meta schema、资源生产/消耗公式、电力结算公式、战斗命中/射线/导弹/护盾/装甲/维修判定、敌人/任务/星系图推进或 RNG。

## QA 回归点

### A. 空存档建造采矿链

- [ ] A01. 清空/重置运行后进入游戏，建造面板默认可打开且无控制台阻断错误。
- [ ] A02. 空存档建造面板能区分采矿、供电、推进、武器、防御、维修、支撑设施角色。
- [ ] A03. 采矿设施卡显示用途、成本、电力影响、可负担状态和已建数量。
- [ ] A04. 供电设施卡显示供电角色和对电力余量的影响。
- [ ] A05. 玩家按推荐或角色提示能完成最小采矿链，不需要猜测按钮含义。
- [ ] A06. 建造采矿链后，右侧状态或健康摘要能反映采矿能力从缺失变为存在。

### B. 资源不足与建造失败

- [ ] B01. 当前资源不足时，建造卡片在尝试前显示缺口或不可负担原因。
- [ ] B02. 资源不足时点击建造不会只表现为无响应，必须出现可读反馈。
- [ ] B03. 多资源同时不足时，提示能指出至少一个主缺口，不刷满屏重复红字。
- [ ] B04. 资源恢复到可负担后，卡片可负担状态刷新。
- [ ] B05. 折扣或加成影响成本时，不足提示基于实际展示成本，而不是原始成本。

### C. 断电与电力压力

- [ ] C01. 建造高耗电设施前，卡片或诊断能提示可能拉低电力余量。
- [ ] C02. 当前断电时，设施详情或健康摘要能说明关键能力受影响。
- [ ] C03. 大量设施断电时，低噪声提示只聚焦核心能力，如采矿、武器、防御、推进。
- [ ] C04. 补充供电后，断电诊断和健康摘要能刷新为较健康状态。
- [ ] C05. 电力产出、消耗、余量在只读 power margin 钩子中可读取。

### D. 远离资源外环与采矿范围

- [ ] D01. 准备建造采矿站时，系统能说明当前位置是否接近资源外环。
- [ ] D02. 采矿站远离资源外环时，提示覆盖不足或采不到资源。
- [ ] D03. 已存在采矿站但全部采不到资源时，低噪声提示能指出“采矿站未覆盖资源”或等价原因。
- [ ] D04. 靠近资源外环放置后，采矿诊断不再显示远离风险。
- [ ] D05. 只读 resource reachability 钩子能列出采矿站与资源覆盖状态。

### E. 武器/护盾/维修状态

- [ ] E01. 选中武器设施时，能看到可用状态、缺电状态或作用边界说明。
- [ ] E02. 选中导弹/远程攻击相关设施时，能看到方向、边界或可攻击状态说明。
- [ ] E03. 敌袭前没有武器或武器断电时，健康摘要能指出火力短板。
- [ ] E04. 护盾设施存在时，玩家能识别当前具备基础防护。
- [ ] E05. 护盾设施缺电或防护不足时，诊断不误导为完全安全。
- [ ] E06. 维修设施存在时，玩家能理解维修覆盖或维修能力状态。
- [ ] E07. 受损设施附近有/无维修覆盖时，诊断结果可区分。

### F. 建造折扣、Meta 与协议加成显示

- [ ] F01. 建造折扣存在时，卡片显示变化后的成本。
- [ ] F02. 折扣显示不改变实际扣费规则，只反映现有结果。
- [ ] F03. Meta 加成影响初始资源、成本或能力时，建造面板能观察到变化。
- [ ] F04. 协议加成影响成本、资源或能力时，建造面板/摘要能观察到变化。
- [ ] F05. 无 Meta/协议加成时，展示回到基线，不残留加成文案。
- [ ] F06. 加成文案不承诺新系统或未实现效果。
