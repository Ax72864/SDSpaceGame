# v0.19.0-resource-repair-chain-evidence 证据方案

## 基本信息

- 版本：`v0.19.0-resource-repair-chain-evidence`。
- 负责人：`/pm-qa-coordinator`。
- 依据：制作人方向文档、v0.19 看板、v0.18.0 坐标诊断后复测证据、最新 `Game/game.js` 资源 / 采矿 / repair / playtest helper。
- 范围：只做证据方案、可见浏览器证据和 PM 记录；不修改 `Game/`、数值、规则、存档、发布产物或旧文档。

## Helper 与规则口径

- `repair` 成本：`metal 44`、`plasma 6`。
- `mining` 成本：`metal 28`、`ore 10`。
- `frame` 成本：`metal 8`；`power` 成本：`metal 22`、`ore 8`；`turret` 成本：`metal 35`、`ore 12`。
- 可用只读 helper：`getBuildPaletteDiagnostics`、`getPlacementDiagnostics`、`getGridScreenPoint`、`getGridInteractionDiagnostics`、`compareGridClickAlignment`、`getResourceReachability`、`snapshot`。
- 禁止使用 `buildAt` 或直接写 `window.__gameState` / storage 构造通过状态。

## 浏览器取证入口

- 使用可见 Chrome + CDP，入口为 `Game/index.html?playtest=1`。
- 使用临时 Chrome profile，记录 `localStorageLength`、`sessionStorageLength`、key 数量和长度摘要。
- 点击前用 UI 暂停稳定坐标；等待收益窗口时通过 UI 恢复运行，再重新暂停取证。
- 输入分类写为浏览器级辅助证据：CDP 物理鼠标事件 + DOM build 按钮点击；不计为真实人工鼠标或真实触控通过。

## 建造链路径

1. 记录初始资源、storage signature、runtime errors、helper 可用性。
2. 扫描半径内 `mining` 候选格的 `rangeHint`，记录最近资源体、资源类型、distance、range、gap、inRange 和 placement reason。
3. 复用 v0.18 建造链：`frame -> power` at `-1,-1`，`frame -> mining` at `-1,1`，`frame -> turret` at `-2,0`。
4. 建成 `mining` 后等待 10 秒，记录等待前后资源、`harvestingCount`、nearest body、gap、statusAlerts、toast 和 runtime errors。
5. 建成 `turret` 后等待 45 秒作为补资源窗口，记录资源增长是否足以建 `repair`。
6. 扩 `frame` at `0,-2`，再尝试 `repair` at `0,-2`。

## 必填证据字段

- 入口与环境：URL、浏览器、viewport、DPR、临时 profile、输入方式。
- 存档安全：local/session storage 长度、key 摘要、是否未清空用户真实存档。
- 点击证据：目标 `gridKey`、clientX/clientY、`aligned`、`screenDelta.distance`、点击前后 cell facility。
- 资源证据：每步 before/after `metal`、`ore`、`gas`、`plasma`、`research`。
- `repair` 证据：cost、have、gap、建成前 frame、建成后 `facility=repair`、失败时 toast/statusAlerts、失败是否扣资源。
- 采矿证据：`mining` 位置、最近资源体、资源类型、distance、range、gap、inRange、`harvestingCount`、等待窗口、收益是否可归因。
- 运行证据：runtime errors、console/unhandled 等价记录、toast、statusAlerts。

## 结论判定

- `repair` 通过：正常 UI 路径下 `frame` 成功替换为 `repair`，资源扣除与成本一致，runtime errors 为 0。
- `repair` 失败但可接受：明确 need/have/gap，提示说明缺什么，且未扣资源、未改变目标格。
- `mining` 通过：至少一个 `mining` 的 `harvestingCount > 0`，nearest target in range，等待后资源增长可归因于采矿。
- `mining` 未通过但证据完成：记录 `all_miners_out_of_range`、gap、等待窗口、资源变化不能归因于采矿，并说明下一步仍阻塞收益闭环。
- 任何 CDP/helper 成功都只归类为浏览器级辅助证据，不替代真实人工鼠标或触控体验。

## 最小修复准入

- 仅当证据证明玩家无法理解缺什么、差多少、下一步怎么补，或采矿收益/覆盖状态不可追踪时，才进入 #5。
- #5 只允许考虑提示、toast/statusAlerts 或只读诊断反馈；不得改资源成本、采矿范围/公式、维修规则、建造合法性、存档、Meta、战斗或发布流程。
