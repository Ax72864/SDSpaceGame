# v0.4.0 任务多样化 PM 看板

## 本轮目标

让 rogue 局内每个关卡拥有真正不同的任务体验：在保持 v0.1.x / v0.2.0 / v0.2.1 / v0.3.0 已验收路径不回潮的前提下，引入拾荒任务与护送任务，并把现有任务系统重构为可扩展的注册表 + 事件分发框架，为后续任务类型扩展铺路。

## 背景与制作人方向

制作人方向文档：`Workspaces/game-producer-lead-designer/v0.4.0-mission-diversity-direction.md`，commit `9f15bbf`。基线为 v0.3.0 框架韧性发布后，当前版本风险级别为项目高风险。

### 方向要点摘要

- 将当前"清剿敌人 + 采矿"主导的单一循环扩展为多任务体验。
- 新增拾荒任务：预生成 `origin = "wreck"` 的古老残骸 fragment，玩家通过 v0.3.0 桥接系统回收指定数量后完成任务。
- 新增护送任务：生成 1 艘友方 NPC 货船，玩家保护其移动到跃迁点，NPC 抵达完成，死亡按数值方案处理。
- 重构任务框架：建立 `OBJECTIVE_TYPES` 注册表和 `notifyObjective(event, payload)` 事件分发；现有 `mine / explore / battle / survive / guardian` 迁入新框架且行为等价。
- 任务选择 seed 化：基于 `state.run.seed` + level 派生 RNG 选择任务，同时保留每个 level 的基础类型 / 权重兜底，终局 level 6 仍固定为 guardian。

### 体验目标

1. **可探索**：玩家进入每个星系时需要通过 HUD / 引导识别任务类型，同一关多次重启的任务选择参考 seed，不再完全固定。
2. **可参与**：拾荒任务提供"搜寻老旧残骸 → 靠近桥接回来"的明确动作；护送任务提供"保护缓慢移动的友方货船"的明确动作。
3. **可重玩**：下一局可能遇到不同任务，形成"再开一局看看下一关是啥"的动机。

### 必做范围

- 建立 `OBJECTIVE_TYPES` 注册表，定义 `make / tick / onEvent... / getDetail / getHint / render` 等任务类型能力。
- 建立 `notifyObjective(event, payload)` 事件分发函数，统一承接采集、击杀、guardian 击杀等进度事件。
- `updateObjective(dt)`、HUD `progressDetail`、`pickNextForObjective`、`buildGuideText` 改为查询注册表，不再硬编码任务 switch。
- `createObjective()` 使用 `state.run.seed` + level 派生 RNG 进行任务选择，并保留 level 节奏兜底权重。
- 现有 5 种任务 `mine / explore / battle / survive / guardian` 在新注册表下行为完全等价。
- 新增拾荒任务 `salvage`：预生成低速 `origin = "wreck"` fragment，HUD 区分古老残骸，仅统计古老残骸桥接。
- 新增护送任务 `escort`：生成友方 NPC 货船、HUD 提示方向 / 距离 / HP、敌方可伤害、玩家炮塔不锁定 NPC。

### 不做范围

- 不做并行多任务。
- 不做防御 / 救援 / 拾遗 / 探险 / 交付等其它新任务类型。
- 不做敌方 NPC AI 多样化。
- 不做设施 / 武器 / 科技升级 / 局外天赋扩展。
- 不做联机 / 异步对战。
- 不修改 `SAVE_KEY = "star-ring-station-meta-v1"`、局外存档结构、局外点数算法。
- 不做 v0.3.1 转入评估池的 9 条非阻塞建议。
- 不修改 v0.3.0 fragment 物理、桥接判定、弹道命中常量。
- 不修改 v0.2.1 三引擎手感、推进器松键不续推、`updateMouseAim` 转向、双模式 WASD。
- 不修改 level 0 首局 60 秒减压、初始资源和旧 meta 兼容。

### 兼容要求

- v0.1.x / v0.2.0 / v0.2.1 / v0.3.0 全部回归点必须通过。
- 现有 5 种任务在新注册表下行为完全等价。
- 终局守护者关 level 6 仍按 guardian 任务执行，不被 seed 化打乱。
- 旧存档 meta 注入后正常读取，包括无 `unlocks`、空 `{}`、仅 `{points:5}` 形态。
- `SAVE_KEY`、局外存档结构和局外点数算法不变。

## 风险定级

- 风险等级：项目高风险。
- 定级原因：触及核心玩法任务生成、任务进度事件、HUD 目标表达、v0.3.0 fragment 桥接系统、友方 NPC 数据结构、敌我识别和关键运行态数据结构。
- 主要风险：现有 5 种任务迁入注册表时可能行为回潮；任务 seed 化可能破坏 level 节奏或终局 guardian；拾荒 fragment 可能污染玩家脱落 fragment 逻辑；护送 NPC 可能被玩家炮塔误伤或引发敌人目标选择空引用；存档 / meta 不应被新运行态字段污染。
- 高风险流程：制作人方向 → PM 拆分 → 主程 ADR → 数值方案 → 核心实现 → 专项审核 → PM 浏览器级 + 主观回归 → 制作人最终决定与发布安排。
- 审核基底约束：专项审核使用 `expert-agent-planner-gpt55`，必须与执行不同基底。

## 缺陷记录格式

- 标题：一句话描述问题。
- 严重程度：S0 阻断 / S1 严重 / S2 一般 / S3 轻微。
- 影响范围：说明影响启动、存档、任务选择、任务进度、拾荒、护送、fragment 桥接、NPC、HUD、WASD、采集、建造、敌人攻击、终局或自由游玩中的哪一部分。
- 复现步骤：按浏览器实机操作顺序列出，包含 seed / level / objective type / 站体形状 / NPC 状态 / fragment 来源 / 玩家操作。
- 实际结果：记录玩家实际看到或系统实际发生的情况，必要时附运行态采样。
- 期望结果：记录按制作人方向、ADR 或数值方案应发生的体验。
- 优先级：P0 / P1 / P2，并说明是否阻塞接受。
- 负责人：明确后续处理成员。
- 当前处理：待分配 / 修复中 / 待审核 / 已回归 / 接受为已知问题。
- 回归要求：修复后尽量按原复现步骤复测，并说明失败路径是否关闭。
