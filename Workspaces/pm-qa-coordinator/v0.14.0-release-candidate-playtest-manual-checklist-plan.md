# v0.14.0-release-candidate-playtest 人工 playtest checklist 与小屏验收方案

## 目标与边界

版本名：`v0.14.0-release-candidate-playtest`。

本方案用于 PM 与 UI 视角协同验收 SSDSpaceGame 发布候选体验证据，重点确认玩家能从空存档首局走完建造、采矿、资源压力、战斗、复盘、星系跳转和 Meta 回到新局，并在真实浏览器和小屏/窄屏窗口下记录可复查结论。

严格边界：

- 本文只定义人工 checklist、记录字段、小屏验收方式、隐藏 checklist 入口建议和通过/失败标准。
- 本轮不修改 `Game/`、`Docs/`、`.cursor/`、`42.md`、`Releases/`、PM 看板、制作人方向文档或其他工作区文件。
- 本方案不宣布发布就绪；真实人工体验、browser/headless、自动检查和未覆盖风险必须分开记录。
- 没有真实浏览器人工体验记录时，结论只能是“发布候选证据不足”。
- 触控若没有真实设备验证，只能记录为“触控假设/未覆盖”，不能写成移动端通过。

## 执行前准备

测试者先记录以下基础信息，后续每条路径都复用同一字段：

- 测试日期/测试者：
- 浏览器与版本：例如 Edge / Chrome / Firefox。
- 操作系统：Windows 10/11 或其他。
- 窗口尺寸：建议至少覆盖 `1366x768`、`900x700`、`760x640`、`420x740`、`390x844`。
- 输入方式：鼠标键盘、触控板、真实触控、浏览器设备模拟。
- 存档状态：空存档、隔离 profile、或记录具体清理方式。
- 控制台错误：无 / warning / error / runtime exception，附摘要。
- 截图/备注：有问题时必须记录截图文件名或文字说明。

每条路径记录字段统一为：

| 字段 | 记录要求 |
| --- | --- |
| 浏览器 | 浏览器名称、版本、是否真实浏览器或 headless |
| 窗口尺寸 | 宽高、DPR、是否属于小屏/窄屏 |
| 控制台错误 | error/warning/runtime exception 摘要，若无则写“无” |
| 视觉可读性 | HUD、面板、提示、按钮、文字折行是否能看懂 |
| 操作是否有反馈 | 点击、键盘、建造、失败、跳转、复盘是否有可见反馈 |
| 是否阻塞发布 | 阻塞 / 非阻塞 polish / 未覆盖 / 通过 |
| 截图/备注 | 截图编号、复现步骤、主观感受或疑问 |

## 人工主路径

### P0-1 空存档首局

起始条件：使用空存档或隔离浏览器 profile 打开游戏。

操作步骤：

1. 打开游戏页面，等待首屏稳定。
2. 观察顶部 HUD、资源条、危险度、关卡进度、建造面板、右侧引导和更多状态入口。
3. 不操作 30 秒，观察是否出现 runtime error、异常警报或 UI 抖动。
4. 点击/展开 `更多状态`，确认状态、设计健康、战斗摘要、Meta 入口存在。

观察重点：

- 首局默认目标和下一步可读，不依赖旧 Meta 进度。
- `resources`、`runDanger`、`runProgress`、`statusAlerts`、`moreStatusPanel` 不遮挡核心画布。
- 鼠标键盘入口可理解，暂停、移动模式、建造入口有明显状态。

结论记录：通过 / 失败 / 未覆盖 / 阻塞发布。

### P0-2 建造采矿链

操作步骤：

1. 查看建造卡，找到采矿、电力、推进、防御或维修相关设施。
2. 选择采矿相关设施，按提示在网格放置。
3. 驾驶进入资源点外环，观察资源采集提示和资源数值变化。
4. 查看设计健康、资源采集引导和建造卡状态是否给出下一步方向。

观察重点：

- 建造卡密度可读，名称、角色、成本、供电、状态、提示不互相挤压。
- 建造成功、失败、选择保持、退出建造模式都有反馈。
- 采矿链启动后，资源变化或资源采集状态可被看见。

结论记录：必须说明玩家是否能独立找到采矿链。

### P0-3 资源不足与断电

操作步骤：

1. 尝试点击当前不可负担的建造卡或连续建造制造资源不足。
2. 观察按钮状态、卡片状态、toast、引导或 `statusAlerts` 是否说明缺少资源。
3. 制造或观察电力压力，确认 HUD 或诊断说明关键能力受影响。
4. 资源恢复或供电改善后，观察错误提示是否刷新、消失或降级。

观察重点：

- 不可负担操作不能表现为“按钮无反应”。
- 断电提示不能暗示规则变化，必须说明受影响能力。
- 资源不足、供电不足的反馈在小屏下仍可读。

结论记录：若玩家无法判断失败原因，默认阻塞发布候选路径。

### P0-4 敌人接近

操作步骤：

1. 正常游玩至敌人接近，或选择能自然触发威胁的星系/阶段。
2. 观察顶部武器行、危险度、右侧战斗状态和 `statusAlerts`。
3. 记录最近/最危险敌人的数量、方向、距离或等价信息是否可读。
4. 敌人解除后继续观察 15 秒，确认旧威胁不会长期残留。

观察重点：

- 安全态不持续红色警报。
- 多敌人场景优先解释最危险对象。
- 战斗提示不被建造面板、右侧面板或星系弹层遮挡。

结论记录：若无法确认敌人接近或威胁来源，阻塞发布候选。

### P0-5 武器、护盾与维修

操作步骤：

1. 建造或确认基础武器、防御/护盾、维修设施。
2. 进入一次交火，观察武器是否攻击、是否提示无目标/超距/缺电/LOS 阻挡。
3. 观察护盾或防御状态是否区分存在、缺电、受损或无护盾。
4. 观察维修设施是否说明有目标、无目标、缺电或覆盖不足。
5. 使用导弹齐射或相关按钮时，确认点击反馈、装填、锁敌或失败原因。

观察重点：

- 战斗状态读取不应改变战斗判定、维修进度或敌人行为。
- 失败原因要比“没打出去”更具体。
- 小屏下按钮仍可点，战斗状态和复盘不被折叠到不可见。

结论记录：任何武器/护盾/维修核心反馈缺失，至少进入阻塞候选评估。

### P0-6 战斗后复盘

操作步骤：

1. 完成一次真实交火，不要求通关。
2. 打开或观察 `combatReviewSummary`、`combatStatusSummary`、设计健康摘要。
3. 记录复盘是否包含损伤、短板、主要威胁、下一步建议等 2-4 条信息。
4. 开始新一轮威胁或继续游玩，确认旧复盘不会污染当前状态。

观察重点：

- 复盘文案短、可读、可折行。
- 复盘不承诺未实现功能，不替玩家自动修复。
- 小屏下复盘区域可滚动，不阻塞继续按钮或跳转入口。

结论记录：复盘缺失不一定直接阻塞，但若玩家无法理解战斗后为何失败，应阻塞发布候选。

### P0-7 星系跳转

操作步骤：

1. 完成当前星系任务或达到可跳转条件。
2. 点击“跃迁下一星系”或打开星系图。
3. 观察星系卡片、风险、描述、预告和取消/停留按钮。
4. 选择目的地并回到局内，观察 HUD、资源、建造和危险度状态。

观察重点：

- `galaxyMapPanel` 在桌面和小屏下都能滚动。
- 星系卡片按钮可点，取消/停留按钮不被底部区域遮挡。
- 跳转状态不能被只读检查推进或改变选择。

结论记录：跳转入口找不到、无法点击、无法回局内均阻塞发布候选。

### P0-8 Meta 回到新局

操作步骤：

1. 触发本局结算、Meta 入口或局外天赋入口。
2. 打开 Meta 面板，查看点数、天赋卡、分类、购买按钮和关闭按钮。
3. 从结算或 Meta 路径回到新局。
4. 观察新局 HUD、建造入口、资源和危险度是否混乱或继承错误状态。

观察重点：

- Meta 面板右侧抽屉在窄屏应变为全屏可滚动。
- 新局返回不能修改 `SAVE_KEY` 或 meta schema。
- “开始新局”“留在终结星系”“打开局外天赋面板”等按钮在小屏可点。

结论记录：如果回到新局后状态混乱，阻塞发布准备。

## 小屏/触控验收方案

### 视口矩阵

- 桌面基线：`1366x768`，鼠标键盘。
- 窄桌面：`900x700`，鼠标键盘。
- 小屏横向临界：`760x640`，确认 `max-width: 760px` 样式。
- 手机竖屏假设：`420x740`，浏览器设备模拟或窄窗口。
- 极窄假设：`390x844`，浏览器设备模拟或窄窗口。
- 真实触控：若没有真实设备，记录为未覆盖，不阻塞代码合入但阻塞“移动端已通过”表述。

### 小屏重点

- HUD：资源、电力、危险度、关卡进度和星系路径不永久遮挡画布核心操作。
- 建造卡密度：两列卡片在 `420px` 和 `390px` 下仍能看见名称、成本、状态和至少一条关键提示。
- `combatStatusSummary` / `combatReviewSummary`：长文本能折行，严重度颜色不丢失，更多状态区域可滚动。
- 底部建造面板：`max-height` 生效，可滚动，按钮不小于可点击目标，选中/不可负担状态可辨认。
- 右侧状态面板：小屏下可滚动，不覆盖全部画布，不让 Meta/星系弹层失去操作。
- 星系图：卡片纵向排列，底部取消按钮可见或可滚动到达。
- Meta 面板：抽屉全宽、tab 可横向滚动，卡片一列显示，关闭按钮可点。
- 结算面板：`combatReview`、Meta 反馈和开始新局按钮不互相遮挡，底部按钮 sticky 时不盖住关键信息。
- 触控：按钮、summary、星系卡、Meta tab、关闭按钮、开始新局按钮可点；画布拖拽和按钮点击不互相抢焦点。

## 游戏内隐藏 checklist / 文档化入口建议

优先级建议：

1. v0.14.0 可先采用 PM 文档化 checklist，避免给普通玩家增加噪声。
2. 如需要游戏内入口，建议只在 `?playtest=1` 或测试 hook 开启时展示。
3. 入口默认隐藏，不占用 HUD 主视觉；可放在 `更多状态` 下方或以小型浮动“PM 检查”按钮出现。
4. 面板只读展示：当前路径节点、记录字段、关键 DOM 可见性、viewport、console/runtime 摘要、storage signature、frame timing 粗采样。
5. 面板不写存档、不改变 RNG、不推进星系、不修改资源/敌人/战斗/Meta。
6. 面板支持复制 JSON 或纯文本记录，但复制失败不能影响游戏。
7. 普通玩家路径下不出现 checklist、测试术语或未完成提示。

最小 UI 形态：

- 入口：`?playtest=1` 后在 `更多状态` 中显示“PM 验收记录”折叠区。
- 面板：默认折叠，展开后展示 8 个路径节点、证据字段和未覆盖风险。
- 样式：沿用现有 panel/details 视觉，字号不高于状态信息，避免抢占战斗警报。
- 关闭：可折叠或随 `?playtest=1` 消失，不保存用户选择。

## 通过/失败标准

### 阻塞发布候选

- 空存档首局无法进入或出现阻塞 runtime exception。
- 玩家无法找到建造入口、无法完成最小采矿链，或采矿成功没有任何可见反馈。
- 资源不足、断电、建造失败表现为无反馈，导致核心路径卡住。
- 敌人接近不可见、威胁来源不可判断，或战斗 HUD 长期误报。
- 武器、护盾、维修核心状态不可读，导致玩家无法理解战斗失败原因。
- 战斗后完全没有复盘或复盘严重误导下一步操作。
- 星系跳转入口不可见、按钮不可点、跳转后无法回到局内。
- Meta 回到新局后 HUD、资源、建造或局外状态混乱。
- 小屏/窄屏下 HUD、建造、战斗、复盘、星系图或 Meta 入口被遮挡到无法继续。
- 只读检查或 playtest hook 写入存档、推进 RNG、改变星系/Meta/战斗状态。
- PM 无法区分自动检查、browser/headless、人工体验和未覆盖风险。

### 非阻塞 polish

- 文案略长但不影响理解和继续操作。
- 小屏下部分次要说明需要滚动才能查看，但核心按钮和失败原因可见。
- 个别 warning 可解释且不影响核心路径，已记录复现条件。
- 战斗复盘建议不够精炼，但能说明损伤或短板。
- 触控真实设备未覆盖，但桌面窄屏和设备模拟通过；结论不得写成移动端通过。
- 截图命名或记录格式不统一，但关键证据完整。

### 接受阈值

- P0 主路径 8 项全部有记录，且无阻塞发布候选问题。
- 小屏矩阵至少覆盖 `760x640`、`420x740`、`390x844` 三种窄屏/小屏参数。
- 每条路径都记录浏览器、窗口尺寸、控制台错误、视觉可读性、操作反馈、是否阻塞、截图/备注。
- 至少一次真实浏览器人工体验完成；否则结论为“证据不足”。
- 所有 S0/S1 缺陷修复后，按原复现步骤复测并记录失败路径已关闭。

## 人工/UI 回归点

### A. 起始与 HUD

- [ ] A01. 空存档打开游戏无阻塞 runtime exception。
- [ ] A02. 首屏能看到资源、电力或等价资源摘要。
- [ ] A03. 首屏能看到危险度、关卡进度和当前星系/路径信息。
- [ ] A04. `runInfo` 不显示过期或矛盾状态。
- [ ] A05. `runDanger` 安全态与警戒态颜色和文案可区分。
- [ ] A06. `weaponsRow` 在无威胁时不制造误报。
- [ ] A07. HUD 不遮挡暂停按钮、画布核心操作或建造入口。
- [ ] A08. 控制台没有影响初始化的 error。

### B. 建造与采矿链

- [ ] B01. 建造面板在桌面和小屏均可见。
- [ ] B02. 建造卡名称、角色、成本、供电和状态可读。
- [ ] B03. 建造卡不可负担状态有视觉区分。
- [ ] B04. 点击建造卡后，建造模式状态有反馈。
- [ ] B05. 放置成功有反馈，且不会误退出到不可理解状态。
- [ ] B06. 放置失败有原因，且失败后仍可继续尝试或退出。
- [ ] B07. 采矿设施或资源链入口能被测试者找到。
- [ ] B08. 资源点颜色说明与实际资源类型一致。
- [ ] B09. 进入资源点外环后，采集状态或资源变化可观察。
- [ ] B10. 资源采集引导不会在小屏被底部建造面板完全盖住。

### C. 资源不足与断电

- [ ] C01. 资源不足时不只是按钮无反应。
- [ ] C02. 缺少哪类资源可读。
- [ ] C03. 不可负担提示在 `420px` 宽度下仍能看见关键字。
- [ ] C04. 电力不足时 HUD 或诊断说明受影响能力。
- [ ] C05. 断电不会让玩家误以为游戏卡死。
- [ ] C06. 资源恢复后，过期不足提示会刷新或降级。
- [ ] C07. 断电提示与武器/护盾/维修状态一致。
- [ ] C08. `statusAlerts` 多条提示时可读，不互相重叠。

### D. 敌人接近与战斗状态

- [ ] D01. 敌人接近时顶部或右侧有威胁提示。
- [ ] D02. 威胁提示能表达数量、距离、方向或等价优先级。
- [ ] D03. 多敌人时最危险对象优先级可理解。
- [ ] D04. 安全态不会持续显示高危红色警报。
- [ ] D05. 敌人解除后旧警报不会长期残留。
- [ ] D06. `combatStatusSummary` 在小屏下能折行。
- [ ] D07. `combatStatusSummary` 严重度颜色可区分。
- [ ] D08. 战斗提示不被 Meta 面板、星系图或建造面板永久遮挡。

### E. 武器、护盾、维修

- [ ] E01. 武器有目标时能看到攻击或装填状态。
- [ ] E02. 武器无目标时原因可读。
- [ ] E03. 武器超距、LOS 阻挡或缺电时至少一种原因可诊断。
- [ ] E04. 导弹齐射按钮可点且有装填/发射/失败反馈。
- [ ] E05. 护盾或防御状态能区分存在、缺电、受损或缺失。
- [ ] E06. 维修设施有目标时能看到维修倾向或状态。
- [ ] E07. 维修设施无目标、缺电或覆盖不足时原因可读。
- [ ] E08. 读取状态不改变战斗、维修或敌人推进。

### F. 战斗后复盘

- [ ] F01. 一次交火后出现复盘或等价总结。
- [ ] F02. 复盘至少说明损伤、主要威胁、短板或下一步建议之一。
