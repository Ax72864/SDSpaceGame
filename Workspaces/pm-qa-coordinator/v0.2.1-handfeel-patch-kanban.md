# v0.2.1 手感补丁任务看板

## 本轮目标

围绕 `42.md` 中仍需真实观察的“敌人过于喜欢撞击”和“三引擎移动仍不灵活”做小范围手感补丁。v0.2.1 不扩大到新系统，先由 PM 完成 5 分钟首局真实 / 浏览器手感回归，记录复现和体感证据；只有回归确认问题仍成立时，才启动数值 / 玩法微调。

## 背景与制作人方向

- 当前基线：`v0.2.0-galactic-rhythm` 已发布；master 后续完成 S2 首局减压微调，并通过发布前完整流程回归。
- PM 既有结论：自动检查与浏览器级完整流程回归无 S0 / S1 阻断；真实 15-30 分钟手玩、敌人撞击倾向、满三引擎灵活度和长期节奏仍未覆盖。
- 制作人本轮决定：下一阶段不做新系统，优先把“敌人撞击倾向”和“三引擎机动仍不灵活”作为 `v0.2.1` 手感补丁候选。
- 处理原则：先验证再调参；若 5 分钟首局回归无法稳定复现或体感可接受，则本补丁可只记录为观察结论，不强行修改数值。

## 风险定级与流程

- 风险等级：中高风险手感补丁。
- 定级原因：候选改动可能触及敌人接近 / 碰撞、推进 / 阻尼 / 转向和 WASD 双模式，属于核心手感与首局压力曲线；但本轮明确禁止新 AI 系统、物理重构和存档结构变化，范围低于 v0.2.0 主版本任务。
- 建议流程：PM 首局 5 分钟手感回归 -> 制作人判断是否启动微调 -> /systems-numerics-designer + /senior-gameplay-engineer 小范围方案与实现 -> /lead-game-programmer 或 expert-agent-planner-gpt55 专项审核 -> PM 回归 -> /game-producer-lead-designer 接受结论。
- 启动门槛：只有 PM 回归记录显示敌人撞击或三引擎机动问题仍影响首局体验，才进入数值 / 玩法微调。

- 禁止修改范围：不得引入新敌人 AI 系统、完整物理重构、移动端重做、新武器 / 新设施、存档结构变更、发布归档；不得修改 `Docs/`、`.cursor/`、`42.md`、`持续开发Prompt.md`、`Releases/`、`Release/` 或无关工作区；不得删除文件；不得执行 git commit / push，除非后续任务明确改变授权。

  - 海盗期望距离：增加 pirate 与玩家站体的期望交战距离，让其更偏向远程绕行 / 射击而不是贴脸。

- 验收标准：
  - 每个候选参数说明修改理由、预期体感收益、可能副作用和回滚方式。

  - 不修改 `SAVE_KEY="star-ring-station-meta-v1"`、`state.meta` 兼容结构或局外点数 / 天赋读取。
- 风险：
  - 降低撞击倾向可能让早期战斗压力过低，削弱海盗威胁。
  - 提升推进响应可能让资源采集、躲避和战斗整体变简单，需要同时观察首局压力。
  - 分离 / 避让若处理不当可能造成敌人抖动、穿模或无法攻击。
