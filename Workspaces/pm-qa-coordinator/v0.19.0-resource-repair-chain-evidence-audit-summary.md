## 一、#5 最小修复边界专项审核

### 1.1 改动清单（基于 `git show 09354db -- Game/game.js`）

仅四处改动：

1. 新增 `formatMissingResourceGaps(missingResources)`（`Game/game.js` 7048 行附近）。函数体只是 `filter` + `map` + `join`，从只读 `missingResources` 数组读出 `gap` 与 `name` 拼接成 `还缺 金属 X、等离子 Y` 形式的字符串，未读写 `state`、`storage`、`window` 或任何全局副作用入口。
2. `getBuildPaletteTipText` tip 文案：`警告：缺X X，当前不能建造` → `警告：当前这座缺X X，暂时不能建造`。仅文案，未改 `entry.affordable / entry.missingResources / entry.powerRiskAfterBuild` 的判断分支与顺序。
3. `buildAt` 中 `setBuildError` 文案：`${def.name} 资源不足：${formatCost(cost)}（建造模式已保持）` → `${def.name} 当前资源不足：还缺 ${missingSummary}（建造模式已保持）`，其中 `missingSummary` 来自只读的 `formatMissingResourceGaps(getBuildPaletteMissingResources(cost)) || formatCost(cost)`。当 missing 为空时回退到原 `formatCost`，保留原行为。
4. `buildStatusAlerts` 中选中建筑的 statusAlert：把内联 `Object.entries(cost).filter(...)` 替换为已有只读 helper `getBuildPaletteMissingResources(cost)`，文案改为 `当前仍选中「...」：下一座还缺 ...（不是刚才建造失败；核心会缓慢产金属）`；空列表分支与 `placementPreview` 分支未动。
