# v0.9.0 局间星系图系统 PM 看板

## 本轮目标

在 v0.8.0 局内随机事件发布后，引入局间星系图系统：玩家在关与关之间从 2-3 个候选星系中选择下一关，形成可记忆、可权衡、与既有任务 / 事件 / 战斗系统叠加的 7 关单局路径。

## 背景与制作人方向

制作人方向文档：`Workspaces/game-producer-lead-designer/v0.9.0-galaxy-map-direction.md`，上游指定 commit `36dae9f`。基线为 HEAD `36dae9f`（v0.8.0 发布后），当前版本风险级别为项目高风险。

### 方向要点摘要

- 新增平行 `GALAXY_TYPES` 注册表，包含 `tradeHub / pirateTerritory / miningBelt / mysteryZone / warFront / techRuin / emptyVoid` 7 种星系类型。
- `galaxyType` 作为权重覆盖层叠加在 `OBJECTIVE_LEVEL_WEIGHTS` / `ENCOUNTER_LEVEL_WEIGHTS` / spawn / resource / hostile-station 之上，不替换既有曲线。
- 跃迁流程从 `objectiveJumpBtn -> nextLevel()` 改为 `objectiveJumpBtn -> galaxyMapPanel -> confirmGalaxyJump -> nextLevel(galaxyType)`。
- `level 0` 起点固定 `emptyVoid`，`level 6` 终末关固定 `warFront`，level 1-5 从 2-3 个候选中选择。
- 视觉系统按 `galaxyType` 分化背景、星云、行星、小行星密度、HUD 当前星系和路径预览。
- 独立 RNG 流使用 `state.run.seed ^ 0xd3b07cf3 + level * 0x6c8e9cf5` 或 ADR 等价方案，不能破坏 objective / encounter / escort seed 稳定性。

### 体验目标

1. **路径决策权**：玩家每次跃迁都能根据当前 HP、资源、武器状态和风险偏好选择下一关去向。
2. **单局轨迹记忆**：玩家能在一局结束后说出自己走过的星系路径，且不同路径的体验差异明显。
3. **与已有系统协同**：v0.4.0 任务、v0.6.0 hostile-station、v0.8.0 随机事件全部保留，星系类型只作为上层偏好和视觉签名叠加。

### 必做范围

- 工程基础：`GALAXY_TYPES` 注册表、`state.run.galaxyMap`、`currentGalaxyType`、`galaxyChoicesShown`、`ensureRunRuntimeState` 默认值、`nextLevel(galaxyType?)`、`generateGalaxyChoices`、`confirmGalaxyJump`、独立 RNG、`__gameTest.resetRun` 清理和 `__gameTest.setGalaxyType` 钩子。
- 权重覆盖层：OBJECTIVE / ENCOUNTER / 敌人 spawn / 资源奖励 / hostile-station 触发概率乘子。
- 视觉差异：`drawBackground`、`drawNebula`、行星、小行星密度按 `galaxyType` 分化。
- UI 层：`#galaxyMapPanel`、候选卡片、Escape 取消、`#galaxyPathEl`、`#currentGalaxyInfoEl`、路径预览 HUD。
- 清理与验证：死代码清理、关键注释更新、v0.9.0 调试钩子文档、节点 A/B/C 中间审查、专项审核、PM 浏览器级 + 主观回归、制作人最终决定。

### 不做范围

- 不破坏 v0.4.0 `OBJECTIVE_TYPES`：注册表 / createObjective / notifyObjective / grantObjectiveReward / 8 处现有调用点完全保留。
- 不破坏 v0.8.0 `ENCOUNTER_TYPES`：注册表 / 5 种事件 / notifyEncounters / applyEncounterReward / seedEncountersForLevel 完全保留，仅在 weight 表层添加 galaxyType 乘子。
- 不破坏 v0.7.0 锁敌 / 战斗操控：priorityTarget / F 键齐射 / salvoSize 等完全保留。
- 不破坏 v0.6.0 hostile-station BOSS 战：assault 触发逻辑保留，仅 galaxyType 影响触发概率乘子。
- 不破坏 v0.5.0 研发系统。
- 不修改 `SAVE_KEY`；`state.run.galaxyMap / currentGalaxyType / galaxyChoicesShown` 仍走 runtime 路径，刷新即重置。
- 不引入新 enemy kind、OBJECTIVE_TYPES、ENCOUNTER_TYPES、building cell 类型、跨局持久化、联机、新 BOSS、跨关持续事件、地图 zoom / 拖拽。

### 兼容要求

- v0.1.x / v0.2.0 / v0.2.1 / v0.3.0 / v0.4.0 / v0.5.0 / v0.6.0 / v0.7.0 / v0.8.0 全部回归点必须通过。
- `nextLevel()` 无参调用默认走 `emptyVoid`，与现有线性行为等价。
- `__gameTest.findEscortSeed / getObjectiveType / sampleAssaultRate / resetRun / getEncounters / forceActivateEncounters / advanceEncounters / tradeEncounter` 默认路径完全等价。
- `SAVE_KEY` 不变；旧存档兼容，缺失新字段时安全默认。
- 玩家选择 `emptyVoid` 后体验精确等价 v0.8.0，emptyVoid 所有乘子均为 1.0。
- Escape 取消 `galaxyMapPanel` 后回到关内继续游戏，任务完成态保持；终末关跃迁固定 `warFront`。

## 风险定级

- 风险等级：项目高风险。
- 原因 1：核心系统扩展，新增 `GALAXY_TYPES`、`state.run.galaxyMap`、跨关路径节点、候选生成和测试钩子。
- 原因 2：跃迁 UI 重做，`objectiveJumpBtn` 不再直接进入下一关，必须处理 galaxyMapPanel、取消、终末关和 objectiveChoicePanel 状态竞争。
- 原因 3：权重表二次分组，OBJECTIVE / ENCOUNTER / spawn / resource / hostile-station 乘子必须叠加而不破坏既有难度曲线。
- 原因 4：视觉系统按 galaxyType 分化，背景、星云、行星、小行星密度和 HUD 路径同时变化，可能影响性能和可读性。
- 原因 5：跨关字段扩展，`state.run` 新字段、旧存档 fallback、resetRun / nextLevel 清理和独立 RNG 流若处理不当会污染旧路径。
- 高风险流程：制作人方向 -> PM 拆分 -> 主程技术 ADR -> 数值方案 -> T1-T6 分批实现 -> 节点 A/B/C 中间审查 -> 专项审核 -> PM 浏览器级 + 主观回归 -> 制作人最终决定与发布。
- 审核基底约束：专项审核使用 `expert-agent-planner-gpt55`，必须与执行不同基底。

## 缺陷记录格式

- 标题：一句话描述问题。
- 严重程度：S0 阻断 / S1 严重 / S2 一般 / S3 轻微。
- 影响范围：说明影响启动、存档、`GALAXY_TYPES`、`state.run.galaxyMap`、`currentGalaxyType`、`galaxyChoicesShown`、`nextLevel(galaxyType?)`、`generateGalaxyChoices`、`confirmGalaxyJump`、独立 RNG、OBJECTIVE 权重、ENCOUNTER 权重、spawn、resource、hostile-station、galaxyMapPanel、路径 HUD、视觉分化、`ENCOUNTER_TYPES`、`OBJECTIVE_TYPES`、研发、锁敌、齐射、WASD、采集、建造、终局或自由游玩中的哪一部分。
- 复现步骤：按浏览器实机操作顺序列出，包含 seed / level / objective type / encounter type / galaxyType / 候选列表 / 玩家选择 / HUD 文案 / 面板状态 / 任务状态 / 奖励变化。
- 实际结果：记录玩家实际看到或系统实际发生的情况，必要时附运行态采样。
- 期望结果：记录按制作人方向、ADR 或数值方案应发生的体验。
- 优先级：P0 / P1 / P2，并说明是否阻塞接受。
- 负责人：明确后续处理成员。
- 当前处理：待分配 / 修复中 / 待审核 / 已回归 / 接受为已知问题。
- 回归要求：修复后尽量按原复现步骤复测，并说明失败路径是否关闭。
