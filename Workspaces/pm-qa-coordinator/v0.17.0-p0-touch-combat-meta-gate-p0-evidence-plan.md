## 证据口径

### 证据类型

- 真实点击：真人在真实浏览器中通过可见 UI 操作完成步骤，优先级最高。
- 真实触控：真人在触控设备或真实触控浏览器环境中完成步骤；可证明手指 hit、滚动、系统浏览器行为。
- 辅助 hook：只读状态、debug hook、测试 hook 或游戏内辅助入口；只能帮助确认状态，不能替代玩家是否看懂、点到、走通。
- CDP：通过 Chrome DevTools Protocol 观察或驱动页面；可补充 browser/runtime/storage 证据，不能等同真人体验通过。
- DevTools/鼠标替代：移动视口、鼠标点击或触控模拟；可证明布局、可见性、鼠标可点性，不能证明真实手指触控、软键盘或系统浏览器行为。
- 未覆盖：本轮没有形成可复查证据；必须说明原因和是否影响发布。

### 结论状态

- 通过：该步骤由真实点击或真实触控完成，关键反馈可见可理解，且无阻塞 runtime error。
- 失败：该步骤已尝试但未达成目标，存在复现现象和实际结果。
- 阻塞：S0/S1 或足以阻断 P0 主路径、交火复盘、Meta 回流或证据可信度的问题。
- 非阻塞风险：不阻断当前 P0，但影响舒适度、可读性、触控可信度或发布信心。
- 辅助覆盖：该步骤主要依赖 hook/CDP/预置状态或替代操作，只能作为补充证据。
- 未覆盖：没有实际执行或证据不足，不能声明通过。

## 空存档与隔离存档策略

- 不得删除、覆盖或清空用户真实存档。
- 首选使用临时浏览器 profile、无痕/隔离 profile、单独浏览器用户数据目录或可明确区分的测试环境。
- 如只能使用现有 profile，必须在记录中写明“现有状态”，记录取证前 `storageSignature`、关键 storage 摘要和可能污染判断的已存在进度。
- 若需要空存档效果，不通过删除真实数据达成；只能使用临时 profile、浏览器级隔离、或明确标记为“非空存档起点”。
- Meta 回流前后必须记录 storage 影响：未写、按预期写入、无法判断、异常写入或因现有状态不可区分。
- 任何会修改 `SAVE_KEY`、meta schema、存档结构或局外成长数据的方案都不属于本 P0 证据方案范围，必须暂停升级。

## P0 成功 / 失败 / 继续补证标准

### 成功标准

- 完整真人 P0 主路径至少闭环一次，并明确入口、建造采矿、资源不足/断电、敌人接近、真实交火、武器/护盾/维修反馈、战后复盘、星系跳转、Meta 点击回流的证据引用。
- 自然交火和战后复盘形成真实浏览器证据，至少覆盖武器开火、护盾/受击或防御缺口、维修状态、`combatReview` 或等价复盘。
- Meta 点击回流形成真实 UI 点击证据，并记录点击前后 storage 影响。
- 触控证据真实可用；若无真机，替代边界说明诚实，且没有发现阻塞级触控/小屏问题。
- runtime errors 无 S0/S1 阻塞；S2/S3 或 polish 项有结论、风险和复测记录。
- PM 证据包能支撑制作人在“通过、继续补证、小修 polish、进入发布准备交接”中做明确决定。

### 失败标准

- 真人 P0 主路径无法闭环，且阻塞点不能通过继续补证解释。
- 入口、建造、交火、复盘、星系或 Meta 任一关键路径存在 S0/S1 阻塞且未修复复测。
- 自然交火、武器/护盾/维修、战后复盘或 Meta 点击回流只能靠 hook/CDP 模拟，缺少真实 UI 体验证据。
- runtime errors、未捕获异常、Promise rejection 或脚本中断影响 P0 可信度。
- 取证过程污染或删除用户真实存档，或无法解释 storage 变化。
- 发现需要修改核心规则、数值、AI、RNG、伤害、维修、护盾、战斗结算、`SAVE_KEY`、meta schema 或发布流程才能过关。

### 继续补证标准

- 主要路径没有明确失败，但缺少可复查证据引用。
- 自然交火等待时间、前置状态或入口可发现性不足，导致证据未覆盖，但没有证明功能失败。
- 触控无真机，只能提供 DevTools/鼠标替代，需要制作人接受边界或安排真实触控补证。
- Meta storage 变化来自自然推进还是真实回流无法判断，需要隔离 profile 或更清晰的前后摘要。
- 小屏风险为非阻塞，但需要按原视口追加截图、视频或人工路径记录。
- 证据类型混杂，无法判断哪些是自然点击、哪些是辅助覆盖，需要补齐字段后再进 #9。
