## 复测过程

### 第一轮：默认窗口

- 目标：按初始推荐路径建造到可采候选。
- 初始推荐：`-2,0 -> -3,0 -> -4,0 -> -5,0 -> -5,-1 -> -5,-2`，目标 `-5,-2`。
- 初始诊断：`pathAffordable=true`，总需求 `metal=76` / `ore=10`。
- 结果：只成功建到 `-4,0`。从 `-5,0` 起，推荐格屏幕坐标命中左侧建造面板而不是 canvas，真实点击未进入游戏画布。
- 结论：默认窗口下失败原因是输入/布局遮挡，不是资源不可支付，也不是 hook 缺失。

### 第二轮：宽窗口

- 目标：排除默认窗口遮挡后，再走真实点击路径。
- 窗口：`1427 x 907`，DPR `2`。
- 初始候选：目标 `-5,4`，初始 `pathAffordable=true`，总需求 `metal=92` / `ore=10`。
- 执行方式：每一步重新读取当前 `ghostPath.stepGridKeys[0]`，点击下一推荐格。
- 实际点击序列：
  - `frame -2,0`
  - `frame -1,-2`
  - `frame 0,-3`
  - `frame 0,-4`
  - `frame 0,-5`
  - `frame 0,-6`
  - `frame 0,-7`
  - `mining 0,-7`
- 所有上述 grid 点击命中 `CANVAS#game`，无 HUD 遮挡。
- 采矿站建成后，`cells` 中存在 `0,-7: mining`，`active=true`，`enabled=true`，`detached=false`。

## 收益归因

| 采样点 | harvestingCount | target | resources | body.amount |
| --- | ---: | --- | --- | ---: |
| 建站后等待前 | `1` | `小行星-3` / `metal` | `metal=92.009, ore=57.315, gas=39.876, plasma=9.016` | `332.533` |

归因判断：

- `harvestingCount` 从建站后保持为 `1`。
- 目标天体 `body.amount` 从 `332.533` 降到 `280.311`，下降 `52.222`。
- 目标资源为 `metal`，玩家 `metal` 从 `92.009` 增至 `151.059`，增加 `59.050`。
- 资源增加与目标天体下降方向一致，且页面无 runtime error，判定取得可归因采矿收益。

## 结论

- #5 采矿收益复测通过：真实输入路径可在宽窗口下按推荐路径建造采矿站，并取得 `harvestingCount > 0`、目标资源下降和资源增加证据。
- v0.20 遗留的“采矿收益不可归因”失败路径本轮关闭。
- #7 采矿收益 gate 可解锁；但 #6 首局 P0 建造 / 采矿 / 资源 / 维修闭环仍需单独验收。

## 残留风险

1. 默认窗口下左侧建造面板会遮挡部分推荐格，玩家真实输入可能仍受布局影响；建议 #9 小屏 / 触控证据重点覆盖 HUD 遮挡。
2. 本轮精确复测的是当前生成路径的等价历史失败路径，未出现原始 `mining=-1,1` 坐标。
3. `mobileApproachHint` 本轮因为路径可支付未触发 UI 文案，仍需在不可达或不可支付局面补专项证据。
