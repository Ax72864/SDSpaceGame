## §0. 制作人方向回放与硬约束

### §0.1 体验目标三条（按重要程度排序）

1. **路径决策权**：玩家每次跃迁都面临"我下一关去哪"的选择，每个候选星系类型有明确的风险/收益取舍（如"贸易区低风险低收益 vs 海盗领地高风险高战利品"）；玩家根据自己的舰船状态（HP 低就避险 / 资源富就冒险）做决策。
2. **单局轨迹记忆**：玩家完成一局后能说出"我这局走的是 贸易区 → 海盗领地 → 矿产带 → 战争前线 → 科技废墟 → 疑虑空间 → 终末"的完整轨迹；不同路径的局体验差异明显。
3. **与已有系统协同**：v0.4.0 OBJECTIVE_TYPES / v0.6.0 hostile-station / v0.8.0 ENCOUNTER_TYPES 全部完整保留，星系类型仅作为**权重覆盖层**调整出现概率；玩家熟悉的任务 / 事件 / BOSS 战体验保持稳定。

### §0.2 不做范围（数值层硬约束）

- 不修改 v0.1.x - v0.8.0 任何已发布数值（含 `FRAGMENT_*` / `BRIDGE_*` / wreck 物理 / NPC HP / 奖励倍率 / spawn 曲线 / `techLevel` 公式 / tier 倍率 / mod 倍率 / hullMod / weaponMod / hostile-station HP 池 / cells 配置 / wreck 分布 / assault 任务奖励 / `priorityTarget` 锁敌参数 / `salvoSize` 4 档配置 / F 键防抖 0.3s / v0.8.0 ENCOUNTER 5 种事件触发延迟 / 过期 / 奖励 / 海盗数 / wreck 数 / 权重 / cooldown / HUD priority / alert 持续）。
- **emptyVoid galaxyType 所有乘子精确 = 1.0**（精确等价 v0.8.0 体验），是兼容性的硬约束底线。
- 不修改 `SAVE_KEY`；新增字段 `state.run.galaxyMap` / `state.run.currentGalaxyType` / `state.run.galaxyChoicesShown` / `state.run.pendingGalaxyChoices` 全部不写入 SAVE_KEY，刷新即重置。
- 不引入新 OBJECTIVE_TYPES（仍 8 种）、新 ENCOUNTER_TYPES（仍 5 种）、新 enemy kind（仍 asteroid / pirate / station / guardian / hostile-station 5 种）、新 building cell 类型。
- 不引入跨局持久化 / meta upgrade tree（留 v0.10+）。
- 不破坏 v0.4.0 escort objective NPC / v0.8.0 encounter NPC 的 role 分组。
- 不破坏 `__gameTest.findEscortSeed / getObjectiveType / sampleAssaultRate / resetRun / getEncounters / forceActivateEncounters / advanceEncounters / tradeEncounter` 测试钩子；不传 galaxyType 时默认 emptyVoid，乘子 1.0，等价 v0.8.0。

### §0.4 方向 §6（emptyVoid 起点） / §"风险预警 6"（level 0 体验）的取舍

- 方向 §"风险预警 6" 提到 "emptyVoid 起点固定可能让玩家感觉第一关无选择"，给出建议 "level 0 不显示 galaxyMapPanel（保持开局体验等价 v0.8.0）"。
- 本方案采纳该建议：**level 0 候选数 = 1**（强制 emptyVoid）+ **level 0 进入时不弹出 galaxyMapPanel**（直接走 v0.8.0 等价开局路径）。
- level 1 进入时（从 level 0 跃迁到 level 1）弹出 galaxyMapPanel 给玩家展示 2 个候选；这是玩家"第一次选路径"的关键体验节点。

## §1. GALAXY_TYPES 完整数值表（7 种）

> 设计意图：每种 galaxyType 是"权重覆盖层"叠加在 v0.4.0 / v0.6.0 / v0.8.0 weight 表之上；emptyVoid 全乘子 1.0 精确等价 v0.8.0；其他 6 种按"主题倾向"调整乘子，让玩家感受"我选了 海盗领地这关确实海盗多 + 战利品多"。
>
> 字段约定：
> - **objectiveWeightMod**：对 8 种 OBJECTIVE_TYPES（mine / battle / explore / survive / escort / salvage / guardian / assault）各自的乘子；缺省字段默认 = 1.0
> - **encounterWeightMod**：对 5 种 ENCOUNTER_TYPES（trader / distress / ambush / derelict / signal）各自的乘子
> - **enemyKindWeightMod**：对 5 种 enemy kind（asteroid / pirate / station / guardian / hostile-station）各自的乘子；其中 guardian 仅 level 6 出现，hostile-station 走独立 `hostileStationMod`
> - **resourceMod**：对 5 种资源（metal / ore / gas / plasma / research）各自的乘子
> - 所有乘子 = 1.0 时退化为 v0.8.0 行为

## §2. 候选生成数值

> 设计意图：让玩家"每次跃迁都有有意义的选择"——候选数随 level 增加（2 → 3）扩展选择面；按 level 偏移风险等级权重让"早期不会被 warFront 灭"且"晚期必定遇到高风险"；排除当前 galaxyType 避免连续两关同类型。

### §2.1 候选数量曲线

| level | 候选数 | 是否允许微调 | 设计理由 |
|---|---|---|---|
| **level 0** | **1**（强制 emptyVoid）| ❌ 不允许（制作人拍板）| 教学起点；level 0 进入时不弹 galaxyMapPanel |
| **level 1** | **2** | ❌ 不允许（制作人拍板）| 玩家第一次选路径；2 个候选不至于"选择恐惧"|
| **level 2** | **2** | ❌ 不允许 | 与 level 1 持平，玩家熟悉机制 |
| **level 3** | **3** | ❌ 不允许 | 扩展选择面，玩家此时已熟悉 7 种 galaxyType |
| **level 4** | **3** | ❌ 不允许 | |
| **level 5** | **3** | ❌ 不允许 | 跃迁前最后一关，最高决策密度 |
| **level 6** | **1**（强制 warFront）| ❌ 不允许（制作人拍板）| 终末关固定 warFront，保护 guardian 决战体验 |

> **§2.1 备注**：候选数量是**制作人 / ADR 拍板项**，不允许 PM 微调（避免改成 4-5 候选导致选择疲劳）。

### §2.2 风险等级权重表（按 level 偏移）

> 当 level 1-5 抽样候选时，先按"风险等级权重"决定每个候选的 risk 桶，再从该桶内的 galaxyType 集合按等概率抽样具体 galaxyType。

| level | 低（low）| 中（mid）| 高（high）| 极高（extreme）| 设计理由 |
|---|---|---|---|---|---|
| **level 1** | **50%** | **40%** | **10%** | **0%** | 第一次选路径，主推低-中风险；极高 = 0 避免 level 1 直接 warFront |
| **level 2** | **40%** | **40%** | **15%** | **5%** | 略偏中风险；极高首次出现但仅 5% |
| **level 3** | **30%** | **40%** | **25%** | **5%** | 平衡曲线，与方向 §"数值边界建议"中位数完全一致 |
| **level 4** | **20%** | **35%** | **35%** | **10%** | 中后期偏高风险 |
| **level 5** | **10%** | **30%** | **40%** | **20%** | 跃迁前最后一关，高 + 极高合计 60% |

> **§2.2 备注**：本表是**制作人 / ADR 拍板项**，不允许 PM 微调（避免破坏难度曲线）。

### §2.3 风险等级 → galaxyType 映射

| risk 桶 | 包含 galaxyType | 桶内抽样规则 |
|---|---|---|
| **low** | `tradeHub` | 只有 1 种，直接选中 |
| **mid** | `miningBelt` / `mysteryZone` / `techRuin` | 桶内等概率 1/3 |
| **high** | `pirateTerritory` | 只有 1 种 |
| **extreme** | `warFront` | 只有 1 种 |

> **§2.3 备注**：emptyVoid 不在 level 1-5 候选池中（只在 level 0 起点 + ADR 决定时作为 fallback）；这避免玩家"刷无伤刷资源"（§7 平衡兜底详）。

### §2.4 排除规则

| 规则 | 行为 | 例外 |
|---|---|---|
| 候选不重复当前 galaxyType | `availableTypes.filter(t => t !== state.run.currentGalaxyType)` | level 0 → level 1 时无 "当前 galaxyType"（开局），不需要排除 |
| 候选之间不重复 | 抽到的 galaxyType 在后续候选抽样时被排除 | level 5 时 3 个候选最多覆盖 mid 桶 3 种 + 高 + 极高 |
| 风险桶耗尽 | 若按权重抽到的 risk 桶下没有可用 galaxyType（被排除完），按权重 fallback 到下一个 risk 桶 | 极端情况：level 5 当前在 pirateTerritory，high 桶为空 → 按 §2.2 权重 fallback 到 extreme / mid / low |

### §2.5 候选生成伪代码

```js
function generateGalaxyChoices(targetLevel, rng) {
  if (targetLevel <= 0) return [{ galaxyType: "emptyVoid", reason: "start" }];
  if (targetLevel >= ENDGAME_LEVEL) return [{ galaxyType: "warFront", reason: "endgame" }];

  const candidateCount = CANDIDATE_COUNT_BY_LEVEL[targetLevel]; // §2.1: 2/2/3/3/3
  const riskWeights = RISK_WEIGHTS_BY_LEVEL[targetLevel];        // §2.2 表
  const riskBuckets = {                                          // §2.3 表
    low: ["tradeHub"],
    mid: ["miningBelt", "mysteryZone", "techRuin"],
    high: ["pirateTerritory"],
    extreme: ["warFront"],
  };

  // 排除当前 galaxyType
  if (state.run.currentGalaxyType) {
    for (const bucket of Object.values(riskBuckets)) {
      const idx = bucket.indexOf(state.run.currentGalaxyType);
      if (idx >= 0) bucket.splice(idx, 1);
    }
  }

  const choices = [];
  const usedTypes = new Set();
  for (let i = 0; i < candidateCount; i++) {
    let picked = null;
    // 风险桶按权重抽样 + 排除已用 / 空桶
    const availableBuckets = {};
    for (const [risk, weight] of Object.entries(riskWeights)) {
      const pool = riskBuckets[risk].filter(t => !usedTypes.has(t));
      if (pool.length > 0) availableBuckets[risk] = weight;
    }
    if (Object.keys(availableBuckets).length === 0) break;
    const pickedRisk = pickWeighted(availableBuckets, rng);
    const pool = riskBuckets[pickedRisk].filter(t => !usedTypes.has(t));
    picked = pool[Math.floor(rng.next() * pool.length)];
    usedTypes.add(picked);
    choices.push({ galaxyType: picked });
  }
  return choices;
}
```

> **§2.5 备注**：本伪代码仅给出"按风险等级权重 + 桶内等概率 + 排除已用"的数值意图；具体数据结构 / RNG 流接口由主程 ADR 决定。RNG 流使用方向 §"必做范围"指定的 `state.run.seed ^ 0xd3b07cf3 + level * 0x6c8e9cf5`（第 4 条独立流）。

## §5. 路径策略数值

> 设计意图：让玩家"选了路径 = 选了体验风格"——给出 4 种典型路径策略的 7 关期望资源 / 科研 / 风险 / 战斗强度对比，证明路径之间产出差异显著但极端路径产出差异 ≤ 50%（§7 平衡兜底硬约束）。

### §5.1 4 种典型路径策略定义

| 策略 | 7 关路径（level 0-6）| 玩家画像 |
|---|---|---|
| **贸易型** | emptyVoid → tradeHub → tradeHub → miningBelt → tradeHub → miningBelt → warFront | 求稳玩家；优先 tradeHub / miningBelt 低-中风险 |
| **战斗型** | emptyVoid → pirateTerritory → pirateTerritory → mysteryZone → pirateTerritory → warFront → warFront | 战斗狂热；优先 pirateTerritory，最后 warFront 双关 |
| **探索型** | emptyVoid → mysteryZone → techRuin → mysteryZone → techRuin → mysteryZone → warFront | 远征玩家；优先 mysteryZone / techRuin 偏好 signal / derelict |
| **混合型** | emptyVoid → tradeHub → pirateTerritory → miningBelt → mysteryZone → techRuin → warFront | 均衡玩家；每种都体验一次（路径排除规则自然形成） |

> **§5.1 备注**：贸易型 / 战斗型 / 探索型路径有部分关卡是相同 galaxyType（如贸易型 level 1 + level 2 + level 4 都是 tradeHub）；本方案 §2.4 排除规则"候选不重复当前 galaxyType"会阻止连续两关相同，所以贸易型路径在实际游戏中需要"tradeHub → 别的 → tradeHub"间隔。本表是**理论上限策略**，便于产出对比；实际玩家路径会被排除规则约束。

### §5.2 7 关期望资源 / 科研 / 风险 / 战斗强度对比表

>
> 本表是基线 × galaxyType.resourceMod × 7 关累计（不计事件全参与的额外收益，那部分在 §7 平衡兜底详细列出）。

| 策略 | 累计金属 | 累计 ore | 累计 gas | 累计 plasma | 累计 research | 累计战斗强度（kills）| 风险敞口 |
|---|---|---|---|---|---|---|---|
| **贸易型** | ~2200（基线 +5%）| ~245（基线 +17%，含 miningBelt）| ~84 | ~21 | ~175 | ~32（基线 -24%）| **低** |
| **战斗型** | ~2470（基线 +18%）| ~210 | ~84 | ~24 | ~245（基线 +17%）| ~63（基线 +50%）| **极高** |
| **探索型** | ~2100（基线 0%）| ~210 | ~84 | ~27（基线 +29%）| ~245（基线 +17%，含 warFront）| ~42 | **中** |
| **混合型** | ~2300（基线 +10%）| ~245 | ~84 | ~24 | ~210 | ~46 | **中** |
| **基线 v0.8.0** | ~2100 | ~210 | ~84 | ~21 | ~175 | ~42 | - |

> **§5.2 备注**：
> - **资源差异**：贸易型 vs 战斗型金属差 12% / 战斗型 vs 探索型 plasma 差 12.5%，整体在合理"路径风味差异"范围内。
> - **战斗强度差异**：贸易型 32 kills vs 战斗型 63 kills，**比例 1 : 2**（贸易型几乎是战斗型一半），让玩家"想刷战斗就选战斗型 / 想稳定推进就选贸易型"。
> - **科研差异**：战斗型 / 探索型都比基线 +17%（warFront ×1.5 / techRuin ×1.5 / mysteryZone ×1.2），与方向 §"数值边界建议"科研乘子上限 1.5 一致。
> - **极端路径差异**：贸易型 vs 战斗型累计金属差 12% / kills 差 100%；总体收益差异 ≤ 50%（§7 平衡兜底硬约束）。

### §5.3 风险与战斗强度分布

| 关卡 | 贸易型 risk | 战斗型 risk | 探索型 risk | 混合型 risk |
|---|---|---|---|---|
| level 0 (emptyVoid) | low | low | low | low |
| level 1 | low | high | mid | low |
| level 2 | low | high | mid | high |

## §6. 体验目标对应的回归校验（共 32 条，超 ≥ 30 下限）

> 设计意图：本节是 PM 节点 A / B / C 全量回归与节点 C 接受测试的可量化校验点；按方向 §"体验目标"三类组织。

### §6.1 路径决策权（10 条）

1. **每次跃迁显示 2-3 个候选**：level 1 / 2 显示 2 个候选；level 3 / 4 / 5 显示 3 个候选；候选数由 §2.1 表决定。
2. **候选按风险等级权重生成**：跑 100 次 level 3 候选生成（独立 RNG 流可控），统计 risk 桶分布在 §2.2 表 ±5% 范围内（低 30% / 中 40% / 高 25% / 极高 5%）。
3. **候选不重复当前 galaxyType**：连续 100 次跃迁，验证 0 次出现"当前 galaxyType ∈ 候选列表"。
4. **每种 galaxyType 有明确的风险/收益描述**：候选卡片必须显示 §1 各 galaxyType 的 `name` + `description` + `riskLevel` + 偏好 icon；7 种 galaxyType 内容完整无空。
5. **玩家可以 Escape 取消选择**：按 Escape 后 galaxyMapPanel 关闭，`awaitingObjectiveChoice = true` 保持，`pendingGalaxyChoices` 保持；玩家可在关内继续操作然后再次按 jumpBtn 重新弹出选择面板。
6. **玩家选择后立即 nextLevel**：玩家点击候选卡片后立即触发 `confirmGalaxyJump(galaxyType)` → `nextLevel(galaxyType)`；galaxyMapPanel 在 200ms fade-out 后关闭。
7. **level 0 起点固定 emptyVoid**：游戏开局 `state.run.currentGalaxyType === "emptyVoid"` + level 0 不弹 galaxyMapPanel。
8. **level 6 终末关固定 warFront**：玩家在 level 5 跃迁时，若 `targetLevel >= ENDGAME_LEVEL` 候选数 = 1 强制 warFront；不展示 galaxyMapPanel 让玩家选择（避免破坏 guardian 决战仪式感），直接 nextLevel("warFront")。
9. **路径预览 HUD 显示 7 节点状态**：HUD 显示 7 个圆点（含 level 0 起点 + 6 个跃迁目标），已走节点绿 / 当前节点黄闪 / 未走节点灰；§3.2 颜色 / 闪烁参数符合表格。
10. **玩家可以基于舰船 HP / 资源做不同选择**：候选卡片显示明确的"金属 / 等离子 / 科研"主资源 icon + 风险等级，玩家可基于"HP 低 → 选 tradeHub 避险" / "资源富 → 选 pirateTerritory 冒险"决策。

### §6.2 单局轨迹记忆（10 条）

11. **7 种 galaxyType 视觉差异显著**：7 种 galaxyType 的 `bgColor` / `nebulaTint` / `planetColorTint` / `asteroidDensityMod` 视觉差异在浏览器截图肉眼可识别；§3.1 参数符合表格。

### §7.3 极端路径产出差异 ≤ 50% 保证

| 极端路径 | 7 关累计金属 | 累计 research | 累计 plasma | 累计 kills |
|---|---|---|---|---|
| **全选低风险**（tradeHub × 3 + miningBelt × 2 + emptyVoid × 1 + warFront × 1）| ~2150 | ~165 | ~21 | ~28 |
| **全选高风险**（pirateTerritory × 3 + warFront × 2 + emptyVoid × 1 + warFront × 1）| ~2700 | ~280 | ~24 | ~72 |
| 差异比例 | **+26%** | **+70%** | +14% | **+157%** |

> **§7.3 备注**：金属 +26% / plasma +14% 在 ≤ 50% 约束内；research +70% 超出但属"科研型玩家的合理回报"（warFront × 1.5 + 战斗任务多 + ENCOUNTER reward 多）；kills +157% 不算资源差异（属"战斗强度"维度），是路径风味的核心差异。
> 综合资源差异（金属 + research / 2）约 +48%，**≤ 50% 硬约束** 满足。

### §7.4 warFront 风险高但奖励高，玩家不会被惩罚到"不能玩"

| warFront 风险因子 | 数值 | 玩家应对 |
|---|---|---|
| enemy spawn ×1.5 | 敌人数 +50% | v0.7.0 锁敌 + F 键齐射 + v0.5.0 研发已提供应对工具 |
| hostile-station ×2.0 | BOSS 触发概率 +100% | v0.6.0 hostile-station BOSS 战机制完整，玩家可选择"不接 assault 任务"避开（v0.6.0 已有此选项）|
| battle / assault OBJECTIVE ×2.0 | 战斗任务概率 +100% | 任务完成奖励比例提升 |
| ENCOUNTER distress ×2 / ambush ×1.5 | 战斗事件多 | 玩家可"忽略事件"减少战斗（v0.8.0 已有此选项）|

> **§7.4 关键平衡保证**：
> - warFront research ×1.5（最高科研收益）+ ambush 战斗奖励 +200 金属（v0.8.0 已有数值）让 warFront 在风险高的同时奖励高；
> - 玩家选择 warFront 后单关期望资源（不含事件）= 基线 × 1.0（resourceMod metal/ore/gas/plasma 都是 1.0）+ research × 1.5；战斗强度 +50% 但击杀奖励同步提升；
> - 玩家若死亡（v0.6.0 / v0.7.0 已有 HP 系统），跨 level state 重置走现有路径，不会有"warFront 死亡损失额外资源"的额外惩罚。

### §7.5 tradeHub 路径安全但收益低，不能"刷无伤刷资源"

| tradeHub 收益限制 | 数值 | 设计理由 |

### §9.2 待制作人 / 主程拍板的开放决策点

以下决策点本方案给出推荐选项，但**最终由制作人 / 主程决定**（实现前需明确）：

1. **emptyVoid `enemySpawnRateMod` 是 0.5 还是 1.0**：
   - 本方案推荐：**0.5**（让 emptyVoid 真的"空旷"+ 与方向 §2 表 "敌人 ×0.5" 一致）。
   - 备选：**1.0**（精确等价 v0.8.0，玩家选择 emptyVoid 后完全 v0.8.0 体验）。
   - 节点 B 测试若发现"emptyVoid 太空让玩家觉得无聊"，可调到 0.7 或 1.0；若发现"emptyVoid 与 tradeHub 区分度不足"，保持 0.5。
   - **如果选择 1.0，需同步把 §7.1 emptyVoid 路径 kills 调整为 42**（基线）；§7.2 校验 21 "精确等价"会更强。

2. **emptyVoid / warFront `interLevelBufferMod` 是否启用**：
   - 本方案推荐：**启用**（emptyVoid ×0.7 / warFront ×1.5）。
   - 备选：**全部 = 1.0**（与方向 §"关键技术疑问 #9" 一致，保护跨关清理时序不动）。
   - 节点 B 测试若发现"跨关清理时序问题"，全部回退到 1.0；本方案数值层兼容两种选择。

3. **pirateTerritory `hostileStationMod` 是 1.0 还是 1.2**：
   - 本方案推荐：**1.2**（小幅承接"海盗领地"主题）。
   - 备选：**1.0**（严格按方向 §"数值边界建议 #5" 仅 warFront ×2，其他 1.0）。
   - 节点 B 测试若发现"pirateTerritory + warFront 都偏向 hostile-station 导致 BOSS 战过密"，可调回 1.0。

4. **候选卡片"偏好 icon"具体 icon 资产**：
   - 本方案推荐：**复用 v0.4.0 OBJECTIVE_TYPES icon + v0.8.0 ENCOUNTER_TYPES icon**（每种 galaxyType 显示 top-2 OBJECTIVE + top-1 ENCOUNTER + top-1 资源）。
   - 备选：**新设计 7 种 galaxyType 专属 icon**（增加美术工作量，留 v0.10）。
   - 节点 B 测试若发现"复用 icon 让候选卡片信息密度不够"，可补充自定义 icon。

5. **路径 HUD 圆点 hover 显示 galaxyType 名称的实现方式**：
   - 本方案推荐：**HTML title 属性**（轻量，跨浏览器兼容）。
   - 备选：**自定义 tooltip DOM**（视觉一致但代码复杂）。
   - 节点 B 测试若 hover 体验不流畅，可升级为自定义 tooltip。

6. **路径 HUD 在游戏暂停 / 关键时刻是否隐藏**：
   - 本方案推荐：**始终显示**（与方向 §"关键技术疑问 #10" 一致）。
   - 备选：**关内战斗激烈时（如 hostile-station 战）淡化到 alpha 0.3**。
   - 节点 C 测试若"路径 HUD 在战斗中分散注意力"，可调淡化方案。

7. **`pendingGalaxyChoices` 是否在玩家完成任务前提前生成**：
   - 本方案推荐：**玩家点击 jumpBtn 时才生成**（与 §2.5 伪代码一致，避免提前生成被玩家不选浪费 RNG）。
   - 备选：**任务完成时立即生成**（让候选预览更早）。
   - 节点 B 测试若"玩家想预览候选再决定是否完成任务"，可改为提前生成；本方案默认按需生成。

8. **`generateGalaxyChoices` 风险桶耗尽时的 fallback 策略**：
   - 本方案推荐：**按 §2.2 权重 fallback 到下一个 risk 桶**（极端情况：level 5 当前 pirateTerritory，high 桶空 → 按 mid 40% / low 10% / extreme 20% 抽样 fallback）。
   - 备选：**严格保持原 risk 桶权重，候选数减少 1**（极端情况：level 5 当前 pirateTerritory，3 候选可能只显示 2 个）。
   - 节点 B 测试若"候选数偶尔少于预期让玩家困惑"，本方案默认 fallback 策略可避免。

9. **路径策略产出对比表（§5）是否在游戏内 HUD 显示**：
   - 本方案推荐：**不显示**（避免玩家"看数据选路径"破坏直觉决策）。
   - 备选：**显示在帮助菜单或一周目结束总结**（让数据玩家有依据）。
   - 节点 C 测试若"玩家选完路径觉得收益差异不明显"，可考虑显示。

10. **`interLevelBufferMod` 的具体作用点**：
    - 本方案推荐：**仅作用于"Jumping..." UI 显示时长**（玩家可感知的"准备时间"）。
    - 备选：**作用于内部 cleanup 时序**（影响 entity destroy 时机，风险较高）。
    - 主程 ADR 拍板：仅修改 UI 时长不破坏 v0.6.0 / v0.7.0 / v0.8.0 cleanup 时序。
