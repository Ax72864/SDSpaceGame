## §0. 制作人方向回放与硬约束

### §0.1 体验目标三条（按重要程度排序）

1. **路径决策权**：玩家每次跃迁面对 2-3 个候选星系，每个候选有明确的"风险/收益"取舍（低风险低收益 vs 高风险高战利品）；玩家根据当前舰船 HP / 资源储备做"避险或冒险"决策。
2. **单局轨迹记忆**：玩家完成一局后能说出完整 7 关轨迹（如"emptyVoid → tradeHub → pirateTerritory → miningBelt → warFront → techRuin → mysteryZone → warFront 终末"）；不同路径产出的局体验差异显著（不是 7 关都长一样）。
3. **与已有系统协同**：v0.4.0 OBJECTIVE_TYPES 8 种 / v0.6.0 hostile-station / v0.8.0 ENCOUNTER_TYPES 5 种 / v0.7.0 锁敌 / v0.5.0 研发 / v0.3.0 fragment dock 全部完整保留；galaxyType 仅作为"权重覆盖层"调整出现概率，玩家熟悉的体验完全稳定。

## §1. GALAXY_TYPES 完整数值表（7 种）

> 设计意图：7 种 galaxyType 形成"风险/收益"四象限+"特化路径"对角：tradeHub / emptyVoid 是低风险低收益避险锚点；pirateTerritory / warFront 是高风险高收益冒险锚点；miningBelt / techRuin / mysteryZone 是中风险特化路径（资源 / 科研 / 探索偏好）。每种 galaxyType 的乘子刻意保留 fallback 路径（emptyVoid 全乘子=1.0），让 v0.8.0 基线完全可复现。

### §1.0 GALAXY_TYPES 字段集

| 字段 | 类型 | 说明 | 取值范围 |
|---|---|---|---|
| `name` | string | 中文显示名 | 4 字以内 |
| `description` | string | 1-2 句简介（候选卡片显示）| ≤ 40 字 |
| `riskLevel` | string | 风险等级 | `"low"` / `"mid"` / `"high"` / `"extreme"` |
| `iconColor` | hex | HUD icon 用色 | 6 位 hex |
| `nebulaTint` | hex | 星云着色 | 6 位 hex |
| `bgColor` | hex | 背景色（drawBackground fillStyle）| 6 位 hex |
| `objectiveWeightMod` | object | 对 8 种 OBJECTIVE 各自乘子 | [0.3, 3.0] |
| `encounterWeightMod` | object | 对 5 种 ENCOUNTER 各自乘子 | [0.3, 3.0] |
| `enemySpawnRateMod` | number | 敌人 spawn 频率乘子 | [0.5, 2.0] |
| `enemyKindWeightMod` | object | 敌人 kind 权重乘子（pirate / station 等）| [0.5, 2.0] |

## §2. 候选生成数值

> 设计意图：让"候选生成"既给玩家选择权又保持难度曲线——level 0 / 6 强制锚点保护开局教学和终末决战体验；level 1-5 候选数量按"早期 2 个 / 中后期 3 个"分段（给玩家逐步学习选择的认知负担）；风险权重按 level 上升渐进偏移，避免 level 1 直接出 warFront 让新玩家挫败。

### §2.1 候选数量分布

| level | 候选数量 | 是否允许微调 | 设计理由 |
|---|---|---|---|
| **level 0** | **1**（强制 emptyVoid）| ❌ 不允许（制作人拍板）| 起点固定教学性低难度，玩家先熟悉基础玩法 |
| **level 1** | **2** | ❌ 不允许（制作人拍板）| 第一次选择，2 个候选降低决策负担 |
| **level 2** | **2** | ❌ 不允许 | 与 level 1 持平，玩家适应"选择"机制 |
| **level 3** | **3** | ❌ 不允许 | 候选扩展到 3 个，玩家开始做"三选一"权衡 |
| **level 4** | **3** | ❌ 不允许 | 与 level 3 持平 |
| **level 5** | **3** | ❌ 不允许 | 跃迁前最后一次选择，3 候选最大化决策空间 |
| **level 6** | **1**（强制 warFront）| ❌ 不允许（制作人拍板）| 终末关固定 BOSS 战体验，保护 guardian 决战 |

> **§2.1 备注**：候选数量是**制作人拍板项**，不允许 PM 微调。强制 level 0=emptyVoid / level 6=warFront 是方向 §"7 种 galaxyType 设计"明确约束，保护开局教学体验和终末战体验。

### §2.2 风险等级权重表（levelBias）

> 候选生成时按"风险等级 → 基线权重 → 按 level 偏移"两层加权抽样：先按 §2.2 表查出当前 level 的"低/中/高/极高"概率分布，再在每个风险等级内的可选 galaxyType 间均匀抽取（已排除当前 galaxyType + 已抽中的 galaxyType）。

| level | 低风险 % | 中风险 % | 高风险 % | 极高风险 % | 是否允许微调 |
|---|---|---|---|---|---|
| **level 1** | **50%** | **40%** | **10%** | **0%** | ❌ 不允许（制作人拍板）|
| **level 2** | **40%** | **40%** | **15%** | **5%** | ❌ 不允许 |
| **level 3** | **30%** | **40%** | **25%** | **5%** | ❌ 不允许 |
| **level 4** | **20%** | **35%** | **35%** | **10%** | ❌ 不允许 |
| **level 5** | **10%** | **30%** | **40%** | **20%** | ❌ 不允许 |

> **§2.2 备注**：
> - level 1 极高风险 0% 是硬约束：避免新玩家第一次选择就直接遇到 warFront 挫败。
> - level 5 高 + 极高 60% 是设计意图：跃迁前最后一次选择应该让玩家面临"是否冒险冲击终末"的压力。
> - 权重表是制作人拍板项（方向 §"数值边界建议" 第 7 / 8 项的展开）；PM 不允许微调单格百分比，但允许 PM 在 §13.1 中调整 level 1 / level 5 整行的"低 ↔ 高"偏向（±5% 范围内）。

### §2.3 风险等级内 galaxyType 映射

> 7 种 galaxyType 按 §1.0 `riskLevel` 字段分组：

| 风险等级 | 包含的 galaxyType | 数量 |
|---|---|---|
| low | tradeHub / emptyVoid | 2 |
| mid | miningBelt / mysteryZone / techRuin | 3 |
| high | pirateTerritory | 1 |
| extreme | warFront | 1 |

> **§2.3 备注**：低风险 2 个 / 中风险 3 个 / 高风险 1 个 / 极高 1 个的分布是方向 §"7 种 galaxyType 设计" 的隐式结构；本方案明确组织。注意 emptyVoid 在 level 1-5 候选生成中**允许出现**（玩家可以主动选"喘息一关"），但不会作为强制候选；level 0 是强制起点不在 level 1-5 抽样范围。

### §2.4 排除规则

| 排除条件 | 说明 |
|---|---|
| **当前 galaxyType** | 候选池排除当前 galaxyType（避免连续两关同类型）|
| **已抽中候选** | 单次候选生成内，每个 galaxyType 最多出现 1 次 |
| **emptyVoid 不强制排除** | level 1-5 候选池保留 emptyVoid（让玩家可选"喘息"），仅 level 0 / 6 强制锚点 |
| **特殊情况兜底**：候选池耗尽 | 抽样过程中若 availableTypes 为空，提前返回当前已抽中的候选（即使数量不足 2 / 3）|

### §2.5 候选生成伪代码

```js
function generateGalaxyChoices(targetLevel, rng) {
  if (targetLevel <= 0) return [{ galaxyType: "emptyVoid", reason: "start" }];
  if (targetLevel >= ENDGAME_LEVEL) return [{ galaxyType: "warFront", reason: "endgame" }];
  
  const candidateCount = CANDIDATE_COUNT_BY_LEVEL[targetLevel]; // §2.1: 1-2=2, 3-5=3
  const riskBias = RISK_BIAS_BY_LEVEL[targetLevel];             // §2.2 表
  
  let available = Object.keys(GALAXY_TYPES);
  if (state.run.currentGalaxyType) {
    available = available.filter(t => t !== state.run.currentGalaxyType);
  }
  
  const choices = [];
  for (let i = 0; i < candidateCount && available.length > 0; i++) {
    // 1. 按风险等级权重抽样
    const targetRisk = pickWeightedRisk(riskBias, rng);
    // 2. 在该风险等级内均匀抽样 galaxyType
    const candidates = available.filter(t => GALAXY_TYPES[t].riskLevel === targetRisk);
    if (candidates.length === 0) {
      // 该风险等级无可选，降级到下一风险等级
      const fallback = pickFallbackRisk(targetRisk, available, rng);
      if (!fallback) break;
      choices.push({ galaxyType: fallback });
      available = available.filter(t => t !== fallback);
    } else {
      const picked = candidates[Math.floor(rng.next() * candidates.length)];
      choices.push({ galaxyType: picked });
      available = available.filter(t => t !== picked);
    }
  }
  return choices;
}
```

> **§2.5 备注**：本伪代码仅给出"按风险等级 + 排除规则抽样"的数值意图；具体数据结构、RNG 流接口、fallback 算法细节由主程 ADR 决定（方向 §"独立 RNG 流" 第 4 条 `seed ^ 0xd3b07cf3 + level * 0x6c8e9cf5`）。

### §2.6 候选生成期望分布预测（10000 次模拟）

## §5. 路径策略数值

> 设计意图：让"不同路径玩法明显不同"——4 类典型路径（贸易型 / 战斗型 / 探索型 / 混合）的资源 / 科研 / 风险 / 战斗强度差异显著但不超过 ±50%（§7 平衡兜底）；让玩家在体验中能说出"这局我走的是贸易型 / 战斗型"。

### §5.1 4 类典型路径定义

| **贸易型** | 优先 tradeHub / emptyVoid；中后期补 1 次 miningBelt | 高金属 / 低 wreck / 中科研 |
| **战斗型** | 优先 pirateTerritory；中后期补 1 次 warFront 中转 | 高金属 + 高 wreck / 中科研 |
| **探索型** | 优先 mysteryZone / techRuin；后期补 1 次 miningBelt | 高 plasma + 高科研 |
| **混合型** | 候选随机选；不刻意偏向 | 中等各项 |

## §6. 体验目标对应的回归校验（共 32 条，≥ 30 下限）

> 设计意图：本节是 PM 节点 A / B / C 全量回归与节点 C 接受测试的可量化校验点；按方向 §"体验目标"三类组织——路径决策权 11 / 单局轨迹记忆 10 / 与已有系统协同 11。

### §6.1 路径决策权（11 条）

1. **每次跃迁显示 2-3 个候选**：level 1-2 跃迁时 `state.run.pendingGalaxyChoices.length === 2`，level 3-5 跃迁时 `state.run.pendingGalaxyChoices.length === 3`；候选数量由 `CANDIDATE_COUNT_BY_LEVEL` 表查找（§2.1）。
2. **候选按风险等级权重生成**：跑 100 次 level 5 候选生成，统计极高风险（warFront）出现率 ≈ 20%，高风险（pirateTerritory）≈ 40%，中风险（3 种平均）≈ 30%，低风险（2 种平均）≈ 10%；与 §2.2 表一致（±5% 容差）。
3. **候选不重复当前 galaxyType**：玩家当前 `state.run.currentGalaxyType === "tradeHub"` 时，跃迁后弹出的 2-3 个候选中没有 tradeHub；候选生成算法 §2.5 排除规则生效。
4. **每种 galaxyType 有明确的风险/收益描述**：候选卡片显示 `name` + `description` + `riskLevel` 标签 + 至少 1 个偏好 icon（如"贸易 ×3"或"战斗 ×2"）；§3.3 卡片字段完整。
5. **玩家可以 Escape 取消选择**：玩家在 galaxyMapPanel 显示时按 Escape，panel fade-out 200ms 后隐藏；`state.run.awaitingObjectiveChoice` 保持 true（玩家可在关内继续操作，然后再次点 jumpBtn 重新弹出选择面板）。
6. **玩家选择后立即 nextLevel**：玩家点击候选卡片触发 `confirmGalaxyJump(galaxyType)`，调用 `nextLevel(galaxyType)`；从点击到 buffer 开始延迟 ≤ 200ms。
7. **level 0 起点固定 emptyVoid**：玩家开局 `state.run.level === 0` 时 `state.run.currentGalaxyType === "emptyVoid"`；不弹 galaxyMapPanel（level 0 进入是 game start，不是 nextLevel）。
8. **level 6 终末关固定 warFront**：玩家 level 5 跃迁完成后 `state.run.level === 6` + `state.run.currentGalaxyType === "warFront"`；玩家从 level 5 跃迁到 level 6 时**不显示 galaxyMapPanel**，直接 `nextLevel("warFront")`。
9. **路径预览 HUD 显示 7 节点状态**：HUD `#galaxyPathEl` 显示 7 个圆点；已走过节点用 §3.2 绿色 `#5ec460`（或 galaxyType iconColor），当前节点用黄色 `#ffcc44` 闪烁周期 0.8s，未走过节点用灰色 `#555555`。
10. **玩家可以基于舰船 HP / 资源做不同选择**：节点 C 主观测试——玩家在 HP < 30% 时选择候选倾向 tradeHub / emptyVoid（避险），HP > 80% + 金属充足时选择候选倾向 pirateTerritory / warFront（冒险）；本条不由代码校验，由 PM 节点 C 主观打分 ≥ 3/5。
11. **候选生成期望分布稳定**：跑 10000 次 level 3 候选生成，统计各 galaxyType 出现频次与 §2.6 期望表一致（±3% 容差）；用 `__gameTest.simulateGalaxyChoices(level, count)` 测试钩子验证。

### §6.2 单局轨迹记忆（10 条）

12. **7 种 galaxyType 视觉差异显著**：玩家进入每种 galaxyType 后 `drawBackground` fillStyle 切换为 §3.1 表对应 `bgColor`；7 种 hex 颜色完全不同；PM 节点 B 屏幕截图对比可视化确认。
13. **每种 galaxyType 名称 + icon HUD 显示**：屏幕左上角 `#currentGalaxyInfoEl` 显示当前 galaxyType `name` + iconColor 实心圆 + risk label；切换 galaxyType 时 fade-out 200ms + fade-in 250ms。
14. **路径预览 HUD 实时更新已走节点**：玩家完成 level N 跃迁到 level N+1 时，HUD `#galaxyPathEl` 中节点 N 从"当前"切换到"已走过"颜色；节点 N+1 从"未走过"切换到"当前"闪烁。
15. **玩家完成一局后路径完整可见**：玩家从 level 0 → 7 完整一局后，HUD 显示 7 个圆点全部为"已走过"色（绿或 galaxyType iconColor）；玩家可在结算时看到完整轨迹。
16. **不同路径的局体验差异明显**：节点 C 主观测试——同一种子（seed）走"全 tradeHub" vs "全 warFront"两局，玩家可说出至少 5 个体验差异点（敌人多少 / 资源多少 / 战斗强度 / 拾荒事件类型 / HUD 色系）。
17. **背景色 / 星云 / 行星按 galaxyType 切换**：玩家进入新 galaxyType 后 `drawBackground` / `drawNebula` / 行星渲染分别使用 §3.1 表对应 `bgColor` / `nebulaTint` / `planetColorTint`；切换在 buffer 结束时一次性完成（不分帧渐变）。
18. **当前 galaxyType 在 HUD 持续可见**：从 buffer 结束到下次 nextLevel，`#currentGalaxyInfoEl` 始终显示当前 galaxyType；玩家随时可看到"我在哪个星系"。
19. **已走过节点高亮 / 未走过灰色**：HUD `#galaxyPathEl` 中 visited=true 节点显示高亮色（绿或 iconColor），visited=false 节点显示灰色 `#555555`；当前 currentNodeId 节点显示黄色闪烁。
20. **候选卡片显示偏好 icon**：候选卡片显示 galaxyType 的偏好 icon——如 tradeHub 卡片显示"🛒×3 trader"、pirateTerritory 卡片显示"⚔️×2 battle"、mysteryZone 卡片显示"📡×3 signal"；玩家可一眼看出"这个星系偏好什么"。
21. **HUD 不遮挡视野**：路径预览 HUD（140 px × 24 px）+ 当前 galaxyType 状态条（160 px × 24 px）放在屏幕右上 + 左上角，**不遮挡屏幕中央 800 px × 600 px 战斗区域**；节点 B 屏幕截图验证。

### §7.4 warFront 风险高但奖励高，玩家不会被惩罚到"不能玩"

| warFront 单关 | 数值 | 玩家应对策略 |
|---|---|---|
| 敌人 spawn ×2 | 比 emptyVoid 4 倍敌人 | 玩家用 v0.7.0 锁敌 + F 键齐射 + cell 升级应对 |
| hostileStation ×2 | 触发概率翻倍 | 玩家可借 assault objective ×2 + research ×1.5 高回报 |
| battle objective ×2 | 主任务偏战斗 | 玩家"对症下药"用战斗装备完成任务 |
| metal ×1.2 / research ×1.5 / plasma ×1.2 | 资源 +20% ~ +50% | 战斗收益显著 |
| crossLevelBufferMod ×1.5（终末关）| 跨关 buffer +50% | 给玩家"凝重决战"心理准备 |

> **§7.4 保证**：warFront 是"高风险高回报"，玩家选择后 7 关之内有足够资源应对（research +50% → 升级 cell + 救援装备）。**不强制玩家选 warFront**（除 level 6 终末关），玩家可在 level 1-5 完全避开 warFront 走低风险路径，但终末关无法避开（设计意图：保护 guardian BOSS 战）。

### §7.5 tradeHub 路径安全但收益低，不能"刷无伤刷资源"

| tradeHub 单关 | 数值 | 玩家"刷资源"路径分析 |
|---|---|---|
| 敌人 spawn ×0.7 | 敌人数量减少 30% | 战斗收益减少 |
| battle objective ×0.5 | 主任务很少出现 battle | 玩家被"逼"走 mine / survive 路径 |
| metal ×1.2 | 单关金属 +20% | 唯一可"刷"的方向 |
| trader ×3.0 | trader 频次 3 倍 | 但 trader 是"换稀有资源"不"刷金属" |
| ambush ×0.3 | ambush 几乎不出现 | 高风险高回报路径被屏蔽 |
| research ×1.0 | 科研基线 | 无加成 |
| plasma ×1.0 | 等离子基线 | 无加成 |

> **§7.5 保证**：玩家走极端贸易型 7 关，金属 +30% 是上限；科研 / 等离子无加成；战斗事件少 + ambush 罕见 → 玩家"无伤"但"得不到稀有资源"。终末关 warFront 时 guardian 战斗强度不变（warFront 终末关数值固定），玩家不能用"贸易型 7 关"刷出 warFront 决战所需的稀有装备。**这与 §"游戏不能被刷"原则一致**。

### §9.2 待制作人 / 主程拍板的开放决策点

以下决策点本方案给出推荐选项，但**最终由制作人 / 主程决定**（实现前需明确）：

1. **resourceMod 字段是 5 资源还是 3 资源**：
   - 本方案推荐：**5 资源（metal / ore / gas / plasma / research）**完整字段；如 v0.8.0 代码仅 3 资源（metal / plasma / research），则 ore / gas 字段保留 1.0 占位等待 v0.10+ 扩展。
   - 节点 A 测试若发现 ore / gas 字段在 game.js 中完全不存在，ADR 可决定"仅保留 3 字段"，本方案对应修订。

2. **emptyVoid asteroidDensityMod=0.5 / enemySpawnRateMod=0.5 是否过低**：
   - 本方案推荐：**0.5（emptyVoid 设计意图 "敌人 ×0.5 / 小行星少"）**。
   - 节点 B 测试若发现"emptyVoid 太空旷感觉无聊"，可调整到 0.6 - 0.7；本方案放在拍板项是为方向 §"7 种 galaxyType 设计" 一致。

3. **warFront 中途出现 vs 终末关的 crossLevelBufferMod 差异**：
   - 本方案推荐：**中途 1.0 / 终末 1.5**；不引入单 galaxyType 双 mod 字段（代码层用 `if (level === ENDGAME_LEVEL) buffer *= 1.5` 兜底）。
   - 节点 B 测试若发现"终末关 buffer 4.5s 太长玩家不耐烦"，可降为 1.2 - 1.3；本方案 1.5 是方向意图。

4. **候选卡片是否显示"风险预告" 数字**：
   - 本方案推荐：**仅显示 risk label（低 / 中 / 高 / 极高）+ 偏好 icon**；不显示具体乘子数字（避免玩家"算 EV"破坏游戏直觉）。
   - 节点 C 测试若发现"玩家选错卡片很挫败 (期望中风险得到 ambush ×2)"，可在卡片底部增加 1-2 行"可能遭遇"提示文字；本方案 v0.9.0 保持极简。

5. **路径预览 HUD 已走节点用 galaxyType iconColor 还是统一绿色**：
   - 本方案推荐：**统一绿色 `#5ec460`**（与方向 §"数值边界建议"一致）；玩家鼠标 hover 可看到节点详情 tooltip。
   - 节点 C 测试若发现"玩家想看色系轨迹"，可改为 galaxyType iconColor 渲染已走节点（§3.2 备注路径）；本方案默认走方向严格颜色定义。

6. **galaxyMapPanel 显示前是否要"短暂高亮当前 galaxyType"提示玩家"我在哪"**：
   - 本方案推荐：**不显式高亮**；玩家从 HUD 左上当前 galaxyType 状态条已知信息。
   - 节点 B 测试若发现"玩家点 jumpBtn 后忘了刚才在哪"，可在 panel 顶部加"上一关：xxx"小条。

7. **候选生成的"风险等级降级"算法**：
   - 本方案推荐：**§2.5 fallback 规则**——targetRisk 无可选时降级到下一风险等级；但具体降级方向（高→中→低 vs 低→中→高）由 ADR 决定。
   - 节点 A 测试需验证：当前 galaxyType=warFront 时排除后 extreme 风险池空，能否正确降级到 high；本方案推荐"就近降级"（高→中 / 中→低 / 极高→高）。

8. **`__gameTest.simulateGalaxyChoices(level, count)` 测试钩子是否引入**：

   - 本方案推荐：**alert 队列保持 absolute time**（不按 galaxyType 缩放）；buffer ×0.7 仅影响 nextLevel 清理时长，不影响 alert 持续时间。

10. **路径策略评分（§5.2 / §5.3）是否需要 in-game 显示给玩家**：
