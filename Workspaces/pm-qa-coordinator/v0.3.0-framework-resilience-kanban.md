# v0.3.0 框架韧性 任务看板

## 本轮目标

让"框架脱落 → 玩家重新接入"成为玩家在一局内**可感知、可操作、可回收**的循环：

1. **可感知**：站体某段链路被打断时，脱落部分明显从站体上"飞出来"成为漂浮残骸；HUD 与音画反馈让玩家立刻意识到"丢了一块"。
2. **可操作**：玩家可以操控站体靠近残骸，把它视作世界中的目标移动 / 围绕；不强制要求精确对齐到 grid。
3. **可回收**：玩家在合适位置扩展 frame 时，能够把附近的残骸整段重新合并回站体，并恢复其上设施功能（包括之前的耐久 / 状态）。

体验风格基线：仍然是 v0.2.0 节奏紧凑的太空站建造，不引入复杂物理模拟；残骸物理简化到"惯性漂移 + 阻尼"层级即可。不强制玩家立刻回收残骸，允许玩家选择放弃；但应让"试图回收"的玩家有可读路径和明确反馈。

## 背景与制作人方向

### 必做范围

- 脱落残骸作为独立浮动实体：把整段脱落 cells 抽离成独立 fragment（建议 `state.fragments[]`），具备自己的世界坐标、速度、角度、角速度、cells 子集；station 主 Map 内不再保留这些 cell。残骸物理为惯性漂移 + 轻度阻尼，无主动 AI；可被敌人 / 玩家投射物物理影响（最小：被弹道误伤减 HP；不必强制）。
- 残骸视觉与 HUD 反馈：残骸用区别于站体的视觉表现，例如外环虚线、灰阶、半透明、闪烁警示。HUD 新增"脱落"提示：最近残骸方向 / 距离 / 设施数量；可复用现有 `statusAlerts` 或新增最小一段。残骸被敌人摧毁或漂离世界边界后应明确消失，并给一次性提示。
- 玩家靠近 + 重新接入：当玩家空间站某个候选 grid 位的世界坐标距任意 fragment cell 的世界坐标 < 阈值（建议约 `CELL * 1.2`）、且 fragment cell 朝向与 station grid 朝向夹角 < 阈值时，扩展 frame 可以"桥接"。桥接成功时整段 fragment 合并回 station Map，保留 facility、hp、frameHp、enabled、active 等运行时状态，并触发成功反馈；桥接失败时按现有 `setBuildError` 风格提示。
- 平衡与可读性兜底：当 fragment 数量过多或玩家短时间内丢失过多 cell 时，给一次性平衡提示并允许玩家通过维修无人机 / 重连恢复部分压力。维修无人机应对 detached cell 提供有限支持，具体策略由数值策划在执行阶段决定。

### 不做范围

- 不做新设施（如新武器 / 新维修类型 / 新建造资源）。
- 不做完整物理碰撞重做（残骸不与小行星 / 行星实体碰撞，仅与站体桥接判定相关）。
- 不做局外天赋树扩展（即使脱落与韧性相关）。
- 不做联机 / 异步对战。
- 不做新关卡任务类型（护送 / 撤离）。
- 不做存档结构变更：`SAVE_KEY="star-ring-station-meta-v1"` 不动；新增运行态字段一律放 `state.run` / 新建 `state.fragments`。
- 不做发布归档，除非制作人在版本验收后另行决定。

### 兼容要求

- v0.1.x：资源彩色外环、采矿范围、采集 HUD、首局减压、`gameOver` reload 重启、局外存档读取。
- v0.2.0：程序化星系生成、`level 0-2` 减压不回潮、`level 6` 守护者与本局结算、自由游玩 spawn 停刷、危险度 / 进度 HUD。
- v0.2.1：linearDamping / angularDamping / coast 阻尼组合、推进器松键不续推、`updateMouseAim` 转向响应。
- WASD 双模式入口、建造与移动解耦、建造失败保持模式。
- `state.meta` 结构与 `loadMeta` 兼容：旧形态可被补齐，新字段不写入 meta。

## 风险定级

- 风险等级：项目高风险任务。
- 定级原因：触及核心 cells 数据结构、连接性 BFS、渲染坐标系、战斗反馈链路；任何回归都可能破坏已可玩闭环。
- 高风险流程：制作人方向 → PM 拆分 → 主程 ADR → 执行 → 专项审核 → PM 回归 → 制作人最终决定。
- 审核基底约束：专项审核必须使用 `expert-agent-planner-gpt55`，与执行不同基底。
- 专项审核原则：审核者只记录结论、问题清单和状态建议，不直接修改 `Game/` 代码或其他文件。

## QA 回归点清单

### v0.1.x 关键回归点

- 启动：页面加载完成，canvas 和 HUD 存在，无运行时异常或 console error。
- 首局减压：新局前 60 秒压力不回潮，不出现连续刷怪压垮新玩家。
- 资源彩色外环：资源天体外环颜色可读，资源类型与 HUD 一致。
- 采矿范围：进入资源外环后可采集，离开范围后采集状态合理停止。
- 采集 HUD：显示正在采集的资源类型、目标和进度。
- 键盘移动：WASD 屏幕方向与朝向前方模式均可响应。
- 建造反馈：建造成功、资源不足、非法位置等反馈可见。
- 敌人攻击：敌人可造成结构伤害，不出现空引用或无效目标卡死。
- `game over` 重启：核心损毁后按既有 reload 重启逻辑可恢复新局。
- 局外存档：旧 meta 可读取并补齐，不清空局外点数和天赋。

### v0.2.0 关键回归点

- 星系生成：`nextLevel()` 后天体布局、资源分布和视觉差异可感知。
- 跃迁：关卡推进后任务、世界和 HUD 状态正确刷新。
- 危险度 HUD：星系危险度显示存在且随关卡变化。
- 本局进度 HUD：关卡目标和进度显示正确。
- 难度曲线：`level 0-2` 减压不回潮，`level 3-4` 压力上升，`level 5-6` 明显紧张但不失控。
- 终局守护者：终局星系可触发守护者并完成目标。
- 结算面板：完成后显示关卡数、奖励和操作按钮。
- 开始新局：结算后开始新局可重置本局状态，不损坏 meta。
- 留在终结星系：自由游玩可继续操作，普通刷怪停刷不破坏。
- 兼容旧存档：旧形态 `meta` 可被 `loadMeta()` 补齐，`SAVE_KEY` 不变。
- 随机 seed 稳定性：同 seed / level 生成结果稳定，不出现空星系、重叠星体或资源不可达。

### v0.2.1 关键回归点

- 三引擎刹停：三引擎配置松键刹停时间 `<= 6s`。
- 三引擎刹车距离：三引擎配置松键刹车距离 `<= 700`。
- 90 度转向：鼠标目标改为 90 度方向后 `<= 3s` 接近目标，无明显过冲。
- 纯 WASD 起步刹停：新手基础移动手感不退步，不出现"推不动"或"刹得过死"。
- 推进器松键：松键后推进器不续推，输入意图清晰。
- `updateMouseAim`：大角度 `1.85x` 非线性转向响应保留，小角度响应不抖动。

### v0.3.0 新增回归点

- Fragment 生成：站体链路被打断后，脱落 cells 从 station 主 Map 抽离为独立 fragment。
- Fragment 漂浮：fragment 具备世界坐标、速度、角度 / 角速度和阻尼，不再跟随站体位移。
- Fragment 限位：fragment 数量、单 fragment cells、总 detached cells 和世界边界清理符合数值方案。
- 桥接成功：玩家在阈值内扩展 frame 可将整段 fragment 合并回 station，保留设施、HP、frameHp、enabled、active 等状态。
- 桥接失败提示：距离 / 角度 / grid 冲突不满足时，出现可读失败提示且建造模式不异常退出。
- HUD 反馈：最近残骸方向、距离、设施数量或警示信息可见；残骸消失 / 被摧毁时有一次性提示。
- 维修无人机对 detached 策略：按数值方案处理刚脱落 fragment，不出现无限维修、跨地图维修或完全无反馈。
- 多 fragment 场景：多个 fragment 同时存在时，桥接目标选择、HUD 提示和清理逻辑可读且稳定。
- 性能不退步：多 fragment 场景无明显帧率恶化、无限循环、距离判定爆炸或浏览器卡死。

## 缺陷记录格式

- 标题：一句话描述问题。
- 严重程度：S0 阻断 / S1 严重 / S2 一般 / S3 轻微。
- 影响范围：说明影响启动、存档、fragment 脱落、fragment 漂浮、桥接、HUD、维修无人机、WASD 模式、采集、建造、敌人攻击、终局或自由游玩中的哪一部分。
- 复现步骤：按浏览器实机操作顺序列出，包含 seed / 关卡 / 站体形状 / 触发脱落方式 / 玩家桥接操作。
- 实际结果：记录玩家实际看到或系统实际发生的情况，必要时附运行态采样。
- 期望结果：记录按制作人方向、ADR 或数值方案应发生的体验。
- 当前处理：待分配 / 修复中 / 待审核 / 已回归 / 接受为已知问题。
- 负责人：明确后续处理成员。
- 回归要求：修复后尽量按原复现步骤复测，并说明失败路径是否关闭。
