# v0.15.0-browser-evidence-gate 人工证据方案与记录模板

## 目标与边界

版本名：`v0.15.0-browser-evidence-gate`。

本文用于 PM 在真实浏览器中收集 SSDSpaceGame v0.15.0 证据闸门材料，覆盖真实浏览器运行态 JSON、P0 人工路径、三视口遮挡/滚动/可点性、已知缺口和发布闸门建议。

严格边界：

- 本文只定义证据目录建议、记录字段、人工路径模板、三视口模板、检查点和 PM 发布闸门建议。
- 证据包只放在 PM 工作区，不进入 `Releases/`。
- 不用 headless、Node/VM、自动检查结果替代真实浏览器和人工体验结论。
- 没有真实浏览器运行态 JSON 时，结论只能是“继续补证”。
- 没有 P0 人工路径记录时，结论只能是“继续补证”。
- 真实触控若未覆盖，必须写入残留风险，不能写成移动端完整通过。

## 证据包目录建议

建议 PM 后续按以下结构归档证据。目录和文件均位于 `Workspaces/pm-qa-coordinator/` 下，不进入 `Releases/`。


命名建议：

- `known-gaps`：保存未覆盖路径、证据可信度风险、阻塞缺陷、非阻塞 polish 和复测状态。

## 发布闸门建议

PM 只给建议，制作人做最终决定。

### 建议进入发布准备

同时满足以下条件时，PM 可建议进入发布准备交接：

- 至少一份真实浏览器运行态 JSON 可复查，且包含 storage signature、frameTiming、runtimeErrors、blocking checks 和关键 DOM/hook 摘要。
- 至少一份 P0 人工路径记录覆盖 8 项主路径，且无 S0/S1 阻塞。
- `760x640`、`420x740`、`390x844` 三视口均有遮挡、滚动、可点性记录，阻塞问题已关闭。
- 所有 S0/S1 缺陷已修复，并按原浏览器、原视口、原步骤复测通过。
- 未覆盖项明确写入 known gaps，且不影响“当前纵向切片可进入发布准备”的判断。
- 证据明确区分自动检查、真实浏览器、headless、Node/VM、人工体验和未覆盖路径。

### 建议继续补证

出现以下任一情况时，PM 只能建议继续补证：

- 没有真实浏览器运行态 JSON。
- 没有 P0 人工路径记录。
- 三视口缺失任一目标视口，且没有等价 viewport 说明。
- runtimeErrors、storage signature 或 frameTiming 关键字段缺失且无法解释。
- 证据无法区分真实浏览器、headless、Node/VM、人工体验或未覆盖项。
- 记录只有结论，没有操作步骤、实际结果、截图/复制摘要或失败原因。

### 建议小修 polish

满足以下条件时，PM 可建议进入小修 polish，而不是直接发布准备：

- 真实浏览器 JSON 和 P0 人工路径已有，但存在非阻塞 S2/S3 或体验 polish。
- 小屏存在可理解但不舒服的问题，例如文案过长、次要说明需滚动、非核心按钮密度偏高。
- 复制摘要、截图命名或记录格式不统一，但关键证据完整。
- 真实触控未覆盖，但三视口和设备模拟证据可复查；结论中不得写移动端完整通过。
- polish 范围不触及核心规则、数值、存档、`SAVE_KEY`、meta schema、发布产物或依赖安装。
