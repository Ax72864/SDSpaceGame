## PM 汇总结论

PM 建议 #3 进入制作人决策：先批准低风险默认层 `A + E + F` 进入 #4 实施，即金属缺口诊断、候选路径或 ghost path、移动 + 重建教学。该默认层不改采矿范围 / 公式、资源布局、成本、存档、Meta schema，也不调整移动物理参数。

若默认层完成后，真实输入连续 3 次新档仍无法在 120 秒内取得 `harvestingCount > 0`，再触发 fallback 判断：优先进入 `D-d tutorial buff` 或 `B 首关资源布局保障` 的高风险流程。fallback 不能与默认层一起静默实施，必须先由制作人批准，再由 PM 拆分、专项执行、专项审核、PM 汇总和制作人最终决定。

## 三份方案共同点

- 三方都承认 v0.20.0 已建立诊断能力，但采矿收益仍未闭环：缺少稳定 `harvestingCount > 0`、目标资源下降或可归因收益。
- 三方都反对在无审批情况下直接修改采矿范围 / 公式、资源布局、数值、存档或发布流程。
- 三方都认为仅显示“还差 X”不足以支持玩家行动，必须把失败原因翻译成“还缺多少资源、下一步在哪里、是否应该移动 / 重建”。
- 三方都要求后续验收必须包含真实输入与小屏或等价触控证据，不能只用 headless、CDP 或 hook 替代玩家路径。
- 三方都要求收益归因：UI 可读反馈只能辅助，PM 仍要用 `harvestingCount > 0`、目标资源下降或等价字段证明采矿收益成立。

## 默认推荐实施范围

默认层 `A + E + F` 仅允许进入以下范围：

- A：金属缺口诊断与可读提示，说明当前路径总金属 / 矿石缺口、采矿站是否会产出、失败原因是什么。
- E：候选路径或 ghost path，优先只展示最佳候选、框架步数、目标格与总成本；小屏可降级为短句。
- F：移动 + 重建教学，提示玩家“驾驶工站靠近资源外环后重建采矿站”，但不修改移动物理、推进器成本、资源布局或建造合法性。

默认层禁止事项：

- 禁止修改 `MINING_RANGE_OFFSET`、`getMiningRange` 或资源外环公式。
- 禁止修改资源生成、首关 seed、station spawn、资源体数量或位置。
- 禁止修改 `frame` / `mining` / `thruster` 成本、起始资源或协议数值。
- 禁止修改存档 / Meta schema、`SAVE_KEY`、`META_SCHEMA_VERSION` 或发布包流程。
- 禁止把理论路径文案写成“必定可建”，除非实现验证当前资源可支付、路径合法且目标未占用。

## fallback 触发条件

默认层实施后，PM #5 / #6 至少按以下方式复测：

- 新档 + balanced 协议，真实鼠标输入，连续 3 次。
- 每次限定 120 秒内尝试取得 `harvestingCount > 0`。
- 记录是否出现目标资源下降、收益归因、失败原因、玩家实际选择了扩建还是移动路径。
- 记录小屏或等价触控路径是否被 HUD、候选格、建造面板或滚动误触阻断。

若连续 3 次新档在 120 秒内仍无法取得 `harvestingCount > 0`，PM 建议触发 fallback 决策：

1. 优先评估 `D-d tutorial buff`：首次玩家、首局、严格隔离的教程资源补偿。该项触及数值，必须走高风险流程。
2. 若制作人认为不应以数值补偿解决，或 D-d 仍不足，再评估 `B 首关资源布局保障`。该项触及资源布局、RNG 与视觉签名，必须走高风险流程。

PM 当前不建议把 D-a/D-b/D-c、C 或推进器 / 移动物理参数修改列为 1.0 fallback 默认池。

## 制作人需要决定的问题

- 是否批准默认层 `A + E + F` 进入 #4 实施。
- fallback 是否保留：默认层失败后，是否允许进入 `D-d tutorial buff` 或 `B 首关资源布局保障` 的高风险流程。
- 若保留 fallback，优先级是先 D-d 后 B，还是直接 B，或者全部推迟到 1.0.x。
- 哪些内容列入 1.0 必过 gate：至少建议包含真实输入 120 秒内 `harvestingCount > 0`、目标资源下降或收益归因、玩家能看懂金属缺口、候选路径 / 移动路径、小屏可读可点。
- 是否明确否决 1.0 阶段修改采矿范围 C、frame 成本、整体起始资源上调、推进器成本 / 物理参数。

## PM 建议的 #4 验收口径

- 自动 / 浏览器证据：`getMiningCoverageDiagnostics` 或等价 JSON 能输出采矿状态、最近资源、候选路径、成本缺口、失败原因；无阻断 runtime error / unhandled rejection。
- 真实输入证据：空档 + balanced 至少 3 次新档，120 秒内取得 `harvestingCount > 0` 或明确触发 fallback。
- 收益归因证据：目标资源下降或收益来源字段成立，不能把自然资源增长误判为采矿收益。
- UI 证据：玩家能看到“当前是否产出、为什么没有、下一步扩建还是移动”；小屏不会遮挡关键按钮或候选格。
- 回滚边界：默认层可单独回滚，不影响存档、Meta、资源生成、数值或发布流程。
