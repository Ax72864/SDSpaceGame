## 0. 使用约定

- 文案默认中文，长度尽量控制在 ≤ 14 个汉字，便于在卡片角标、健康摘要短项和低噪声提示中显示。
- 凡是“暂定值”均在表中明确标记 `〔暂定〕`，等 T1-T4 实装后由 PM 回归回调。
- 阈值只用于“展示分级 / 是否触发提示”，与底层结算无关。若实装时发现阈值与实际体验偏差超过一档，应由数值策划单独回调，不允许在实现任务里改动。
- 文案表里出现的“电力 / 推力 / 维修速率 / 护盾 / 射程”等读数，全部按 `TYPES[*].baseStats` 现有字段输出，不引入新概念。
- 所有诊断字段需要可在 headless DOM 或只读测试钩子中读取，字段名建议采用 `kebab-case` 或既有 `data-*` 风格，由主程在 #3 技术边界中冻结，本文档只给“显示名”和“候选 key”。

- **收益 / 作用**：玩家应该“看到 + 听懂”的核心效果，只从 `baseStats` 读取，不引入新指标。
- **常见不工作原因**：每种设施最常见的 1-3 条失效原因，用于选中诊断和低噪声提示。

### 1.4 文案边界规则

- “电力 +10 / 耗电 3 / 副产电 +2”全部直接拼接 `baseStats` 字段，不允许换算成“百分比覆盖率”等新概念。
- 处理器和等离子的“转化比率”按 `baseStats.produceRatio` 显示为“1 矿 → 0.72 金属”这类玩家可读形式，**不**新增隐藏字段。
- 武器的射程、伤害、冷却显示需带单位（`m / 伤害 / s`），但**不**展示“DPS”，避免误导玩家成本对比。
- 维修站显示“维修速率 / 骨架维修速率 / 冷却”，**不**写“治疗量”这种概念漂移词。

## 2. 空间站健康摘要 6 类阈值（只用于诊断分级）

健康摘要由 4-6 个短项组成，本表给出 6 类候选项；UI 视空间可只展示 4-5 项，PM 在 #7 实装时根据高度限制裁剪，但顺序固定按下表。

> 通用约定：
> - 健康分级共 3 档：`良好` / `注意` / `短板`（不要再细分，避免噪声）。
> - 所有阈值仅用于决定文案颜色和短项标题，不进入实际玩法结算。
> - 阈值取“当前帧只读快照”，不做平滑；如果某指标在 60 帧内反复跳档，由实装在 #7 增加 0.5s 防抖，但本文档不规定平滑公式。

### 2.1 电力余量 `power-margin`

| 分级 | 触发条件（基于现有 `powerOut` 总和 `P_out` 与 `powerUse` 总和 `P_use`） | 短项文案 | 颜色暗示 |
|---|---|---|---|
| 良好 | `P_out - P_use ≥ 5` 且无断电设施 | 电力余量充足 | 中性 / 蓝 |
| 注意 | `0 ≤ P_out - P_use < 5`，或断电设施数 ∈ [1, 2] | 电力紧张 | 黄 |
| 短板 | `P_out < P_use`，或断电设施数 ≥ 3，或核心耗电链（采矿/武器/护盾/维修任意一类）整组断电 | 多处断电 | 红 |

说明：`P_out - P_use ≥ 5` 的 5 是〔暂定〕值，等同“一台 turret(3) + 缓冲 2”。实装后 PM 若发现“注意”长期挂着，可下调到 3。

### 2.2 采矿能力 `mining-coverage`

| 分级 | 触发条件 | 短项文案 |
|---|---|---|
| 良好 | 至少 1 台 `mining` 通电、连通且范围内（`MINING_RANGE_OFFSET = 130`）有可采资源 | 采矿覆盖良好 |
| 注意 | 有 `mining` 但≥ 50% 处于“范围内无资源 / 断电 / 未连通”状态 | 采矿覆盖不足 |
| 短板 | 无 `mining`，或全部 `mining` 不可工作 | 无有效采矿 |

说明：50% 阈值用于避免“1 台不工作”就刷红；〔暂定〕。

### 2.3 推进能力 `thrust-capacity`

| 分级 | 触发条件 | 短项文案 |
|---|---|---|
| 良好 | 通电 `thruster` 数量 ≥ 2，且每台未脱离 | 推进充足 |
| 注意 | 通电 `thruster` = 1，或 ≥ 2 但其中 ≥ 1 台断电 | 推进偏弱 |
| 短板 | 0 台通电推进器（含全部断电 / 全部脱离） | 缺少推进 |

说明：不引入“质心偏移度”阈值，避免本版本误装变成数值修改。质心相关只在选中诊断给“相对核心 X/Y 偏移：±N 格”这类只读读数。

### 2.4 火力 `firepower`

| 分级 | 触发条件 | 短项文案 |
|---|---|---|
| 良好 | 通电 `turret` + `missile` ≥ 2，且至少 1 台为 `turret`（保证自动开火） | 火力充足 |
| 注意 | 仅 1 台武器通电，或仅有 `missile` 而无 `turret`（高度依赖玩家手动） | 火力单一 |
| 短板 | 无任何通电武器，或全部武器断电 | 缺少火力 |

### 2.5 防御 `defense`

| 分级 | 触发条件 | 短项文案 |
|---|---|---|
| 良好 | 至少 1 台 `shield` 通电，且 `armor` 数量 ≥ 2 | 防御充足 |
| 注意 | 仅有 `shield` 但无 `armor`，或仅有 `armor` 但无 `shield`，或 `shield` 当前已被打破回复中 | 防御偏薄 |
| 短板 | 既无通电 `shield` 也无 `armor`（即“硬骨头都没有”） | 缺少防御 |

### 2.6 维修 `repair`

| 分级 | 触发条件 | 短项文案 |
|---|---|---|
| 良好 | 至少 1 台 `repair` 通电，且当前无受损设施 / 或受损设施都在维修范围内 | 维修覆盖良好 |
| 注意 | 有 `repair` 但 ≥ 1 台受损设施不在覆盖内，或 `repair` 断电 | 维修缺口 |
| 短板 | 0 台 `repair`，且当前已有受损设施 | 无维修能力 |

## 3. 低噪声提示优先级与触发阈值

> 提示池在“关键风险出现”时才触发；普通低效（例如 1 台机器断电）不进入提示池，只反映在健康摘要里。
> 同时触发多条时，按下表 `优先级 P` 从小到大显示**最多 2 条**，其余进入 tooltip 或 debug snapshot。

| 优先级 P | 提示 key | 触发阈值（与 §2 一致） | 短文案 | 解除条件 |
|---|---|---|---|---|
| 1 | `enemy-incoming-no-defense` | 检测到敌人 / encounter / 30s 内将进入战斗节点，且 §2.4 火力 = 短板或 §2.5 防御 = 短板 | 敌袭临近：无火力或无防御 | 火力与防御任一回到“注意”及以上，或战斗结束 |
| 2 | `jump-no-thrust` | 玩家打开跃迁 / 星系图准备跳跃，且 §2.3 推进 = 短板 | 跃迁前缺少推进 | 至少 1 台推进通电，或关闭跃迁窗口 |
| 3 | `mass-blackout` | 断电设施数 ≥ 3，或核心能力（采矿/武器/护盾/维修）任意一类“整组断电” | 多处断电影响核心能力 | 断电设施数 ≤ 1 且无整组断电 |
| 4 | `mining-no-target` | 已建 `mining` ≥ 1，且**全部** `mining` 处于“范围内无资源” | 采矿站未覆盖资源 | 至少 1 台 `mining` 范围内出现资源 |
| 5 | `weapons-offline` | 已建武器（turret/missile）≥ 1，且全部断电 | 武器全部断电 | 至少 1 台武器恢复供电 |
| 6 | `shield-offline` | 已建 `shield` ≥ 1，且全部 `shield` 断电 | 护盾断电 | 至少 1 台护盾恢复供电 |
| 7 | `repair-gap` | §2.6 维修 = 注意 / 短板，且当前累计受损设施 ≥ 3 | 维修跟不上损伤 | 受损设施 ≤ 1，或维修恢复覆盖 |
| 8 | `resource-shortfall` | 玩家在建造面板尝试建造时主资源 < 实际成本（含折扣）且差距 ≥ 30%（〔暂定〕） | 资源不足，差 N 单位 | 资源回补到可建造水平 |

通用降噪规则：
- 每条提示触发后**至少冷却 8s** 才能再次出现（〔暂定〕；战斗节点可缩短到 4s）；冷却由 #8 实装控制，不进入存档。
- 提示文案禁止出现“自动优化”“一键修复”“为你建造”这类承诺，所有提示必须是“说出问题”，不替玩家下决策。
- 不允许出现“提示 list 持续刷屏”：若 8s 内累计已显示 ≥ 3 条不同提示，应抑制后续低优先级提示，只保留 P ≤ 3 的关键提示。

## 4. Meta / 协议加成显示文案边界

目标：让玩家“看到 Meta / 协议改了什么”，但**不**重新解释数值系统，**不**承诺新机制。

### 4.1 通用规则

- 出现位置：建造卡片成本行、设施详情副标题、健康摘要顶部一行（可选）。
- 文案形式必须是“现状陈述”：`已受局外加成影响` / `已受开局协议影响`，**不**写“你将获得 +X%”这种动态承诺式语句。
- 数字展示采用“最终值（原值）”格式，例如：`金属 18（22）`。原值括号中显示，颜色降阶，避免与新成本竞争注意力。
- 若同时受 Meta 与协议两类加成影响，副标题写成：`已受局外与协议影响`，并以 tooltip 展开列出生效项；最多展示 3 条，超过则显示 `+N 项`。
- **禁止**显示百分比修饰公式（例如 `成本 ×0.85`），玩家只看到结果，不看到数值系统。

### 4.2 文案样例（〔暂定〕）

| 场景 | 副标题 | 展开 tooltip 示例 |
|---|---|---|
| Meta 影响初始资源 | 已受局外加成影响 | 开局金属 +20 / 气体 +5 |
| 开局协议影响成本 | 已受开局协议影响 | 采矿设施成本 ×0.85（按 `START_PROTOCOL_FACILITY_DISCOUNT_MAP` 真实键） |
| Meta + 协议共同 | 已受局外与协议影响 | （列出最多 3 条，第 4 条起合并为 `+N 项`） |
| 无任何加成 | （不显示副标题） | — |
| 协议加成被解除 / 切换存档 | 副标题消失 | 不残留旧文案 |

### 4.3 文案禁词

- 禁止：`系统已升级` / `新机制` / `自动平衡` / `为你优化`。
- 禁止把折扣后的成本写成“真实成本”而把原成本写成“伪成本”——必须保证玩家能在 1 秒内看出哪个数会被扣。
- 禁止在 Meta / 协议未生效时显示“括号原值”，避免玩家误以为有加成。

## 5. 低噪声 vs 健康摘要 vs 选中诊断的职责分工

| 渠道 | 出现时机 | 内容粒度 | 解除时机 | 不允许做什么 |
|---|---|---|---|---|
| 建造卡片 | 玩家在建造面板浏览或悬停时 | 设施级（角色 / 成本 / 电力 / 是否可建） | 关闭面板 | 不显示战斗 / 跃迁倒计时；不提示 RNG 走向 |
| 选中诊断 | 玩家点选已有设施或预览建造位置 | 单格 / 单设施级（连接 / 范围 / 缺电 / 缺料 / 方向） | 取消选中 | 不显示全站汇总；不写“你应该建什么” |
| 健康摘要 | 持续显示在右侧状态 / 更多状态 | 全站 4-6 项（§2） | 状态变化时刷新分级 | 不替代低噪声提示；不出现具体设施名 |
| 低噪声提示 | 触发阈值满足且关键路径阻断时 | 全站级 1-2 条（§3） | 阈值解除 + 冷却结束 | 不堆叠 ≥ 3 条；不承诺自动修复；不显示在战斗 HUD 正中央 |

## 6. 暂定值与回调清单

| 字段 | 当前值 | 何时回调 | 回调判据 |
|---|---|---|---|
| 电力余量“良好”阈值 | `P_out - P_use ≥ 5` | T1-T3 实装并跑空存档 1 次 | 若“注意”长时间挂起，可下调到 3 |
| 采矿覆盖“注意”比例 | `≥ 50% mining 不工作` | T2 + T3 完成后 | 若 1 台不工作就跳红，则调到 60% |
| 推进“良好”台数 | 通电 ≥ 2 | T3 完成后 | 玩家若觉得 2 台仍不够大体量空间站，则升至 3 |
| 火力“良好”定义 | ≥ 2 武器且含 turret | T3 完成后 | 若 missile 单流派玩家觉得被错误标黄，可加“仅 missile 但 ≥ 3 台 = 良好”分支 |
| 提示冷却 | 8s（战斗内 4s） | T4 完成后 | 若同类提示反复刷屏，则延长至 12s |
| 资源不足触发阈值 | 差距 ≥ 30% | T4 完成后 | 若提示过频，提高到 40% |
