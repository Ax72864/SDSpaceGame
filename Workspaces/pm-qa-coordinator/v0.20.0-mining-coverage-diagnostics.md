# v0.20.0 采矿覆盖只读诊断 helper

## 名称与入口

- Helper：`window.__gameTest.playtest.getMiningCoverageDiagnostics(options)`
- 内部实现：`computeMiningCoverageDiagnostics(options)`（只读，不导出到全局）

## 用法示例

```javascript
// 默认：基于工站/最近资源，返回最多 12 个入圈候选格
window.__gameTest.playtest.getMiningCoverageDiagnostics()

// 聚焦某座矿站
window.__gameTest.playtest.getMiningCoverageDiagnostics({ minerGridKey: "-1,1" })

// 指定目标资源名，扩大候选搜索
window.__gameTest.playtest.getMiningCoverageDiagnostics({
  targetBodyName: "小行星-2",
  maxCandidates: 16,
  searchPadding: 12
})
```

## 返回字段摘要

| 字段 | 说明 |
| --- | --- |
| `readOnly` | 恒为 `true` |
| `miningStatus` | `minerCount/activeCount/harvestingCount` 等 |
| `reachability` | 与 `getResourceReachability().overall` 同口径 |
| `miners` | 每座矿站的 `target/nearest/gap/reasonKey` |
| `nearestResource` | 参考位置到最近资源外环的 `distance/range/gap` |
| `candidates[]` | 入圈候选格：`gridKey/cellState/expandable/miningRange/expansionPath` |
| `bestCandidate` | 优先可扩建且入圈的候选 |
| `suggestedExpansionPath` | `frameSteps` 框架链 + `finalAction` |
| `failure` | 不改状态的阻塞原因与 `nextAction` |
| `rules` | `MINING_RANGE_OFFSET=130`，公式 `body.r + 130` |

## 边界与禁止行为

- 只读：不写入 `state`、不读写 `localStorage/sessionStorage`、不调用 `buildAt`、不改变资源/收益。
- 候选格搜索范围受 `searchPadding` 与已连接结构边界限制；固定格不可跨 seed 复用。
- `suggestedExpansionPath` 仅模拟框架邻接扩建，不考虑资源成本、电力或战斗阻断。
- 不替代可见浏览器人工操作证据；供 PM/玩家理解“采不到 / 往哪扩建 / 是否需重新选址”。

## 最小 UI 增强

`resourceGuide` 在矿站就绪但未入圈时追加一行：

> 当前矿站不会产出；需沿框架扩建到彩色外环内，或驾驶工站靠近资源后重建采矿站。

不改采矿范围、公式，不赠送资源。
