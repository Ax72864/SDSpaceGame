# v0.2.0 星系节奏任务看板

## 本轮目标

建立 `v0.2.0-galactic-rhythm`（星系节奏）版本任务包记录，确保“跃迁下一星系”真正产生新世界，并让一局游戏形成起承转合的关卡节奏。

## 背景与制作人方向

- 版本号：`v0.2.0-galactic-rhythm`。
- 体验目标：让“跃迁下一星系”真的产生新世界，让一局游戏有起承转合的节奏。
- 必做范围一：跃迁产生新星系。每次 `nextLevel()` 重新生成 `state.world.bodies`，包含天体数量、位置、半径、资源分布；不同关卡允许出现富矿星系、气体星云、等离子风暴等资源倾向；恒星颜色和背景星点密度需要形成肉眼可见差异；现有固定布局 `initWorld` 需要改造为按 `level` 与 random seed 生成。
- 必做范围二：关卡难度爬升。`spawnWave` / `spawnEnemy` 需要把 `level` 纳入 spawn 频率、敌人比例、HP、伤害曲线；早期关卡 `level 0-2` 保持当前减压手感，中期 `level 3-4` 压力上升，后期 `level 5-6` 明显紧张；HUD 需要给出可感知的星系危险度提示。
- 必做范围三：本局终局节奏。在推荐触发关卡数 N 后进入“本局终结”星系，任务可采用“击毁本局守护者”或“撤离信标”；完成后显示本局结算面板，包含完成关卡数、累计奖励、`+50%` 局外点数加成，并提供“开始新局”和“留在终结星系自由游玩”两个按钮。
- 兼容要求：不得破坏 v0.1.x 已有资源彩色外环、采矿范围、采集 HUD、首局减压体验和 `gameOver` 的 `location.reload` 重启逻辑；现有 `state.meta` 与 `SAVE_KEY="star-ring-station-meta-v1"` 必须向下兼容，绝对不可清空玩家点数和天赋。

## 风险定级

- 风险等级：高风险任务包。
- 定级原因：本任务触及核心关卡推进、世界生成、敌人压力曲线、本局结算和持久化兼容边界，任何回归都可能破坏已有可玩闭环。
- 高风险流程：制作人方向 -> PM 拆分 -> /senior-systems-engineer 实现 -> expert-agent-planner-gpt55 专项审核 -> PM 回归 -> 制作人最终决定。
- 审核基底约束：执行任务由 /senior-systems-engineer 承担；专项审核必须使用 expert-agent-planner-gpt55，避免审核与执行同基底。
- 严格范围边界：本任务包仅允许后续实现阶段修改 `Game/index.html`、`Game/style.css`、`Game/game.js`；不得修改 `Docs/`、`.cursor/`、`42.md`、`持续开发Prompt.md`、`Releases/`、`Release/` 或其他 `Workspaces/<其他成员>/`。

### v0.2.0 新增回归点

- 星系生成：`level 0-6` 至少覆盖多个关卡，确认每次跃迁生成不同 `state.world.bodies`。
- 资源分布差异：富矿、气体、等离子或类似星系类型能体现不同资源倾向。
- 视觉差异：恒星颜色、背景星点密度或星系调色板能让玩家肉眼识别关卡变化。
- 跃迁流程：普通关卡完成后进入下一星系，任务状态和 HUD 不残留旧星系错误信息。
- 危险度 HUD：危险等级图标或文字随关卡提升变化，位置不遮挡核心操作。
- 本局进度 HUD：能看出当前关卡进度或距终局的距离。
- 难度曲线：`level 0-2` 保持减压，`level 3-4` 明显上升，`level 5-6` 明显紧张。
- 终局触发：达到推荐 N 后进入终结星系，并出现不同于普通跃迁的目标。
- 终局任务：守护者或撤离信标目标可完成，不出现无敌、不可达、无法交互或无法判定完成。
- 结算面板：完成关卡数、累计奖励、`+50%` 局外点数加成显示正确。
- 开始新局：点击后 reload 重启，并保留合法局外奖励。
- 留在终结星系：点击后不 reload，玩家可以继续自由游玩。
- 兼容旧存档：使用旧 `state.meta` 时不报错，新增字段如有默认值不破坏旧数据。
- 随机 seed 稳定性：同一局内生成结果不会每帧抖动，不同关卡不会因 seed 缺失生成完全相同世界。
