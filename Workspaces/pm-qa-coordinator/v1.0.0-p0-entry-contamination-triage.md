## 根因判断

不是普通玩家入口逻辑问题，根因在 `playtest/release-candidate` 验收入口自身：

1. `initReleaseCandidatePlaytestEntry()` 在 `playtest=1` 下会自动打开 panel。
2. `openReleaseCandidatePanel()` 在无历史结果时会自动触发 `refreshReleaseCandidateAutoChecks()`。
3. `refreshReleaseCandidateAutoChecks()` 会调用 `runReleaseCandidateChecks()`，其中 `sampleReleaseFrameTiming()` 采样期间 run 仍可自然推进。
4. 在 PM 尚未接管输入时，run 可能自然推进到失败结算（核心摧毁）或完成目标后写入 `SAVE_KEY`（`grantObjectiveReward -> grantMetaPoints -> saveMeta`），从而污染 #6 空档首局取证。

## 修复策略（最小改动）

仅改 `Game/game.js` 的 playtest/release-candidate 入口行为，不改普通玩法规则：

- 新增 `activateReleaseCandidatePlaytestHold()`：`playtest=1` 入口默认暂停 run（`state.paused=true`，按钮文案切到“继续”）。
- 调整 `openReleaseCandidatePanel()`：去掉“首次打开自动跑检查”，改为仅提示“手动点刷新自动检查”。
