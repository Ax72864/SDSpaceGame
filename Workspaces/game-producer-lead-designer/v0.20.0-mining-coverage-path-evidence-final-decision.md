## 最终决定

接受 `v0.20.0-mining-coverage-path-evidence` 为内部阶段完成：采矿覆盖只读诊断能力已经建立，当前规则下候选路径、建议扩建路径和失败原因比 v0.19 更清楚。

本决定不代表完整 P0 通过。采矿收益路径仍未闭环，不能进入发布准备，不打 tag，不归档 `Releases/`，不解锁自然交火、战后复盘或 Meta 回流补证。

## 玩家与团队收益

- 玩家侧不再只看到“还差 X”，现有反馈已经能解释当前矿站不会产出，需要扩建到彩色外环内，或驾驶工站靠近资源后重建采矿站。
- 团队侧能拿到候选格、`suggestedExpansionPath`、nearest resource、distance、range、gap、`allOutOfRange` 和失败原因，用于判断“采不到 / 往哪扩建 / 为什么仍失败”。
- 证据侧明确区分了可见浏览器 / CDP、只读 hook、输入失败、状态污染、成本阻断和收益归因不足，避免把自然资源增长误判为采矿收益。

## 仍不舒服 / 未闭环

- 没有取得 `harvestingCount > 0`、稳定 `target` 或目标资源 `body.amount` 下降证据，采矿收益未形成。
- helper 建议路径理论可入圈，但真实铺设在候选前被金属成本阻断，`suggestedExpansionPath` 不能直接等同真实可支付路径。
- 原 `mining=-1,1` 输入稳定性仍有问题，HUD 覆盖、CDP 点击递送和相机 / 工站移动会让证据不稳定。
- 真实人工鼠标与真实触控未覆盖，现有证据不能包装成人类体验通过。
- 自然交火、战后复盘和 Meta 回流仍未解锁，基础建设收益链还不能支撑下一层体验。

## 下一阶段建议

进入 `v0.21.0-mining-coverage-product-decision` 或等价阶段，按高风险流程评估采矿覆盖产品方案。

下一阶段必须先回答：继续在现有规则下追求更稳的真实输入 / 成本可支付路径，还是调整资源、采矿、移动、布局，或提供更强的“当前无法采到，需要重新选址 / 推进资源外环”体验引导。

如果方案触及采矿范围 / 公式、资源生成公式、资源布局、移动能力、建造合法性、数值、存档、Meta 或发布资产，必须升级项目高风险流程：制作人确认方向，PM 重新拆分，专项成员执行与审核，PM 汇总后再由制作人决定。

## 制作人接受结论

v0.20.0 的价值成立：它把采矿问题从“只知道还差距离”推进为“团队知道候选路径、失败原因和证据缺口”。但它还没有把玩家收益路径闭环，不能当作采矿 P0 成功，也不能进入发布链路。

下一轮继续追采矿覆盖产品方案；在收益闭环或阻塞体验被真正接受前，不允许用战斗、复盘或 Meta 内容绕过基础建设链。
