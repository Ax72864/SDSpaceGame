# v0.6.0 敌方空间站 制作人方向

版本名：`v0.6.0-hostile-station`
基线：HEAD `b810455`（v0.5.0 研发与升级发布后）
风险级别：**项目高风险**（涉及核心玩法 + 关键数据结构扩展 + 新 enemy 实体类型）

## 本轮目标

让玩家在每局 level 3-5 中段遭遇 **BOSS 级敌方空间站** 决战：cells 化的多目标击杀让玩家有战术决策深度（先打武器 / 先打动力 / 直击核心）；击毁后生成大量 wreck fragment 与 v0.3.0 拾荒系统强协同，让中期决战节点形成"升级 build → 决战 → 拾荒奖励 → 进入下个星系"的完整爽快闭环，填补 v0.4.0 guardian（终局）之前的 BOSS 体验空白。

## 体验目标

按重要程度排序：

1. **可感知 BOSS 决战**：玩家在 level 3-5 遇到 hostile-station 时，能立刻感受到"这不是普通杂兵"——视觉规模大、攻击多点、生命池厚、被打击有 cells 逐块脱落的反馈；击毁后有相机震屏 / 大量 wreck 爆出 / 任务完成感。
2. **可战术的攻击决策**：玩家可以选择"先打武器 cell 让它停火"或"先打 power cell 削弱"或"直击核心快速终结"；同 build 在不同战术下表现不同，让 v0.5.0 研发系统的 build 决策有了"针对 BOSS"的具体使用场景。
3. **可衔接的奖励循环**：击毁敌方空间站后生成 6-12 个 wreck fragment，与 v0.3.0 拾荒系统天然衔接（玩家拖去停靠 → 合并到 station → 获得新设施），让"决战 → 战利品 → 进入下个星系"形成单局节奏高潮。

## 必做范围

### 工程基础（先做这些再做战斗与任务）

- **`enemy.kind = "hostile-station"` 新类型**：扩展 `state.enemies` 元素 schema，hostile-station 类型在标准 enemy 字段（`kind / x / y / vx / vy / angle / hp / r`）基础上**新增 `cells: Map<key, cell>` 子结构**，cells 复用 v0.3.0 fragment 的 cells 数据结构与 `createCell` 工厂。
- **HP 池 = cells 总和**：hostile-station 没有"整体 hp"概念（或仅保留 `hp = sum(cells.hp)` 作为聚合用于显示），击杀判定改为"所有非装甲 cells 被破坏"或"core cell 被破坏"（具体由主程 ADR 决定）。
- **`damageHostileStationCell` 函数**：仿 v0.3.0 `damageFragmentCell` 实现 cell 粒度命中扣血；玩家弹道命中 hostile-station 时优先做 cell 命中（仿 `getHostileStationCellHit(projectile)`）。
- **复用 `makeWreckFragment` 死亡爆 wreck**：hostile-station 被击毁后生成 6-12 个 wreck fragment，`origin = "enemy-wreck"` 字段标记来源（与玩家 `origin="player"` / salvage `origin="wreck"` 区分），便于后续视觉 / 平衡差异化。

### AI 与战斗（沿用 guardian / station 现有模板）

- **静态悬停 AI**：hostile-station 在 `enemyUsesStationAi` 中标记，沿用 470 距离悬停模板；`accel` / `drag` 调到极小让它**近完全静止**（不引入 v0.6.0 移动型敌舰的复杂度）。
- **多武器槽自然产生**：cells 中装 `turret` / `missile` 等武器 cell（数据复用 player TYPES），每个武器 cell 独立 reload + 独立 fire；击毁该武器 cell 即停火该武器（直接的战术反馈）。
- **召唤手下**：保留现有 `station` kind 的"召唤 pirate minion"机制；hostile-station 每 6-10 秒召唤 1-2 个 pirate minion，并由 `spawnCap = 3-5` 限制总数（与 guardian 一致风格）。
- **死亡效果**：相机震屏（仿 `triggerGuardianDeathEffect` 加强版）+ 6-12 个 wreck fragment 爆出 + 粒子。

### 任务与生成节奏

- **新任务类型 `assault`**：在 `OBJECTIVE_TYPES` 注册，"摧毁敌方空间站"任务，在 level 3-5 出现概率 25-40%（具体由数值方案决定）；任务奖励含 metal + research（researchReward 较高，约 9-10，介于 battle 5 与 guardian 12 之间）。
- **任务激活生成**：hostile-station 仅由 `OBJECTIVE_TYPES.assault.make` 主动 spawn，不污染 `LEVEL_SPAWN_RATIO` 随机波；spawn 位置仿 `spawnGuardianNearStation`（远距离安全生成，让玩家有时间准备）。
- **任务事件**：`notifyObjective("hostileStationKilled", payload)` 新事件加入 `OBJECTIVE_EVENT_HANDLERS`；`assault` 任务 `onEnemyKilled` 监听 `enemy.kind === "hostile-station"` 完成。
- **奖励倍率**：`rewardMultiplier` 约 1.4-1.6（介于 battle 1.0 与 guardian 2.0 之间）；保留 v0.4.0 totalObjectiveRewardBase 不动语义。

### HUD 与反馈

- **hostile-station 出现警报**：仿 v0.4.0 guardian 警报，在 hostile-station spawn 时 HUD 红色警报 "敌方空间站接近"。
- **距离 / HP 指示**：HUD 显示 hostile-station 当前 HP%（基于 cells 总 HP 比例）+ 距离。
- **击毁庆祝**：击毁后 1-2 秒"主目标已摧毁"提示 + wreck 爆出动画。

## 不做范围

- **不做移动型敌方空间站**：v0.6.0 敌方空间站完全悬停静态，移动型留 v0.7+（涉及独立 vel / target / AI 状态机重构）。
- **不做敌方空间站 cells 升级 / 改造系统**：玩家研发系统不延伸到敌方 cells；hostile-station cells 是固定配置。
- **不做敌方 NPC 阵营**（敌方手下作为 enemy 而非 npc 处理；v0.4.0 escort npc 系统不动）。
- **不做新武器机制**（近防 / 激光 / AOE 类）：hostile-station 武器全部复用现有 fireProjectile + missile 弹道路径。
- **不做 enemy / npc 抽象重构**：保留两套独立 schema；v0.6.0 仅扩展 enemy.kind。
- **不修改 guardian**：终局 BOSS 地位保持，与 hostile-station 体感差异化（guardian 仍是单 hp 池，hostile-station 是 cells 化）。
- **不修改 v0.5.0 研发系统**：玩家研发 / 升级 / 改造对 hostile-station 自动生效（damage / range / reload 已通过 getCellStat 应用）。
- **不破坏 v0.4.0 任务系统**：仅新增 `assault` 任务类型，其他 7 种任务保留。
- **不修改 SAVE_KEY** / 局外存档结构。
- **不做 v0.5.1 候选项**（shield regen base / thruster mass / 数值微调 / hover 提示完善）。
- **不做联机 / 异步对战**（game desc §9 / §10）。

## 兼容要求

- v0.1.x / v0.2.0 / v0.2.1 / v0.3.0 / v0.4.0 / v0.5.0 全部回归点必须通过。
- 现有 4 种 enemy kind（asteroid / pirate / station / guardian）行为完全等价；新增 hostile-station 不影响既有 enemy 路径。
- v0.3.0 fragment 系统（player + wreck origin）行为不破坏；新增 `enemy-wreck` origin 与 `player` / `wreck` 区分但共用 cells 数据结构。
- v0.4.0 任务系统（7 种 OBJECTIVE_TYPES）保留；新增 `assault` 为第 8 种。
- v0.5.0 研发系统（getCellStat / 升级 / 改造 / hullMod / weaponMod）对玩家攻击 hostile-station 自动生效。
- SAVE_KEY 不变；旧存档兼容（无 assault / hostileStation 字段时默认）。
- 终局 guardian 在 level 6 仍然出现，与 hostile-station 在 level 3-5 不冲突。

## 关键技术疑问（给主程）

请主程在 ADR 中明确回答以下 8 项决定：

1. **hostile-station cells 数据结构**：完全复用 `fragment.cells`（含 createCell / damageFragmentCell / computeFragmentEdgeKeys）还是新写一份？**建议完全复用**，hostile-station 在数据结构上等同 fragment（cells Map + 中心点 + 旋转），仅 enemy.kind 字段区分行为；这样未来移动型敌方空间站也能用 fragment 的 angle / angularVel。
2. **击杀判定**：cells 全部破坏 / core cell 破坏 / HP 池低于阈值，三选一？**建议 core cell 破坏即死**，与玩家 station 死亡判定一致（玩家 station 失去 core 后崩溃）；这让"先打 core 速胜"成为可行战术。
3. **武器 cell 行为**：hostile-station 的武器 cell 是直接复用 player turret update 逻辑（getCellStat + fireProjectile）还是单独写一套？**建议复用**，让 v0.5.0 研发系统的 damage / reload / range stat 在 hostile-station 武器上自然有效（但敌方 cells 不应被玩家 research 升级，需要 isEnemyOwned 隔离）。
4. **HP 与平衡**：hostile-station 各 cell HP / 武器 damage / 总 HP 池区间？建议总 HP 池 1500-3500（按 level 缩放），单 cell HP 150-300，武器 damage 与 pirate 同档（10-15）；具体由数值方案决定。
5. **AI 静态程度**：hostile-station accel / drag 调到多少算"近完全静止"？建议 `accel ≤ 5`、`drag ≥ 0.5`（让位置漂移几像素但视觉静止）；保留 angle 缓慢自转（视觉活力，仿 fragment.angularVel）。
6. **召唤手下**：固定 pirate minion 召唤间隔 / cap 区间？建议 `spawnInterval` 7-10 秒、`spawnCap` 3-5、minion 跟随 hostile-station 周围而非直接索敌玩家（避免 minion 被 BOSS 召唤后立刻冲向玩家显得突兀）。
7. **wreck 爆出**：死亡时 wreck 数量 / 设施分布？建议 6-12 个 wreck fragment，设施分布偏战斗（`ENEMY_WRECK_FACILITY_WEIGHTS` 中 turret 50% / shield 25% / armor 15% / repair 10%），让玩家拾荒后能用于强化 build。
8. **生成节奏与冷却**：assault 任务在 level 3-5 出现概率？是否限制"同一星系仅出现 1 次 hostile-station"避免连续刷？建议 level 3 概率 25% / level 4 概率 35% / level 5 概率 30%，单星系上限 1 次。

## 数值边界建议（给数值策划）

- **hostile-station 总 HP 池**：1500 / 2200 / 3000（按 level 3/4/5 缩放）；单 cell HP 150-300（按 cell 类型）。
- **cells 配置**：5-8 个 cells，含 1 core + 1-2 power + 2-3 turret + 1 shield + 0-1 armor（具体配置由数值策划定）。
- **武器 stat**：turret damage 12-15、reload 1.6s、range 480-550（与 pirate 同档但更准）。
- **召唤手下**：spawnInterval 7-10s、spawnCap 3-5、minion = pirate。
- **assault 任务奖励**：metal base 35-45 / researchReward 9-10 / rewardMultiplier 1.4-1.6。
- **assault level 权重**：`OBJECTIVE_LEVEL_WEIGHTS` level 3 概率 25% / level 4 概率 35% / level 5 概率 30%；单星系上限 1 次。
- **wreck 爆出**：6-12 个 fragment / 设施分布 turret 50% / shield 25% / armor 15% / repair 10%。
- **AI 静止参数**：accel 3-5、drag 0.4-0.6、angularVel 0.1-0.3 rad/s（缓慢自转）。
- **击毁判定阈值**：core cell HP 归零即死。

## 高风险流程要求

按团队规则，本任务属于核心玩法 + 新 enemy 实体类型 + cells 化扩展，**必须走完整高风险流程**：

1. 制作人方向（本文档）commit + push 后才能进入下一步。
2. PM 拆分 v0.6.0 任务看板（7 个核心任务节点，参考 v0.5.0 看板风格）。
3. 主程 ADR 必须先 commit + push 才能进入实现。
4. 数值方案与 ADR 可并行启动，但必须在实现前 commit。
5. 实现建议拆分 T1 hostile-station 实体与 cells 基础（含 spawn / 渲染 / 命中）/ T2 AI 与武器 / T3 死亡与 wreck 爆出 / T4 assault 任务与生成节奏 / T5 HUD 反馈与平衡调参 / T6 死代码清理；每个子任务后做最小验证。
6. 实现完成后进入专项审核（与执行不同基底 agent），通过后 PM 完整回归。
7. 通过后制作人最终决定接受 + 是否触发发布。
