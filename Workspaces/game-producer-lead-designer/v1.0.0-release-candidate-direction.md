## 1.0 最小玩家承诺

`v1.0.0-release-candidate` 的目标不是兑现原始设计中的全部太空站 Roguelike 长线内容，而是交付一个单人首局可玩的垂直切片。玩家从空档进入游戏后，必须能理解并完成：

- 建造：从核心扩展框架，并建出推进 P0 所需的关键设施。
- 采矿收益：玩家能看懂采矿条件，成功让采矿站产生可归因收益，并把收益用于后续规划。
- 资源规划：玩家能根据金属、电力等基础资源约束决定先建什么、停什么、补什么。
- 维修：玩家遭遇损伤后能理解维修价值，并看到维修链路产生有效反馈。
- 基础防御 / 交火：玩家能遭遇并处理基础威胁，看到炮塔、防御设施或站体损伤带来的结果。
- 战后复盘：战斗或阶段结束后，玩家能看到本轮表现、损失、收益和下一步选择。
- 星系推进：玩家能完成首个星系阶段，并理解继续推进到下一个阶段的入口。
- Meta 回流：玩家结束或完成首局后，能看到局外奖励、解锁或成长入口，并理解下一局为什么会更强或有新选择。

交付候选还必须满足：真实浏览器无阻断错误，有真实输入与小屏证据，有可交付发布包和说明。没有这些证据时，只能算内部阶段完成，不能包装为 1.0 发布。

## 当前不可发布阻塞

- 采矿收益未闭环：尚未取得稳定 `harvestingCount > 0`、目标资源下降或可归因收益证据。
- 采矿覆盖路径产品方案未决：当前 helper 能解释候选路径和失败原因，但真实可支付、可理解、可操作的玩家方案还未确定。
- 真实人工鼠标 / 触控未覆盖：现有可见浏览器、CDP 与只读 hook 证据不能替代玩家真实输入体验。
- 自然交火 / 复盘 / Meta 未解锁：基础建设收益链未闭环前，不能跳到下一层体验补证。
- 完整 P0 未闭环：从空档进入、建造、采矿、维修、防御、复盘、星系推进到 Meta 回流的整条路径尚未完成验收。
- 正式发布包 / Release 归档未做：未构建可交付包，未准备 `Releases/` 目录说明，不能创建发布 tag。

## 必做范围

- 高风险采矿覆盖产品方案评估与实施：先决定是调整采矿范围 / 公式、资源布局、移动能力、建造合法性、数值，还是保留规则并改善可达路径与引导；决策后再执行。
- P0 首局完整证据：从空档开始，覆盖建造、采矿收益、资源规划、维修、基础防御 / 交火、战后复盘、星系推进、Meta 回流。
- 真实浏览器 / 真实输入 / 小屏证据：至少覆盖桌面真实鼠标路径与小屏触控或等价触控模式，记录阻断错误、输入失败和可读性问题。
- 自动检查和 release candidate checks：保留必要自动检查，明确哪些是单元 / 静态 / 浏览器级检查，哪些不能替代人工体验。
- 发布包构建与 `Releases/` 归档：通过发布闸门后，准备可直接运行的构建、版本说明、commit / tag / 构建时间 / 平台信息、已知问题和后续方向。
- 专项审核与制作人最终批准：涉及高风险项的实现必须完成专项审核，PM 汇总后由制作人决定是否进入 tag 与归档。

## 不做范围

- 不承诺联机、2-4 人合作、P2P 大厅或联机天赋。
- 不承诺异步对战、玩家空间站同步或 AI 控制对战站。
- 不承诺完整长期 Roguelike 内容量、完整天赋树、完整设施库、完整关卡池或 15-30 分钟正式局长。
- 不承诺实现 `Docs/game desc.txt` 中所有原始设计条目；1.0 只守住单人首局垂直切片。
- 不在证据不足、采矿收益未闭环、P0 未完整通过或发布包未归档的状态下强行发布。

## 风险流程

以下内容属于项目高风险项：采矿范围 / 公式、资源布局、移动能力、建造合法性、数值、存档 / Meta schema。

高风险项必须执行流程：制作人方向 -> PM 拆分 -> 专项方案 -> 执行 -> 审核 -> PM 汇总 -> 制作人决定。

在 PM 拆分前，不允许执行成员直接修改上述规则；在制作人最终决定前，不允许把高风险变更纳入 1.0 发布候选。

## 1.0 Gate 接受标准

### 玩家路径 Gate

- 玩家从空档进入首局，无需读外部说明即可理解核心、框架、设施建造和基础资源含义。
- 玩家能成功完成一次可归因采矿收益，并将收益用于下一步建造或修复。
- 玩家能经历一次基础威胁、看到损伤 / 防御 / 维修反馈，并理解是否需要调整站体。
- 玩家能完成首个星系阶段，进入战后复盘，再进入星系推进或下一阶段选择。
- 玩家能在首局结束或阶段完成后看到 Meta 回流入口，并理解局外奖励或成长对下一局的意义。

### 质量 Gate

- 真实浏览器体验无阻断 runtime error、unhandled rejection 或卡死。
- 桌面真实鼠标路径通过；小屏触控或等价触控模式至少完成首局 P0 主路径。
- HUD、提示、复盘和 Meta 文案能解释玩家当前状态、失败原因和下一步目标。
- 采矿、建造、维修、战斗、复盘、Meta 任一关键路径失败时，玩家能得到可理解反馈，而不是沉默失败。

### 技术 Gate

- 自动检查与 release candidate checks 通过，且检查范围和未覆盖风险写入 PM 汇总。
- 存档与 Meta schema 稳定，空档、旧档或异常档处理不会阻断首局进入。
- 调试 helper、测试入口和临时日志不污染正式体验；需要保留的观测能力必须有明确调试边界。
- 构建流程可重复，发布包能在 Windows 环境直接运行或按说明启动。

### 发布 Gate

- PM 完成 `v1.0.0-release-candidate` 总看板、专项审核汇总和 release candidate checks 结论。
- 制作人根据完整证据给出最终批准，明确是否允许 tag 与 `Releases/` 归档。
- 发布包包含可运行游戏、版本说明、主要变更、对应 commit、tag、构建时间、构建平台、已知问题和后续优化方向。
- 只有四类 Gate 全部通过后，才允许创建 `v1.0.0-release-candidate` 对应 tag 并归档发布产物。
