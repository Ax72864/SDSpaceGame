## 制作人结论

批准默认层 `A + E + F` 进入 #4 采矿方案实现与自检：

- `A`：金属缺口诊断。让玩家知道当前采矿失败是缺金属、缺路径、缺矿站位置，还是资源目标不可达。
- `E`：候选路径或 ghost path。给出下一步可理解的候选建造方向，但必须区分“建议路径”和“当前资源可支付路径”。
- `F`：移动 + 重建教学。把驾驶工站靠近资源外环，再重建采矿站的路径讲清楚，作为与扩建框架并列的玩家选择。

本次批准的核心目标不是降低采矿规则难度，而是让玩家能看懂为什么没有收益、下一步该怎么做，并通过真实输入取得可归因采矿收益证据。

## 默认层禁止事项

默认实现层明确禁止 #4 修改以下内容：

- 采矿范围或采矿公式。
- 首关资源布局、资源生成或 seed 保障。
- `frame`、`mining`、`thruster` 等建造成本。
- 移动物理、推进器推力、速度或操控参数。
- 建造合法性规则。
- 存档、Meta schema 或兼容结构。

如实现过程中发现必须触碰上述任一项，#4 立即停止扩大实现范围，退回 PM 拆分并重新走高风险决策。

## fallback 决策

`D-d tutorial buff` 与 `B 首关资源布局保障` 暂不批准实现，也不得随 #4 默认层一起开发。

二者仅作为后续高风险预案保留。触发条件为：

- 默认层 `A + E + F` 已完成实现、自检和基础复测。
- PM 使用真实输入连续 3 次新档验证。
- 每次新档在 120 秒内仍无法取得 `harvestingCount > 0` 或等价可归因采矿收益证据。

只有同时满足以上条件，才允许重新进入高风险流程：制作人方向 -> PM 拆分 -> 专项方案 -> 执行自检 -> 专项审核 -> PM 汇总 -> 制作人再次决定。

## 1.0 Gate

1.0 采矿 gate 的最低通过条件是取得采矿收益可归因证据：

- 真实输入路径下出现 `harvestingCount > 0` 或等价收益证据。
- 证据能说明收益来自采矿行为，而不是自然增长、采样误判或调试 hook。
- 玩家能从 UI 理解采矿站当前是否产出、为什么失败、下一步是扩建候选路径还是移动 + 重建。

在采矿收益可归因证据取得前，不允许继续 #7 自然交火 / 防御 / 维修 / 战后复盘补证。战斗和复盘证据不能替代基础建设链路成立。

## 产品接受口径

本决策接受 #3 方案汇总，允许 #4 开始低风险默认层实现。玩家收益是：首次卡在采矿时不再只看到沉默失败，而是能获得明确缺口、候选方向和移动重建路径。

仍不舒服的点：如果真实输入下资源仍不可达，默认层只能解释失败，不能保证收益闭环。该风险必须由 #4/#5/#6 的真实输入证据关闭；若关闭失败，再按高风险流程评估 fallback。
