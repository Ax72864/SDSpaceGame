# v0.7.0 战斗操控 制作人方向

版本名：`v0.7.0-combat-control`
基线：HEAD `242b9ea`（v0.6.0 敌方空间站发布后）
风险级别：**项目中高风险**（涉及核心战斗交互 + 输入系统扩展 + HUD 重构）

## 本轮目标

把已经在自动追鼠标的 `state.station.angle` 升级为玩家的"战斗操作信号"——加上**武器装填可视化**、**右键锁敌优先目标**和**键盘手动齐射**三套机制，让 v0.6.0 hostile-station BOSS 决战和日常战斗都从"按按钮等敌死"升级为"看节奏选目标卡时机扣扳机"，让 v0.5.0 研发系统的 build 选择、v0.6.0 战术决策（先打武器/直击核心）有可操作的工具。

## 体验目标

按重要程度排序：

1. **看得见战斗节奏**：玩家随时清楚 N 个导弹井中哪些 ready / 哪些还在装填倒计时，turret cell 哪些因 LOS 遮挡而停火；让"我有多少火力可调度"成为玩家随时能扫一眼就知道的信息。
2. **能锁定优先目标**：玩家右键点击敌方 BOSS 关键 cell（如 hostile-station 的 core / 武器 turret）或杂兵核心目标，所有 turret / missile / shield 都自动优先打它；让 v0.6.0 战术决策（先打武器/直击 core）从"靠运气"变成"主动控制"。
3. **能控制齐射时机**：玩家可以选择按 F 键"扣扳机"齐射就绪的导弹井，也可以选择"先发 2 枚试探 / 留 4 枚应急"；让导弹资源管理从"一按全梭"变成"节奏选择"。

## 必做范围

### 工程基础（先做这些再做战斗与 HUD）

- **`state.input.priorityTarget` 字段**：在 `state.input` 中新增 `{ enemy, setAt, validUntil }` 结构，存储玩家右键锁定的优先敌人；超时（12 秒）或敌死或距离 > 800 时自动清空。
- **`selectTarget(origin, range)` 函数**：包装现有 `nearestEnemy(origin, range)`，优先返回 `state.input.priorityTarget.enemy`（若在 range 内 + hp > 0 + 未超时），否则回退到原 `nearestEnemy` 逻辑。
- **`state.missile.salvoSize` 字段**：新增"齐射规模"配置（1 / 2 / 3 / all），玩家可通过键盘 `[` `]` 在四档之间循环调整；初始值 1（保持"逐发"作为新默认体验，避免老玩家一次性梭光全部导弹井被惩罚）。
- **`state.input.aimingRightButton` 守卫**：鼠标 `pointerdown` / `pointerup` / `pointermove` 必须按 `event.button` 区分左右键，左键继续走原 `state.drag` / `handleCanvasClick` 流程，右键单独走 `priorityTarget` 设定流程。

### 武器系统改造

- **`updateTurret` 使用 `selectTarget`**：将 `nearestEnemy(origin, range)` 改为 `selectTarget(origin, range)`，自动尊重玩家锁敌；`hasLineOfSight` 检查保留（侧面装甲战术不破坏）。
- **`updateShield` 使用 `selectTarget`**：`shieldDirection` 内的 `nearestEnemy(420)` 改为 `selectTarget(cellWorldPos, 420)`，让玩家右键锁定的敌方威胁能引导护盾朝向。
- **`fireMissiles` 改为 salvoSize 模式**：默认仅发射 `min(salvoSize, readyCount)` 个最早装填好的导弹井（按 reload === 0 排序，先就绪的先发）；保留 UI 按钮 + 新增 F 键绑定；保持现有 nearestEnemy 路径 fallback。

### 输入系统扩展

- **鼠标右键 = 锁敌 / 取消锁敌**：右键点击敌人（CELL × 1.5 半径内）→ 设定 `priorityTarget`；右键空白处 → 清空 `priorityTarget`。
- **键盘 F 键 = 手动齐射**：在 `bindInput` 内绑定 F 键到 `fireMissiles()`（避开已占用的 Space / Enter / WASD / 1-9 / 0 / - / =）。
- **键盘 `[` `]` = 调整齐射规模**：`[` 减小 salvoSize（最小 1），`]` 增大 salvoSize（最大 all = 999 / 表示全部 ready）；在 HUD 显示当前 salvoSize。
- **保留所有现有按键**：Space / Enter / WASD / 1-9 / 0 / - / = / 左键 / 鼠标拖动旋转 station 完全保留行为。

### HUD 与视觉反馈

- **主 HUD 武器装填指示器**：在资源条下方新增"武器"行：
  - 导弹井：`导弹井 X/Y 就绪 · 齐射 N · 下一发 Y 秒`
  - 炮塔：`炮塔 X/Y 工作 · K 个被自机遮挡`（K = 因 LOS 遮挡未开火的 turret 数量）
- **selectedCellPanel 内 reload 进度条**：选中 missile / turret cell 时，状态文字下方显示 reload 进度条（0-100%）。
- **优先目标视觉锁定框**：在 `renderEnemies` 之前 / 之中，给 `state.input.priorityTarget.enemy` 渲染红色十字准星 / 锁定圆环（区别于普通选中），让玩家清晰知道"我锁了谁"。
- **导弹井 cell 渲染时 ready 标识**：在 `renderStation` cell 渲染分支中，missile cell 装填完成时显示亮色发光环。
- **turret 被 LOS 遮挡视觉**：turret cell 当前帧因 LOS 遮挡未开火时，在 turret cell 边框瞬时显示**橙色半透明环**（持续 0.3-0.5 秒）作为视觉警告——不阻止任何行为，只让玩家知道"这个炮塔被挡住了"。

## 不做范围

- **不改炮塔朝向**：turret 仍朝最近敌人（或玩家锁定目标），不改为"基于 station angle + cell 局部偏移"——避免破坏现有 LOS 战术与玩家 build 学习成本。
- **不改 shield 方向性**：shield 仍保持现有 360° 近全向 dot >= -0.1 拦截——避免破坏 v0.6.0 hostile-station 弹幕平衡。
- **不引入新武器类型**：保留现有 turret / missile / shield 三种主战武器。
- **不引入弹药 / 能量 / 充能等新资源系统**。
- **不修改 v0.5.0 研发系统 / v0.6.0 hostile-station 实体**。
- **不修改 SAVE_KEY** / 局外存档结构。
- **不破坏 v0.6.0 hostile-station BOSS 战平衡**：所有改动是"玩家工具升级"而非"敌方/玩家数值改动"，BOSS 战难度感受不下降。
- **不做联机 / 异步对战**（game desc §9 / §10）。
- **不做局外天赋树重构**（game desc §8）。
- **不做新探索 / 地图系统**（game desc §7）。

## 关键技术疑问（给主程）

请主程在 ADR 中明确回答以下 8 项决定：

1. **`priorityTarget` 数据结构**：直接 reference enemy 对象 `{ enemy, setAt, validUntil }` 还是用 `enemyId` 弱引用？**建议直接 reference**（enemy 死后由 `selectTarget` 检查 `hp > 0` 自动失效，避免 ID 管理复杂度）。
2. **右键事件优先级**：鼠标右键 + Space + Enter + 左键 + 拖动旋转五种输入并存时的优先级链？**建议**：右键独立通道（仅 priorityTarget 设定）/ 左键 + 拖动维持原逻辑 / 键盘单独通道。需要确保 `event.button === 2` + `event.preventDefault()` + `contextmenu` 事件抑制（避免浏览器右键菜单弹出）。
3. **`salvoSize` 默认值与边界**：默认 1 还是 all？**建议默认 1**（让玩家自主升级到 all 而非默认全梭体验）；边界 1 / 2 / 3 / all（999）四档；持久化到 `state.meta` 还是仅 run 内？**建议仅 run 内**，跨 run 重置为 1，避免老玩家被新机制突袭。
4. **`fireMissiles` 排序逻辑**：salvoSize < readyCount 时优先发射哪些导弹井？**建议按 cell key 字典序**（确定性 + 玩家可预期，"左边的先发"）；不按"最早装填好的"避免多次按 F 选择性 bias 难以理解。
5. **优先目标视觉锁定框**：在 enemy 渲染前后 / 顶部覆盖层 / HUD 标记？**建议**：场景层渲染（覆盖在 enemy 之上，但在 HUD 之下），样式：红色十字准星 + 旋转脉冲圆环，区别于现有 cellSelect / hostile-station 渲染。
6. **HUD 武器行布局**：放在资源条下方还是右侧 mini panel？**建议**：资源条下方，单行水平布局，避免占用任务/告警空间；移动设备 / 小屏可考虑响应式。
7. **F 键 / `[` `]` 键的可发现性**：游戏内是否需要 onboarding 教学？**建议**：仅在 HUD 武器行末尾加灰色提示文字"按 F 齐射 · [/]调档"，不引入额外教学层。
8. **LOS 遮挡视觉警告频次**：每帧炮塔被挡都显示橙色环还是只在"刚被挡上"的瞬间显示？**建议**：仅在"被挡的瞬间"短暂显示 0.5 秒（用 `cell._losBlockedAt` 时间戳节流），持续遮挡不闪烁。

## 数值边界建议（给数值策划）

- **`priorityTarget` 有效期**：12 秒（玩家锁敌后 12 秒不操作自动失效）；可在 ±20% 范围内微调。
- **`priorityTarget` 锁敌距离**：玩家鼠标坐标 / CELL × 1.5 半径内（即 ≈ 48 像素）的敌人可被锁定；范围内多个敌人时取最近。
- **`priorityTarget` 失效距离**：锁定后敌人距离 > 800 px 自动清空（玩家自动脱战）。
- **`salvoSize` 边界**：1 / 2 / 3 / all（999）四档循环，初始 1。
- **F 键防抖**：连按间隔最短 0.3 秒，避免误操作连发。
- **HUD reload 进度条**：精度 1%（每 10ms 更新）；颜色：0-25% 暗红 / 25-75% 黄 / 75-100% 亮绿。
- **LOS 遮挡视觉警告**：橙色环持续 0.5 秒，alpha 衰减；同一 cell 5 秒内不重复触发。
- **优先目标锁定框**：红色十字 / 半径 = enemy.r × 1.5（hostile-station 用 80 px）/ 旋转角速度 1 rad/s / pulse 周期 1.0 秒。

## 中风险流程要求

按团队规则，本任务属于核心战斗系统扩展 + 输入系统扩展 + HUD 重构，**走中风险流程**（比 v0.6.0 高风险流程简化，但仍需 ADR + 数值方案 + PM 节点回归 + 制作人决定）：

1. 制作人方向（本文档）commit + push 后才能进入下一步。
2. PM 拆分 v0.7.0 任务看板（6-7 个核心任务节点，参考 v0.6.0 看板风格简化）。
3. 主程 ADR 必须先 commit + push 才能进入实现。
4. 数值方案可与 ADR 并行启动，但必须在实现前 commit。
5. 实现建议拆分 T1 工程基础（state.input / selectTarget / salvoSize 字段）/ T2 武器系统改造（updateTurret/updateShield/fireMissiles 接入 selectTarget）/ T3 输入扩展（右键 / F / [/]）/ T4 HUD 装填指示器 / T5 视觉反馈（锁定框 / ready 标识 / LOS 警告）/ T6 死代码清理；每个子任务后做最小验证。
6. 实现完成后进入专项审核（与执行不同基底 agent），通过后 PM 完整回归。
7. 通过后制作人最终决定接受 + 是否触发发布。
