## 制作人结论

进入 `v0.20.0-mining-coverage-path-evidence`。

v0.19 已把采矿问题从“系统可能坏了”收敛为“当前矿站没有覆盖资源外环”：矿站已建成且 active，但 `harvestingCount=0`、`target=null`、`allOutOfRange=true`，等待窗口内资源增长不能归因于采矿。本阶段的核心目标不是扩玩法，而是证明玩家如何让矿站覆盖资源外环并产生可归因收益；若现有规则路径不可达，则必须让 UI / 诊断明确解释“当前位置采不到 / 需要重新选址”。

本阶段继续关闭发布闸门。采矿覆盖与收益闭环前，不解锁自然交火、战后复盘、Meta 回流补证。

## 核心目标

证明玩家在当前规则下，能否通过扩建、移动或重新选址，让矿站进入资源外环并形成可归因收益。

必须回答两个问题：

1. 成功路径是否存在：玩家从原路径或一个明确替代路径出发，能否建成 active 矿站，使资源目标进入采矿范围，并看到 `harvestingCount > 0`、`target` 指向资源、等待前后资源增量可归因于采矿。
2. 若成功路径不存在或现有 UI 无法引导玩家抵达：UI / 只读诊断是否能解释当前位置为什么采不到，以及下一步应扩建、移动、重新选址，还是放弃当前点位。

## 玩家收益

- 玩家不只看到“还差 X”，还知道当前矿站能不能通过扩建、移动或选址进入资源外环。
- 玩家能区分“暂时距离不够，可以继续靠近 / 扩建”和“当前位置采不到，需要重新选址”。
- 玩家能理解下一步该做什么：继续补框架、调整站体位置、选择更靠近资源外环的位置，或接受当前规则下不可达。
- 团队能把资源增长从核心自然产出中分离出来，证明收益是否来自采矿站。

## 必做范围

1. PM 看板：由 PM 建立 v0.20.0 看板，拆分负责人、验收标准、状态、风险和阻塞项。
2. 采矿覆盖路径方案：梳理原路径 `mining=-1,1` 的距离 / range / gap，并提出至少一条可尝试的覆盖路径；路径可以是扩建路径、移动路径或重新选址路径，但不得默认改规则。
3. 可见浏览器证据：用可见浏览器记录玩家路径、关键 UI 状态、等待窗口和资源变化；CDP / hook 只能作为辅助输入或诊断，不能包装成人类体验通过。
4. 只读路径 / 距离诊断：记录 `minerCount`、`activeCount`、`harvestingCount`、`target`、最近资源、distance、range、gap、`allOutOfRange`、等待前后资源数值和收益归因。
5. 必要最小反馈修复：若规则路径可达但 UI 表达不足，只允许做最小反馈修复，让玩家理解“当前采不到 / 还差多少 / 下一步怎么选址”。不得改采矿公式、范围、资源布局或移动能力。
6. 原路径复测：复测 v0.19 的原路径，确认维修链不回潮，采矿阻塞是否仍复现，最小反馈是否准确解释阻塞。
7. 专项审核：由专项审核确认改动未越界，尤其检查采矿范围 / 公式、资源布局、移动能力、建造合法性、存档和发布目录。
8. PM 汇总：PM 汇总通过、未通过、未覆盖、runtime errors、证据质量和残留风险。
9. 制作人决定：基于 PM 汇总决定本阶段接受、继续追采矿路径，或升级高风险流程。

## 不做范围

- 默认不改采矿范围、采矿公式、资源生成公式、资源布局、移动能力、建造合法性或存档结构。
- 不新增玩法、建筑、敌人、战斗规则、Meta 系统或局外成长内容。
- 不进入自然交火、战后复盘、Meta 回流补证。
- 不发布、不打 tag、不归档 `Releases/`。
- 不把 CDP / hook / 只读 helper 证据包装成人类体验通过。

## 风险与升级条件

本阶段为中风险，因为它触及基础建设因果链和采矿收益证据，但默认只做路径验证、诊断和最小反馈。

以下任一情况出现，立即升级高风险流程，停止本轮直接修改：

- 必须扩大或重算采矿范围才能覆盖资源外环。
- 必须调整资源外环、行星 / 小行星布局或资源生成位置。
- 必须改变移动能力、推进能力、站体坐标规则或相机 / 点击映射，玩家才能抵达采矿范围。
- 必须修改资源生成公式、采矿收益公式或设施工作条件。
- 必须修改建造合法性、存档结构、schema 或历史数据兼容逻辑。

高风险流程要求制作人重新确认方向，PM 重新拆分任务，专项成员审核后再决定是否进入规则改动；不能在 v0.20.0 本轮直接顺手改。

## 接受标准

本阶段接受必须满足以下条件之一：

1. 形成至少一条可复测的采矿覆盖成功路径，并证明收益归因：
   - 矿站 active，且资源目标进入采矿范围。
   - `harvestingCount > 0`，`target` 指向明确资源。
   - 等待窗口前后资源变化能排除或标注核心自然产出，说明采矿收益来自哪里。
   - 可见浏览器证据能复现路径，不只依赖 CDP / hook。
2. 或明确证明现有规则下不可达，并完成最小反馈闭环：
   - 只读诊断能说明 nearest resource、distance、range、gap、`allOutOfRange`。
   - UI 能让玩家理解“当前位置采不到 / 需要重新选址”或等价阻塞原因。
   - 玩家下一步行动清楚，不再只看到孤立的“还差 X”。

同时必须满足：

- runtime errors 为 `0`，console errors 为 `0`，unhandled rejections 为 `0`。
- v0.19 已通过的维修链和原建造路径不回潮。
- 不修改本阶段不做范围中的核心规则和发布资产。
- 证据中明确区分自动检查、浏览器级检查、CDP 辅助、真实人工体验和未覆盖路径。

## 下一阶段条件

只有当采矿覆盖 / 收益闭环完成，或现有规则不可达但反馈已能让玩家理解阻塞，才允许进入下一层体验验证。

下一阶段可讨论：

- 自然交火：验证敌人压力是否能建立建造和维修价值。
- 战后复盘：让玩家理解损伤、维修、资源消耗和失败原因。
- Meta 回流补证：在局内基础建设因果链成立后，再验证局外奖励是否支撑下一局。

如果 v0.20.0 仍无法关闭采矿覆盖 / 收益证据，不允许以“战斗更有趣”为理由跳过基础建设链。
