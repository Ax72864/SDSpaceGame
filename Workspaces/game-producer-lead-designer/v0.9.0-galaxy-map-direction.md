# v0.9.0 局间星系图系统 制作人方向

版本名：`v0.9.0-galaxy-map`
基线：HEAD `3a89628`（v0.8.0 局内随机事件发布后）
风险级别：**项目高风险**（局间结构重构 + 跃迁 UI 重做 + 权重表二次分组 + 视觉系统按 galaxyType 分化 + 跨关字段扩展）

## 本轮目标

在 v0.7.0 战斗操控 + v0.8.0 局内事件之上，引入**局间星系图系统**——玩家不再"线性进入下一关"，而是在**2-3 个候选星系**中选择目的地，每个星系有独立的敌人 / 任务 / 事件偏好和视觉风格；7 关一局的结构不变，但玩家走出的路径是"独属于本局的轨迹"，把维度从"局内深度（v0.4.0 任务 + v0.8.0 事件）"扩展到"局间结构（玩家路径选择）"。

## 体验目标

按重要程度排序：

1. **路径决策权**：玩家每次跃迁都面临"我下一关去哪"的选择，每个候选星系类型有明确的风险/收益取舍（如"贸易区低风险低收益 vs 海盗领地高风险高战利品"）；玩家根据自己的舰船状态（HP 低就避险 / 资源富就冒险）做决策。
2. **单局轨迹记忆**：玩家完成一局后能说出"我这局走的是 贸易区 → 海盗领地 → 矿产带 → 战争前线 → 科技废墟 → 疑虑空间 → 终末"的完整轨迹；不同路径的局体验差异明显（不是 7 关都长一样）。
3. **与已有系统协同**：v0.4.0 OBJECTIVE_TYPES / v0.6.0 hostile-station / v0.8.0 ENCOUNTER_TYPES 全部完整保留，星系类型仅作为**权重覆盖层**调整出现概率；玩家熟悉的任务 / 事件 / BOSS 战体验保持稳定，新维度叠加在旧维度之上。

## 核心机制

### 1. 平行 GALAXY_TYPES 注册表（不破坏现有 weight 表）

**关键架构决策**：v0.9.0 必须采用**权重覆盖层**而非替换 `OBJECTIVE_LEVEL_WEIGHTS` / `ENCOUNTER_LEVEL_WEIGHTS`。

理由：
- 现有 weight 表是"按 level 决定整局节奏"，是稳定的难度曲线基线
- galaxyType 是"按玩家选择微调当前关"，叠加在 level 基线之上
- 替换式实现会破坏现有 `__gameTest.getObjectiveType / sampleAssaultRate` 等测试钩子
- 覆盖式实现保留 fallback 路径，旧存档 / 旧路径完全等价 v0.8.0

实施清单：
- 新建 `GALAXY_TYPES = { tradeHub, pirateTerritory, miningBelt, mysteryZone, warFront, techRuin, emptyVoid }` 注册表
- 每个 galaxyType 字段：`name / description / riskLevel / iconColor / nebulaTint / objectiveWeightMod / encounterWeightMod / enemySpawnMod / resourceMod / cssClass`
- 新建 `state.run.galaxyMap: { nodes: [], currentNodeId, visitedNodeIds }` 数据结构
- 新建 `state.run.pendingGalaxyChoices: []`（玩家可选的 2-3 个候选）
- 新建 `state.run.currentGalaxyType: string | null`（当前关的 galaxyType，level 0 默认 emptyVoid）
- `getObjectiveWeightTable(level)` 内部追加 galaxyType 权重乘子调整
- `seedEncountersForLevel(level)` 内部追加 galaxyType 权重乘子调整
- `spawnWave` / 敌人 spawn rate 内部追加 galaxyType 修正

### 2. 7 种 galaxyType 设计

| 编号 | 名称 | riskLevel | OBJECTIVE 偏好 | ENCOUNTER 偏好 | 敌人/资源/视觉 |
|---|---|---|---|---|---|
| G1 | **稳定贸易区**（tradeHub） | 低 | mine + survive 偏好 | trader ×3 | 敌人 ×0.7 / 金属 ×1.2 / 蓝色调 |
| G2 | **海盗领地**（pirateTerritory） | 高 | battle 偏好 | ambush ×2 / distress ×1.5 | pirate ×1.5 / 金属奖励 ×1.3 / 红色调 |
| G3 | **矿产带**（miningBelt） | 中 | mine 偏好 ×2 | derelict ×1.5 | asteroid ×2 / 矿物 ×1.5 / 黄褐色调 |
| G4 | **疑虑空间**（mysteryZone） | 中 | explore 偏好 | signal ×3 / trader ×0.5 | 敌人组合多样化 / 等离子 ×1.3 / 紫色调 |
| G5 | **战争前线**（warFront） | 极高 | battle + assault 偏好 ×2 | ambush ×1.5 / distress ×2 | hostile-station 概率 ×2 / 科研 ×1.5 / 橙红色调 |
| G6 | **科技废墟**（techRuin） | 中 | salvage 偏好 ×2 | derelict ×2 / signal ×1.5 | wreck cell 偏 plasma / 科研 ×1.5 / 青绿色调 |
| G7 | **空旷星域**（emptyVoid） | 低 | 全任务等概率 | 全事件 ×0.5 | 敌人 ×0.5 / 跨关 buffer ×0.7（更快跃迁）/ 灰色调 |

`level 0` 起点固定 `emptyVoid`（教学性低难度起点）；`level 6` 终末关固定 `warFront`（保护 guardian 决战体验）；level 1-5 由玩家从 2-3 个候选中选。

### 3. 跃迁选择流程

当前流程（v0.8.0）：
1. 任务完成 → `awaitingObjectiveChoice=true`
2. 玩家点击 `objectiveJumpBtn` → 直接调 `nextLevel()`

v0.9.0 新流程：
1. 任务完成 → `awaitingObjectiveChoice=true`
2. 玩家点击 `objectiveJumpBtn` → **弹出 #galaxyMapPanel**（不立即 nextLevel）
3. 显示 2-3 个候选 galaxyType 卡片
4. 玩家点击候选卡片 → `confirmGalaxyJump(nodeId)` → `nextLevel(galaxyType)`
5. `nextLevel(galaxyType)` 将传入的 galaxyType 写入 `state.run.currentGalaxyType` 后照常清理 + createObjective + seedEncountersForLevel

**关键约束**：
- 旧路径 `nextLevel()` 不传参时 fallback 到 `emptyVoid`（兼容 `__gameTest.resetRun` 等测试调用）
- `objectiveChoicePanel` 流程完全保留（仅在玩家点击 jumpBtn 后增加 galaxyMapPanel 步骤）
- 终末关（level 6 → guardian）不显示 galaxyMapPanel，直接 nextLevel(warFront)
- 玩家可以 Escape 取消选择，回到关内继续游戏（但任务已 complete，跃迁仍 pending）

### 4. 候选星系生成算法

每次显示 galaxyMapPanel 前调用 `generateGalaxyChoices(level, rng)`：

```javascript
function generateGalaxyChoices(targetLevel, rng) {
  if (targetLevel <= 0) return [{ galaxyType: "emptyVoid", reason: "start" }];
  if (targetLevel >= ENDGAME_LEVEL) return [{ galaxyType: "warFront", reason: "endgame" }];
  
  // level 1-5：随机抽 2-3 个候选
  const candidateCount = targetLevel < 3 ? 2 : 3;
  const availableTypes = Object.keys(GALAXY_TYPES).filter(t => t !== "emptyVoid");
  // 排除当前 galaxyType（避免连续两关同类型）
  if (state.run.currentGalaxyType) {
    const idx = availableTypes.indexOf(state.run.currentGalaxyType);
    if (idx >= 0) availableTypes.splice(idx, 1);
  }
  
  // 按 level 难度曲线调整权重（高 level 偏向高风险星系）
  const weights = {};
  for (const t of availableTypes) {
    const risk = GALAXY_TYPES[t].riskLevel; // "low" | "mid" | "high" | "extreme"
    weights[t] = baseWeights[risk] * levelBias[targetLevel];
  }
  
  const choices = [];
  for (let i = 0; i < candidateCount && availableTypes.length > 0; i++) {
    const picked = pickWeighted(weights, rng);
    if (!picked) break;
    choices.push({ galaxyType: picked });
    delete weights[picked];
  }
  return choices;
}
```

### 5. 视觉差异（按 galaxyType 分化）

每个 galaxyType 有独立视觉签名：
- **背景色**：天空底色（`drawBackground` 内的 fillStyle）按 galaxyType 切换
- **星云**：色调 / 密度按 galaxyType 切换（`drawNebula` 添加 `tintColor` 参数）
- **行星样式**：tradeHub 蓝色星球 / pirateTerritory 暗红 / miningBelt 黄褐 / mysteryZone 紫 / warFront 橙红 / techRuin 青绿 / emptyVoid 灰
- **小行星密度**：miningBelt 多 / emptyVoid 少 / 其他中等
- **HUD 状态条**：左上角显示当前 galaxyType 名称 + icon + 风险等级

### 6. 路径预览 HUD

玩家可以在 HUD 看到：
- **已走过的路径**：`#galaxyPathEl` 显示 emptyVoid → tradeHub → ... 链式 7 个节点（已走过的高亮 / 未走过的灰色 / 当前节点闪烁）
- **下一关预告**：选择完候选后 HUD 显示"下一关：xxx"直到 nextLevel
- **路径决策时**：galaxyMapPanel 显示候选 + 风险预告 + 主要偏好 icon

### 7. 数据结构

```javascript
state.run.galaxyMap = {
  nodes: [
    { id: "node-0", level: 0, galaxyType: "emptyVoid", visited: true },
    { id: "node-1", level: 1, galaxyType: "tradeHub", visited: true },
    // ... 玩家走过的节点
  ],
  currentNodeId: "node-1",
  pendingChoices: [],  // 当前可选的下一关候选
};

state.run.currentGalaxyType = "tradeHub";  // 当前关 galaxyType
state.run.galaxyChoicesShown = false;     // 当前关 galaxyMapPanel 是否已展示（避免重复）
```

## 关键技术疑问（给主程）

请主程在 ADR 中明确回答以下 10 项决定：

1. **GALAXY_TYPES 字段集**：每个 galaxyType 的完整字段（name / description / riskLevel / iconColor / nebulaTint / objectiveWeightMod / encounterWeightMod / enemySpawnMod / resourceMod / cssClass）。是否引入额外字段（如 `bgmHint / spawnDelayMod`）？**建议**：上述 10 字段，不再添加。
2. **`state.run.galaxyMap` 数据结构**：详细字段清单 + ensureRunRuntimeState 兼容默认值。**建议**：见方向 §7 草案，主程可微调字段命名。
3. **权重乘子作用时机**：是覆盖式（替换 base 权重）还是叠加式（乘子相乘）？**建议**：叠加式 `final = base × galaxyMod`，乘子 ≥ 0，galaxyMod=1.0 时等价 v0.8.0。
4. **跃迁流程改造点**：在 `objectiveJumpBtn` 的 click handler 内 hook 是更安全还是改造 `nextLevel()` 入口？**建议**：handler hook（保留 nextLevel 原 API 兼容性），handler 内若 `currentGalaxyType !== "endgame"` 弹出 galaxyMapPanel，否则直接 `nextLevel("warFront")`。
5. **galaxyMapPanel DOM 结构**：是 `<dialog>` 还是 `<div role="dialog">` + overlay？卡片是 grid 还是 flex？**建议**：复用 `objectiveChoicePanel` 的 div + overlay 模式，卡片用 flex row（最多 3 卡片可在屏幕宽度内并排）。
6. **取消跃迁的状态恢复**：Escape 取消后是否保持 `awaitingObjectiveChoice=true`？**建议**：保持（玩家可以在关内继续操作，然后再次点 jumpBtn 重新弹出选择面板）。
7. **视觉分化的渲染时机**：`drawBackground` 全帧重绘还是按 galaxyType 切换 cache？**建议**：全帧重绘（性能开销可忽略），按 galaxyType 选 fillStyle / nebulaTint。
8. **现有 `__gameTest.getObjectiveType(seed, level)` 测试钩子兼容**：传入 galaxyType 后输出会变化吗？**建议**：钩子新增可选第 3 参数 `galaxyType?`，默认 `emptyVoid`（乘子 1.0），完全等价 v0.8.0。
9. **跨关 buffer 时间是否调整**：emptyVoid 跃迁是否需要更短 buffer？**建议**：保持现有 buffer 时间不变（避免触发跨关清理时序问题），仅在 HUD 显示上做 galaxyType 提示。
10. **路径预览 HUD 持久性**：是每关都显示还是 toggle 显示？**建议**：始终显示（在 HUD 右上角小条），但折叠为 7 个圆点 + 当前节点亮 + 已走节点淡（避免遮挡视野）。

## 数值边界建议（给数值策划）

- **OBJECTIVE 权重乘子范围**：0.3 - 3.0（避免 0 完全禁用 / 避免 > 3 过度偏好）
- **ENCOUNTER 权重乘子范围**：0.3 - 3.0
- **敌人 spawn 乘子范围**：0.5 - 2.0
- **资源乘子范围**：0.7 - 1.5
- **hostile-station 触发乘子**：0.5 - 2.0（warFront ×2，其他 1.0）
- **候选数量**：level 0 = 1（强制 emptyVoid）/ level 1-2 = 2 / level 3-5 = 3 / level 6 = 1（强制 warFront）
- **风险等级权重**（候选生成）：低 30% / 中 40% / 高 25% / 极高 5%
- **levelBias**：低 level 偏低风险 / 高 level 偏高风险（避免 level 1 直接出 warFront）
- **路径 HUD 圆点**：直径 12px，已走 / 当前 / 未走分别为 #5ec460 / #ffcc44 / #555555
- **候选卡片尺寸**：280px 宽 × 380px 高（参考 v0.5.0 升级面板卡片）
- **GalaxyMap 出现 / 消失动画**：fade-in 250ms / fade-out 200ms

## 风险预警

1. **跃迁流程改造可能破坏 `__gameTest.resetRun` / `findEscortSeed`**：必须保留 `nextLevel()` 无参调用兼容路径。
2. **权重乘子叠加可能造成数值失衡**：需数值方案明确"emptyVoid 全乘子 1.0"约束 + 其他 galaxyType 极值（如 warFront assault ×2）测试边界。
3. **UI 层 galaxyMapPanel 与 objectiveChoicePanel 状态竞争**：需明确两者的显示顺序（先 objectiveChoicePanel 再 galaxyMapPanel）+ Escape 取消时各自状态。
4. **视觉分化可能影响性能**：drawBackground / drawNebula 每帧调用，galaxyType 切换不应引入新的 cache 失效。
5. **路径预览 HUD 可能遮挡视野**：右上角小条 + 7 个圆点，CSS 避免大尺寸 / 透明度过低不可读。
6. **emptyVoid 起点固定可能让玩家感觉"第一关无选择"**：可考虑 level 1 直接给 2 个候选，level 0 不显示 galaxyMapPanel（保持开局体验等价 v0.8.0）。
