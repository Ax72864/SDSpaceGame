## 玩家收益

本阶段的收益不是玩家拿到一个可发布版本，而是团队明确了首局补证失败点和正确下一步：不能再把 hook、CDP 或辅助构造状态当作真人体验通过。

这使后续工作更聚焦：先证明玩家能真实完成首局最小建设链，再继续证明交火、复盘和 Meta 回流，而不是在证据不足时进入发布流程。

## 仍不舒服的点

玩家需要能自然完成以下路径：

1. 扩框架。
2. 在空框架上建设施。
3. 形成采矿、武装和维修能力。
4. 进入交火并看到战后复盘。
5. 从结算或局外路径完成 Meta 回流。

现在证据不足。尤其是“扩框架 -> 在已有空框架上建设施”仍缺正确真实操作复测；后面的自然交火、战后复盘和 Meta 回流也因此没有资格判通过。

## 下一阶段建议

建议推进 `v0.18.0-p0-build-chain-retest` 或等价下一阶段。

优先级如下：

1. 用正确真实操作复测建造链：先扩出 `frame`，再点击该已有 `frame` 替换 `power` / `mining` / `turret` / `repair`，逐步记录 `gridKey`、`facility`、资源变化、toast / statusAlerts 和 runtime errors。
2. 如果正确操作仍失败，再补足新手提示、点击反馈或最小 UI/交互 polish，并按原失败路径复测。
3. 建造 / 采矿链闭环后，再继续自然交火、武器 / 护盾 / 维修反馈、战后复盘证据。
4. 交火和复盘成立后，再补 Meta 真实点击回流和 storage 影响记录。
5. 真实触控仍需单独覆盖；若无真机，只能诚实记录替代边界，不能包装为真触控通过。
