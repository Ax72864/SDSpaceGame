## 不批准发布准备的原因

- 完整 #6 首局 P0 建设链仍未通过：建造、采矿、资源规划、维修没有形成可交付闭环。
- 采矿收益只在单项 gate 通过；P0 连续路径中采矿不稳定，出现过短暂收益后回到 out-of-range，追加复测也未稳定捕获 harvest。
- repair 路径未闭环：维修站未建成，`repair_missing` / `repairerCount=0`，受损对象存在但维修反馈无效。
- 真实人工鼠标、小屏触控或等价触控体验未覆盖；CDP 鼠标事件和只读 hook 只能作为辅助证据。
- #7 自然交火 / 防御 / 维修 / 战后复盘、#8 星系推进与 Meta、#9 真实输入 / 触控、#10 release candidate 检查、#11 发布包 / Release 归档方案仍被阻塞。
- 当前没有发布包、tag、`Releases/` 归档或制作人最终发布批准证据。

## 下一阶段方向

继续 `v1.0.0` 修复，不开新版本号。下一阶段只追一个目标：让 #6 首局 P0 建设链稳定通过，再解锁 #7 及后续发布 gate。

优先修复顺序：

1. P0 建设链稳定性：玩家或 PM 复测能从空档按可理解路径完成框架、采矿站、补电 / 防御、维修准备。
2. 工站保持采矿范围：采矿站建成后，后续补设施、移动或暂停恢复不应把采矿状态推回不可理解的 out-of-range。
3. repair 路径：维修站可建、维修目标可发现、维修反馈可见，失败时有明确原因。
4. 真实输入：桌面人工鼠标路径和小屏触控或等价触控路径必须覆盖 P0 主路径。

如果默认层引导与输入修复仍不能稳定关闭 #6，必须回到高风险产品决策，重新评估 `tutorial buff`、首关资源布局、移动 / 停靠规则、采矿范围或相关数值。任何这类改动都不能由执行成员自行加码。
