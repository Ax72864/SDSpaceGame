# 团队成员表

本文档汇总当前项目的 Cursor subagent 成员。成员配置位于 `.cursor/agents/`，调用时使用对应的 `/agent-name`。

| 成员 | 调用名 | 核心能力 | 主要职责 |
| --- | --- | --- | --- |
| 制作人兼主策划 | `/game-producer-lead-designer` | 游戏立项、核心循环设计、体验目标收敛、版本范围控制、跨职能任务拆解 | 决定产品方向和阶段目标，定义 MVP 与迭代范围，审核玩法体验是否清晰、可玩、可交付 |
| 执行兼数值策划 | `/systems-numerics-designer` | 规则落地、数值表设计、关卡参数、成长曲线、奖励掉落、平衡性分析 | 将主策划方向转成可实现的规则、公式、配置字段和验收标准，负责调参建议与边界案例 |
| UI 美术 | `/ui-artist` | HUD、按钮、弹窗、图标、界面层级、可读性、交互反馈和轻量动效设计 | 设计玩家界面信息结构和视觉表现，保证操作清晰、反馈明确，并为 UI 资产提出规格建议 |
| 资产兼主美术 | `/art-director-asset-artist` | 整体美术风格、角色场景资产、道具特效、色板规范、资源清单和美术一致性 | 制定视觉方向，拆分资产需求与优先级，控制资源复用、制作成本和风格统一 |
| 资深玩法技术 | `/senior-gameplay-engineer` | 核心玩法实现、输入响应、状态机、碰撞判定、反馈触发、快速原型 | 将玩法设计实现为可玩的交互逻辑，负责手感、参数可调性、玩法稳定性和基础自检 |
| 资深系统技术 | `/senior-systems-engineer` | 工程结构、资源加载、配置、存档、性能优化、构建流程和工具链维护 | 保障项目工程稳定、运行效率和基础系统质量，处理系统风险、性能问题和构建发布问题 |
| 主程 | `/lead-game-programmer` | 技术方案评审、架构决策、代码质量把关、任务拆分、风险控制和疑难问题攻关 | 负责技术方向和关键决策，评审公共架构、核心代码、数据结构、发布风险和跨模块影响 |
| PM 兼测试 | `/pm-qa-coordinator` | 版本计划、任务跟进、验收标准、测试用例、缺陷复现、回归检查和发布清单 | 维护任务闭环与质量标准，跟踪负责人、进度、风险和验收结果，保证版本可测试、可交付 |
| IT 工程师 | `/it-engineer` | 软件安装、开发环境配置、工具交付、系统维护、版本验证和危险操作识别 | 为团队准备安全稳定的开发环境，处理工具链、SDK、CLI、依赖和本机环境问题，并控制系统级风险 |

## 分工原则

- 产品方向、玩法目标和版本边界由 `/game-producer-lead-designer` 负责最终判断。
- 任务拆分、测试验收、回归和发布清单由 `/pm-qa-coordinator` 负责闭环。
- 技术架构、公共代码、关键风险和复杂疑难问题由 `/lead-game-programmer` 把关。
- 玩法实现优先交给 `/senior-gameplay-engineer`，工程系统问题优先交给 `/senior-systems-engineer`。
- 规则、数值、配置和关卡参数优先交给 `/systems-numerics-designer`。
- 界面体验交给 `/ui-artist`，整体视觉和资产规划交给 `/art-director-asset-artist`。
- 环境安装、工具配置、权限和系统风险交给 `/it-engineer`；涉及高风险操作时必须先确认。
